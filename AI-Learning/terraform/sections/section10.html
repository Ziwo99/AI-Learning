Section 10: Gestion des déploiements

  • Revenir à la fiche du cours
  • Section précédente
  • Section suivante

La gestion des déploiements est une étape essentielle dans l'utilisation de Terraform. Elle permet de déployer des applications et de configurer les variables d'environnement nécessaires. De plus, elle offre la possibilité de gérer les mises à jour et les rollbacks de manière efficace.

Déploiement d'applications

Le déploiement d'applications avec Terraform se fait en utilisant des ressources spécifiques qui permettent de créer et de configurer les instances nécessaires. Ces ressources peuvent être des serveurs, des conteneurs ou d'autres types d'infrastructures.

Voici un exemple de code Terraform pour déployer une application sur un serveur :

resource "aws_instance" "example" {
    ami           = "ami-0c94855ba95c71c99"
    instance_type = "t2.micro"

    tags = {
      Name = "example-instance"
    }
  }

Dans cet exemple, nous utilisons la ressource "aws_instance" pour créer une instance EC2 sur AWS. Nous spécifions l'AMI (Amazon Machine Image) à utiliser, le type d'instance et ajoutons des tags pour identifier notre instance.

Configuration des variables d'environnement

La configuration des variables d'environnement est une étape importante pour déployer des applications avec Terraform. Les variables d'environnement permettent de stocker des informations sensibles ou spécifiques à l'environnement, telles que des clés d'API ou des URL de base de données.

Voici un exemple de code Terraform pour configurer des variables d'environnement :

resource "aws_instance" "example" {
    ami           = "ami-0c94855ba95c71c99"
    instance_type = "t2.micro"

    tags = {
      Name = "example-instance"
    }

    provisioner "remote-exec" {
      inline = [
        "export DB_HOST=${var.db_host}",
        "export API_KEY=${var.api_key}"
      ]
    }
  }

Dans cet exemple, nous utilisons la ressource "aws_instance" pour créer une instance EC2 sur AWS. Nous utilisons également le provisioner "remote-exec" pour exécuter des commandes à distance sur l'instance. Nous configurons les variables d'environnement DB_HOST et API_KEY en utilisant les valeurs définies dans les variables Terraform.

Gestion des mises à jour et des rollbacks

Terraform offre des fonctionnalités avancées pour gérer les mises à jour et les rollbacks de manière efficace. Lorsque vous apportez des modifications à votre infrastructure, Terraform compare l'état actuel avec l'état souhaité et détermine les actions nécessaires pour atteindre cet état.

Si une mise à jour est nécessaire, Terraform effectuera les modifications de manière séquentielle et sécurisée. En cas d'échec, Terraform peut automatiquement effectuer un rollback en revenant à l'état précédent.

Voici un exemple de code Terraform pour gérer les mises à jour et les rollbacks :

resource "aws_instance" "example" {
    ami           = "ami-0c94855ba95c71c99"
    instance_type = "t2.micro"

    tags = {
      Name = "example-instance"
    }

    lifecycle {
      create_before_destroy = true
    }
  }

Dans cet exemple, nous utilisons la ressource "aws_instance" pour créer une instance EC2 sur AWS. Nous utilisons également le bloc "lifecycle" pour spécifier que la nouvelle instance doit être créée avant que l'ancienne instance ne soit détruite. Cela permet d'assurer un déploiement en douceur et un rollback automatique en cas d'échec.
