Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. La POO offre de nombreux avantages, tels que la réutilisabilité du code, la modularité, l'encapsulation des données et la facilité de maintenance.

Classes et objets en Lua

En Lua, la POO est mise en œuvre à l'aide de tables. Une table peut être utilisée pour représenter une classe, et ses clés peuvent être utilisées pour représenter les attributs et les méthodes de la classe. Pour créer un objet, nous créons simplement une instance de la table représentant la classe.

Voici un exemple de création d'une classe en Lua :

Person = {name = "", age = 0}

function Person:new(name, age)
    local obj = {}
    setmetatable(obj, self)
    self.__index = self
    obj.name = name
    obj.age = age
    return obj
end

function Person:sayHello()
    print("Hello, my name is " .. self.name .. " and I am " .. self.age .. " years old.")
end

local person = Person:new("John", 25)
person:sayHello()

Dans cet exemple, nous créons une classe Person avec deux attributs (name et age) et une méthode (sayHello). La méthode new est utilisée pour créer une nouvelle instance de la classe, et la méthode sayHello est utilisée pour afficher un message de salutation.

Héritage et polymorphisme

En Lua, l'héritage peut être réalisé en utilisant le mécanisme de métatable. Une métatable est une table spéciale qui définit le comportement d'une autre table. En utilisant les métatables, nous pouvons définir une relation d'héritage entre deux classes.

Voici un exemple d'héritage en Lua :

Animal = {name = ""}

function Animal:new(name)
    local obj = {}
    setmetatable(obj, self)
    self.__index = self
    obj.name = name
    return obj
end

function Animal:sayHello()
    print("Hello, my name is " .. self.name)
end

Cat = Animal:new()

function Cat:sayHello()
    print("Meow, my name is " .. self.name)
end

local animal = Animal:new("Animal")
local cat = Cat:new("Whiskers")

animal:sayHello()
cat:sayHello()

Dans cet exemple, nous avons une classe Animal avec une méthode sayHello, et une classe Cat qui hérite de la classe Animal. La classe Cat redéfinit la méthode sayHello pour afficher un message spécifique aux chats. Lorsque nous créons une instance de la classe Cat, elle hérite automatiquement des attributs et des méthodes de la classe Animal.

Encapsulation et modularité

En Lua, l'encapsulation peut être réalisée en utilisant des fonctions locales et des tables locales. Les fonctions locales ne sont accessibles que dans le contexte où elles sont définies, et les tables locales ne sont accessibles que dans le fichier où elles sont définies.

Voici un exemple d'encapsulation en Lua :

local Counter = {}

function Counter:new()
    local count = 0

    local function increment()
        count = count + 1
    end

    local function decrement()
        count = count - 1
    end

    local function getCount()
        return count
    end

    return {
        increment = increment,
        decrement = decrement,
        getCount = getCount
    }
end

local counter = Counter:new()
counter:increment()
counter:increment()
counter:decrement()
print(counter.getCount())

Dans cet exemple, nous créons une classe Counter qui utilise des fonctions locales pour encapsuler la variable count. Les fonctions increment, decrement et getCount sont les seules façons d'interagir avec la variable count. Cela permet de protéger la variable count et d'éviter les accès non autorisés.
