Syntaxe de base et structure d'un programme Kotlin

La syntaxe de base de Kotlin est similaire à celle d'autres langages de programmation tels que Java. Un programme Kotlin est composé de fonctions, de classes et d'expressions.

Voici un exemple de structure de base d'un programme Kotlin :

fun main() {
    // Code principal du programme
}

Dans cet exemple, la fonction main() est la fonction principale du programme. C'est le point d'entrée de l'exécution du programme.

Types de données et variables

Kotlin prend en charge différents types de données tels que les entiers, les nombres à virgule flottante, les chaînes de caractères, les booléens, etc.

Voici quelques exemples de déclaration de variables avec des types de données différents :

val age: Int = 25
val name: String = "John Doe"
var balance: Double = 100.50

Dans cet exemple, age est une variable immuable (déclarée avec val) de type entier (Int), name est une variable immuable de type chaîne de caractères (String), et balance est une variable mutable (déclarée avec var) de type nombre à virgule flottante (Double).

Opérateurs et expressions

Kotlin prend en charge différents opérateurs tels que les opérateurs arithmétiques, les opérateurs de comparaison, les opérateurs logiques, etc.

Voici quelques exemples d'utilisation d'opérateurs en Kotlin :

val a = 10
val b = 5

val sum = a + b
val difference = a - b
val product = a * b
val quotient = a / b

val isGreater = a > b
val isEqual = a == b

val logicalAnd = (a > 0) && (b > 0)
val logicalOr = (a > 0) || (b > 0)

Dans cet exemple, nous utilisons différents opérateurs pour effectuer des opérations arithmétiques, des comparaisons et des opérations logiques.

Structures de contrôle (boucles, conditions)

Les structures de contrôle permettent de contrôler le flux d'exécution d'un programme. Kotlin prend en charge les structures de contrôle telles que les boucles et les conditions.

Voici quelques exemples d'utilisation de structures de contrôle en Kotlin :

// Boucle while
var i = 0
while (i < 10) {
    println(i)
    i++
}

// Boucle for
for (i in 1..10) {
    println(i)
}

// Condition if
val x = 5
if (x > 0) {
    println("Positive")
} else if (x < 0) {
    println("Negative")
} else {
    println("Zero")
}

Dans cet exemple, nous utilisons une boucle while pour afficher les nombres de 0 à 9, une boucle for pour afficher les nombres de 1 à 10, et une condition if pour afficher si un nombre est positif, négatif ou nul.

Fonctions et lambdas

Les fonctions sont des blocs de code réutilisables qui effectuent une tâche spécifique. Kotlin prend en charge la déclaration de fonctions et l'utilisation de fonctions anonymes appelées lambdas.

Voici un exemple de déclaration de fonction en Kotlin :

fun greet(name: String) {
    println("Hello, $name!")
}

// Appel de la fonction
greet("John")

Dans cet exemple, nous déclarons une fonction greet() qui prend un paramètre name de type chaîne de caractères. La fonction affiche un message de salutation en utilisant le nom passé en argument.

Voici un exemple d'utilisation de lambda en Kotlin :

val numbers = listOf(1, 2, 3, 4, 5)

numbers.forEach { number ->
    println(number)
}

Dans cet exemple, nous utilisons la fonction forEach() pour itérer sur les éléments d'une liste et afficher chaque élément à l'aide d'une lambda.

Gestion des exceptions

La gestion des exceptions permet de gérer les erreurs qui se produisent pendant l'exécution d'un programme. Kotlin prend en charge la gestion des exceptions à l'aide des blocs try, catch et finally.

Voici un exemple de gestion des exceptions en Kotlin :

fun divide(a: Int, b: Int): Int {
    try {
        return a / b
    } catch (e: ArithmeticException) {
        println("Division by zero is not allowed")
    } finally {
        println("Division operation completed")
    }
    
    return 0
}

// Appel de la fonction
val result = divide(10, 0)

Dans cet exemple, nous définissons une fonction divide() qui effectue une division entre deux nombres. Si une exception ArithmeticException se produit (division par zéro), nous affichons un message d'erreur. Le bloc finally est exécuté quel que soit le résultat de la division.
