Section 6 : Les fonctionnalités avancées de Kotlin

Section 6 : Les fonctionnalités avancées de Kotlin

Delegates

Les délégués sont une fonctionnalité puissante de Kotlin qui permettent de déléguer la mise en œuvre d'une propriété à une autre classe. Cela permet de réutiliser du code et de simplifier la gestion des propriétés.

Voici un exemple de délégué :

class Example {
    var property: String by Delegate()
}

class Delegate {
    operator fun getValue(thisRef: Any?, property: KProperty<*>): String {
        return "Value from delegate"
    }

    operator fun setValue(thisRef: Any?, property: KProperty<*>, value: String) {
        println("Setting value to $value")
    }
}

fun main() {
    val example = Example()
    println(example.property) // Affiche "Value from delegate"
    example.property = "New value" // Affiche "Setting value to New value"
}

Property accessors

Les accessoires de propriété permettent de personnaliser le comportement lors de la lecture et de l'écriture d'une propriété. Ils sont utilisés pour contrôler l'accès aux propriétés et effectuer des opérations supplémentaires lors de la lecture ou de l'écriture.

Voici un exemple d'accessoires de propriété :

class Example {
    var property: String = ""
        get() = field.toUpperCase()
        set(value) {
            field = value.trim()
        }
}

fun main() {
    val example = Example()
    example.property = "   Hello World   "
    println(example.property) // Affiche "HELLO WORLD"
}

Inline functions

Les fonctions en ligne sont des fonctions qui sont copiées directement à l'endroit où elles sont appelées, plutôt que d'être appelées normalement. Cela permet d'éviter les appels de fonction supplémentaires et d'améliorer les performances.

Voici un exemple de fonction en ligne :

inline fun measureTime(block: () -> Unit) {
    val startTime = System.currentTimeMillis()
    block()
    val endTime = System.currentTimeMillis()
    println("Execution time: ${endTime - startTime} ms")
}

fun main() {
    measureTime {
        // Code à mesurer
        for (i in 1..1000000) {
            println(i)
        }
    }
}

Reflection

La réflexion est une fonctionnalité qui permet d'inspecter et de manipuler des objets à l'exécution. Kotlin offre un support intégré pour la réflexion, ce qui facilite l'accès aux informations sur les classes, les propriétés et les fonctions.

Voici un exemple d'utilisation de la réflexion :

class Example {
    var property: String = "Hello"

    fun printProperty() {
        println(property)
    }
}

fun main() {
    val example = Example()
    val property = Example::class.java.getDeclaredField("property")
    property.isAccessible = true
    property.set(example, "World")
    example.printProperty() // Affiche "World"
}

Annotations

Les annotations sont des métadonnées qui peuvent être ajoutées à du code Kotlin. Elles permettent de fournir des informations supplémentaires sur les éléments du code, comme les classes, les propriétés et les fonctions.

Voici un exemple d'utilisation d'une annotation :

@Target(AnnotationTarget.CLASS)
@Retention(AnnotationRetention.RUNTIME)
annotation class MyAnnotation

@MyAnnotation
class Example

fun main() {
    val example = Example()
    val annotation = example.javaClass.getAnnotation(MyAnnotation::class.java)
    println(annotation) // Affiche "@MyAnnotation()"
}
