Les modules et la gestion des dépendances

Les modules sont une fonctionnalité clé de TypeScript qui permet d'organiser et de réutiliser du code. Ils permettent de diviser le code en plusieurs fichiers, ce qui facilite la maintenance et la collaboration entre développeurs.

Pour créer un module en TypeScript, il suffit d'utiliser le mot-clé export devant les éléments que l'on souhaite rendre accessibles depuis d'autres fichiers. Par exemple :

// fichier module.ts
export function add(a: number, b: number): number {
  return a + b;
}

export const PI = 3.14;

Dans un autre fichier, on peut ensuite importer et utiliser les éléments exportés :

// fichier main.ts
import { add, PI } from './module';

console.log(add(2, 3)); // affiche 5
console.log(PI); // affiche 3.14

Il est également possible d'importer tous les éléments exportés d'un module en utilisant l'étoile (*) :

// fichier main.ts
import * as module from './module';

console.log(module.add(2, 3)); // affiche 5
console.log(module.PI); // affiche 3.14

En ce qui concerne la gestion des dépendances, TypeScript utilise le gestionnaire de paquets npm. Pour installer une dépendance, il suffit d'exécuter la commande npm install nom-de-la-dependance. Par exemple :

npm install lodash

Une fois la dépendance installée, on peut l'importer et l'utiliser dans notre code TypeScript :

// fichier main.ts
import * as _ from 'lodash';

const numbers = [1, 2, 3, 4, 5];
const doubledNumbers = _.map(numbers, n => n * 2);

console.log(doubledNumbers); // affiche [2, 4, 6, 8, 10]

Les génériques en TypeScript

Les génériques sont une fonctionnalité puissante de TypeScript qui permet de créer des composants réutilisables et flexibles. Ils permettent de définir des types de données génériques qui peuvent être utilisés avec différents types de données spécifiques.

Pour déclarer une fonction ou une classe générique, on utilise le symbole <T> après le nom de la fonction ou de la classe. Par exemple :

function identity<T>(arg: T): T {
  return arg;
}

const result = identity<number>(5);
console.log(result); // affiche 5

Ici, la fonction identity prend un argument de type T et renvoie une valeur du même type. Lorsque l'on appelle la fonction, on spécifie le type de données souhaité entre les chevrons (<>).

Les génériques peuvent également être utilisés avec les classes :

class Box<T> {
  private value: T;

  constructor(value: T) {
    this.value = value;
  }

  getValue(): T {
    return this.value;
  }
}

const box = new Box<string>('Hello');
console.log(box.getValue()); // affiche 'Hello'

Ici, la classe Box est générique et peut être utilisée avec n'importe quel type de données. Lorsque l'on crée une instance de la classe, on spécifie le type de données souhaité entre les chevrons (<>).

Les décorateurs

Les décorateurs sont une fonctionnalité avancée de TypeScript qui permet d'ajouter des métadonnées et de modifier le comportement des classes, des méthodes, des propriétés, etc. Ils sont utilisés dans des frameworks populaires tels que Angular.

Pour utiliser un décorateur, on utilise le symbole @ suivi du nom du décorateur. Par exemple :

@log
class MyClass {
  // ...
}

function log(target: any) {
  console.log('Class logged:', target);
}

Ici, le décorateur @log est appliqué à la classe MyClass. Lorsque la classe est définie, le décorateur est exécuté et affiche le message 'Class logged:' suivi de la cible (la classe).

Les décorateurs peuvent également prendre des paramètres :

@log('debug')
class MyClass {
  // ...
}

function log(level: string) {
  return function(target: any) {
    console.log('Class logged with level', level + ':', target);
  }
}

Ici, le décorateur @log('debug') est appliqué à la classe MyClass avec le paramètre 'debug'. Lorsque la classe est définie, le décorateur est exécuté avec le paramètre et affiche le message 'Class logged with level debug:' suivi de la cible (la classe).

Les types avancés (Union types, Intersection types, Literal types, etc.)

TypeScript propose plusieurs types avancés qui permettent de définir des types de données plus complexes et précis.

Les Union types permettent de définir un type qui peut être l'un des types spécifiés. Par exemple :

function printId(id: number | string) {
  console.log('ID:', id);
}

printId(1); // affiche ID: 1
printId('abc'); // affiche ID: abc

Ici, la fonction printId peut prendre un argument de type number ou string.

Les Intersection types permettent de combiner plusieurs types en un seul. Par exemple :

type Person = {
  name: string;
  age: number;
};

type Employee = {
  company: string;
};

type EmployeePerson = Person & Employee;

const employee: EmployeePerson = {
  name: 'John Doe',
  age: 30,
  company: 'Acme Corp'
};

Ici, le type EmployeePerson est une intersection des types Person et Employee. L'objet employee doit donc avoir les propriétés définies dans les deux types.

Les Literal types permettent de définir des types qui ne peuvent prendre que des valeurs spécifiques. Par exemple :

type Direction = 'left' | 'right' | 'up' | 'down';

function move(direction: Direction) {
  console.log('Moving', direction);
}

move('left'); // affiche Moving left
move('up'); // affiche Moving up
move('diagonal'); // erreur de compilation, la valeur 'diagonal' n'est pas autorisée

Ici, le type Direction ne peut prendre que les valeurs 'left', 'right', 'up' ou 'down'.

Il existe d'autres types avancés tels que les Conditional types, les Mapped types, les Index types, etc. qui permettent de manipuler et de transformer les types de données de manière avancée.
