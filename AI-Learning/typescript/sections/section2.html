Types de données et déclarations de variables

En TypeScript, les types de données sont utilisés pour définir le type d'une variable. Cela permet au compilateur TypeScript de détecter les erreurs de type lors de la compilation du code.

Il existe plusieurs types de données primitifs en TypeScript :

  • number : représente les nombres, qu'ils soient entiers ou à virgule flottante.
  • string : représente une séquence de caractères.
  • boolean : représente une valeur booléenne, soit true soit false.
  • any : représente un type de données non spécifié, ce qui permet de contourner la vérification de type.
  • void : représente l'absence de valeur.
  • null et undefined : représentent les valeurs nulles et non définies respectivement.

La déclaration de variables en TypeScript se fait à l'aide du mot-clé let ou const. Par exemple :

let age: number = 25;
const name: string = "John Doe";
let isStudent: boolean = true;
let person: any = "John Doe"; // person peut être de n'importe quel type de données
let data: void = undefined;
let value: null = null;

Les fonctions en TypeScript

En TypeScript, les fonctions sont déclarées à l'aide du mot-clé function. Elles peuvent prendre des paramètres et retourner une valeur.

Voici un exemple de déclaration de fonction en TypeScript :

function add(a: number, b: number): number {
  return a + b;
}

let result: number = add(5, 10);
console.log(result); // Output: 15

Les paramètres de la fonction peuvent également avoir des types optionnels ou par défaut :

function greet(name: string, age?: number): void {
  console.log("Hello, " + name);
  if (age) {
    console.log("You are " + age + " years old.");
  }
}

greet("John"); // Output: Hello, John
greet("Jane", 25); // Output: Hello, Jane, You are 25 years old.

Les classes et les objets

En TypeScript, les classes sont utilisées pour créer des objets. Une classe est une structure qui définit les propriétés et les méthodes d'un objet.

Voici un exemple de déclaration de classe en TypeScript :

class Person {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  greet(): void {
    console.log("Hello, my name is " + this.name + " and I am " + this.age + " years old.");
  }
}

let person1: Person = new Person("John Doe", 25);
person1.greet(); // Output: Hello, my name is John Doe and I am 25 years old.

Les propriétés et les méthodes d'une classe peuvent être définies avec des modificateurs d'accès tels que public, private ou protected.

Les interfaces en TypeScript

Les interfaces en TypeScript sont utilisées pour définir la structure d'un objet. Elles définissent les propriétés et les méthodes que l'objet doit implémenter.

Voici un exemple de déclaration d'interface en TypeScript :

interface Shape {
  color: string;
  area(): number;
}

class Circle implements Shape {
  radius: number;

  constructor(radius: number) {
    this.radius = radius;
  }

  area(): number {
    return Math.PI * this.radius * this.radius;
  }
}

let circle: Circle = new Circle(5);
circle.color = "red";
console.log(circle.area()); // Output: 78.53981633974483

Les interfaces peuvent également être utilisées pour définir des types de fonction :

interface Greet {
  (name: string): void;
}

let greet: Greet = function(name: string): void {
  console.log("Hello, " + name);
}

greet("John"); // Output: Hello, John
