Structure d'un projet Django

Un projet Django est composé de plusieurs applications qui travaillent ensemble pour créer un site web. Chaque application est un module autonome qui peut être réutilisé dans d'autres projets Django.

Pour créer un projet Django, vous pouvez utiliser la commande suivante :

django-admin startproject nom_du_projet

Cela créera un répertoire contenant tous les fichiers nécessaires pour votre projet Django.

Voici la structure de base d'un projet Django :

nom_du_projet/
    manage.py
    nom_du_projet/
        __init__.py
        settings.py
        urls.py
        wsgi.py

Le fichier manage.py est un script qui vous permet d'interagir avec votre projet Django. Vous pouvez l'utiliser pour lancer le serveur de développement, exécuter des commandes de gestion de base de données, etc.

Le fichier settings.py contient les paramètres de configuration de votre projet Django. Vous pouvez y définir des variables telles que la base de données à utiliser, les fichiers statiques, les modèles de sécurité, etc.

Le fichier urls.py contient les URL de votre projet Django. Vous pouvez y définir les correspondances entre les URL et les vues de votre application.

Le fichier wsgi.py est utilisé pour déployer votre projet Django sur un serveur web compatible avec WSGI.

Les modèles

Les modèles sont des classes Python qui définissent la structure et le comportement des données dans votre application Django. Ils sont utilisés pour créer des tables dans la base de données et pour effectuer des opérations de lecture et d'écriture sur ces tables.

Voici un exemple de modèle Django :

from django.db import models

class Article(models.Model):
    title = models.CharField(max_length=200)
    content = models.TextField()
    pub_date = models.DateTimeField(auto_now_add=True)

Dans cet exemple, nous avons défini un modèle Article avec trois champs : title, content et pub_date. Le champ title est un champ de texte court, le champ content est un champ de texte long et le champ pub_date est un champ de date et d'heure qui est automatiquement défini lors de la création de l'article.

Une fois que vous avez défini vos modèles, vous pouvez les utiliser pour effectuer des opérations de base de données, telles que la création, la lecture, la mise à jour et la suppression d'objets. Django fournit une API ORM (Object-Relational Mapping) qui vous permet d'interagir avec la base de données en utilisant des objets Python plutôt que des requêtes SQL brutes.

Les vues

Les vues sont des fonctions ou des classes Python qui définissent la logique de traitement des requêtes HTTP dans votre application Django. Elles sont responsables de la récupération des données nécessaires à partir des modèles, de la génération de réponses HTTP et de la redirection des utilisateurs vers d'autres vues.

Voici un exemple de vue Django :

from django.shortcuts import render
from django.http import HttpResponse

def hello(request):
    return HttpResponse("Hello, world!")

Dans cet exemple, nous avons défini une vue hello qui renvoie une réponse HTTP contenant le texte "Hello, world!". Cette vue peut être associée à une URL spécifique dans le fichier urls.py de votre projet Django.

Les vues peuvent également prendre des paramètres de requête, accéder aux données de session, effectuer des opérations de base de données, etc. Elles offrent une grande flexibilité pour traiter les requêtes HTTP de manière personnalisée.

Les templates

Les templates sont des fichiers HTML qui définissent la structure et le contenu des pages web dans votre application Django. Ils sont utilisés pour générer des réponses HTTP dynamiques en insérant des données provenant des modèles et des vues.

Voici un exemple de template Django :

<h1>{{ article.title }}</h1>
<p>{{ article.content }}</p>
<p>Publié le {{ article.pub_date }}</p>

Dans cet exemple, nous avons utilisé des balises spéciales entourées de doubles accolades pour insérer les valeurs des champs du modèle Article. Lorsque ce template est rendu, les balises spéciales seront remplacées par les valeurs réelles des champs.

Les templates peuvent également inclure des boucles, des conditions, des filtres et d'autres fonctionnalités pour rendre la génération de contenu plus flexible et dynamique.

Les URL patterns

Les URL patterns sont des expressions régulières qui définissent les correspondances entre les URL et les vues dans votre application Django. Ils sont utilisés pour diriger les requêtes HTTP vers les vues appropriées en fonction de l'URL demandée.

Voici un exemple de fichier urls.py avec des URL patterns :

from django.urls import path

from . import views

urlpatterns = [
    path('hello/', views.hello),
    path('articles/', views.article_list),
    path('articles/<int:article_id>/', views.article_detail),
]

Dans cet exemple, nous avons défini trois URL patterns :

  • hello/ correspond à la vue hello que nous avons définie précédemment.
  • articles/ correspond à la vue article_list qui affiche une liste d'articles.
  • articles/<int:article_id>/ correspond à la vue article_detail qui affiche les détails d'un article spécifique en utilisant son identifiant.

Les URL patterns peuvent également prendre des paramètres de requête, des expressions régulières personnalisées et d'autres fonctionnalités pour rendre la gestion des URL plus flexible.
