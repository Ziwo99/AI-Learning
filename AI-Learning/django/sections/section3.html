Les migrations

Les migrations sont un mécanisme intégré à Django qui permet de gérer les modifications de schéma de base de données de manière incrémentielle. Elles permettent de garder la base de données synchronisée avec les modèles de l'application.

Pour créer une migration, vous pouvez utiliser la commande suivante :

python manage.py makemigrations

Cette commande va analyser les modèles de votre application et générer un fichier de migration correspondant à chaque modification détectée.

Une fois les migrations créées, vous pouvez les appliquer à la base de données en utilisant la commande :

python manage.py migrate

Cette commande va exécuter les migrations enregistrées dans la base de données, en les appliquant dans l'ordre chronologique.

Les requêtes avec Django ORM

Django ORM (Object-Relational Mapping) est une couche d'abstraction qui permet d'interagir avec la base de données en utilisant des objets Python plutôt que des requêtes SQL brutes.

Pour effectuer des requêtes avec Django ORM, vous pouvez utiliser la classe QuerySet, qui représente une requête sur une table de la base de données.

Par exemple, pour récupérer tous les objets d'un modèle, vous pouvez utiliser la méthode all() :

Model.objects.all()

Vous pouvez également filtrer les objets en utilisant la méthode filter() :

Model.objects.filter(attribute=value)

Il est également possible d'effectuer des opérations plus complexes, telles que des jointures, des regroupements ou des tris, en utilisant les méthodes appropriées de la classe QuerySet.

Les formulaires

Les formulaires sont un élément clé de la gestion des données dans Django. Ils permettent de créer des interfaces utilisateur pour saisir et valider les données avant de les enregistrer dans la base de données.

Pour créer un formulaire dans Django, vous devez définir une classe héritant de la classe Form ou ModelForm, en fonction de vos besoins.

Par exemple, pour créer un formulaire basé sur un modèle, vous pouvez utiliser la classe ModelForm :

from django import forms
from .models import MyModel

class MyModelForm(forms.ModelForm):
    class Meta:
        model = MyModel
        fields = '__all__'

Une fois le formulaire défini, vous pouvez l'utiliser dans vos vues pour afficher le formulaire à l'utilisateur, récupérer les données saisies et les valider.

Les gestionnaires d'administration

Le gestionnaire d'administration de Django est un outil puissant qui permet de gérer les données de l'application via une interface d'administration générée automatiquement.

Pour activer le gestionnaire d'administration, vous devez enregistrer vos modèles dans le fichier admin.py de votre application :

from django.contrib import admin
from .models import MyModel

admin.site.register(MyModel)

Une fois les modèles enregistrés, vous pouvez accéder à l'interface d'administration en utilisant l'URL "/admin" de votre application.

Le gestionnaire d'administration offre de nombreuses fonctionnalités, telles que la recherche, le filtrage, le tri, l'édition en masse, etc. Il est également possible de personnaliser l'interface d'administration en utilisant les classes ModelAdmin.
