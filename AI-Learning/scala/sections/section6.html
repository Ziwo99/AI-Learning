Introduction à la programmation concurrente

La programmation concurrente est une technique qui permet d'exécuter plusieurs tâches simultanément, améliorant ainsi les performances et l'efficacité d'une application. En Scala, la programmation concurrente est prise en charge par différentes fonctionnalités intégrées dans le langage.

Threads et synchronisation

Un thread est une unité d'exécution indépendante qui peut être exécutée en parallèle avec d'autres threads. En Scala, vous pouvez créer et gérer des threads en utilisant la classe Thread de la bibliothèque standard.

La synchronisation est un mécanisme qui permet de contrôler l'accès concurrentiel aux ressources partagées. En Scala, vous pouvez utiliser des blocs synchronisés et des verrous pour garantir l'exécution ordonnée des threads.

import scala.concurrent._
import ExecutionContext.Implicits.global

val future = Future {
  // Code exécuté de manière asynchrone dans un thread séparé
  // ...
}

// Attente du résultat du future
val result = Await.result(future, Duration.Inf)

Utilisation des futures et des promesses

Les futures et les promesses sont des abstractions puissantes pour la programmation concurrente en Scala. Un future représente un résultat potentiellement disponible à l'avenir, tandis qu'une promesse est un moyen de produire ce résultat.

import scala.concurrent._
import ExecutionContext.Implicits.global

val promise = Promise[String]()

val future = promise.future

future.onComplete {
  case Success(result) => println("Résultat : " + result)
  case Failure(exception) => println("Erreur : " + exception.getMessage)
}

// Résolution de la promesse avec une valeur
promise.success("Hello, World!")

Acteurs et modèles d'acteurs

Les acteurs sont des entités concurrentes qui communiquent entre elles en envoyant des messages. Chaque acteur possède son propre état interne et peut réagir aux messages qu'il reçoit.

En Scala, vous pouvez utiliser l'API d'acteurs fournie par la bibliothèque Akka pour implémenter des systèmes concurrents basés sur le modèle d'acteurs.

import akka.actor._

class MyActor extends Actor {
  def receive = {
    case message: String => println("Message reçu : " + message)
    case _ => println("Message non reconnu")
  }
}

val system = ActorSystem("MySystem")
val myActor = system.actorOf(Props[MyActor], "myActor")

myActor ! "Hello, World!"
