Mixins et traits

Les mixins et les traits sont des fonctionnalités clés de la programmation orientée objet en Scala. Un mixin est une classe qui peut être mélangée avec d'autres classes pour ajouter des fonctionnalités supplémentaires. Un trait est une unité de code réutilisable qui peut être mélangée avec des classes pour ajouter des comportements spécifiques.

Les mixins et les traits permettent une composition flexible des fonctionnalités, ce qui facilite la réutilisation du code et la création de hiérarchies de classes plus complexes.

Voici un exemple d'utilisation de mixins en Scala :

trait Logger {
  def log(message: String): Unit = {
    println(s"[${this.getClass.getSimpleName}] $message")
  }
}

class UserService extends Logger {
  def createUser(username: String, password: String): Unit = {
    // Logique de création d'utilisateur
    log(s"Utilisateur créé : $username")
  }
}

val userService = new UserService()
userService.createUser("john.doe", "password")

Dans cet exemple, la classe UserService utilise le mixin Logger pour ajouter la fonctionnalité de journalisation. La méthode log du mixin peut être appelée à partir de la classe UserService.

Pattern matching

Le pattern matching est une fonctionnalité puissante de Scala qui permet de faire correspondre des valeurs à des modèles et d'exécuter des actions en fonction de ces correspondances. Il peut être utilisé pour effectuer des opérations conditionnelles plus complexes que les simples instructions if et else.

Voici un exemple d'utilisation du pattern matching en Scala :

def matchNumber(num: Int): String = num match {
  case 0 => "Zéro"
  case 1 => "Un"
  case 2 => "Deux"
  case _ => "Autre"
}

println(matchNumber(1)) // Affiche "Un"

Dans cet exemple, la fonction matchNumber utilise le pattern matching pour faire correspondre la valeur de num à différents modèles. Si num correspond à l'un des modèles spécifiés, la correspondance est effectuée et la valeur correspondante est renvoyée.

Classes abstraites et interfaces

Les classes abstraites et les interfaces sont des concepts importants de la programmation orientée objet en Scala. Une classe abstraite est une classe qui ne peut pas être instanciée directement et qui peut contenir des méthodes abstraites, c'est-à-dire des méthodes sans implémentation. Une interface est une spécification de comportement qui peut être implémentée par des classes.

Voici un exemple d'utilisation de classes abstraites et d'interfaces en Scala :

abstract class Animal {
  def sound(): String
}

class Cat extends Animal {
  def sound(): String = "Meow"
}

trait Walks {
  def walk(): Unit = {
    println("Walking...")
  }
}

class Dog extends Animal with Walks {
  def sound(): String = "Woof"
}

val cat = new Cat()
println(cat.sound()) // Affiche "Meow"

val dog = new Dog()
println(dog.sound()) // Affiche "Woof"
dog.walk() // Affiche "Walking..."

Dans cet exemple, la classe abstraite Animal définit une méthode abstraite sound, qui doit être implémentée par les classes concrètes qui en héritent. La classe Cat et la classe Dog sont des exemples de classes concrètes qui héritent de Animal et implémentent la méthode sound.

Le trait Walks définit une méthode walk qui peut être mélangée avec d'autres classes. La classe Dog utilise le trait Walks pour ajouter la fonctionnalité de marche.

Gestion des exceptions

La gestion des exceptions est une partie importante de la programmation en Scala. Les exceptions sont utilisées pour signaler des erreurs ou des conditions exceptionnelles qui se produisent pendant l'exécution d'un programme.

Voici un exemple de gestion des exceptions en Scala :

try {
  val result = 10 / 0
  println(result)
} catch {
  case e: ArithmeticException => println("Erreur de division par zéro")
} finally {
  println("Fin de l'exécution")
}

Dans cet exemple, le code à l'intérieur du bloc try est exécuté. Si une exception se produit, le code à l'intérieur du bloc catch correspondant est exécuté. Dans ce cas, une exception ArithmeticException est attrapée et un message d'erreur est affiché. Le bloc finally est exécuté quelle que soit l'issue du bloc try et du bloc catch.

Méthodes implicites

Les méthodes implicites sont une fonctionnalité avancée de Scala qui permet d'ajouter des méthodes à des classes existantes sans les modifier. Les méthodes implicites sont utilisées pour étendre les fonctionnalités des classes sans avoir à les sous-classer ou à les modifier directement.

Voici un exemple d'utilisation de méthodes implicites en Scala :

class Circle(radius: Double) {
  def area(): Double = math.Pi * radius * radius
}

implicit class CircleExtensions(circle: Circle) {
  def circumference(): Double = 2 * math.Pi * circle.radius
}

val circle = new Circle(5)
println(circle.area()) // Affiche l'aire du cercle
println(circle.circumference()) // Affiche la circonférence du cercle

Dans cet exemple, la classe Circle représente un cercle avec un rayon donné. La méthode implicite CircleExtensions ajoute une méthode circumference à la classe Circle. Cette méthode peut être appelée directement sur une instance de Circle, même si elle n'est pas définie dans la classe elle-même.
