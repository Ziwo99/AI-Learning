Section 3 : Programmation orientée objet en Scala

Classes et objets

Les classes et les objets sont les éléments fondamentaux de la programmation orientée objet en Scala. Une classe est un modèle qui définit les propriétés et les comportements d'un objet, tandis qu'un objet est une instance spécifique d'une classe.

Voici un exemple de définition de classe en Scala :

class Person(var name: String, var age: Int) {
  def greet(): Unit = {
    println(s"Hello, my name is $name and I am $age years old.")
  }
}

val person = new Person("John", 25)
person.greet()

Dans cet exemple, nous avons défini une classe "Person" avec deux propriétés : "name" de type String et "age" de type Int. La classe a également une méthode "greet()" qui affiche un message de salutation en utilisant les propriétés de l'objet.

Héritage et polymorphisme

L'héritage permet à une classe d'hériter des propriétés et des comportements d'une autre classe. Cela permet de créer une hiérarchie de classes où les classes dérivées (ou sous-classes) héritent des caractéristiques de la classe de base (ou superclasse).

Voici un exemple d'héritage en Scala :

class Animal(var name: String) {
  def makeSound(): Unit = {
    println("The animal makes a sound.")
  }
}

class Dog(name: String) extends Animal(name) {
  override def makeSound(): Unit = {
    println("The dog barks.")
  }
}

val animal: Animal = new Dog("Buddy")
animal.makeSound()

Dans cet exemple, nous avons une classe de base "Animal" avec une méthode "makeSound()". Nous avons ensuite une classe dérivée "Dog" qui hérite de la classe "Animal" et redéfinit la méthode "makeSound()". Lorsque nous créons un objet de type "Dog" et l'assignons à une variable de type "Animal", nous pouvons toujours appeler la méthode "makeSound()", mais elle sera exécutée selon l'implémentation de la classe dérivée.

Traits et mixins

Les traits sont des unités de code réutilisables en Scala. Ils peuvent être utilisés pour définir des comportements communs qui peuvent être mélangés (mixins) dans des classes. Les mixins permettent d'ajouter des fonctionnalités supplémentaires à une classe sans avoir à hériter d'une autre classe.

Voici un exemple d'utilisation de traits en Scala :

trait Printable {
  def print(): Unit
}

class Document extends Printable {
  override def print(): Unit = {
    println("Printing the document...")
  }
}

val document = new Document()
document.print()

Dans cet exemple, nous avons défini un trait "Printable" avec une méthode "print()". Ensuite, nous avons une classe "Document" qui étend le trait "Printable" et implémente la méthode "print()". Ainsi, la classe "Document" peut utiliser le comportement défini dans le trait "Printable".

Gestion des exceptions

La gestion des exceptions permet de gérer les erreurs et les situations exceptionnelles qui peuvent se produire pendant l'exécution d'un programme. En Scala, les exceptions sont gérées à l'aide des blocs "try-catch" et "try-finally".

Voici un exemple de gestion des exceptions en Scala :

try {
  val result = 10 / 0
  println(result)
} catch {
  case e: ArithmeticException => println("Division by zero error.")
} finally {
  println("Finally block executed.")
}

Dans cet exemple, nous essayons de diviser 10 par 0, ce qui provoque une exception "ArithmeticException". Le bloc "catch" capture cette exception et affiche un message d'erreur approprié. Le bloc "finally" est exécuté quel que soit le résultat de l'opération, ce qui permet de nettoyer les ressources ou d'exécuter des actions finales.
