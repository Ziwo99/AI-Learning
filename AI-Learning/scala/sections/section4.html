Fonctions de première classe et fonctions anonymes en Scala

En Scala, les fonctions sont des citoyens de première classe, ce qui signifie qu'elles peuvent être traitées comme des valeurs. Cela permet de les passer en tant que paramètres, de les assigner à des variables et de les retourner en tant que résultats d'autres fonctions.

Les fonctions anonymes, également appelées lambdas, sont des fonctions sans nom qui peuvent être définies directement dans le code sans avoir besoin de les nommer explicitement. Elles sont souvent utilisées pour des opérations simples ou des transformations de données.

Voici un exemple de déclaration d'une fonction de première classe et d'une fonction anonyme en Scala :

val addition: (Int, Int) => Int = (a, b) => a + b

val resultat = addition(3, 5) // Le résultat sera 8

Fonctions d'ordre supérieur en Scala

Les fonctions d'ordre supérieur sont des fonctions qui prennent d'autres fonctions en tant que paramètres ou qui retournent des fonctions en tant que résultats. Elles permettent de créer des abstractions plus puissantes et de composer des fonctions de manière flexible.

Voici un exemple d'utilisation d'une fonction d'ordre supérieur en Scala :

def appliquerOperation(a: Int, b: Int, operation: (Int, Int) => Int): Int = {
  operation(a, b)
}

val addition: (Int, Int) => Int = (a, b) => a + b

val resultat = appliquerOperation(3, 5, addition) // Le résultat sera 8

Immutabilité et immutabilité en Scala

En Scala, les variables peuvent être déclarées comme immutables en utilisant le mot-clé val ou comme mutables en utilisant le mot-clé var. L'immutabilité est encouragée en Scala car elle favorise la programmation fonctionnelle et facilite la gestion des états.

Les objets immutables ne peuvent pas être modifiés une fois qu'ils ont été créés. Au lieu de cela, une nouvelle instance est créée avec les modifications souhaitées. Cela garantit que les données restent cohérentes et facilite le développement de code parallèle et concurrent.

Voici un exemple d'utilisation de variables immutables en Scala :

val x = 5
// x ne peut pas être modifié

var y = 10
// y peut être modifié

y = y + 1 // y est maintenant égal à 11

Collections et manipulation de données en Scala

Scala offre une large gamme de collections pour manipuler des données de manière fonctionnelle. Les collections peuvent être immutables ou mutables, selon les besoins.

Voici quelques exemples de collections couramment utilisées en Scala :

  • Liste (List) : une séquence ordonnée d'éléments, où chaque élément peut apparaître plusieurs fois.
  • Ensemble (Set) : une collection d'éléments uniques, sans ordre spécifique.
  • Map (Map) : une collection d'associations clé-valeur, où chaque clé est unique.

Les collections en Scala offrent de nombreuses opérations de manipulation de données, telles que le filtrage, le tri, la transformation, la réduction, etc. Ces opérations peuvent être effectuées de manière fonctionnelle en utilisant des fonctions d'ordre supérieur.

Voici un exemple d'utilisation de la collection List en Scala :

val nombres = List(1, 2, 3, 4, 5)

val carres = nombres.map(x => x * x) // Le résultat sera List(1, 4, 9, 16, 25)

val somme = nombres.reduce((a, b) => a + b) // Le résultat sera 15
