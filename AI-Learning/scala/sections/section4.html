Section 4: Programmation fonctionnelle en Scala

Fonctions de première classe et fonctions d'ordre supérieur en Scala

En Scala, les fonctions sont des valeurs de première classe, ce qui signifie qu'elles peuvent être assignées à des variables, passées en tant que paramètres à d'autres fonctions et retournées en tant que résultats d'autres fonctions. Les fonctions de première classe permettent une programmation plus expressive et flexible.

Les fonctions d'ordre supérieur sont des fonctions qui prennent d'autres fonctions en tant que paramètres ou retournent des fonctions en tant que résultats. Elles permettent de créer des abstractions puissantes et de manipuler des fonctions de manière générique.

Voici un exemple de fonction de première classe en Scala :

val doubler: Int => Int = (x: Int) => x * 2

Dans cet exemple, la variable "doubler" est assignée à une fonction anonyme qui prend un entier en paramètre et retourne le double de cet entier.

Voici un exemple de fonction d'ordre supérieur en Scala :

def appliquerDeuxFois(f: Int => Int, x: Int): Int = f(f(x))

val doubler: Int => Int = (x: Int) => x * 2

val resultat = appliquerDeuxFois(doubler, 5) // Le résultat est 20

Dans cet exemple, la fonction "appliquerDeuxFois" prend une fonction "f" en paramètre, ainsi qu'un entier "x". Elle applique la fonction "f" deux fois sur l'entier "x" et retourne le résultat. La fonction "doubler" est ensuite passée en tant que paramètre à la fonction "appliquerDeuxFois", ce qui donne un résultat de 20.

Immutabilité et immutabilité en Scala

En Scala, les variables peuvent être déclarées comme immuables en utilisant le mot-clé "val" ou comme mutables en utilisant le mot-clé "var". Les variables immuables ne peuvent pas être réassignées une fois qu'elles ont été initialisées, ce qui favorise la programmation fonctionnelle et rend le code plus sûr et prévisible.

Voici un exemple de variable immuable en Scala :

val x: Int = 5
x = 10 // Erreur de compilation : réassignation d'une variable immuable

Dans cet exemple, la variable "x" est déclarée comme immuable avec le mot-clé "val". Une tentative de réassignation de la variable "x" entraînera une erreur de compilation.

En revanche, les variables mutables peuvent être réassignées après leur initialisation en utilisant le mot-clé "var". Bien que les variables mutables puissent être pratiques dans certains cas, il est recommandé d'utiliser des variables immuables autant que possible pour éviter les effets de bord et rendre le code plus prévisible.

Fonctions récursives en Scala

En Scala, les fonctions récursives sont des fonctions qui s'appellent elles-mêmes pendant leur exécution. La récursion est un concept clé en programmation fonctionnelle et permet de résoudre des problèmes de manière élégante et concise.

Voici un exemple de fonction récursive en Scala qui calcule la factorielle d'un nombre :

def factorielle(n: Int): Int = {
  if (n == 0) 1
  else n * factorielle(n - 1)
}

val resultat = factorielle(5) // Le résultat est 120

Dans cet exemple, la fonction "factorielle" s'appelle elle-même avec un argument réduit à chaque itération jusqu'à ce que la condition de base (n == 0) soit atteinte. La factorielle de 5 est calculée en multipliant 5 par la factorielle de 4, puis par la factorielle de 3, et ainsi de suite, jusqu'à ce que la factorielle de 0 soit atteinte, qui est égale à 1.

Manipulation de collections en Scala

Scala offre de puissantes fonctionnalités pour manipuler des collections de données, telles que les listes, les tableaux et les ensembles. Les opérations de manipulation de collections en Scala sont basées sur des fonctions d'ordre supérieur, ce qui permet d'écrire du code concis et expressif.

Voici quelques exemples d'opérations de manipulation de collections en Scala :

val nombres = List(1, 2, 3, 4, 5)

// Filtrer les nombres pairs
val nombresPairs = nombres.filter(_ % 2 == 0) // Le résultat est List(2, 4)

// Transformer les nombres en leur carré
val carres = nombres.map(x => x * x) // Le résultat est List(1, 4, 9, 16, 25)

// Réduire les nombres en les additionnant
val somme = nombres.reduce(_ + _) // Le résultat est 15

Dans ces exemples, la méthode "filter" est utilisée pour filtrer les éléments d'une liste en fonction d'un prédicat, la méthode "map" est utilisée pour appliquer une transformation à chaque élément d'une liste, et la méthode "reduce" est utilisée pour réduire une liste à une seule valeur en utilisant une opération spécifiée.

Scala offre également d'autres opérations de manipulation de collections telles que "flatMap", "fold", "groupBy" et bien d'autres, qui permettent de résoudre une grande variété de problèmes de manière élégante et efficace.
