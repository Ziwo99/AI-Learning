Section 4 : Programmation fonctionnelle en Scala

Section 4: Programmation fonctionnelle en Scala

Fonctions de haut niveau

Les fonctions de haut niveau en Scala sont des fonctions qui peuvent être passées en tant que paramètres à d'autres fonctions ou retournées en tant que résultats. Elles sont considérées comme des valeurs de première classe. Cela signifie que les fonctions peuvent être traitées de la même manière que les autres types de données.

Voici un exemple de fonction de haut niveau en Scala :

def applyOperation(x: Int, y: Int, operation: (Int, Int) => Int): Int = {
  operation(x, y)
}

def add(x: Int, y: Int): Int = {
  x + y
}

val result = applyOperation(5, 3, add)
println(result) // Output: 8

Fonctions anonymes et lambdas

Les fonctions anonymes en Scala sont des fonctions qui n'ont pas de nom. Elles sont également appelées lambdas. Les fonctions anonymes peuvent être utilisées pour simplifier le code en évitant la nécessité de définir une fonction séparée.

Voici un exemple d'utilisation d'une fonction anonyme en Scala :

val numbers = List(1, 2, 3, 4, 5)

val squaredNumbers = numbers.map(x => x * x)
println(squaredNumbers) // Output: List(1, 4, 9, 16, 25)

Fonctions récursives

Les fonctions récursives en Scala sont des fonctions qui s'appellent elles-mêmes. Elles sont souvent utilisées pour résoudre des problèmes qui peuvent être décomposés en sous-problèmes plus petits.

Voici un exemple de fonction récursive en Scala pour calculer le factoriel d'un nombre :

def factorial(n: Int): Int = {
  if (n == 0) 1
  else n * factorial(n - 1)
}

val result = factorial(5)
println(result) // Output: 120

Immutabilité et immutabilité partielle

L'immutabilité en Scala signifie que les objets ne peuvent pas être modifiés une fois qu'ils ont été créés. Cela permet d'éviter les effets de bord indésirables et facilite la programmation concurrente.

Voici un exemple d'utilisation de l'immutabilité en Scala :

val list = List(1, 2, 3, 4, 5)
val newList = list.map(x => x * 2)
println(newList) // Output: List(2, 4, 6, 8, 10)

L'immutabilité partielle en Scala signifie que certains éléments d'un objet peuvent être modifiés, tandis que d'autres restent immuables. Cela permet de combiner les avantages de l'immutabilité avec la flexibilité de la modification sélective.

Fonctions d'ordre supérieur

Les fonctions d'ordre supérieur en Scala sont des fonctions qui prennent d'autres fonctions en tant que paramètres ou retournent des fonctions en tant que résultats. Elles permettent de créer des fonctions génériques et réutilisables.

Voici un exemple d'utilisation d'une fonction d'ordre supérieur en Scala :

def applyTwice(f: Int => Int, x: Int): Int = {
  f(f(x))
}

def increment(x: Int): Int = {
  x + 1
}

val result = applyTwice(increment, 5)
println(result) // Output: 7
