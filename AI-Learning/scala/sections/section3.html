Section 3: Programmation orientée objet en Scala

Classes et objets en Scala

En Scala, la programmation orientée objet est au cœur du langage. Les classes et les objets sont les éléments fondamentaux de la programmation orientée objet en Scala.

Une classe est un modèle ou un plan pour créer des objets. Elle définit les propriétés et les comportements communs à un ensemble d'objets. Pour créer une classe en Scala, utilisez le mot-clé class suivi du nom de la classe.

Voici un exemple de déclaration de classe en Scala :

class Personne(nom: String, age: Int) {
  def afficherDetails(): Unit = {
    println("Nom: " + nom)
    println("Age: " + age)
  }
}

Dans cet exemple, nous avons défini une classe Personne avec deux propriétés : nom de type String et age de type Int. La classe a également une méthode afficherDetails qui affiche les détails de la personne.

Pour créer un objet à partir d'une classe en Scala, utilisez le mot-clé new suivi du nom de la classe et des arguments appropriés :

val personne1 = new Personne("Alice", 25)
personne1.afficherDetails()

Ce code crée un nouvel objet Personne avec le nom "Alice" et l'âge 25, puis appelle la méthode afficherDetails pour afficher les détails de la personne.

Héritage et polymorphisme en Scala

En Scala, vous pouvez utiliser l'héritage pour créer des classes dérivées à partir d'une classe de base existante. La classe dérivée hérite des propriétés et des comportements de la classe de base.

Pour définir une classe dérivée en Scala, utilisez le mot-clé extends suivi du nom de la classe de base :

class Etudiant(nom: String, age: Int, numeroEtudiant: String) extends Personne(nom, age) {
  def afficherNumeroEtudiant(): Unit = {
    println("Numéro étudiant: " + numeroEtudiant)
  }
}

Dans cet exemple, nous avons défini une classe Etudiant qui hérite de la classe Personne. La classe Etudiant a une propriété supplémentaire numeroEtudiant et une méthode afficherNumeroEtudiant qui affiche le numéro d'étudiant.

Le polymorphisme est également pris en charge en Scala. Cela signifie que vous pouvez utiliser une instance d'une classe dérivée là où une instance de la classe de base est attendue.

Voici un exemple :

val personne2: Personne = new Etudiant("Bob", 20, "12345")
personne2.afficherDetails()

Dans cet exemple, nous avons créé un objet Etudiant et l'avons assigné à une variable de type Personne. Cela est possible car la classe Etudiant hérite de la classe Personne. Nous pouvons ensuite appeler la méthode afficherDetails sur cet objet, qui est en réalité une instance de la classe Etudiant.

Traits et mixins en Scala

En plus de l'héritage, Scala prend en charge les traits, qui sont des unités de code réutilisables qui peuvent être mélangées dans des classes pour ajouter des fonctionnalités supplémentaires.

Pour définir un trait en Scala, utilisez le mot-clé trait suivi du nom du trait :

trait Loggable {
  def log(message: String): Unit = {
    println("Log: " + message)
  }
}

Dans cet exemple, nous avons défini un trait Loggable avec une méthode log qui affiche un message de journalisation.

Pour mélanger un trait dans une classe en Scala, utilisez le mot-clé with :

class Service extends Loggable {
  def doSomething(): Unit = {
    log("Doing something...")
  }
}

Dans cet exemple, la classe Service mélange le trait Loggable en utilisant le mot-clé with. Cela signifie que la classe Service a maintenant accès à la méthode log définie dans le trait Loggable.

Gestion des exceptions en Scala

En Scala, vous pouvez gérer les exceptions à l'aide des blocs try, catch et finally.

Voici un exemple :

try {
  // Code susceptible de générer une exception
  val resultat = 10 / 0
} catch {
  case e: ArithmeticException => println("Erreur de division par zéro")
} finally {
  // Code exécuté toujours, qu'il y ait eu une exception ou non
  println("Fin du bloc try-catch")
}

Dans cet exemple, nous essayons de diviser 10 par 0, ce qui génère une exception ArithmeticException. Le bloc catch capture cette exception et affiche un message d'erreur approprié. Le bloc finally est exécuté après le bloc try et le bloc catch, qu'il y ait eu une exception ou non.

Il est également possible de lancer des exceptions personnalisées en utilisant le mot-clé throw. Voici un exemple :

def diviser(a: Int, b: Int): Int = {
  if (b == 0) {
    throw new IllegalArgumentException("Division par zéro")
  } else {
    a / b
  }
}

Dans cet exemple, nous avons défini une fonction diviser qui prend deux entiers en argument. Si le deuxième argument est zéro, nous lançons une exception IllegalArgumentException avec un message d'erreur personnalisé.
