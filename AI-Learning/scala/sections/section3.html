Classes et objets en Scala

La programmation orientée objet (POO) est un paradigme de programmation largement utilisé dans le développement logiciel. Scala est un langage de programmation qui prend en charge pleinement la POO. En Scala, vous pouvez créer des classes et des objets pour modéliser des entités du monde réel et définir leur comportement à l'aide de méthodes et de propriétés.

Pour créer une classe en Scala, utilisez le mot-clé class suivi du nom de la classe et des parenthèses vides. Par exemple, voici comment vous pouvez définir une classe Person avec une propriété name : 

class Person {
  var name: String = ""
}

Dans cet exemple, nous avons déclaré une propriété name de type String dans la classe Person. La propriété est initialisée avec une valeur vide.

Pour créer un objet à partir d'une classe en Scala, utilisez le mot-clé new suivi du nom de la classe et des parenthèses vides. Par exemple, voici comment vous pouvez créer un objet person à partir de la classe Person : 

val person = new Person()

Dans cet exemple, nous avons créé un nouvel objet person à partir de la classe Person. L'objet a accès à toutes les propriétés et méthodes définies dans la classe.

Héritage et polymorphisme en Scala

L'héritage est un concept clé de la POO qui permet à une classe d'hériter des propriétés et des méthodes d'une autre classe. Scala prend en charge l'héritage simple, ce qui signifie qu'une classe peut hériter d'une seule autre classe.

Pour définir une classe qui hérite d'une autre classe en Scala, utilisez le mot-clé extends. Par exemple, voici comment vous pouvez définir une classe Employee qui hérite de la classe Person : 

class Employee extends Person {
  var salary: Double = 0.0
}

Dans cet exemple, la classe Employee hérite de la classe Person et ajoute une nouvelle propriété salary de type Double.

Le polymorphisme est un autre concept clé de la POO qui permet à une classe de se comporter comme une autre classe de la même hiérarchie d'héritage. En Scala, vous pouvez utiliser le polymorphisme en définissant des méthodes avec le même nom dans différentes classes de la hiérarchie d'héritage.

Par exemple, supposons que nous ayons une classe Shape avec une méthode area qui calcule l'aire de la forme. Nous pouvons ensuite définir des sous-classes telles que Rectangle et Circle qui héritent de la classe Shape et redéfinissent la méthode area pour calculer l'aire spécifique à chaque forme. 

class Shape {
  def area(): Double = 0.0
}

class Rectangle(width: Double, height: Double) extends Shape {
  override def area(): Double = width * height
}

class Circle(radius: Double) extends Shape {
  override def area(): Double = math.Pi * radius * radius
}

Dans cet exemple, nous avons défini une méthode area dans la classe Shape qui renvoie 0.0 par défaut. Les sous-classes Rectangle et Circle redéfinissent la méthode area pour calculer l'aire spécifique à chaque forme.

Traits et mixins en Scala

Les traits sont une fonctionnalité puissante de Scala qui permettent de définir des comportements réutilisables et de les ajouter à des classes. Un trait est similaire à une interface en Java, mais il peut également contenir des implémentations de méthodes.

Pour définir un trait en Scala, utilisez le mot-clé trait. Par exemple, voici comment vous pouvez définir un trait Logger avec une méthode log : 

trait Logger {
  def log(message: String): Unit = {
    println(s"[LOG] $message")
  }
}

Dans cet exemple, nous avons défini un trait Logger avec une méthode log qui imprime un message de journalisation préfixé par "[LOG]".

Pour ajouter un trait à une classe en Scala, utilisez le mot-clé with. Par exemple, voici comment vous pouvez ajouter le trait Logger à la classe Person : 

class Person extends Logger {
  var name: String = ""
}

Dans cet exemple, la classe Person hérite du trait Logger et a maintenant accès à la méthode log.

Gestion des exceptions en Scala

La gestion des exceptions est une partie importante de la programmation pour gérer les erreurs et les situations exceptionnelles. En Scala, vous pouvez utiliser des blocs try et catch pour gérer les exceptions.

Voici comment vous pouvez utiliser un bloc try et catch pour gérer une exception en Scala : 

try {
  // Code susceptible de générer une exception
} catch {
  case ex: Exception =>
    // Gestion de l'exception
}

Dans cet exemple, le code à l'intérieur du bloc try est susceptible de générer une exception. Si une exception est levée, elle est capturée par le bloc catch et vous pouvez y gérer l'exception en utilisant le motif case ex: Exception.

Vous pouvez également utiliser des blocs finally pour exécuter du code qui doit être exécuté, qu'une exception soit levée ou non. Par exemple : 

try {
  // Code susceptible de générer une exception
} catch {
  case ex: Exception =>
    // Gestion de l'exception
} finally {
  // Code à exécuter toujours
}

Dans cet exemple, le code à l'intérieur du bloc finally est exécuté après le bloc try et le bloc catch, qu'une exception soit levée ou non.
