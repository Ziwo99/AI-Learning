Types avancés (type paramétré, type supérieur, type inférieur)

Les types avancés en Scala permettent de définir des contraintes supplémentaires sur les types de données utilisés dans un programme. Ils offrent une flexibilité et une expressivité accrues lors de la manipulation des types.

Le type paramétré permet de définir une classe ou une fonction qui peut être utilisée avec différents types de données. Par exemple, on peut définir une classe générique de liste :

class Liste[T] {
  // implémentation de la liste
}

Le type supérieur permet de spécifier que le type utilisé doit être une sous-classe d'un certain type. Par exemple, on peut définir une fonction qui accepte uniquement des objets de type Animal ou de ses sous-classes :

def afficherAnimal[A >: Animal](animal: A): Unit = {
  println(animal)
}

Le type inférieur permet de spécifier que le type utilisé doit être une super-classe d'un certain type. Par exemple, on peut définir une fonction qui accepte uniquement des objets de type Chien ou de ses super-classes :

def afficherChien[A <: Chien](chien: A): Unit = {
  println(chien)
}

Macros

Les macros en Scala permettent de générer du code à la compilation. Elles offrent une puissante fonctionnalité de méta-programmation, permettant de transformer et de générer du code source Scala à partir d'un code source Scala existant.

Les macros sont définies à l'aide de la macro-annotation @macro et peuvent être utilisées pour effectuer des opérations complexes à la compilation, telles que la génération de code répétitif, la vérification statique des types ou la transformation de code source.

import scala.language.experimental.macros
import scala.reflect.macros.blackbox.Context

object Macros {
  def hello: Unit = macro helloImpl

  def helloImpl(c: Context): c.Expr[Unit] = {
    import c.universe._
    c.Expr(q"""println("Hello, World!")""")
  }
}

Dans cet exemple, la macro hello génère du code qui imprime "Hello, World!" à la compilation.

Réflexion et introspection

La réflexion en Scala permet d'inspecter et de manipuler les types et les valeurs à l'exécution. Elle offre une puissante fonctionnalité de méta-programmation, permettant de créer des programmes qui peuvent s'adapter dynamiquement à différentes situations.

La réflexion peut être utilisée pour obtenir des informations sur les classes, les méthodes, les champs et les annotations, ainsi que pour créer de nouvelles instances de classes, appeler des méthodes et accéder aux champs.

import scala.reflect.runtime.universe._

val mirror = runtimeMirror(getClass.getClassLoader)
val classSymbol = mirror.staticClass("com.example.MyClass")
val classMirror = mirror.reflectClass(classSymbol)
val constructor = classSymbol.primaryConstructor.asMethod
val constructorMirror = classMirror.reflectConstructor(constructor)
val instance = constructorMirror() // création d'une nouvelle instance de MyClass

Dans cet exemple, la réflexion est utilisée pour obtenir le symbole de classe de com.example.MyClass, puis pour créer une nouvelle instance de cette classe en appelant son constructeur principal.

Interopérabilité avec Java

Scala est compatible avec Java, ce qui signifie que vous pouvez utiliser des bibliothèques Java existantes dans vos programmes Scala et vice versa. Scala offre une syntaxe concise et expressive pour interagir avec du code Java, ce qui facilite l'interopérabilité entre les deux langages.

Pour utiliser une classe Java dans Scala, il suffit d'importer la classe et de l'utiliser comme n'importe quelle autre classe Scala :

import com.example.MyJavaClass

val javaObject = new MyJavaClass()
javaObject.doSomething()

De même, pour utiliser une classe Scala dans Java, il suffit de l'importer et de l'utiliser comme n'importe quelle autre classe Java :

import com.example.MyScalaClass;

MyScalaClass scalaObject = new MyScalaClass();
scalaObject.doSomething();

Scala offre également des fonctionnalités avancées d'interopérabilité avec Java, telles que la conversion automatique des types, la prise en charge des annotations Java et la possibilité d'étendre des classes Java dans Scala.
