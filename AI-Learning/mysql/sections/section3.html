Introduction aux requêtes SQL

Les requêtes SQL (Structured Query Language) sont utilisées pour interagir avec une base de données relationnelle. Elles permettent de récupérer, modifier, supprimer ou insérer des données dans une base de données. SQL est un langage standardisé et largement utilisé dans l'industrie pour gérer les données.

Sélection des données avec SELECT

La clause SELECT est utilisée pour récupérer des données spécifiques à partir d'une table ou d'une vue dans une base de données. Elle permet de spécifier les colonnes à récupérer et les critères de filtrage des données. Voici un exemple de requête SELECT pour récupérer toutes les colonnes d'une table "users" : 

SELECT * FROM users;

Cette requête récupérera toutes les lignes de la table "users" et affichera toutes les colonnes.

Filtrage des données avec WHERE

La clause WHERE est utilisée pour filtrer les données lors de l'exécution d'une requête SELECT. Elle permet de spécifier des conditions pour sélectionner uniquement les lignes qui répondent à ces conditions. Voici un exemple de requête SELECT avec une clause WHERE pour récupérer les utilisateurs dont l'âge est supérieur à 18 ans : 

SELECT * FROM users WHERE age > 18;

Cette requête récupérera toutes les lignes de la table "users" où la colonne "age" est supérieure à 18.

Tri des données avec ORDER BY

La clause ORDER BY est utilisée pour trier les résultats d'une requête SELECT selon une ou plusieurs colonnes. Elle peut trier les données par ordre croissant (ASC) ou décroissant (DESC). Voici un exemple de requête SELECT avec une clause ORDER BY pour trier les utilisateurs par ordre alphabétique de leur nom : 

SELECT * FROM users ORDER BY name ASC;

Cette requête récupérera toutes les lignes de la table "users" et les triera par ordre alphabétique croissant du nom.

Jointures de tables

Les jointures de tables permettent de combiner les données de plusieurs tables en fonction de leurs relations. Elles sont utilisées pour récupérer des données qui sont réparties sur plusieurs tables. Voici un exemple de requête SELECT avec une jointure de tables pour récupérer les utilisateurs et leurs commandes associées : 

SELECT users.name, orders.order_date
FROM users
JOIN orders ON users.id = orders.user_id;

Cette requête récupérera les noms des utilisateurs et les dates de leurs commandes en combinant les données des tables "users" et "orders" en fonction de la correspondance entre les colonnes "id" de la table "users" et "user_id" de la table "orders".

Agrégation des données avec GROUP BY

La clause GROUP BY est utilisée pour regrouper les données en fonction de certaines colonnes et d'effectuer des opérations d'agrégation sur ces groupes de données. Elle est souvent utilisée avec des fonctions d'agrégation telles que COUNT, SUM, AVG, etc. Voici un exemple de requête SELECT avec une clause GROUP BY pour compter le nombre de commandes par utilisateur : 

SELECT users.name, COUNT(orders.id) as total_orders
FROM users
JOIN orders ON users.id = orders.user_id
GROUP BY users.name;

Cette requête récupérera les noms des utilisateurs et le nombre total de leurs commandes en utilisant la fonction COUNT pour compter les identifiants des commandes, puis en regroupant les données par nom d'utilisateur.

Manipulation des données avec INSERT, UPDATE et DELETE

Les requêtes INSERT, UPDATE et DELETE sont utilisées pour manipuler les données dans une base de données. INSERT est utilisée pour insérer de nouvelles lignes dans une table, UPDATE est utilisée pour mettre à jour des lignes existantes et DELETE est utilisée pour supprimer des lignes. Voici des exemples de requêtes INSERT, UPDATE et DELETE : 

INSERT INTO users (name, age) VALUES ('John Doe', 25);

UPDATE users SET age = 30 WHERE name = 'John Doe';

DELETE FROM users WHERE name = 'John Doe';

La première requête insérera une nouvelle ligne dans la table "users" avec le nom "John Doe" et l'âge de 25 ans. La deuxième requête mettra à jour l'âge de l'utilisateur "John Doe" à 30 ans. La troisième requête supprimera toutes les lignes de la table "users" où le nom est "John Doe".
