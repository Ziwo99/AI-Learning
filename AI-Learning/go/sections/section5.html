Gestion des erreurs avec les erreurs prédéfinies

En Go, la gestion des erreurs est une partie essentielle du processus de développement. Les erreurs peuvent survenir à différents niveaux de l'exécution d'un programme et il est important de les détecter et de les gérer correctement.

Go fournit des erreurs prédéfinies qui peuvent être utilisées pour signaler et gérer les erreurs. Ces erreurs sont représentées par le type d'interface "error" qui est défini dans le package "errors". Pour créer une nouvelle erreur, vous pouvez utiliser la fonction "errors.New()" en passant une chaîne de caractères décrivant l'erreur. 

package main

import (
    "errors"
    "fmt"
)

func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("division by zero")
    }
    return a / b, nil
}

func main() {
    result, err := divide(10, 0)
    if err != nil {
        fmt.Println("Error:", err)
        return
    }
    fmt.Println("Result:", result)
}

Dans cet exemple, nous avons une fonction "divide" qui effectue une division entre deux nombres. Si le diviseur est égal à zéro, nous retournons une nouvelle erreur en utilisant la fonction "errors.New()". Dans la fonction "main", nous appelons la fonction "divide" avec un diviseur égal à zéro, ce qui génère une erreur. Nous vérifions ensuite si une erreur s'est produite en utilisant la syntaxe "if err != nil" et affichons l'erreur à l'écran.

Utilisation des paniques et des récupérations

En plus de la gestion des erreurs, Go offre également la possibilité de déclencher des paniques et de les récupérer. Une panique est une situation exceptionnelle qui se produit lorsqu'une erreur critique se produit et que le programme ne peut pas continuer à s'exécuter normalement.

Pour déclencher une panique, vous pouvez utiliser la fonction "panic()" en passant une valeur qui décrit la panique. Une fois qu'une panique est déclenchée, l'exécution normale du programme est arrêtée et le programme commence à remonter la pile d'appels pour trouver un point de récupération. 

package main

import "fmt"

func main() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("Recovered:", r)
        }
    }()

    panic("Something went wrong!")
}

Dans cet exemple, nous utilisons la fonction "panic()" pour déclencher une panique avec le message "Something went wrong!". Nous utilisons également la fonction "recover()" dans une fonction différée pour récupérer la panique. La fonction "recover()" renvoie la valeur passée à "panic()" si une panique a été déclenchée, sinon elle renvoie "nil". Dans notre cas, nous affichons simplement le message de récupération à l'écran.

Il est important de noter que les paniques ne doivent être utilisées que dans des situations exceptionnelles où le programme ne peut pas continuer à s'exécuter normalement. Dans la plupart des cas, il est préférable d'utiliser la gestion des erreurs pour gérer les situations d'erreur prévisibles.
