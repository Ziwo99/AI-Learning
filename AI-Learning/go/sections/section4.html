Concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. En Go, les objets sont créés à l'aide de structures (structs) et les méthodes sont attachées à ces structures pour leur donner des comportements spécifiques.

Les concepts clés de la POO en Go sont :

  • Encapsulation : les données et les méthodes associées sont regroupées dans une structure pour former un objet.
  • Héritage : les structures peuvent être imbriquées les unes dans les autres pour créer des relations de parenté.
  • Polymorphisme : les interfaces permettent de définir des comportements communs à plusieurs structures différentes.

Définition de structures et méthodes

En Go, une structure est définie à l'aide du mot-clé type suivi du nom de la structure et de ses champs. Par exemple :

type Person struct {
  Name string
  Age  int
}

Les méthodes sont définies en associant une fonction à une structure. Par exemple :

func (p *Person) SayHello() {
  fmt.Println("Hello, my name is", p.Name)
}

Dans cet exemple, la méthode SayHello() est attachée à la structure Person et peut être appelée sur une instance de cette structure.

Interfaces et polymorphisme

Les interfaces en Go permettent de définir des comportements communs à plusieurs structures différentes. Une interface est définie en spécifiant les méthodes qu'une structure doit implémenter pour être considérée comme satisfaisant cette interface.

Par exemple, nous pouvons définir une interface Animal avec une méthode MakeSound() :

type Animal interface {
  MakeSound()
}

Une structure peut implémenter cette interface en définissant la méthode MakeSound(). Par exemple :

type Dog struct {
  Name string
}

func (d Dog) MakeSound() {
  fmt.Println(d.Name, "says woof!")
}

En utilisant cette interface, nous pouvons traiter une instance de Dog comme un Animal :

func main() {
  var animal Animal
  animal = Dog{Name: "Fido"}
  animal.MakeSound() // Output: Fido says woof!
}

Gestion des erreurs avec les interfaces

En Go, la gestion des erreurs est souvent réalisée à l'aide d'interfaces. Une interface error est définie dans la bibliothèque standard de Go et peut être utilisée pour représenter une erreur.

Les fonctions peuvent retourner une valeur de type error pour indiquer qu'une erreur s'est produite. Par exemple :

func Divide(x, y float64) (float64, error) {
  if y == 0 {
    return 0, errors.New("division by zero")
  }
  return x / y, nil
}

Lors de l'appel de cette fonction, nous pouvons vérifier si une erreur s'est produite en comparant la valeur de retour à nil. Par exemple :

result, err := Divide(10, 0)
if err != nil {
  fmt.Println("An error occurred:", err)
} else {
  fmt.Println("Result:", result)
}

En utilisant cette approche, les erreurs peuvent être gérées de manière centralisée et les fonctions peuvent être chaînées pour effectuer des opérations supplémentaires en cas d'erreur.
