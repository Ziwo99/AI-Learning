Les classes et les objets

Les classes et les objets sont des éléments fondamentaux de la programmation orientée objet en Python. Une classe est un modèle ou un plan pour créer des objets, tandis qu'un objet est une instance spécifique d'une classe.

Pour définir une classe en Python, utilisez le mot-clé class suivi du nom de la classe. Par exemple :

class Personne:
    def __init__(self, nom, age):
        self.nom = nom
        self.age = age

    def afficher_infos(self):
        print("Nom :", self.nom)
        print("Age :", self.age)

personne1 = Personne("Alice", 25)
personne1.afficher_infos()

Dans cet exemple, nous avons défini une classe Personne avec un constructeur __init__ qui initialise les attributs nom et age. La méthode afficher_infos affiche les informations de la personne. Ensuite, nous avons créé un objet personne1 de la classe Personne et appelé la méthode afficher_infos pour afficher les informations.

L'héritage et le polymorphisme

L'héritage permet à une classe d'hériter des attributs et des méthodes d'une autre classe. La classe qui hérite est appelée la classe fille ou sous-classe, tandis que la classe dont elle hérite est appelée la classe parent ou superclasse.

Voici un exemple d'héritage en Python :

class Animal:
    def __init__(self, nom):
        self.nom = nom

    def parler(self):
        raise NotImplementedError("La méthode parler doit être implémentée par les sous-classes.")

class Chien(Animal):
    def parler(self):
        return "Woof!"

class Chat(Animal):
    def parler(self):
        return "Meow!"

chien = Chien("Bobby")
print(chien.parler())

chat = Chat("Minou")
print(chat.parler())

Dans cet exemple, nous avons une classe parent Animal avec une méthode parler qui est définie mais lève une exception NotImplementedError. Les classes filles Chien et Chat héritent de la classe Animal et implémentent leur propre version de la méthode parler. Lorsque nous créons des objets de ces classes et appelons la méthode parler, nous obtenons la sortie appropriée pour chaque animal.

Les exceptions et la gestion des erreurs

Les exceptions sont des événements qui se produisent lorsqu'une erreur se produit pendant l'exécution d'un programme. En Python, les exceptions sont gérées à l'aide des blocs try et except.

Voici un exemple de gestion d'exception en Python :

try:
    x = 10 / 0
except ZeroDivisionError:
    print("Erreur : division par zéro")

Dans cet exemple, nous essayons de diviser 10 par zéro, ce qui provoque une exception ZeroDivisionError. Le bloc try tente d'exécuter le code, et si une exception se produit, le bloc except est exécuté avec le type d'exception spécifié.

Les générateurs et les itérateurs

Les générateurs et les itérateurs sont des concepts importants en Python pour travailler avec des séquences de données.

Un générateur est une fonction spéciale qui retourne un itérable, c'est-à-dire un objet qui peut être itéré (par exemple, une boucle for peut être utilisée pour parcourir ses éléments). Les générateurs sont utilisés pour créer des séquences de données de manière efficace.

Voici un exemple de générateur en Python :

def generateur_carres(n):
    for i in range(n):
        yield i ** 2

carres = generateur_carres(5)
for carre in carres:
    print(carre)

Dans cet exemple, nous avons défini une fonction generateur_carres qui utilise le mot-clé yield pour retourner les carrés des nombres de 0 à n. Lorsque nous appelons cette fonction, nous obtenons un générateur qui peut être utilisé pour itérer sur les carrés.

Un itérateur est un objet qui implémente les méthodes spéciales __iter__ et __next__. L'itérateur est utilisé pour parcourir les éléments d'un itérable.

Voici un exemple d'itérateur en Python :

class Compteur:
    def __init__(self, debut, fin):
        self.debut = debut
        self.fin = fin

    def __iter__(self):
        return self

    def __next__(self):
        if self.debut < self.fin:
            valeur = self.debut
            self.debut += 1
            return valeur
        else:
            raise StopIteration

compteur = Compteur(1, 5)
for nombre in compteur:
    print(nombre)

Dans cet exemple, nous avons défini une classe Compteur qui implémente les méthodes spéciales __iter__ et __next__. Lorsque nous itérons sur un objet Compteur, la méthode __next__ est appelée pour obtenir les valeurs suivantes jusqu'à ce que la condition de fin soit atteinte.

Les décorateurs

Les décorateurs sont des fonctions qui prennent une autre fonction en argument et étendent son comportement sans la modifier directement. Les décorateurs sont utilisés pour ajouter des fonctionnalités supplémentaires à une fonction existante.

Voici un exemple de décorateur en Python :

def decorateur(fonction):
    def fonction_decoree():
        print("Avant l'appel de la fonction")
        fonction()
        print("Après l'appel de la fonction")

    return fonction_decoree

@decorateur
def ma_fonction():
    print("Ma fonction")

ma_fonction()

Dans cet exemple, nous avons défini une fonction decorateur qui prend une autre fonction en argument et renvoie une fonction décorée. La fonction décorée ajoute du code supplémentaire avant et après l'appel de la fonction d'origine. Lorsque nous utilisons le décorateur @decorateur avant la définition de la fonction ma_fonction, la fonction ma_fonction est automatiquement passée à la fonction decorateur et la fonction décorée est utilisée lors de l'appel.
