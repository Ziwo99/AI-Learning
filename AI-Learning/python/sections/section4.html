Introduction à la POO

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. La POO permet de regrouper des données et des fonctionnalités connexes dans des objets, ce qui facilite la réutilisation du code, la modularité et la maintenance.

Classes et objets

Une classe est une structure de données qui définit les propriétés et les comportements d'un objet. Les propriétés sont représentées par des variables appelées attributs, et les comportements sont représentés par des fonctions appelées méthodes. Pour créer un objet à partir d'une classe, on utilise le mot-clé class suivi du nom de la classe, suivi de parenthèses.

class Personne:
    def __init__(self, nom, age):
        self.nom = nom
        self.age = age

    def se_presenter(self):
        print("Je m'appelle", self.nom, "et j'ai", self.age, "ans.")

personne1 = Personne("Alice", 25)
personne1.se_presenter()  # Affiche "Je m'appelle Alice et j'ai 25 ans."

Dans cet exemple, nous avons créé une classe Personne avec deux attributs nom et age, ainsi qu'une méthode se_presenter qui affiche une présentation de la personne. Ensuite, nous avons créé un objet personne1 à partir de la classe Personne et appelé sa méthode se_presenter.

Héritage et polymorphisme

L'héritage est un concept clé de la POO qui permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe qui hérite d'une autre classe est appelée une classe dérivée ou une sous-classe, et la classe dont elle hérite est appelée une classe de base ou une superclasse. La sous-classe hérite tous les attributs et les méthodes de la superclasse, et peut également ajouter de nouveaux attributs et méthodes ou modifier ceux existants.

class Employe(Personne):
    def __init__(self, nom, age, salaire):
        super().__init__(nom, age)
        self.salaire = salaire

    def se_presenter(self):
        super().se_presenter()
        print("Je gagne", self.salaire, "euros par mois.")

employe1 = Employe("Bob", 30, 2000)
employe1.se_presenter()  # Affiche "Je m'appelle Bob et j'ai 30 ans. Je gagne 2000 euros par mois."

Dans cet exemple, nous avons créé une classe Employe qui hérite de la classe Personne. La classe Employe a un attribut supplémentaire salaire et une méthode se_presenter qui affiche la présentation de l'employé ainsi que son salaire. Nous avons ensuite créé un objet employe1 à partir de la classe Employe et appelé sa méthode se_presenter.

Gestion des exceptions

La gestion des exceptions est un mécanisme qui permet de gérer les erreurs et les situations exceptionnelles dans un programme. En Python, les exceptions sont des objets qui représentent des erreurs ou des événements indésirables. On peut utiliser des blocs try et except pour gérer les exceptions et exécuter un code de secours en cas d'erreur.

try:
    resultat = 10 / 0
except ZeroDivisionError:
    print("Erreur : division par zéro.")

print("Le programme continue.")

Dans cet exemple, nous essayons de diviser 10 par zéro, ce qui provoque une exception de type ZeroDivisionError. Nous utilisons un bloc try pour entourer le code susceptible de provoquer une exception, et un bloc except pour spécifier le type d'exception que nous voulons gérer. Si une exception de ce type est levée, le code dans le bloc except est exécuté. Dans tous les cas, le programme continue son exécution après le bloc try-except.

Ce sont là quelques-uns des concepts fondamentaux de la programmation orientée objet en Python. La POO offre de nombreux avantages et est largement utilisée dans l'industrie du développement logiciel. En comprenant ces concepts et en les appliquant dans vos projets, vous serez en mesure de créer des programmes plus modulaires, réutilisables et faciles à maintenir.
