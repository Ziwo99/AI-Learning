Concepts de base de la POO

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. La POO repose sur quatre concepts fondamentaux :

  • Encapsulation : Il s'agit de la capacité de regrouper des données et des méthodes dans une seule entité appelée classe. L'encapsulation permet de cacher les détails internes de la classe et de fournir une interface pour interagir avec l'objet.
  • Héritage : Il permet à une classe d'hériter des propriétés et des méthodes d'une autre classe appelée classe parent. Cela permet de réutiliser le code existant et de créer des hiérarchies de classes.
  • Polymorphisme : Il permet à des objets de différentes classes d'être traités de manière uniforme. Cela signifie qu'un objet peut prendre différentes formes en fonction du contexte.
  • Abstraction : Il s'agit de la capacité de représenter des objets du monde réel sous forme de classes et d'ignorer les détails non pertinents. L'abstraction permet de simplifier la complexité du système.

Classes et objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les propriétés et les méthodes communes à tous les objets de cette classe. Pour créer un objet, on utilise le mot-clé class suivi du nom de la classe :

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def greet(self):
        print(f"Hello, my name is {self.name} and I am {self.age} years old.")

person1 = Person("John", 25)
person1.greet()  # Output: Hello, my name is John and I am 25 years old.

Dans cet exemple, nous avons créé une classe Person avec les propriétés name et age, ainsi qu'une méthode greet pour saluer la personne.

Pour créer un objet à partir de cette classe, nous utilisons le constructeur spécial __init__ qui est appelé automatiquement lors de la création d'un nouvel objet. Les propriétés de l'objet sont définies à l'aide du mot-clé self.

Héritage et polymorphisme

L'héritage permet à une classe d'hériter des propriétés et des méthodes d'une autre classe appelée classe parent. Cela permet de réutiliser le code existant et de créer des hiérarchies de classes.

Voici un exemple d'héritage :

class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        raise NotImplementedError("Subclass must implement abstract method")

class Dog(Animal):
    def speak(self):
        print("Woof!")

class Cat(Animal):
    def speak(self):
        print("Meow!")

dog = Dog("Buddy")
dog.speak()  # Output: Woof!

cat = Cat("Whiskers")
cat.speak()  # Output: Meow!

Dans cet exemple, nous avons une classe parent Animal avec une méthode speak qui est déclarée comme une méthode abstraite en levant une exception NotImplementedError. Les classes enfants Dog et Cat héritent de la classe Animal et implémentent la méthode speak de manière spécifique à chaque classe.

Le polymorphisme permet à des objets de différentes classes d'être traités de manière uniforme. Cela signifie qu'un objet peut prendre différentes formes en fonction du contexte. Dans l'exemple ci-dessus, nous pouvons traiter à la fois un chien et un chat comme des animaux, car ils héritent tous deux de la classe Animal.

Encapsulation et abstraction

L'encapsulation permet de regrouper des données et des méthodes dans une seule entité appelée classe. Cela permet de cacher les détails internes de la classe et de fournir une interface pour interagir avec l'objet.

Voici un exemple d'encapsulation :

class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year
        self.__mileage = 0

    def drive(self, miles):
        self.__mileage += miles

    def get_mileage(self):
        return self.__mileage

car = Car("Toyota", "Camry", 2020)
car.drive(100)
car.drive(200)
print(car.get_mileage())  # Output: 300

Dans cet exemple, la propriété __mileage est définie avec deux traits de soulignement avant son nom, ce qui la rend privée. Cela signifie qu'elle ne peut être accédée que depuis l'intérieur de la classe. Pour accéder à cette propriété depuis l'extérieur de la classe, nous utilisons une méthode publique get_mileage.

L'abstraction permet de représenter des objets du monde réel sous forme de classes et d'ignorer les détails non pertinents. Cela permet de simplifier la complexité du système. Dans l'exemple ci-dessus, nous avons créé une classe Car qui représente une voiture. Les détails internes de la voiture, tels que le moteur, la transmission, etc., sont abstraits et ne sont pas exposés à l'utilisateur de la classe.

Gestion des exceptions

La gestion des exceptions permet de gérer les erreurs et les situations exceptionnelles qui peuvent se produire lors de l'exécution d'un programme. En Python, les exceptions sont gérées à l'aide des blocs try et except.

Voici un exemple de gestion des exceptions :

try:
    num1 = int(input("Enter a number: "))
    num2 = int(input("Enter another number: "))
    result = num1 / num2
    print(f"The result is {result}")
except ValueError:
    print("Invalid input. Please enter a valid number.")
except ZeroDivisionError:
    print("Cannot divide by zero.")

Dans cet exemple, nous demandons à l'utilisateur d'entrer deux nombres. Si l'utilisateur entre une valeur invalide (par exemple, une chaîne de caractères au lieu d'un nombre), une exception de type ValueError est levée. Si l'utilisateur entre zéro comme deuxième nombre, une exception de type ZeroDivisionError est levée. Dans chaque cas, le programme affiche un message d'erreur approprié.

Il est également possible de capturer toutes les exceptions avec un bloc except sans spécifier le type d'exception :

try:
    # Code susceptible de lever une exception
except:
    # Code à exécuter en cas d'exception

Cependant, il est généralement recommandé de spécifier le type d'exception que vous souhaitez gérer, afin de ne pas masquer d'autres erreurs potentielles.
