Les générateurs et les itérables

Les générateurs et les itérables sont des concepts avancés en Python qui permettent de créer des objets qui peuvent être itérés. Un itérable est un objet qui peut être parcouru séquentiellement, tandis qu'un générateur est une fonction spéciale qui produit une séquence de valeurs à la demande.

Les générateurs sont souvent utilisés pour générer des séquences de valeurs de manière efficace, car ils ne calculent les valeurs que lorsque cela est nécessaire. Cela permet d'économiser de la mémoire et d'améliorer les performances.

Voici un exemple de générateur simple qui génère les nombres de 1 à 5 :

def generate_numbers():
    for i in range(1, 6):
        yield i

numbers = generate_numbers()

for number in numbers:
    print(number)

Dans cet exemple, la fonction generate_numbers() est un générateur qui utilise le mot-clé yield pour retourner les valeurs une par une. Lorsque la boucle for itère sur l'objet numbers, chaque valeur est générée à la demande.

Les itérables, quant à eux, sont des objets qui implémentent la méthode spéciale __iter__() et qui peuvent être utilisés dans une boucle for. Voici un exemple d'itérable simple qui génère les nombres de 1 à 5 :

class Numbers:
    def __init__(self):
        self.start = 1
        self.end = 6

    def __iter__(self):
        return self

    def __next__(self):
        if self.start >= self.end:
            raise StopIteration
        else:
            number = self.start
            self.start += 1
            return number

numbers = Numbers()

for number in numbers:
    print(number)

Dans cet exemple, la classe Numbers implémente les méthodes spéciales __iter__() et __next__(), ce qui en fait un itérable. Lorsque la boucle for itère sur l'objet numbers, la méthode __next__() est appelée à chaque itération pour obtenir la prochaine valeur.

Les décorateurs

Les décorateurs sont une fonctionnalité avancée en Python qui permet de modifier le comportement d'une fonction existante en lui ajoutant du code supplémentaire. Les décorateurs sont souvent utilisés pour ajouter des fonctionnalités telles que la journalisation, la gestion des erreurs ou la mesure du temps d'exécution.

Voici un exemple de décorateur simple qui mesure le temps d'exécution d'une fonction :

import time

def measure_time(func):
    def wrapper(*args, **kwargs):
        start_time = time.time()
        result = func(*args, **kwargs)
        end_time = time.time()
        execution_time = end_time - start_time
        print(f"Execution time: {execution_time} seconds")
        return result
    return wrapper

@measure_time
def calculate_sum(a, b):
    time.sleep(1)
    return a + b

result = calculate_sum(5, 10)
print(result)

Dans cet exemple, la fonction measure_time() est un décorateur qui mesure le temps d'exécution de la fonction passée en argument. Le décorateur est appliqué à la fonction calculate_sum() à l'aide de la syntaxe @measure_time, ce qui signifie que la fonction calculate_sum() sera automatiquement enveloppée par le code du décorateur.

Les métaclasses

Les métaclasses sont un concept avancé en Python qui permet de définir le comportement des classes. Une métaclass est une classe dont les instances sont des classes. Les métaclasses sont souvent utilisées pour modifier le comportement de l'héritage, ajouter des attributs ou des méthodes aux classes, ou encore pour créer des classes dynamiquement.

Voici un exemple simple de métaclass qui ajoute un attribut à toutes les classes définies avec cette métaclass :

class MyMeta(type):
    def __new__(cls, name, bases, attrs):
        attrs['my_attribute'] = 42
        return super().__new__(cls, name, bases, attrs)

class MyClass(metaclass=MyMeta):
    pass

print(MyClass.my_attribute)

Dans cet exemple, la classe MyMeta est une métaclass qui redéfinit la méthode spéciale __new__() pour ajouter l'attribut my_attribute à toutes les classes définies avec cette métaclass. La classe MyClass est définie avec la métaclass MyMeta, ce qui fait que l'attribut my_attribute est présent dans la classe MyClass.

Les métaclasses sont un sujet avancé en Python et leur utilisation doit être soigneusement considérée. Elles offrent une grande flexibilité, mais peuvent également rendre le code plus complexe et difficile à comprendre.
