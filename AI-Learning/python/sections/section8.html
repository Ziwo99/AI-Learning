Section 8: Les exceptions en Python

Introduction aux exceptions en Python

En programmation, une exception est une erreur qui se produit pendant l'exécution d'un programme. Python utilise des exceptions pour gérer ces erreurs et les traiter de manière appropriée. Les exceptions permettent de détecter et de gérer les erreurs de manière efficace, en évitant l'arrêt brutal du programme.

Les exceptions en Python sont des objets qui représentent une erreur spécifique. Lorsqu'une erreur se produit, Python génère une exception qui peut être capturée et traitée par le programme.

La gestion des exceptions en Python

La gestion des exceptions en Python se fait à l'aide des blocs try-except. Le bloc try contient le code susceptible de générer une exception, tandis que le bloc except définit le code à exécuter en cas d'exception.

Voici un exemple de bloc try-except en Python :

try:
    # Code susceptible de générer une exception
    result = 10 / 0
except ZeroDivisionError:
    # Code à exécuter en cas d'exception
    print("Erreur : division par zéro")

Dans cet exemple, le code à l'intérieur du bloc try tente de diviser 10 par zéro, ce qui génère une exception de type ZeroDivisionError. Le bloc except capture cette exception et exécute le code à l'intérieur, affichant le message "Erreur : division par zéro".

Les exceptions personnalisées en Python

En plus des exceptions prédéfinies, Python permet également de créer des exceptions personnalisées. Les exceptions personnalisées sont utiles lorsque vous souhaitez gérer des erreurs spécifiques à votre programme.

Voici un exemple de création d'une exception personnalisée en Python :

class MonException(Exception):
    pass

try:
    # Code susceptible de générer une exception personnalisée
    raise MonException("Une erreur s'est produite")
except MonException as e:
    # Code à exécuter en cas d'exception personnalisée
    print("Erreur personnalisée :", str(e))

Dans cet exemple, nous créons une classe MonException qui hérite de la classe Exception. Ensuite, nous utilisons le mot-clé raise pour générer une instance de cette exception personnalisée. Le bloc except capture cette exception et exécute le code à l'intérieur, affichant le message "Erreur personnalisée : Une erreur s'est produite".

Il est recommandé d'utiliser des exceptions personnalisées pour gérer les erreurs spécifiques à votre programme, afin de rendre votre code plus clair et plus facile à comprendre.
