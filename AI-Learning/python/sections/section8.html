Section 8: La programmation orientée objet en Python

Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code de manière plus modulaire. En POO, on crée des objets qui représentent des entités du monde réel et qui possèdent des caractéristiques (attributs) et des comportements (méthodes).

En Python, tout est un objet. Les types de données de base comme les entiers, les chaînes de caractères et les listes sont des objets, tout comme les classes que nous allons créer.

Les classes et les objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les attributs et les méthodes que les objets de cette classe auront.

Voici un exemple de classe en Python :

class Voiture:
    def __init__(self, marque, modele):
        self.marque = marque
        self.modele = modele

    def demarrer(self):
        print("La voiture démarre")

    def arreter(self):
        print("La voiture s'arrête")

Dans cet exemple, nous avons créé une classe "Voiture" avec deux attributs "marque" et "modele", ainsi que deux méthodes "demarrer" et "arreter".

Pour créer un objet à partir de cette classe, nous utilisons le mot-clé "new" :

ma_voiture = Voiture("Renault", "Clio")

Nous pouvons ensuite accéder aux attributs et appeler les méthodes de cet objet :

print(ma_voiture.marque)  # Affiche "Renault"
ma_voiture.demarrer()  # Affiche "La voiture démarre"

L'héritage et le polymorphisme

L'héritage est un concept clé de la POO qui permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe qui hérite d'une autre classe est appelée une classe dérivée ou une sous-classe, tandis que la classe dont elle hérite est appelée une classe de base ou une superclasse.

Le polymorphisme est un autre concept important de la POO qui permet à des objets de différentes classes d'être traités de manière similaire. Cela signifie que nous pouvons utiliser une classe dérivée là où une classe de base est attendue.

Voici un exemple d'héritage en Python :

class Animal:
    def __init__(self, nom):
        self.nom = nom

    def parler(self):
        raise NotImplementedError("La méthode parler doit être implémentée dans les classes dérivées")

class Chien(Animal):
    def parler(self):
        print("Wouaf !")

class Chat(Animal):
    def parler(self):
        print("Miaou !")

Dans cet exemple, nous avons une classe de base "Animal" avec une méthode "parler" qui est définie mais lève une exception. Les classes dérivées "Chien" et "Chat" héritent de la classe "Animal" et implémentent leur propre version de la méthode "parler".

Voici comment utiliser ces classes :

mon_chien = Chien("Rex")
mon_chat = Chat("Minou")

mon_chien.parler()  # Affiche "Wouaf !"
mon_chat.parler()  # Affiche "Miaou !"

Les méthodes spéciales

En Python, il existe des méthodes spéciales qui commencent et se terminent par deux underscores. Ces méthodes sont utilisées pour effectuer des opérations spécifiques sur les objets, comme l'initialisation, la représentation sous forme de chaîne de caractères, la comparaison, etc.

Voici quelques exemples de méthodes spéciales :

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"({self.x}, {self.y})"

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

Dans cet exemple, nous avons une classe "Point" avec une méthode spéciale "__init__" pour l'initialisation, une méthode spéciale "__str__" pour la représentation sous forme de chaîne de caractères et une méthode spéciale "__eq__" pour la comparaison.

Voici comment utiliser ces méthodes :

point1 = Point(1, 2)
point2 = Point(1, 2)

print(point1)  # Affiche "(1, 2)"
print(point1 == point2)  # Affiche "True"
