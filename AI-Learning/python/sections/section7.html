Section 7: Les exceptions en Python

Introduction aux exceptions

Les exceptions sont des événements qui se produisent lors de l'exécution d'un programme Python et qui interrompent le flux normal d'exécution. Elles sont utilisées pour gérer les erreurs et les situations exceptionnelles. Lorsqu'une exception se produit, le programme s'arrête et affiche un message d'erreur correspondant à l'exception.

Les exceptions en Python sont représentées par des objets. Chaque type d'exception est une classe différente, dérivée de la classe de base Exception. Les exceptions peuvent être générées par le programme lui-même ou par le système Python lorsqu'une erreur se produit.

La gestion des exceptions

La gestion des exceptions permet de contrôler le comportement du programme lorsqu'une exception se produit. Elle permet d'intercepter les exceptions, d'exécuter un code spécifique en cas d'exception et de reprendre l'exécution normale du programme.

La structure de base pour gérer les exceptions en Python est la suivante :

try:
    # Code susceptible de générer une exception
except ExceptionType:
    # Code à exécuter en cas d'exception
else:
    # Code à exécuter si aucune exception n'est levée
finally:
    # Code à exécuter systématiquement, qu'une exception soit levée ou non

Le bloc try contient le code susceptible de générer une exception. Si une exception se produit, le bloc except correspondant est exécuté. Si aucune exception n'est levée, le bloc else est exécuté. Le bloc finally est exécuté systématiquement, qu'une exception soit levée ou non.

Les exceptions personnalisées

En plus des exceptions prédéfinies en Python, il est possible de créer ses propres exceptions personnalisées. Cela permet de gérer des situations spécifiques qui ne sont pas couvertes par les exceptions standard.

Pour créer une exception personnalisée, il faut définir une nouvelle classe dérivée de la classe de base Exception ou d'une de ses sous-classes. La nouvelle classe peut avoir des attributs et des méthodes supplémentaires pour gérer spécifiquement l'exception.

Voici un exemple de création d'une exception personnalisée :

class MonException(Exception):
    def __init__(self, message):
        self.message = message

try:
    raise MonException("Une erreur s'est produite.")
except MonException as e:
    print(e.message)

Dans cet exemple, la classe MonException est définie avec un attribut message. Lorsque l'exception est levée, l'attribut message est initialisé avec le message spécifié. Lorsque l'exception est interceptée, le message est affiché.
