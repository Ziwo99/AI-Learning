Section 7: Les exceptions en Python

Introduction aux exceptions

Les exceptions en Python sont des événements qui se produisent lorsqu'une erreur se produit pendant l'exécution d'un programme. Elles permettent de gérer les erreurs de manière contrôlée et d'éviter que le programme ne se termine de manière inattendue.

Les exceptions peuvent être générées par le système ou par le programme lui-même. Par exemple, une division par zéro génère une exception de type ZeroDivisionError, tandis qu'une erreur de syntaxe génère une exception de type SyntaxError.

La gestion des exceptions

La gestion des exceptions en Python se fait à l'aide des blocs try et except. Le bloc try permet d'encadrer le code susceptible de générer une exception, tandis que le bloc except permet de spécifier comment gérer cette exception.

Voici un exemple de gestion d'exception :

try:
    # Code susceptible de générer une exception
    result = 10 / 0
except ZeroDivisionError:
    # Gestion de l'exception de division par zéro
    print("Erreur : division par zéro")

Dans cet exemple, si une exception de type ZeroDivisionError est générée lors de la division par zéro, le message "Erreur : division par zéro" sera affiché.

Les exceptions personnalisées

En plus des exceptions prédéfinies, il est possible de créer ses propres exceptions personnalisées en définissant de nouvelles classes d'exceptions. Cela permet de gérer des erreurs spécifiques à votre programme.

Voici un exemple de création d'une exception personnalisée :

class MonException(Exception):
    pass

try:
    # Code susceptible de générer une exception personnalisée
    raise MonException("Une erreur s'est produite")
except MonException as e:
    # Gestion de l'exception personnalisée
    print("Erreur personnalisée :", str(e))

Dans cet exemple, une exception de type MonException est créée et levée à l'aide du mot-clé raise. L'exception est ensuite gérée dans le bloc except et le message d'erreur est affiché.
