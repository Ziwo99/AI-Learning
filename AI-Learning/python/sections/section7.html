Section 7 : La programmation orientée objet en Python

Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. En POO, les objets sont utilisés pour représenter des entités du monde réel, avec leurs attributs (variables) et leurs comportements (méthodes).

En Python, tout est un objet. Les types de données de base tels que les entiers, les flottants, les chaînes de caractères, etc., sont des objets. De plus, vous pouvez créer vos propres classes pour définir de nouveaux types d'objets.

Classes et objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les attributs et les méthodes que les objets de cette classe auront. Pour créer un objet à partir d'une classe, on utilise le mot-clé class suivi du nom de la classe, suivi de parenthèses (facultatives) contenant les classes parentes.

Voici un exemple de classe simple en Python :

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def say_hello(self):
        print("Hello, my name is", self.name)

# Création d'un objet de la classe Person
person1 = Person("Alice", 25)

# Appel de la méthode say_hello de l'objet
person1.say_hello()

Dans cet exemple, nous avons créé une classe Person avec deux attributs : name et age. La méthode spéciale __init__ est appelée lors de la création d'un nouvel objet et permet d'initialiser ses attributs. La méthode say_hello affiche un message de salutation avec le nom de la personne.

Héritage et polymorphisme

L'héritage est un concept clé de la POO qui permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe qui hérite d'une autre classe est appelée une classe dérivée ou une sous-classe, et la classe dont elle hérite est appelée une classe de base ou une superclasse.

Le polymorphisme est un autre concept important de la POO qui permet à des objets de différentes classes d'être traités de manière uniforme. Cela signifie que vous pouvez utiliser une classe de base pour manipuler des objets de différentes classes dérivées, en appelant les mêmes méthodes sur ces objets.

Voici un exemple d'héritage et de polymorphisme en Python :

class Animal:
    def __init__(self, name):
        self.name = name

    def make_sound(self):
        pass

class Dog(Animal):
    def make_sound(self):
        print("Woof!")

class Cat(Animal):
    def make_sound(self):
        print("Meow!")

# Création d'objets de différentes classes dérivées
animal1 = Animal("Animal")
dog1 = Dog("Dog")
cat1 = Cat("Cat")

# Appel de la méthode make_sound sur les objets
animal1.make_sound()  # Ne fait rien
dog1.make_sound()     # Affiche "Woof!"
cat1.make_sound()     # Affiche "Meow!"

Dans cet exemple, nous avons une classe de base Animal avec une méthode make_sound qui ne fait rien. Les classes dérivées Dog et Cat héritent de la classe Animal et redéfinissent la méthode make_sound pour afficher des sons spécifiques à chaque animal. Les objets animal1, dog1 et cat1 sont créés et la méthode make_sound est appelée sur chacun d'eux.

Encapsulation et abstraction

L'encapsulation est un principe de la POO qui consiste à regrouper les données et les méthodes qui les manipulent au sein d'une même entité appelée classe. Cela permet de cacher les détails internes de la classe et de fournir une interface publique pour interagir avec les objets de cette classe.

L'abstraction est un autre principe de la POO qui consiste à simplifier la complexité en masquant les détails inutiles et en se concentrant sur les aspects essentiels. En utilisant l'abstraction, vous pouvez créer des classes qui représentent des concepts abstraits plutôt que des entités concrètes.

Voici un exemple d'encapsulation et d'abstraction en Python :

class Car:
    def __init__(self, brand, model):
        self.brand = brand
        self.model = model

    def start_engine(self):
        print("Engine started")

    def drive(self):
        print("Driving")

    def stop_engine(self):
        print("Engine stopped")

# Création d'un objet de la classe Car
car1 = Car("Toyota", "Corolla")

# Appel des méthodes sur l'objet
car1.start_engine()
car1.drive()
car1.stop_engine()

Dans cet exemple, nous avons une classe Car qui représente une voiture. Les attributs brand et model sont encapsulés à l'intérieur de la classe. Les méthodes start_engine, drive et stop_engine fournissent une interface publique pour interagir avec l'objet. L'utilisateur n'a pas besoin de connaître les détails internes de la classe pour utiliser ces méthodes.
