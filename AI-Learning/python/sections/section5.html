Section 5: Gestion des erreurs et exceptions

Introduction aux exceptions

En programmation, une exception est une erreur ou une condition anormale qui se produit pendant l'exécution d'un programme. Lorsqu'une exception se produit, le programme s'arrête immédiatement et affiche un message d'erreur correspondant à l'exception.

Python fournit un mécanisme de gestion des exceptions qui permet de détecter et de gérer ces erreurs de manière contrôlée. Cela permet d'éviter que le programme ne se bloque complètement en cas d'erreur et de prendre des mesures appropriées pour gérer l'exception.

Les exceptions peuvent être générées par le programme lui-même ou par des erreurs système, telles que l'ouverture d'un fichier inexistant ou la division par zéro.

Gestion des erreurs avec les blocs try-except

En Python, la gestion des exceptions se fait à l'aide de blocs try-except. Un bloc try contient le code susceptible de générer une exception, tandis qu'un bloc except définit le code à exécuter en cas d'exception.

Voici la syntaxe générale d'un bloc try-except :

try:
    # Code susceptible de générer une exception
except ExceptionType:
    # Code à exécuter en cas d'exception de type ExceptionType

Le bloc except peut également être suivi d'une clause else, qui est exécutée si aucune exception n'est levée dans le bloc try :

try:
    # Code susceptible de générer une exception
except ExceptionType:
    # Code à exécuter en cas d'exception de type ExceptionType
else:
    # Code à exécuter si aucune exception n'est levée

Il est également possible de capturer plusieurs types d'exceptions dans un seul bloc except :

try:
    # Code susceptible de générer une exception
except (ExceptionType1, ExceptionType2):
    # Code à exécuter en cas d'exception de type ExceptionType1 ou ExceptionType2

Utilisation des exceptions personnalisées

En plus des exceptions prédéfinies dans Python, il est possible de créer ses propres exceptions personnalisées pour gérer des situations spécifiques dans un programme. Pour cela, il suffit de créer une classe dérivée de la classe de base Exception.

Voici un exemple de création d'une exception personnalisée :

class CustomException(Exception):
    def __init__(self, message):
        self.message = message

try:
    raise CustomException("Une erreur personnalisée s'est produite.")
except CustomException as e:
    print(e.message)

Dans cet exemple, nous créons une classe CustomException dérivée de la classe Exception. Nous définissons également un constructeur qui prend un message en paramètre et l'assigne à l'attribut message de l'exception.

Ensuite, nous utilisons le mot-clé raise pour lever une instance de notre exception personnalisée. Enfin, nous capturons cette exception avec un bloc except et affichons son message.
