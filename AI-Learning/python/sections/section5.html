Introduction à la POO

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code de manière plus modulaire et réutilisable. En POO, le code est organisé autour d'objets qui représentent des entités du monde réel ou des concepts abstraits. Chaque objet est une instance d'une classe, qui définit ses caractéristiques (attributs) et ses comportements (méthodes).

Classes et objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les attributs et les méthodes communs à tous les objets de cette classe. Pour créer un objet, on utilise le mot-clé "class" suivi du nom de la classe et de parenthèses vides. Les attributs sont des variables qui stockent les données spécifiques à chaque objet, tandis que les méthodes sont des fonctions qui définissent les actions que les objets de la classe peuvent effectuer.

class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def greet(self):
        print("Hello, my name is " + self.name + " and I am " + str(self.age) + " years old.")

person1 = Person("Alice", 25)
person1.greet()  # Output: Hello, my name is Alice and I am 25 years old.

Héritage et polymorphisme

L'héritage est un mécanisme qui permet à une classe d'hériter des attributs et des méthodes d'une autre classe appelée classe parent ou superclasse. La classe qui hérite est appelée classe enfant ou sous-classe. Cela permet de réutiliser le code existant et de créer des classes plus spécialisées. Le polymorphisme est la capacité d'un objet à prendre différentes formes en fonction du contexte. Il permet d'utiliser une classe enfant de manière interchangeable avec sa classe parent.

class Animal:
    def __init__(self, name):
        self.name = name

    def make_sound(self):
        pass

class Dog(Animal):
    def make_sound(self):
        print("Woof!")

class Cat(Animal):
    def make_sound(self):
        print("Meow!")

dog = Dog("Buddy")
cat = Cat("Whiskers")

dog.make_sound()  # Output: Woof!
cat.make_sound()  # Output: Meow!

Encapsulation et abstraction

L'encapsulation est un principe de la POO qui consiste à regrouper les données et les méthodes qui les manipulent au sein d'une même entité appelée classe. Cela permet de cacher les détails d'implémentation et de protéger les données sensibles. L'abstraction est un mécanisme qui permet de définir des classes abstraites qui ne peuvent pas être instanciées directement, mais qui servent de modèles pour d'autres classes. Elle permet de définir des méthodes abstraites qui doivent être implémentées par les classes dérivées.

from abc import ABC, abstractmethod

class Shape(ABC):
    @abstractmethod
    def area(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

rectangle = Rectangle(5, 3)
print(rectangle.area())  # Output: 15

Cette section a couvert les concepts fondamentaux de la programmation orientée objet en Python. En comprenant ces concepts, vous serez en mesure de créer des classes, d'utiliser l'héritage et le polymorphisme, ainsi que d'encapsuler et d'abstraire votre code de manière efficace. La POO est largement utilisée dans le développement logiciel et est un élément clé de la programmation Python.
