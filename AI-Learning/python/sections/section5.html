Section 5 : Programmation orientée objet en Python

Classes et objets

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. En Python, tout est un objet, y compris les types de données primitifs tels que les entiers, les chaînes de caractères et les booléens.

Pour créer une classe en Python, utilisez le mot-clé class suivi du nom de la classe. Par exemple :

class Personne:
    def __init__(self, nom, age):
        self.nom = nom
        self.age = age

    def dire_bonjour(self):
        print("Bonjour, je m'appelle", self.nom, "et j'ai", self.age, "ans.")

# Création d'un objet de la classe Personne
personne1 = Personne("Alice", 25)

# Appel de la méthode dire_bonjour de l'objet personne1
personne1.dire_bonjour()

Dans cet exemple, nous avons créé une classe Personne avec un constructeur __init__ qui initialise les attributs nom et age de l'objet. La méthode dire_bonjour affiche un message de salutation en utilisant les attributs de l'objet.

Héritage et polymorphisme

L'héritage est un concept clé de la POO qui permet de créer de nouvelles classes en utilisant des classes existantes. Une classe qui hérite d'une autre classe est appelée une classe dérivée ou une sous-classe, tandis que la classe dont elle hérite est appelée une classe de base ou une superclasse.

En Python, l'héritage est réalisé en spécifiant la classe de base entre parenthèses après le nom de la classe dérivée. Par exemple :

class Employe(Personne):
    def __init__(self, nom, age, salaire):
        super().__init__(nom, age)
        self.salaire = salaire

    def dire_bonjour(self):
        super().dire_bonjour()
        print("Je suis un employé et je gagne", self.salaire, "euros par mois.")

# Création d'un objet de la classe Employe
employe1 = Employe("Bob", 30, 2000)

# Appel de la méthode dire_bonjour de l'objet employe1
employe1.dire_bonjour()

Dans cet exemple, la classe Employe hérite de la classe Personne. Le constructeur de la classe Employe utilise la méthode super() pour appeler le constructeur de la classe de base et initialiser les attributs nom et age. La méthode dire_bonjour de la classe Employe utilise également la méthode super() pour appeler la méthode dire_bonjour de la classe de base avant d'afficher des informations supplémentaires propres à la classe Employe.

Le polymorphisme est un autre concept important de la POO qui permet à des objets de différentes classes d'être traités de manière interchangeable. En Python, le polymorphisme est automatiquement pris en charge grâce à la dynamique du typage.

Encapsulation et abstraction

L'encapsulation est un principe de la POO qui consiste à regrouper des données et des méthodes connexes dans une classe, en les rendant inaccessibles depuis l'extérieur de la classe, sauf par le biais de méthodes spécifiques.

En Python, l'encapsulation est réalisée en utilisant des conventions de nommage pour indiquer la visibilité des attributs et des méthodes. Par convention, un attribut ou une méthode dont le nom commence par un seul tiret bas (_) est considéré comme "protégé" et ne doit pas être accédé depuis l'extérieur de la classe. Un attribut ou une méthode dont le nom commence par deux tirets bas (__) est considéré comme "privé" et ne peut être accédé que depuis l'intérieur de la classe.

L'abstraction est un autre principe de la POO qui consiste à cacher les détails d'implémentation complexes derrière une interface simple et facile à utiliser. En Python, l'abstraction est réalisée en utilisant des classes abstraites et des méthodes abstraites.

Une classe abstraite est une classe qui ne peut pas être instanciée directement, mais qui sert de modèle pour d'autres classes. En Python, une classe abstraite peut être créée en dérivant de la classe ABC du module abc. Une méthode abstraite est une méthode qui doit être implémentée par toutes les sous-classes de la classe abstraite. En Python, une méthode abstraite peut être créée en utilisant le décorateur @abstractmethod du module abc.

from abc import ABC, abstractmethod

class Forme(ABC):
    @abstractmethod
    def aire(self):
        pass

class Rectangle(Forme):
    def __init__(self, longueur, largeur):
        self.longueur = longueur
        self.largeur = largeur

    def aire(self):
        return self.longueur * self.largeur

# Création d'un objet de la classe Rectangle
rectangle1 = Rectangle(5, 3)

# Appel de la méthode aire de l'objet rectangle1
print("L'aire du rectangle est", rectangle1.aire())

Dans cet exemple, la classe Forme est une classe abstraite qui définit une méthode abstraite aire. La classe Rectangle est une sous-classe de la classe Forme qui implémente la méthode aire en calculant l'aire du rectangle. L'objet rectangle1 est créé à partir de la classe Rectangle et la méthode aire est appelée pour afficher l'aire du rectangle.
