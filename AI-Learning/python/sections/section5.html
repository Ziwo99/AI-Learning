Section 5: Gestion des erreurs et exceptions

Les erreurs courantes en Python

En Python, il est courant de rencontrer certaines erreurs lors de l'exécution de notre code. Ces erreurs peuvent être causées par des erreurs de syntaxe, des erreurs de logique ou des erreurs d'exécution. Voici quelques-unes des erreurs courantes en Python :

1. SyntaxError : Cette erreur se produit lorsque le code ne respecte pas la syntaxe du langage Python. Par exemple, oublier de fermer une parenthèse ou d'utiliser un mot-clé réservé comme nom de variable.

2. NameError : Cette erreur se produit lorsque Python ne parvient pas à trouver une variable ou une fonction spécifiée. Cela peut être dû à une faute de frappe dans le nom de la variable ou à une variable non définie.

3. TypeError : Cette erreur se produit lorsque une opération est effectuée sur des types de données incompatibles. Par exemple, additionner une chaîne de caractères avec un entier.

4. IndexError : Cette erreur se produit lorsque vous essayez d'accéder à un élément d'une liste ou d'un tableau en utilisant un index qui est en dehors de la plage valide.

5. ValueError : Cette erreur se produit lorsque vous essayez de convertir une valeur dans un type incompatible. Par exemple, convertir une chaîne de caractères en entier alors qu'elle ne contient pas de chiffres.

Les exceptions et leur gestion

En Python, les exceptions sont utilisées pour gérer les erreurs qui se produisent lors de l'exécution du code. Les exceptions permettent de détecter et de gérer les erreurs de manière appropriée, plutôt que de simplement arrêter l'exécution du programme.

Pour gérer les exceptions en Python, nous utilisons les blocs try-except. Le code qui peut générer une exception est placé dans le bloc try, et le code qui gère l'exception est placé dans le bloc except. Voici un exemple :

try:
    # Code susceptible de générer une exception
    result = 10 / 0
except ZeroDivisionError:
    # Code de gestion de l'exception
    print("Erreur : division par zéro")

Dans cet exemple, le code tente de diviser 10 par zéro, ce qui génère une exception de type ZeroDivisionError. Le bloc except spécifie le type d'exception à gérer, et le code à l'intérieur du bloc except est exécuté lorsque cette exception se produit.

Il est également possible d'utiliser plusieurs blocs except pour gérer différents types d'exceptions. Par exemple :

try:
    # Code susceptible de générer une exception
    result = int("abc")
except ValueError:
    # Code de gestion de l'exception ValueError
    print("Erreur : conversion impossible")
except TypeError:
    # Code de gestion de l'exception TypeError
    print("Erreur : type incompatible")

Dans cet exemple, le code tente de convertir la chaîne de caractères "abc" en entier, ce qui génère une exception de type ValueError. Le bloc except ValueError est exécuté dans ce cas, et le bloc except TypeError serait exécuté si une exception de type TypeError se produisait.

Il est également possible d'utiliser un bloc except sans spécifier le type d'exception, ce qui permet de capturer toutes les exceptions. Cependant, cela peut rendre le débogage plus difficile, car vous ne saurez pas quel type d'exception s'est produit.

En plus des blocs try-except, il est possible d'utiliser les blocs try-except-finally pour exécuter du code qui doit être exécuté qu'une exception se produise ou non. Le bloc finally est exécuté après le bloc try et le bloc except, qu'une exception se soit produite ou non.

try:
    # Code susceptible de générer une exception
    result = 10 / 2
except ZeroDivisionError:
    # Code de gestion de l'exception
    print("Erreur : division par zéro")
finally:
    # Code exécuté après le bloc try et le bloc except
    print("Fin de l'exécution")

Dans cet exemple, le code divise 10 par 2, ce qui ne génère pas d'exception. Le bloc finally est exécuté après le bloc try et le bloc except, et affiche "Fin de l'exécution".

En résumé, la gestion des erreurs et des exceptions en Python permet de détecter et de gérer les erreurs de manière appropriée. Les blocs try-except sont utilisés pour entourer le code susceptible de générer une exception, et les blocs finally sont utilisés pour exécuter du code après le bloc try et le bloc except, qu'une exception se soit produite ou non.
