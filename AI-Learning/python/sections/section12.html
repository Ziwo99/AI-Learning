Section 12: Les tests unitaires en Python

Introduction aux tests unitaires en Python

Les tests unitaires sont une pratique essentielle dans le développement logiciel. Ils permettent de vérifier le bon fonctionnement des différentes parties d'un programme de manière isolée. En Python, nous utilisons le module intégré unittest pour écrire et exécuter des tests unitaires.

Les bibliothèques de tests unitaires en Python

En Python, il existe plusieurs bibliothèques de tests unitaires populaires, telles que unittest, pytest et doctest. Dans cette formation, nous nous concentrerons sur l'utilisation de la bibliothèque unittest.

L'écriture de tests unitaires en Python

Pour écrire des tests unitaires en Python, nous devons créer une classe qui hérite de la classe unittest.TestCase. Chaque méthode de cette classe représente un test unitaire et doit commencer par le préfixe test_. À l'intérieur de chaque méthode de test, nous utilisons les méthodes d'assertion fournies par unittest.TestCase pour vérifier les résultats attendus.

import unittest

class MyTestCase(unittest.TestCase):
    def test_addition(self):
        result = 2 + 2
        self.assertEqual(result, 4)

    def test_subtraction(self):
        result = 5 - 3
        self.assertEqual(result, 2)

if __name__ == '__main__':
    unittest.main()

Dans cet exemple, nous avons créé une classe MyTestCase qui contient deux méthodes de test : test_addition et test_subtraction. À l'intérieur de chaque méthode, nous utilisons la méthode d'assertion assertEqual pour vérifier les résultats attendus.

Pour exécuter les tests unitaires, nous utilisons la fonction unittest.main() qui exécute tous les tests définis dans la classe.

Il est également possible d'organiser les tests en utilisant des méthodes spéciales telles que setUp et tearDown, qui sont exécutées avant et après chaque méthode de test, respectivement. Ces méthodes peuvent être utilisées pour préparer l'environnement de test et nettoyer les ressources utilisées.

En utilisant unittest, nous pouvons également effectuer des tests de comparaison de chaînes, de listes, de dictionnaires, etc., en utilisant les méthodes d'assertion appropriées telles que assertEqual, assertNotEqual, assertIn, assertNotIn, etc.

Il est recommandé d'écrire des tests unitaires pour chaque fonction ou méthode que vous développez, afin de vous assurer que votre code fonctionne correctement et de faciliter la détection des erreurs lors de modifications ultérieures.
