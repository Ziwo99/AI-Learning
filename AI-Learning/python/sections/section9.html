Section 9: Déploiement d'applications Python

Virtualenv

Virtualenv est un outil qui permet de créer des environnements Python isolés. Cela signifie que vous pouvez avoir plusieurs environnements Python sur votre machine, chacun avec ses propres packages et dépendances, sans qu'ils n'interfèrent les uns avec les autres.

Pour créer un nouvel environnement virtuel avec Virtualenv, utilisez la commande suivante :

virtualenv myenv

Cela créera un nouveau dossier appelé "myenv" qui contiendra l'environnement virtuel.

Pour activer l'environnement virtuel, utilisez la commande suivante :

source myenv/bin/activate

Vous verrez maintenant que votre invite de commande est préfixée par le nom de votre environnement virtuel.

Pour désactiver l'environnement virtuel, utilisez la commande suivante :

deactivate

Docker

Docker est une plateforme open-source qui permet d'automatiser le déploiement d'applications dans des conteneurs logiciels. Les conteneurs Docker sont légers, portables et autonomes, ce qui les rend idéaux pour le déploiement d'applications Python.

Pour utiliser Docker, vous devez d'abord installer Docker sur votre machine. Une fois installé, vous pouvez créer un fichier appelé "Dockerfile" qui contient les instructions pour construire votre conteneur Docker.

Voici un exemple de Dockerfile pour une application Python simple :

FROM python:3.9

WORKDIR /app

COPY requirements.txt .

RUN pip install -r requirements.txt

COPY . .

CMD ["python", "app.py"]

Ce Dockerfile utilise l'image de base Python 3.9, copie le fichier requirements.txt dans le conteneur, installe les dépendances, copie le reste des fichiers de l'application dans le conteneur, puis exécute le fichier app.py.

Pour construire l'image Docker, utilisez la commande suivante :

docker build -t myapp .

Cela construira l'image Docker avec le tag "myapp".

Pour exécuter le conteneur Docker, utilisez la commande suivante :

docker run myapp

Heroku

Heroku est une plateforme de cloud qui permet de déployer et de gérer des applications web. Elle prend en charge de nombreux langages de programmation, y compris Python.

Pour déployer une application Python sur Heroku, vous devez d'abord créer un compte Heroku et installer l'outil en ligne de commande Heroku.

Une fois cela fait, vous pouvez créer un fichier appelé "Procfile" qui contient les commandes pour exécuter votre application sur Heroku.

Voici un exemple de Procfile pour une application Flask :

web: gunicorn app:app

Ce Procfile indique à Heroku d'utiliser Gunicorn pour exécuter l'application Flask.

Pour déployer l'application sur Heroku, utilisez les commandes suivantes :

heroku login
heroku create
git push heroku main
heroku ps:scale web=1
heroku open

Cela vous connectera à votre compte Heroku, créera une nouvelle application, poussera votre code vers Heroku, mettra en place une instance de votre application et ouvrira l'URL de votre application dans votre navigateur.
