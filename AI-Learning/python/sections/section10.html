Section 10 : Les tests et débogage en Python

Introduction aux tests unitaires

Les tests unitaires sont une méthode de vérification de la fonctionnalité d'une unité de code, telle qu'une fonction ou une classe, de manière isolée. Ils permettent de s'assurer que chaque unité de code fonctionne correctement et de détecter les éventuelles erreurs ou bugs.

Voici un exemple de test unitaire en Python :

import unittest

def add(a, b):
    return a + b

class TestAdd(unittest.TestCase):
    def test_add(self):
        self.assertEqual(add(2, 3), 5)
        self.assertEqual(add(0, 0), 0)
        self.assertEqual(add(-1, 1), 0)

if __name__ == '__main__':
    unittest.main()

Dans cet exemple, nous avons une fonction add() qui additionne deux nombres. Nous utilisons la bibliothèque unittest pour créer une classe de test TestAdd qui hérite de la classe TestCase. Nous définissons ensuite une méthode test_add() qui contient les assertions pour vérifier les résultats attendus de la fonction add(). En exécutant ce fichier, les tests seront exécutés et les résultats seront affichés.

Utilisation de l'outil de débogage

L'outil de débogage en Python permet d'identifier et de résoudre les erreurs dans le code. Il permet de suivre l'exécution du programme pas à pas, d'inspecter les valeurs des variables et d'identifier les erreurs de logique.

Voici un exemple d'utilisation de l'outil de débogage en Python :

def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n-1)

print(factorial(5))

Dans cet exemple, nous avons une fonction factorial() qui calcule le factoriel d'un nombre. Si nous exécutons ce code, nous obtiendrons une erreur de dépassement de la limite de récursion. Pour déboguer ce code, nous pouvons utiliser l'outil de débogage en ajoutant un point d'arrêt à la ligne où l'erreur se produit, puis en exécutant le programme pas à pas pour identifier la cause de l'erreur.

Gestion des erreurs et exceptions

La gestion des erreurs et exceptions en Python permet de gérer les situations d'erreur de manière contrôlée. Elle permet d'intercepter les erreurs, d'afficher des messages d'erreur personnalisés et de prendre des mesures appropriées pour continuer l'exécution du programme.

Voici un exemple de gestion des erreurs et exceptions en Python :

try:
    num1 = int(input("Entrez un nombre : "))
    num2 = int(input("Entrez un autre nombre : "))
    result = num1 / num2
    print("Le résultat est :", result)
except ValueError:
    print("Erreur : veuillez entrer des nombres valides.")
except ZeroDivisionError:
    print("Erreur : division par zéro.")

Dans cet exemple, nous utilisons le bloc try-except pour gérer les erreurs potentielles lors de la conversion des entrées en nombres et lors de la division. Si une erreur de valeur non valide se produit, une exception de type ValueError sera levée et le message d'erreur correspondant sera affiché. Si une division par zéro se produit, une exception de type ZeroDivisionError sera levée et le message d'erreur correspondant sera affiché.
