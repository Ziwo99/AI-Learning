Gestion des sessions dans Sinatra

Les sessions sont un mécanisme essentiel pour gérer l'état des utilisateurs dans une application web. Dans Sinatra, la gestion des sessions est très simple grâce à l'utilisation du module intégré "sessions".

Pour activer les sessions dans votre application Sinatra, vous devez ajouter la ligne suivante au début de votre fichier principal : 

enable :sessions

Une fois les sessions activées, vous pouvez stocker des informations spécifiques à chaque utilisateur en utilisant la variable de session. Par exemple, pour stocker le nom d'utilisateur d'un utilisateur connecté, vous pouvez utiliser la syntaxe suivante : 

session[:username] = "JohnDoe"

Vous pouvez également accéder aux informations de session à partir de n'importe quelle route ou vue en utilisant la même syntaxe. Par exemple, pour afficher le nom d'utilisateur stocké dans la session, vous pouvez utiliser : 

<%= session[:username] %>

Authentification des utilisateurs

L'authentification des utilisateurs est un aspect crucial de nombreuses applications web. Sinatra facilite l'implémentation de l'authentification en utilisant des méthodes de filtrage et des sessions.

Pour créer un système d'authentification basique, vous pouvez utiliser un formulaire de connexion qui envoie les informations d'identification de l'utilisateur à une route spécifique. Dans cette route, vous pouvez vérifier les informations d'identification et, si elles sont valides, stocker l'identifiant de l'utilisateur dans la session. 

post '/login' do
  username = params[:username]
  password = params[:password]

  # Vérifier les informations d'identification de l'utilisateur
  if username == "admin" && password == "password"
    session[:user_id] = 1
    redirect '/dashboard'
  else
    redirect '/login'
  end
end

Dans cet exemple, si les informations d'identification sont valides, l'identifiant de l'utilisateur est stocké dans la session et l'utilisateur est redirigé vers le tableau de bord. Sinon, l'utilisateur est redirigé vers la page de connexion.

Utilisation de middlewares pour la gestion des sessions

Les middlewares sont des composants logiciels qui s'exécutent entre la requête et la réponse dans une application web. Ils sont utilisés pour effectuer des tâches telles que la gestion des sessions.

Dans Sinatra, vous pouvez utiliser le middleware intégré "Rack::Session::Cookie" pour gérer les sessions de manière transparente. Pour l'utiliser, vous devez ajouter la ligne suivante au début de votre fichier principal : 

use Rack::Session::Cookie, secret: "your_secret_key"

Assurez-vous de remplacer "your_secret_key" par une clé secrète unique pour votre application.

Une fois le middleware activé, vous pouvez accéder aux informations de session de la même manière qu'avec l'approche précédente.
