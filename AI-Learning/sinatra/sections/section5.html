Utilisation de vues dans Sinatra

Les vues dans Sinatra sont utilisées pour afficher le contenu HTML aux utilisateurs. Elles sont généralement écrites en utilisant le langage de template ERB (Embedded Ruby), qui permet d'insérer du code Ruby directement dans le HTML.

Voici un exemple de vue simple dans Sinatra :

<%# views/hello.erb %>
<h1>Bonjour, <%= @name %>!</h1>

Dans cet exemple, la variable d'instance @name est utilisée pour afficher le nom de l'utilisateur dans la vue. Le code Ruby est encadré par les balises <% et %> pour indiquer qu'il doit être évalué.

Utilisation de templates avec ERB

Les templates avec ERB permettent de réutiliser du code HTML commun à plusieurs vues. Ils peuvent être utilisés pour créer des layouts, des headers, des footers, etc.

Voici un exemple de template avec ERB :

<%# views/layout.erb %>
<!DOCTYPE html>
<html>
<head>
  <title>Mon Application Sinatra</title>
</head>
<body>
  <%= yield %>
</body>
</html>

Dans cet exemple, la balise <%= yield %> est utilisée pour indiquer où le contenu spécifique à chaque vue doit être inséré.

Utilisation de layouts pour la mise en page

Les layouts sont utilisés pour définir la mise en page générale de l'application. Ils peuvent contenir des éléments communs à toutes les vues, tels que des headers, des footers, des menus, etc.

Voici un exemple de layout dans Sinatra :

<%# views/layout.erb %>
<!DOCTYPE html>
<html>
<head>
  <title>Mon Application Sinatra</title>
</head>
<body>
  <header>
    <h1>Mon Application Sinatra</h1>
  </header>
  
  <nav>
    <a href="/home">Accueil</a>
    <a href="/about">À propos</a>
    <a href="/contact">Contact</a>
  </nav>
  
  <main>
    <%= yield %>
  </main>
  
  <footer>
    <p>© 2022 Mon Application Sinatra. Tous droits réservés.</p>
  </footer>
</body>
</html>

Dans cet exemple, le layout définit une structure de base pour toutes les vues de l'application, avec un header, une navigation, un contenu principal et un footer.
