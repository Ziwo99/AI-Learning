Définition des routes dans Sinatra

Les routes dans Sinatra sont utilisées pour définir les URL que l'application peut gérer. Une route est associée à une action spécifique de l'application, qui sera exécutée lorsque l'URL correspondante sera appelée.

Les routes dans Sinatra sont définies à l'aide de la méthode get, qui prend en paramètre l'URL à laquelle la route est associée et une action à exécuter. Par exemple, la route suivante associe l'URL "/hello" à une action qui affiche le message "Hello, World!" :

get '/hello' do
  "Hello, World!"
end

Il est également possible de définir des routes pour d'autres méthodes HTTP telles que post, put et delete. Par exemple, la route suivante associe l'URL "/users" à une action qui crée un nouvel utilisateur en utilisant la méthode HTTP POST :

post '/users' do
  # Créer un nouvel utilisateur
end

Gestion des différentes méthodes HTTP

Dans Sinatra, la gestion des différentes méthodes HTTP se fait en utilisant les méthodes correspondantes, telles que get, post, put et delete. Chaque méthode est associée à une route spécifique et à une action à exécuter lorsque cette route est appelée avec la méthode HTTP correspondante.

Par exemple, la route suivante utilise la méthode get pour afficher la liste des utilisateurs :

get '/users' do
  # Afficher la liste des utilisateurs
end

De même, la route suivante utilise la méthode post pour créer un nouvel utilisateur :

post '/users' do
  # Créer un nouvel utilisateur
end

Utilisation des paramètres de requête

Les paramètres de requête sont utilisés pour transmettre des données à une route via l'URL. Dans Sinatra, les paramètres de requête sont accessibles à l'aide de la variable params.

Par exemple, la route suivante utilise les paramètres de requête pour afficher le nom de l'utilisateur passé en tant que paramètre :

get '/hello' do
  "Hello, #{params['name']}!"
end

Si l'URL "/hello?name=John" est appelée, la route affichera "Hello, John!".

Utilisation des paramètres de route

Les paramètres de route sont utilisés pour capturer des parties variables d'une URL. Dans Sinatra, les paramètres de route sont définis en utilisant le symbole ":" suivi du nom du paramètre.

Par exemple, la route suivante utilise un paramètre de route pour afficher le profil de l'utilisateur correspondant à l'ID passé en tant que paramètre :

get '/users/:id' do
  # Afficher le profil de l'utilisateur avec l'ID correspondant à params['id']
end

Si l'URL "/users/123" est appelée, la route affichera le profil de l'utilisateur avec l'ID 123.

Les paramètres de route peuvent également être utilisés dans d'autres parties de l'URL, par exemple :

get '/users/:id/edit' do
  # Afficher le formulaire d'édition pour l'utilisateur avec l'ID correspondant à params['id']
end

Si l'URL "/users/123/edit" est appelée, la route affichera le formulaire d'édition pour l'utilisateur avec l'ID 123.
