Utilisation de modèles de données dans Sinatra

Les modèles de données sont un élément essentiel dans le développement d'applications web avec Sinatra. Ils permettent de représenter et de manipuler les données de manière structurée. Dans Sinatra, les modèles de données sont généralement créés à l'aide d'une bibliothèque d'ORM (Object-Relational Mapping) telle que ActiveRecord ou Sequel.

Pour utiliser des modèles de données dans Sinatra, il est nécessaire de définir une classe pour chaque table de la base de données que vous souhaitez manipuler. Cette classe doit hériter de la classe de modèle fournie par la bibliothèque ORM que vous avez choisie.

Par exemple, si vous utilisez ActiveRecord, vous pouvez définir une classe de modèle pour une table "users" de la manière suivante :

class User < ActiveRecord::Base
  # Définition des associations, validations, etc.
end

Une fois que vous avez défini vos classes de modèle, vous pouvez les utiliser dans vos routes pour effectuer des opérations de création, lecture, mise à jour et suppression de données.

Connexion à une base de données

Pour pouvoir utiliser des modèles de données dans Sinatra, il est nécessaire de configurer une connexion à la base de données. Cette étape peut varier en fonction de la bibliothèque ORM que vous utilisez.

Par exemple, si vous utilisez ActiveRecord, vous pouvez configurer la connexion à la base de données en spécifiant les informations de connexion dans un fichier de configuration, tel que "database.yml". Une fois la configuration effectuée, vous pouvez établir la connexion en appelant la méthode establish_connection dans votre application Sinatra.

require 'sinatra/activerecord'

class MyApp < Sinatra::Base
  configure :development do
    set :database, { adapter: 'sqlite3', database: 'myapp.db' }
  end

  # Établir la connexion à la base de données
  ActiveRecord::Base.establish_connection(settings.database)
end

Assurez-vous d'adapter la configuration de la base de données en fonction de votre environnement de développement (par exemple, "development", "production", etc.) et des informations de connexion spécifiques à votre base de données.

Création, lecture, mise à jour et suppression de données

Une fois que vous avez configuré la connexion à la base de données et défini vos classes de modèle, vous pouvez commencer à manipuler les données à l'aide des opérations CRUD (Create, Read, Update, Delete).

Voici quelques exemples d'opérations courantes :

Création de données

user = User.new(name: 'John', email: 'john@example.com')
user.save

Lecture de données

users = User.all
user = User.find(1)

Mise à jour de données

user = User.find(1)
user.name = 'John Doe'
user.save

Suppression de données

user = User.find(1)
user.destroy

Il est important de noter que ces exemples sont basés sur l'utilisation d'ActiveRecord. Si vous utilisez une autre bibliothèque ORM telle que Sequel, les méthodes et la syntaxe peuvent être légèrement différentes.
