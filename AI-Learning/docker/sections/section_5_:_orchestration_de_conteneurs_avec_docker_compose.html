Section 5 : Orchestration de conteneurs avec Docker Compose

Introduction à Docker Compose

Docker Compose est un outil qui permet de définir et de gérer des applications multi-conteneurs. Il facilite le déploiement et la gestion de plusieurs conteneurs Docker en utilisant un fichier de configuration simple et facile à comprendre. Avec Docker Compose, vous pouvez spécifier les services, les réseaux et les volumes nécessaires pour votre application, et les déployer en un seul clic.

Création d'un fichier de configuration Docker Compose

Pour commencer à utiliser Docker Compose, vous devez créer un fichier de configuration appelé "docker-compose.yml". Ce fichier contient les informations nécessaires pour déployer et gérer vos conteneurs. Voici un exemple de fichier de configuration Docker Compose pour une application web : 

version: '3'
services:
  web:
    build: .
    ports:
      - "80:80"
    volumes:
      - ./app:/app
  db:
    image: mysql
    environment:
      - MYSQL_ROOT_PASSWORD=secret
      - MYSQL_DATABASE=myapp

Dans cet exemple, nous avons deux services : "web" et "db". Le service "web" est construit à partir du répertoire actuel (".") et est exposé sur le port 80 de l'hôte. Le répertoire "app" est monté en tant que volume dans le conteneur. Le service "db" utilise l'image MySQL et définit les variables d'environnement pour le mot de passe root et la base de données.

Déploiement et gestion de plusieurs conteneurs avec Docker Compose

Une fois que vous avez créé votre fichier de configuration Docker Compose, vous pouvez déployer et gérer vos conteneurs en utilisant la commande "docker-compose". Voici quelques commandes utiles : 

# Déployer les conteneurs
docker-compose up

# Déployer les conteneurs en arrière-plan
docker-compose up -d

# Arrêter les conteneurs
docker-compose down

# Afficher les journaux des conteneurs
docker-compose logs

# Exécuter une commande dans un conteneur
docker-compose exec <service> <command>

Avec ces commandes, vous pouvez facilement déployer, arrêter, afficher les journaux et exécuter des commandes dans vos conteneurs Docker Compose. Cela facilite grandement la gestion de vos applications multi-conteneurs.

En conclusion, Docker Compose est un outil puissant pour l'orchestration de conteneurs Docker. Il simplifie le déploiement et la gestion de vos applications multi-conteneurs en utilisant un fichier de configuration simple. En utilisant Docker Compose, vous pouvez facilement créer des environnements de développement, de test et de production cohérents et reproductibles.
