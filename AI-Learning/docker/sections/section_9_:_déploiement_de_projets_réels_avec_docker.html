Section 9 : Déploiement de projets réels avec Docker

Déploiement d'une application web avec Docker

Lorsque vous souhaitez déployer une application web avec Docker, vous devez d'abord créer une image Docker contenant tous les composants nécessaires à votre application. Cette image sera ensuite utilisée pour exécuter des conteneurs Docker qui hébergeront votre application. Pour créer une image Docker, vous devez d'abord rédiger un fichier Dockerfile. Ce fichier contient les instructions pour construire votre image. Voici un exemple de Dockerfile pour une application web Node.js : 

FROM node:14

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD [ "npm", "start" ]
Dans cet exemple, nous utilisons l'image de base "node:14" qui contient Node.js. Nous définissons également le répertoire de travail à "/app" et copions les fichiers package.json et package-lock.json dans ce répertoire. Ensuite, nous exécutons la commande "npm install" pour installer les dépendances de notre application. Enfin, nous copions tous les fichiers de l'application dans le répertoire de travail, exposons le port 3000 et exécutons la commande "npm start" pour démarrer l'application. Une fois que vous avez rédigé votre Dockerfile, vous pouvez construire l'image Docker en exécutant la commande suivante dans le répertoire contenant le Dockerfile : 
docker build -t nom_image .
Après avoir construit l'image, vous pouvez exécuter un conteneur Docker à partir de cette image en utilisant la commande suivante : 
docker run -p 3000:3000 nom_image
Cela exposera le port 3000 du conteneur Docker sur le port 3000 de votre machine hôte, vous permettant d'accéder à votre application web via votre navigateur.

Déploiement d'une base de données avec Docker

Lorsque vous souhaitez déployer une base de données avec Docker, vous pouvez utiliser une image Docker préexistante contenant la base de données de votre choix, ou créer votre propre image personnalisée. Pour utiliser une image préexistante, vous pouvez rechercher sur le Docker Hub (https://hub.docker.com/) une image correspondant à la base de données que vous souhaitez utiliser. Par exemple, si vous souhaitez utiliser MySQL, vous pouvez rechercher l'image "mysql" sur le Docker Hub. Une fois que vous avez trouvé l'image souhaitée, vous pouvez exécuter un conteneur Docker à partir de cette image en utilisant la commande suivante : 

docker run -p 3306:3306 -e MYSQL_ROOT_PASSWORD=password mysql
Cela exposera le port 3306 du conteneur Docker sur le port 3306 de votre machine hôte et définira le mot de passe root de MySQL sur "password". Si vous souhaitez créer votre propre image personnalisée, vous devez rédiger un fichier Dockerfile contenant les instructions pour construire votre image. Par exemple, voici un Dockerfile pour une base de données MongoDB : 
FROM mongo:4

COPY init.js /docker-entrypoint-initdb.d/

EXPOSE 27017
Dans cet exemple, nous utilisons l'image de base "mongo:4" qui contient MongoDB. Nous copions également un fichier "init.js" dans le répertoire "/docker-entrypoint-initdb.d/" de l'image. Ce fichier sera exécuté lors du démarrage de la base de données et peut être utilisé pour initialiser des données. Une fois que vous avez rédigé votre Dockerfile, vous pouvez construire l'image Docker en exécutant la commande suivante dans le répertoire contenant le Dockerfile : 
docker build -t nom_image .
Après avoir construit l'image, vous pouvez exécuter un conteneur Docker à partir de cette image en utilisant la commande suivante : 
docker run -p 27017:27017 nom_image
Cela exposera le port 27017 du conteneur Docker sur le port 27017 de votre machine hôte, vous permettant d'accéder à votre base de données MongoDB.

Déploiement d'une architecture microservices avec Docker

Lorsque vous souhaitez déployer une architecture microservices avec Docker, vous devez créer une image Docker pour chaque service de votre architecture. Chaque image contiendra les composants nécessaires pour exécuter un service spécifique. Pour créer une image Docker pour un service, vous devez rédiger un Dockerfile spécifique à ce service. Ce fichier contient les instructions pour construire l'image du service. Voici un exemple de Dockerfile pour un service de gestion des utilisateurs : 

FROM node:14

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD [ "npm", "start" ]
Dans cet exemple, nous utilisons l'image de base "node:14" qui contient Node.js. Nous définissons également le répertoire de travail à "/app" et copions les fichiers package.json et package-lock.json dans ce répertoire. Ensuite, nous exécutons la commande "npm install" pour installer les dépendances du service. Enfin, nous copions tous les fichiers du service dans le répertoire de travail, exposons le port 3000 et exécutons la commande "npm start" pour démarrer le service. Une fois que vous avez rédigé le Dockerfile pour chaque service, vous pouvez construire les images Docker en exécutant la commande suivante dans le répertoire contenant chaque Dockerfile : 
docker build -t nom_image_service .
Après avoir construit les images, vous pouvez exécuter des conteneurs Docker à partir de ces images en utilisant la commande suivante pour chaque service : 
docker run -p port_service:port_service nom_image_service
Cela exposera le port spécifique du service du conteneur Docker sur le port spécifique de votre machine hôte, vous permettant d'accéder à chaque service individuellement.
