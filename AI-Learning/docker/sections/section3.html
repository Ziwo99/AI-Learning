Images Docker

Les images Docker sont des modèles de base utilisés pour créer des conteneurs Docker. Elles contiennent tout le nécessaire pour exécuter une application, y compris le code, les dépendances, les variables d'environnement et les fichiers système. Les images Docker sont créées à partir d'un fichier appelé Dockerfile, qui spécifie les étapes nécessaires pour construire l'image.

Création d'une image Docker

Pour créer une image Docker, vous devez d'abord créer un fichier Dockerfile. Ce fichier contient une série d'instructions qui indiquent à Docker comment construire l'image. Les instructions les plus couramment utilisées sont FROM, COPY, RUN et CMD.

FROM ubuntu:latest
COPY . /app
RUN make /app
CMD python /app/app.py

Dans cet exemple, nous utilisons l'image de base Ubuntu, copions le contenu du répertoire actuel dans le répertoire /app de l'image, exécutons la commande make pour construire l'application et définissons la commande par défaut pour exécuter le fichier app.py.

Utilisation d'images Docker existantes

Vous pouvez également utiliser des images Docker existantes à partir du Docker Hub, qui est un registre public d'images Docker. Pour utiliser une image existante, vous devez spécifier son nom dans le Dockerfile ou lors de la création d'un conteneur.

FROM nginx:latest

Dans cet exemple, nous utilisons l'image Nginx la plus récente à partir du Docker Hub.

Conteneurs Docker

Les conteneurs Docker sont des instances exécutables d'images Docker. Ils sont isolés les uns des autres et peuvent être démarrés, arrêtés, supprimés et gérés indépendamment. Chaque conteneur a son propre système de fichiers, ses propres processus et ses propres ressources.

Création d'un conteneur Docker

Pour créer un conteneur Docker, vous devez spécifier l'image à utiliser et les options de configuration souhaitées. Vous pouvez également lier des ports, monter des volumes et définir des variables d'environnement.

docker run -d -p 8080:80 --name my-container nginx:latest

Dans cet exemple, nous créons un conteneur à partir de l'image Nginx, le démarrons en arrière-plan, liant le port 8080 de l'hôte au port 80 du conteneur et lui donnons le nom "my-container".

Gestion des conteneurs Docker

Vous pouvez gérer vos conteneurs Docker en utilisant diverses commandes Docker. Par exemple, vous pouvez démarrer, arrêter, supprimer, suspendre, reprendre et redémarrer des conteneurs.

docker start my-container
docker stop my-container
docker rm my-container

Dans cet exemple, nous démarrons le conteneur "my-container", l'arrêtons et le supprimons.

Registres Docker

Les registres Docker sont des dépôts centralisés où vous pouvez stocker et partager des images Docker. Il existe des registres publics, tels que le Docker Hub, ainsi que des registres privés que vous pouvez configurer et gérer vous-même.

Utilisation de registres Docker publics

Pour utiliser un registre Docker public, vous devez spécifier le nom de l'image avec le nom du registre. Par exemple, pour utiliser une image à partir du Docker Hub :

docker pull nginx:latest

Cette commande télécharge l'image Nginx la plus récente à partir du Docker Hub.

Configuration d'un registre Docker privé

Pour configurer un registre Docker privé, vous devez d'abord installer et configurer Docker Registry sur votre serveur. Ensuite, vous pouvez pousser des images vers le registre privé et les utiliser comme n'importe quelle autre image Docker.

docker push my-registry/my-image:tag

Cette commande pousse l'image "my-image" avec l'étiquette "tag" vers le registre privé "my-registry".
