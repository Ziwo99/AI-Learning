Section 3: Concepts de base de Docker

Images Docker

Une image Docker est un modèle de base utilisé pour créer des conteneurs Docker. Elle contient tous les fichiers nécessaires pour exécuter une application, y compris le système d'exploitation, les bibliothèques, les dépendances et le code de l'application elle-même.

Pour créer une image Docker, vous pouvez utiliser un fichier appelé Dockerfile, qui contient les instructions pour construire l'image. Le Dockerfile spécifie les étapes nécessaires pour installer les dépendances, copier les fichiers de l'application et configurer l'environnement.

Voici un exemple de Dockerfile pour une application Node.js :

FROM node:12
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
CMD ["npm", "start"]

Création d'une image Docker

Pour créer une image Docker à partir d'un Dockerfile, vous pouvez utiliser la commande docker build. Cette commande va lire le Dockerfile, exécuter les instructions et créer une nouvelle image Docker.

Voici un exemple de commande pour créer une image Docker :

docker build -t monapp:1.0 .

Cette commande crée une nouvelle image Docker avec le tag "monapp:1.0" en utilisant le Dockerfile situé dans le répertoire courant.

Utilisation d'images Docker existantes

Vous pouvez également utiliser des images Docker existantes pour exécuter des conteneurs. Docker Hub est un registre public qui contient de nombreuses images Docker prêtes à l'emploi.

Pour utiliser une image Docker existante, vous pouvez utiliser la commande docker run. Cette commande va télécharger l'image si elle n'est pas déjà présente sur votre système, puis exécuter un conteneur basé sur cette image.

Voici un exemple de commande pour exécuter un conteneur à partir d'une image Docker :

docker run -d -p 8080:80 nginx

Cette commande exécute un conteneur basé sur l'image "nginx" et le relie au port 8080 de votre machine hôte.

Gestion des images Docker

Pour gérer les images Docker sur votre système, vous pouvez utiliser différentes commandes Docker.

Voici quelques commandes utiles :

  • docker images : affiche la liste des images Docker présentes sur votre système.
  • docker rmi <image_id> : supprime une image Docker spécifique.
  • docker prune : supprime toutes les images Docker inutilisées.

Conteneurs Docker

Un conteneur Docker est une instance en cours d'exécution d'une image Docker. Il s'agit d'un environnement isolé qui contient tous les fichiers et dépendances nécessaires pour exécuter une application.

Pour créer un conteneur Docker, vous pouvez utiliser la commande docker run. Cette commande va exécuter un nouveau conteneur basé sur une image Docker spécifique.

Voici un exemple de commande pour exécuter un conteneur Docker :

docker run -d -p 8080:80 nginx

Cette commande exécute un conteneur basé sur l'image "nginx" et le relie au port 8080 de votre machine hôte.

Création d'un conteneur Docker

Pour créer un conteneur Docker, vous pouvez utiliser la commande docker run avec des options supplémentaires pour spécifier le nom de l'image, les ports à exposer, les volumes à monter, etc.

Voici un exemple de commande pour créer un conteneur Docker :

docker run -d -p 8080:80 -v /chemin/vers/dossier:/app nginx

Cette commande crée un conteneur basé sur l'image "nginx", relie le port 8080 de votre machine hôte au port 80 du conteneur, et monte le dossier "/chemin/vers/dossier" du système hôte dans le dossier "/app" du conteneur.

Gestion des conteneurs Docker

Pour gérer les conteneurs Docker sur votre système, vous pouvez utiliser différentes commandes Docker.

Voici quelques commandes utiles :

  • docker ps : affiche la liste des conteneurs Docker en cours d'exécution.
  • docker stop <container_id> : arrête un conteneur Docker spécifique.
  • docker rm <container_id> : supprime un conteneur Docker spécifique.
  • docker logs <container_id> : affiche les logs d'un conteneur Docker spécifique.

Communication entre les conteneurs Docker

Les conteneurs Docker peuvent communiquer entre eux en utilisant des réseaux Docker. Chaque conteneur peut être connecté à un ou plusieurs réseaux Docker, ce qui leur permet de se parler et de partager des ressources.

Pour créer un réseau Docker, vous pouvez utiliser la commande docker network create. Cette commande va créer un nouveau réseau Docker.

Voici un exemple de commande pour créer un réseau Docker :

docker network create monreseau

Cette commande crée un nouveau réseau Docker appelé "monreseau".

Registres Docker

Un registre Docker est un service qui stocke et distribue des images Docker. Il existe des registres publics, tels que Docker Hub, qui contiennent de nombreuses images prêtes à l'emploi, ainsi que des registres privés que vous pouvez configurer pour stocker vos propres images.

Pour utiliser un registre Docker, vous pouvez utiliser la commande docker pull pour télécharger une image à partir du registre, et la commande docker push pour envoyer une image vers le registre.

Voici un exemple de commande pour télécharger une image à partir d'un registre Docker :

docker pull nginx

Cette commande télécharge l'image "nginx" à partir du registre Docker par défaut (Docker Hub).

Utilisation des registres Docker publics

Pour utiliser un registre Docker public, vous pouvez simplement spécifier le nom de l'image avec le nom du registre. Par exemple, pour télécharger une image à partir de Docker Hub, vous pouvez utiliser la commande docker pull suivie du nom de l'image :

docker pull nginx

Cette commande télécharge l'image "nginx" à partir de Docker Hub.

Configuration d'un registre Docker privé

Pour configurer un registre Docker privé, vous pouvez utiliser le logiciel open-source Docker Registry. Vous pouvez installer Docker Registry sur votre propre infrastructure et le configurer pour stocker vos propres images Docker.

Voici les étapes générales pour configurer un registre Docker privé :

  1. Installez Docker Registry sur votre serveur.
  2. Configurez Docker Registry pour utiliser un certificat SSL (facultatif mais recommandé pour la sécurité).
  3. Configurez Docker pour qu'il utilise votre registre privé en ajoutant l'option --insecure-registry à la configuration Docker.
  4. Créez une image Docker et poussez-la vers votre registre privé en utilisant la commande docker push.

Une fois que votre registre privé est configuré, vous pouvez utiliser la commande docker pull pour télécharger des images à partir de votre registre, en spécifiant le nom de l'image avec le nom de votre registre privé.

docker pull monregistre/nginx

Cette commande télécharge l'image "nginx" à partir de votre registre privé.
