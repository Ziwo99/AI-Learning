Démarrage et arrêt des conteneurs Docker

Les conteneurs Docker peuvent être démarrés et arrêtés à l'aide de la commande docker start et docker stop. Par exemple, pour démarrer un conteneur nommé "my_container", vous pouvez exécuter la commande suivante :

docker start my_container

De même, pour arrêter le conteneur, vous pouvez utiliser la commande :

docker stop my_container

Il est également possible de démarrer et arrêter plusieurs conteneurs en même temps en spécifiant leurs noms séparés par des espaces.

Gestion des ressources des conteneurs Docker

Les ressources des conteneurs Docker, telles que la mémoire et le CPU, peuvent être gérées à l'aide de la commande docker run. Vous pouvez spécifier les limites de ressources en utilisant les options --memory et --cpus. Par exemple, pour limiter un conteneur à 1 Go de mémoire et 2 CPU, vous pouvez exécuter la commande suivante :

docker run --memory=1g --cpus=2 my_image

Il est également possible de limiter les ressources d'un conteneur en cours d'exécution à l'aide de la commande docker update. Par exemple, pour limiter un conteneur nommé "my_container" à 512 Mo de mémoire, vous pouvez exécuter la commande suivante :

docker update --memory=512m my_container

Gestion des volumes Docker

Les volumes Docker permettent de stocker des données persistantes en dehors des conteneurs. Ils peuvent être utilisés pour partager des fichiers entre les conteneurs ou pour stocker des données de base de données. Pour créer un volume Docker, vous pouvez utiliser la commande docker volume create. Par exemple, pour créer un volume nommé "my_volume", vous pouvez exécuter la commande suivante :

docker volume create my_volume

Une fois le volume créé, vous pouvez le monter dans un conteneur en utilisant l'option --volume de la commande docker run. Par exemple, pour monter le volume "my_volume" dans un conteneur nommé "my_container", vous pouvez exécuter la commande suivante :

docker run --volume=my_volume:/path/in/container my_image

Les volumes Docker peuvent également être utilisés pour partager des fichiers entre l'hôte et les conteneurs. Pour monter un répertoire de l'hôte dans un conteneur, vous pouvez utiliser l'option --volume avec le chemin absolu du répertoire sur l'hôte. Par exemple, pour monter le répertoire "/path/on/host" dans un conteneur nommé "my_container", vous pouvez exécuter la commande suivante :

docker run --volume=/path/on/host:/path/in/container my_image

Gestion du réseau des conteneurs Docker

Les conteneurs Docker peuvent être connectés à différents réseaux pour permettre la communication entre eux. Par défaut, les conteneurs sont connectés au réseau par défaut de Docker. Vous pouvez créer vos propres réseaux en utilisant la commande docker network create. Par exemple, pour créer un réseau nommé "my_network", vous pouvez exécuter la commande suivante :

docker network create my_network

Une fois le réseau créé, vous pouvez connecter un conteneur à ce réseau en utilisant l'option --network de la commande docker run. Par exemple, pour connecter un conteneur nommé "my_container" au réseau "my_network", vous pouvez exécuter la commande suivante :

docker run --network=my_network my_image

Les conteneurs connectés au même réseau peuvent communiquer entre eux en utilisant les noms de conteneurs comme noms d'hôtes. Par exemple, si vous avez deux conteneurs nommés "container1" et "container2" connectés au même réseau, vous pouvez utiliser les noms "container1" et "container2" pour la communication entre les conteneurs.

Surveillance et journalisation des conteneurs Docker

Les conteneurs Docker peuvent être surveillés à l'aide de la commande docker stats. Cette commande affiche des informations en temps réel sur l'utilisation des ressources des conteneurs, telles que la mémoire, le CPU et le réseau. Par exemple, pour surveiller tous les conteneurs en cours d'exécution, vous pouvez exécuter la commande suivante :

docker stats

Les journaux des conteneurs Docker peuvent être consultés à l'aide de la commande docker logs. Cette commande affiche les journaux de sortie des conteneurs, ce qui peut être utile pour le débogage et la surveillance. Par exemple, pour afficher les journaux d'un conteneur nommé "my_container", vous pouvez exécuter la commande suivante :

docker logs my_container
