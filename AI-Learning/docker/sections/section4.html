Réseaux Docker

Les réseaux Docker permettent aux conteneurs de communiquer entre eux et avec le monde extérieur. Chaque conteneur Docker peut être connecté à un ou plusieurs réseaux Docker.

Configuration des réseaux Docker

Pour configurer un réseau Docker, vous pouvez utiliser la commande suivante :

docker network create nom_du_reseau

Cela créera un nouveau réseau Docker avec le nom spécifié.

Communication entre les conteneurs sur différents réseaux

Pour permettre la communication entre les conteneurs sur différents réseaux Docker, vous pouvez utiliser la fonctionnalité de liaison de réseaux Docker. Voici comment le faire :

docker network connect nom_du_reseau1 nom_du_conteneur2

Cela connectera le conteneur2 au réseau1, permettant ainsi la communication entre les conteneurs sur ces deux réseaux.

Volumes Docker

Les volumes Docker permettent de stocker des données persistantes en dehors des conteneurs. Ils sont utilisés pour partager des données entre les conteneurs ou pour stocker des données qui doivent survivre à la suppression d'un conteneur.

Création et utilisation de volumes Docker

Pour créer un volume Docker, vous pouvez utiliser la commande suivante :

docker volume create nom_du_volume

Cela créera un nouveau volume Docker avec le nom spécifié.

Pour utiliser un volume Docker dans un conteneur, vous pouvez spécifier le volume lors de la création du conteneur :

docker run -v nom_du_volume:chemin_du_volume nom_de_l_image

Cela montera le volume spécifié dans le chemin spécifié à l'intérieur du conteneur.

Partage de volumes entre les conteneurs

Pour partager un volume entre plusieurs conteneurs, vous pouvez spécifier le même volume lors de la création de chaque conteneur :

docker run -v nom_du_volume:chemin_du_volume1 nom_de_l_image1
docker run -v nom_du_volume:chemin_du_volume2 nom_de_l_image2

Cela permettra aux deux conteneurs d'accéder aux mêmes données stockées dans le volume.

Gestion des ressources système avec Docker

Docker permet de limiter les ressources CPU et mémoire utilisées par les conteneurs. Cela permet de garantir que les conteneurs ne monopolisent pas toutes les ressources du système.

Limiter les ressources CPU et mémoire des conteneurs

Pour limiter les ressources CPU et mémoire d'un conteneur, vous pouvez utiliser les options suivantes lors de la création du conteneur :

docker run --cpus=nombre_de_cpus --memory=quantité_de_mémoire nom_de_l_image

Cela limitera le conteneur à utiliser le nombre de CPUs spécifié et la quantité de mémoire spécifiée.

Surveillance des ressources système

Pour surveiller les ressources système utilisées par les conteneurs Docker, vous pouvez utiliser la commande suivante :

docker stats

Cela affichera les statistiques en temps réel de tous les conteneurs en cours d'exécution, y compris l'utilisation de la CPU, de la mémoire et du réseau.
