Section 3 : Utilisation de conteneurs Docker

Création d'un conteneur Docker

La création d'un conteneur Docker est une étape essentielle dans l'utilisation de Docker. Pour créer un conteneur Docker, vous devez d'abord avoir une image Docker appropriée. Une image Docker est un modèle de base à partir duquel les conteneurs sont créés.

Pour créer un conteneur Docker, vous pouvez utiliser la commande docker run suivie du nom de l'image Docker que vous souhaitez utiliser. Par exemple :

docker run nginx

Cette commande crée un conteneur à partir de l'image Docker "nginx". Le conteneur sera exécuté en utilisant les paramètres par défaut de l'image.

Vous pouvez également spécifier des options supplémentaires lors de la création d'un conteneur Docker. Par exemple, vous pouvez spécifier un nom pour le conteneur en utilisant l'option --name. Voici un exemple :

docker run --name mon-conteneur nginx

Cette commande crée un conteneur à partir de l'image Docker "nginx" et lui donne le nom "mon-conteneur".

Gestion des conteneurs Docker

Une fois que vous avez créé des conteneurs Docker, vous pouvez les gérer en utilisant différentes commandes Docker.

Pour afficher la liste des conteneurs en cours d'exécution, vous pouvez utiliser la commande docker ps. Cette commande affiche des informations telles que le nom du conteneur, l'image utilisée, le statut du conteneur, etc. Voici un exemple :

docker ps

Pour arrêter un conteneur en cours d'exécution, vous pouvez utiliser la commande docker stop suivie du nom ou de l'ID du conteneur. Par exemple :

docker stop mon-conteneur

Cette commande arrête le conteneur nommé "mon-conteneur".

Pour supprimer un conteneur, vous pouvez utiliser la commande docker rm suivie du nom ou de l'ID du conteneur. Par exemple :

docker rm mon-conteneur

Cette commande supprime le conteneur nommé "mon-conteneur".

Utilisation des images Docker

Les images Docker sont la base des conteneurs Docker. Elles contiennent tous les fichiers et les configurations nécessaires pour exécuter un conteneur.

Pour rechercher des images Docker disponibles, vous pouvez utiliser la commande docker search suivie du nom de l'image que vous recherchez. Par exemple :

docker search ubuntu

Cette commande recherche des images Docker contenant le mot-clé "ubuntu". Elle affiche une liste d'images correspondantes avec des informations supplémentaires.

Pour télécharger une image Docker, vous pouvez utiliser la commande docker pull suivie du nom de l'image que vous souhaitez télécharger. Par exemple :

docker pull nginx

Cette commande télécharge l'image Docker "nginx" sur votre système.

Partage d'images Docker

Une fois que vous avez créé ou téléchargé une image Docker, vous pouvez la partager avec d'autres utilisateurs ou la publier sur un registre Docker public ou privé.

Pour partager une image Docker, vous devez d'abord la taguer avec un nom approprié. Vous pouvez utiliser la commande docker tag suivie du nom de l'image et du nouveau nom/tag. Par exemple :

docker tag mon-image utilisateur/mon-image

Cette commande ajoute un nouveau tag "utilisateur/mon-image" à l'image Docker "mon-image".

Pour publier une image Docker sur un registre, vous pouvez utiliser la commande docker push suivie du nom/tag de l'image et du nom du registre. Par exemple :

docker push utilisateur/mon-image

Cette commande publie l'image Docker "utilisateur/mon-image" sur le registre spécifié.

Vous pouvez également partager des images Docker en exportant et en important des fichiers d'archive. Pour exporter une image Docker, vous pouvez utiliser la commande docker save suivie du nom/tag de l'image et du nom du fichier de sortie. Par exemple :

docker save -o mon-image.tar utilisateur/mon-image

Cette commande exporte l'image Docker "utilisateur/mon-image" dans le fichier "mon-image.tar".

Pour importer une image Docker à partir d'un fichier d'archive, vous pouvez utiliser la commande docker load suivie du nom du fichier d'entrée. Par exemple :

docker load -i mon-image.tar

Cette commande importe l'image Docker à partir du fichier "mon-image.tar".
