Encapsulation

L'encapsulation est un concept clé de la programmation orientée objet (POO) qui permet de regrouper des données et des méthodes associées dans une seule entité appelée classe. En Dart, l'encapsulation est réalisée en déclarant des variables et des méthodes comme étant privées, protégées ou publiques.

Les variables privées sont accessibles uniquement à l'intérieur de la classe qui les définit. Elles sont déclarées en utilisant le mot-clé "var" ou "final" avec un underscore (_) devant leur nom. Par exemple :

class Person {
  var _name;
  
  void _privateMethod() {
    // Code du méthode privée
  }
}

Les variables protégées sont accessibles à l'intérieur de la classe qui les définit et de ses sous-classes. Elles sont déclarées en utilisant le mot-clé "var" ou "final" avec un underscore (_) devant leur nom. Par exemple :

class Person {
  var _name;
  
  void _privateMethod() {
    // Code du méthode privée
  }
}

class Employee extends Person {
  var _salary;
  
  void _protectedMethod() {
    // Code du méthode protégée
  }
}

Les variables publiques sont accessibles à partir de n'importe où dans le programme. Elles sont déclarées en utilisant le mot-clé "var" ou "final" sans underscore (_) devant leur nom. Par exemple :

class Person {
  var name;
  
  void publicMethod() {
    // Code du méthode publique
  }
}

Héritage

L'héritage est un autre concept clé de la POO qui permet à une classe d'hériter des propriétés et des méthodes d'une autre classe appelée classe parent. En Dart, l'héritage est réalisé en utilisant le mot-clé "extends". Par exemple :

class Animal {
  void eat() {
    print('L'animal mange.');
  }
}

class Dog extends Animal {
  void bark() {
    print('Le chien aboie.');
  }
}

Dans cet exemple, la classe "Dog" hérite de la classe "Animal" et a accès à la méthode "eat()". Elle ajoute également sa propre méthode "bark()".

Polymorphisme

Le polymorphisme est un concept qui permet à une classe d'être utilisée comme une instance de sa classe parent ou de l'une de ses sous-classes. En Dart, le polymorphisme est réalisé en utilisant des classes abstraites et des méthodes abstraites.

Une classe abstraite est une classe qui ne peut pas être instanciée directement, mais qui peut être utilisée comme type de données. Elle est déclarée en utilisant le mot-clé "abstract". Par exemple :

abstract class Animal {
  void makeSound();
}

class Dog extends Animal {
  void makeSound() {
    print('Le chien aboie.');
  }
}

class Cat extends Animal {
  void makeSound() {
    print('Le chat miaule.');
  }
}

Dans cet exemple, la classe "Animal" est une classe abstraite avec une méthode abstraite "makeSound()". Les classes "Dog" et "Cat" héritent de la classe "Animal" et implémentent la méthode "makeSound()".

Interfaces

Les interfaces sont un moyen de définir un contrat que les classes doivent respecter. En Dart, les interfaces sont réalisées en utilisant des classes abstraites avec des méthodes abstraites.

Une classe abstraite avec des méthodes abstraites est utilisée comme une interface. Les classes qui implémentent cette interface doivent fournir une implémentation pour toutes les méthodes abstraites. Par exemple :

abstract class Animal {
  void makeSound();
}

class Dog implements Animal {
  void makeSound() {
    print('Le chien aboie.');
  }
}

class Cat implements Animal {
  void makeSound() {
    print('Le chat miaule.');
  }
}

Dans cet exemple, la classe "Animal" est utilisée comme une interface avec une méthode abstraite "makeSound()". Les classes "Dog" et "Cat" implémentent cette interface en fournissant une implémentation pour la méthode "makeSound()".

Mixins

Les mixins sont un moyen de réutiliser du code entre différentes classes sans utiliser l'héritage. En Dart, les mixins sont réalisés en utilisant le mot-clé "with".

Un mixin est une classe qui fournit des fonctionnalités supplémentaires à d'autres classes. Il est utilisé en l'incluant avec le mot-clé "with" dans une classe. Par exemple :

mixin Swim {
  void swim() {
    print('L'animal nage.');
  }
}

class Dolphin extends Animal with Swim {
  // Code de la classe Dolphin
}

Dans cet exemple, le mixin "Swim" fournit la méthode "swim()" à la classe "Dolphin". La classe "Dolphin" peut maintenant utiliser la méthode "swim()" sans hériter de la classe "Swim".
