Section 6: Accès aux données avec ADO.NET

Introduction à ADO.NET

ADO.NET est une technologie de Microsoft qui permet d'accéder aux bases de données relationnelles à partir d'applications C#. Il fournit une interface pour se connecter aux bases de données, exécuter des requêtes SQL, récupérer et manipuler les données.

Pour utiliser ADO.NET, vous devez d'abord ajouter une référence à la bibliothèque System.Data dans votre projet C#. Ensuite, vous pouvez utiliser les classes fournies par ADO.NET pour interagir avec la base de données.

Connexion à une base de données

Pour se connecter à une base de données avec ADO.NET, vous devez utiliser la classe SqlConnection. Voici un exemple de code pour établir une connexion à une base de données :

string connectionString = "Data Source=nomServeur;Initial Catalog=nomBaseDeDonnées;User ID=nomUtilisateur;Password=motDePasse";
SqlConnection connection = new SqlConnection(connectionString);
connection.Open();

Assurez-vous de remplacer les valeurs "nomServeur", "nomBaseDeDonnées", "nomUtilisateur" et "motDePasse" par les informations appropriées pour votre base de données.

Exécution de requêtes SQL

Pour exécuter des requêtes SQL avec ADO.NET, vous devez utiliser la classe SqlCommand. Voici un exemple de code pour exécuter une requête SELECT :

string query = "SELECT * FROM nomTable";
SqlCommand command = new SqlCommand(query, connection);
SqlDataReader reader = command.ExecuteReader();

while (reader.Read())
{
    // Traiter les données récupérées
}

reader.Close();

Assurez-vous de remplacer "nomTable" par le nom de la table de votre base de données.

Utilisation des transactions

Les transactions permettent de regrouper plusieurs opérations de base de données en une seule unité logique. ADO.NET prend en charge les transactions à l'aide de la classe SqlTransaction. Voici un exemple de code pour utiliser des transactions :

SqlTransaction transaction = connection.BeginTransaction();

try
{
    // Effectuer des opérations de base de données

    transaction.Commit();
}
catch (Exception ex)
{
    transaction.Rollback();
    // Gérer l'erreur
}
finally
{
    transaction.Dispose();
}

Assurez-vous d'inclure vos opérations de base de données à l'intérieur du bloc try. Si une erreur se produit, la transaction sera annulée en appelant la méthode Rollback(). Si tout se passe bien, la transaction sera validée en appelant la méthode Commit().
