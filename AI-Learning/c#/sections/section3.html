Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. La POO repose sur quatre principes fondamentaux : l'encapsulation, l'abstraction, l'héritage et le polymorphisme.

Classes et objets

En C#, une classe est une structure de données qui définit les propriétés et les comportements d'un objet. Les propriétés sont des variables qui décrivent l'état de l'objet, tandis que les comportements sont des méthodes qui définissent les actions que l'objet peut effectuer. Pour créer un objet à partir d'une classe, on utilise le mot-clé "new".

public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    public void SayHello()
    {
        Console.WriteLine("Hello, my name is " + Name + " and I am " + Age + " years old.");
    }
}

Person person = new Person();
person.Name = "John";
person.Age = 25;
person.SayHello();

Encapsulation et abstraction

L'encapsulation est un mécanisme qui permet de regrouper les données et les méthodes qui les manipulent au sein d'une même entité, appelée classe. Cela permet de cacher les détails internes de la classe et de ne rendre accessibles que les fonctionnalités nécessaires à l'utilisation de l'objet. L'abstraction est un concept qui consiste à représenter les caractéristiques essentielles d'un objet sans se soucier des détails d'implémentation.

public class BankAccount
{
    private decimal balance;

    public void Deposit(decimal amount)
    {
        balance += amount;
    }

    public void Withdraw(decimal amount)
    {
        if (amount <= balance)
        {
            balance -= amount;
        }
        else
        {
            Console.WriteLine("Insufficient funds.");
        }
    }
}

BankAccount account = new BankAccount();
account.Deposit(1000);
account.Withdraw(500);

Héritage et polymorphisme

L'héritage est un mécanisme qui permet de créer de nouvelles classes à partir de classes existantes. La classe existante est appelée classe de base ou classe parent, tandis que la nouvelle classe est appelée classe dérivée ou classe enfant. La classe dérivée hérite des propriétés et des comportements de la classe parent, et peut également ajouter de nouvelles fonctionnalités ou modifier les fonctionnalités existantes. Le polymorphisme est la capacité d'un objet à prendre différentes formes en fonction du contexte.

public class Shape
{
    public virtual void Draw()
    {
        Console.WriteLine("Drawing a shape.");
    }
}

public class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle.");
    }
}

public class Rectangle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a rectangle.");
    }
}

Shape shape1 = new Circle();
Shape shape2 = new Rectangle();
shape1.Draw();
shape2.Draw();

Interfaces et classes abstraites

Une interface est une collection de membres (propriétés, méthodes, événements) qui définissent un contrat que les classes doivent respecter. Une classe abstraite est une classe qui ne peut pas être instanciée et qui sert de modèle pour d'autres classes. Les classes qui héritent d'une classe abstraite doivent implémenter ses membres abstraits. Les interfaces et les classes abstraites permettent de définir des comportements communs à plusieurs classes.

public interface IShape
{
    void Draw();
}

public abstract class Shape
{
    public abstract void Draw();
}

public class Circle : Shape, IShape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle.");
    }
}

public class Rectangle : Shape, IShape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a rectangle.");
    }
}

IShape shape1 = new Circle();
IShape shape2 = new Rectangle();
shape1.Draw();
shape2.Draw();

Gestion des exceptions

La gestion des exceptions permet de détecter et de traiter les erreurs qui se produisent pendant l'exécution d'un programme. En C#, les exceptions sont des objets qui représentent des erreurs ou des situations exceptionnelles. On utilise des blocs try-catch pour capturer et gérer les exceptions. Le bloc try contient le code qui peut générer une exception, tandis que le bloc catch permet de spécifier le code à exécuter en cas d'exception.

try
{
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[4]);
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine("An error occurred: " + ex.Message);
}
