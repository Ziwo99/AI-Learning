Section 3: Programmation orientée objet en C#

Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. En C#, la POO est au cœur du langage et est largement utilisée pour développer des applications robustes et modulaires.

Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. Les objets ont des propriétés (attributs) et des comportements (méthodes). La POO permet de regrouper des données et des fonctionnalités connexes dans des objets, ce qui facilite la compréhension et la maintenance du code.

Classes et objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les propriétés et les comportements communs à tous les objets de cette classe. En C#, une classe est déclarée à l'aide du mot-clé "class" suivi du nom de la classe. 

public class Person
{
    // Propriétés
    public string Name { get; set; }
    public int Age { get; set; }
    
    // Méthodes
    public void SayHello()
    {
        Console.WriteLine("Hello, my name is " + Name + " and I am " + Age + " years old.");
    }
}

Un objet est une instance d'une classe. Pour créer un objet, nous utilisons le mot-clé "new" suivi du nom de la classe et des parenthèses. Nous pouvons ensuite accéder aux propriétés et aux méthodes de l'objet à l'aide de l'opérateur ".". 

// Création d'un objet Person
Person person = new Person();

// Accès aux propriétés
person.Name = "John";
person.Age = 25;

// Appel d'une méthode
person.SayHello(); // Output: Hello, my name is John and I am 25 years old.

Encapsulation et abstraction

L'encapsulation est un concept clé de la POO qui consiste à regrouper les données et les fonctionnalités connexes dans une classe et à cacher les détails d'implémentation à l'extérieur de la classe. Cela permet de protéger les données et de garantir leur intégrité en empêchant un accès direct non autorisé.

En C#, nous utilisons des modificateurs d'accès tels que "public", "private" et "protected" pour contrôler l'accès aux membres d'une classe. Par défaut, les membres sont "private" et ne peuvent être accédés que depuis l'intérieur de la classe.

L'abstraction est un autre concept clé de la POO qui consiste à représenter les caractéristiques essentielles d'un objet sans se soucier des détails d'implémentation. En C#, nous utilisons des classes abstraites et des interfaces pour créer des abstractions. 

public abstract class Shape
{
    // Méthode abstraite
    public abstract double CalculateArea();
}

public class Circle : Shape
{
    public double Radius { get; set; }
    
    public override double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }
}

Dans cet exemple, la classe abstraite "Shape" définit une méthode abstraite "CalculateArea()" qui doit être implémentée par toutes les classes dérivées. La classe "Circle" hérite de la classe "Shape" et implémente la méthode "CalculateArea()" en calculant l'aire d'un cercle.

Héritage et polymorphisme

L'héritage est un mécanisme de la POO qui permet à une classe d'hériter des propriétés et des comportements d'une autre classe. En C#, une classe peut hériter d'une seule classe de base, mais peut implémenter plusieurs interfaces.

Le polymorphisme est un autre concept clé de la POO qui permet à un objet d'être traité comme un objet de son type de base ou comme un objet de l'un de ses types dérivés. Cela permet d'écrire du code générique qui peut être utilisé avec différents types d'objets. 

public class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("The animal makes a sound.");
    }
}

public class Dog : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("The dog barks.");
    }
}

public class Cat : Animal
{
    public override void MakeSound()
    {
        Console.WriteLine("The cat meows.");
    }
}

Dans cet exemple, la classe "Animal" définit une méthode virtuelle "MakeSound()" qui peut être redéfinie par les classes dérivées. Les classes "Dog" et "Cat" héritent de la classe "Animal" et redéfinissent la méthode "MakeSound()" pour produire des sons spécifiques à chaque animal.

Le polymorphisme nous permet d'appeler la méthode "MakeSound()" sur un objet de type "Animal" et d'exécuter la version appropriée de la méthode en fonction du type réel de l'objet. 

Animal animal1 = new Animal();
Animal animal2 = new Dog();
Animal animal3 = new Cat();

animal1.MakeSound(); // Output: The animal makes a sound.
animal2.MakeSound(); // Output: The dog barks.
animal3.MakeSound(); // Output: The cat meows.

Interfaces et classes abstraites

Les interfaces et les classes abstraites sont utilisées pour créer des abstractions en C#. Une interface définit un contrat que les classes doivent respecter, tandis qu'une classe abstraite fournit une implémentation partielle ou complète des fonctionnalités.

Une classe peut implémenter plusieurs interfaces, mais ne peut hériter que d'une seule classe abstraite. Les interfaces sont déclarées à l'aide du mot-clé "interface" suivi du nom de l'interface, tandis que les classes abstraites sont déclarées à l'aide du mot-clé "abstract" suivi du nom de la classe. 

public interface IShape
{
    double CalculateArea();
}

public abstract class Shape : IShape
{
    public abstract double CalculateArea();
}

public class Circle : Shape
{
    public double Radius { get; set; }
    
    public override double CalculateArea()
    {
        return Math.PI * Radius * Radius;
    }
}

Dans cet exemple, l'interface "IShape" définit une méthode "CalculateArea()" que les classes doivent implémenter. La classe abstraite "Shape" implémente l'interface "IShape" et fournit une implémentation partielle de la méthode "CalculateArea()". La classe "Circle" hérite de la classe "Shape" et implémente la méthode "CalculateArea()" pour calculer l'aire d'un cercle.

Ceci conclut la section sur la programmation orientée objet en C#. Nous avons couvert les concepts clés tels que les classes, les objets, l'encapsulation, l'abstraction, l'héritage, le polymorphisme, les interfaces et les classes abstraites. Ces concepts sont essentiels pour développer des applications modulaires et extensibles en C#.
