Section 4: Gestion des exceptions

Introduction aux exceptions

Lorsque vous développez des applications en C#, il est important de prendre en compte la gestion des exceptions. Une exception est une erreur qui se produit pendant l'exécution d'un programme et qui peut interrompre son flux normal. Les exceptions peuvent être causées par divers facteurs, tels que des erreurs de syntaxe, des erreurs de logique ou des erreurs d'exécution.

En C#, les exceptions sont gérées à l'aide de blocs try-catch. Un bloc try est utilisé pour entourer le code qui peut générer une exception, tandis qu'un bloc catch est utilisé pour capturer et gérer l'exception. Si une exception se produit dans le bloc try, le programme saute immédiatement au bloc catch correspondant et exécute le code à l'intérieur de ce bloc.

Voici un exemple de code qui illustre l'utilisation des blocs try-catch pour gérer une exception : 

try {
    // Code susceptible de générer une exception
} catch (Exception ex) {
    // Code de gestion de l'exception
}

Gestion des exceptions avec try-catch

Dans la gestion des exceptions, il est important de capturer et de traiter les exceptions de manière appropriée. Lorsque vous utilisez un bloc catch, vous pouvez spécifier le type d'exception que vous souhaitez capturer. Cela vous permet de gérer différents types d'exceptions de manière spécifique.

Voici un exemple de code qui illustre la gestion des exceptions avec try-catch en utilisant différents blocs catch pour différents types d'exceptions : 

try {
    // Code susceptible de générer une exception
} catch (DivideByZeroException ex) {
    // Code de gestion de l'exception DivideByZeroException
} catch (FileNotFoundException ex) {
    // Code de gestion de l'exception FileNotFoundException
} catch (Exception ex) {
    // Code de gestion de toutes les autres exceptions
}

Utilisation des exceptions personnalisées

En plus des exceptions prédéfinies dans le langage C#, vous pouvez également créer vos propres exceptions personnalisées. Les exceptions personnalisées sont utiles lorsque vous souhaitez gérer des erreurs spécifiques à votre application.

Pour créer une exception personnalisée, vous devez créer une classe qui dérive de la classe Exception. Vous pouvez ajouter des propriétés et des méthodes supplémentaires à votre exception personnalisée pour fournir des informations supplémentaires sur l'erreur.

Voici un exemple de code qui illustre la création et l'utilisation d'une exception personnalisée : 

public class MonExceptionPersonnalisee : Exception {
    public MonExceptionPersonnalisee(string message) : base(message) {
        // Initialisation de la classe de base avec le message d'erreur
    }
}

try {
    // Code susceptible de générer une exception personnalisée
    throw new MonExceptionPersonnalisee("Une erreur s'est produite !");
} catch (MonExceptionPersonnalisee ex) {
    // Code de gestion de l'exception personnalisée
}

Il est important de noter que vous devez utiliser le mot-clé throw pour générer une exception personnalisée. Vous pouvez également ajouter des informations supplémentaires à votre exception en utilisant les propriétés de la classe Exception, telles que Message, InnerException, StackTrace, etc.
