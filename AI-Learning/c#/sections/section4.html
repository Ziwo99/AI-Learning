Section 4: Gestion des exceptions en C#

Introduction aux exceptions

Les exceptions sont des événements qui se produisent lorsqu'une erreur ou une condition anormale se produit pendant l'exécution d'un programme. En C#, les exceptions sont utilisées pour gérer ces situations et permettre au programme de réagir de manière appropriée.

Lorsqu'une exception se produit, le programme peut être interrompu et une trace d'exécution détaillée de l'erreur peut être affichée. Cela permet aux développeurs de comprendre ce qui s'est passé et de corriger le problème.

Gestion des exceptions avec try-catch

En C#, la gestion des exceptions se fait à l'aide des blocs try-catch. Le code potentiellement problématique est placé dans le bloc try, et si une exception se produit, elle est capturée et gérée dans le bloc catch.

Voici un exemple de code utilisant try-catch pour gérer une exception de division par zéro : 


try {
    int result = 10 / 0;
    Console.WriteLine(result);
}
catch (DivideByZeroException ex) {
    Console.WriteLine("Une exception de division par zéro s'est produite : " + ex.Message);
}

Dans cet exemple, le code tente de diviser 10 par zéro, ce qui provoque une exception de division par zéro. Cette exception est capturée dans le bloc catch, où un message d'erreur approprié est affiché.

Utilisation des exceptions personnalisées

En plus des exceptions prédéfinies telles que DivideByZeroException, il est également possible de créer des exceptions personnalisées en C#. Cela permet aux développeurs de définir des erreurs spécifiques à leur application et de les gérer de manière appropriée.

Voici un exemple de code montrant comment créer et utiliser une exception personnalisée : 


public class CustomException : Exception {
    public CustomException(string message) : base(message) { }
}

try {
    throw new CustomException("Une erreur personnalisée s'est produite !");
}
catch (CustomException ex) {
    Console.WriteLine("Une exception personnalisée s'est produite : " + ex.Message);
}

Dans cet exemple, une classe CustomException est créée en dérivant de la classe Exception. Cette classe personnalisée peut avoir des propriétés et des méthodes supplémentaires pour gérer l'erreur de manière spécifique.

Ensuite, une instance de CustomException est lancée à l'aide du mot-clé throw, et elle est capturée et gérée dans le bloc catch approprié.

Il est recommandé d'utiliser des exceptions personnalisées lorsque des erreurs spécifiques à votre application se produisent, afin de pouvoir les gérer de manière appropriée et de fournir des informations précises sur l'erreur.
