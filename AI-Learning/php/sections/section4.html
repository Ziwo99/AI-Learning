Introduction à la POO

La programmation orientée objet (POO) est un paradigme de programmation qui permet d'organiser et de structurer son code de manière plus modulaire et réutilisable. En POO, on travaille avec des objets qui sont des instances de classes. Une classe est un modèle ou un plan qui définit les propriétés et les comportements des objets.

Classes et objets

Une classe est une structure qui regroupe des variables (appelées propriétés ou attributs) et des fonctions (appelées méthodes) qui agissent sur ces variables. Pour créer un objet à partir d'une classe, on utilise le mot-clé new.

class Person {
    // Propriétés
    public $name;
    public $age;

    // Méthodes
    public function sayHello() {
        echo "Hello, my name is " . $this->name;
    }
}

// Création d'un objet
$person = new Person();
$person->name = "John";
$person->age = 25;
$person->sayHello(); // Output: Hello, my name is John

Encapsulation, héritage et polymorphisme

L'encapsulation est un concept qui consiste à regrouper les données et les méthodes qui les manipulent au sein d'une même entité, la classe. On utilise des modificateurs d'accès (public, protected, private) pour définir la visibilité des propriétés et des méthodes.

L'héritage permet de créer de nouvelles classes en se basant sur une classe existante. La classe existante est appelée classe parente ou superclasse, et la nouvelle classe est appelée classe enfant ou sous-classe. La classe enfant hérite des propriétés et des méthodes de la classe parente, et peut en ajouter de nouvelles ou les modifier.

Le polymorphisme permet à des objets de différentes classes d'être traités de manière uniforme. Cela signifie qu'un objet peut être utilisé comme une instance de sa classe, mais aussi comme une instance d'une de ses classes parentes.

Interfaces et traits

Une interface est une structure qui définit un ensemble de méthodes que les classes qui l'implémentent doivent obligatoirement définir. Une classe peut implémenter plusieurs interfaces.

Un trait est une sorte de classe partielle qui peut être réutilisée dans plusieurs classes. Un trait peut contenir des méthodes et des propriétés, et ces dernières peuvent être utilisées par les classes qui utilisent le trait.

Gestion des erreurs et exceptions

En PHP, les erreurs sont des problèmes qui se produisent lors de l'exécution du code et qui empêchent le bon fonctionnement de l'application. Les exceptions sont des erreurs qui peuvent être gérées de manière plus précise et contrôlée.

Pour gérer les erreurs et les exceptions, on utilise les blocs try et catch. Le code qui peut générer une exception est placé dans le bloc try, et le code qui gère l'exception est placé dans le bloc catch.

try {
    // Code susceptible de générer une exception
    throw new Exception("Une erreur s'est produite");
} catch (Exception $e) {
    // Gestion de l'exception
    echo "Erreur : " . $e->getMessage();
}

Il est également possible de créer ses propres exceptions en définissant des classes qui étendent la classe Exception.

Ceci conclut la section sur la programmation orientée objet en PHP. Nous avons vu les concepts fondamentaux de la POO, tels que les classes, les objets, l'encapsulation, l'héritage, le polymorphisme, les interfaces, les traits, ainsi que la gestion des erreurs et des exceptions.
