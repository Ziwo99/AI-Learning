Structure d'une application Ruby on Rails

Une application Ruby on Rails suit une structure spécifique qui facilite le développement et la maintenance du code. Voici la structure de base d'une application Ruby on Rails :

myapp/
  ├── app/
  │   ├── assets/
  │   ├── controllers/
  │   ├── models/
  │   └── views/
  ├── config/
  ├── db/
  ├── lib/
  ├── log/
  ├── public/
  ├── test/
  └── tmp/

Chaque répertoire a un rôle spécifique :

  • app/ : Contient les fichiers liés à la logique de l'application, tels que les contrôleurs, les modèles et les vues.
  • config/ : Contient les fichiers de configuration de l'application, tels que les routes et les paramètres de base de données.
  • db/ : Contient les fichiers de migration de la base de données.
  • lib/ : Contient les bibliothèques et les modules personnalisés.
  • log/ : Contient les fichiers de journalisation de l'application.
  • public/ : Contient les fichiers statiques accessibles publiquement, tels que les images et les feuilles de style.
  • test/ : Contient les tests unitaires et fonctionnels de l'application.
  • tmp/ : Contient les fichiers temporaires générés par l'application.

Création d'un nouveau projet

Pour créer un nouveau projet Ruby on Rails, utilisez la commande suivante :

rails new nom_du_projet

Cette commande génère une nouvelle application Ruby on Rails avec une structure de base et les fichiers nécessaires.

Les routes

Les routes dans Ruby on Rails définissent les URL et les actions correspondantes dans les contrôleurs. Elles sont définies dans le fichier config/routes.rb. Voici un exemple de route :

get '/articles', to: 'articles#index'

Cette route définit l'URL /articles qui est associée à l'action index du contrôleur articles.

Les contrôleurs

Les contrôleurs dans Ruby on Rails gèrent les actions et les interactions avec les modèles et les vues. Voici un exemple de contrôleur :

class ArticlesController < ApplicationController
  def index
    @articles = Article.all
  end

  def show
    @article = Article.find(params[:id])
  end

  def create
    @article = Article.new(article_params)
    if @article.save
      redirect_to @article
    else
      render 'new'
    end
  end

  private

  def article_params
    params.require(:article).permit(:title, :content)
  end
end

Ce contrôleur contient des actions telles que index, show et create. Chaque action effectue des opérations spécifiques, telles que la récupération des articles, l'affichage d'un article spécifique et la création d'un nouvel article.

Les vues

Les vues dans Ruby on Rails sont responsables de l'affichage des données aux utilisateurs. Elles utilisent le langage de template ERB (Embedded Ruby) pour combiner du code Ruby avec du HTML. Voici un exemple de vue :

<h1><%= @article.title %></h1>
<p><%= @article.content %></p>

Cette vue affiche le titre et le contenu d'un article en utilisant les variables d'instance @article.title et @article.content.

Les modèles

Les modèles dans Ruby on Rails représentent les données de l'application et gèrent les interactions avec la base de données. Voici un exemple de modèle :

class Article < ApplicationRecord
  validates :title, presence: true
  validates :content, presence: true
end

Ce modèle définit une classe Article qui est liée à la table articles dans la base de données. Il utilise des validations pour s'assurer que les champs title et content sont présents.

Les migrations

Les migrations dans Ruby on Rails permettent de gérer les modifications de la structure de la base de données. Voici un exemple de migration :

class CreateArticles < ActiveRecord::Migration[6.0]
  def change
    create_table :articles do |t|
      t.string :title
      t.text :content

      t.timestamps
    end
  end
end

Cette migration crée une table articles avec les colonnes title, content et les timestamps created_at et updated_at.

Les associations entre modèles

Les associations entre modèles dans Ruby on Rails permettent de définir des relations entre les différentes tables de la base de données. Voici un exemple d'association :

class Article < ApplicationRecord
  has_many :comments
end

class Comment < ApplicationRecord
  belongs_to :article
end

Cette association indique qu'un article peut avoir plusieurs commentaires (has_many :comments) et qu'un commentaire appartient à un article spécifique (belongs_to :article).
