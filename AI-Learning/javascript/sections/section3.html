Section 3: Programmation orientée objet en Javascript

Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets similaires.

En POO, les objets sont utilisés pour encapsuler des données et des fonctionnalités connexes. Cela permet de créer du code modulaire, réutilisable et facile à maintenir.

Voici un exemple de code utilisant la POO en Javascript :

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    sayHello() {
        console.log('Hello, my name is ' + this.name);
    }
}

let person1 = new Person('John', 25);
person1.sayHello(); // Output: Hello, my name is John

Les objets et les classes

En POO, les objets sont des instances de classes. Une classe est un modèle ou un plan pour créer des objets similaires. Elle définit les propriétés et les méthodes que les objets de cette classe auront.

Voici un exemple de code utilisant des objets et des classes en Javascript :

class Car {
    constructor(brand, color) {
        this.brand = brand;
        this.color = color;
    }

    startEngine() {
        console.log('The ' + this.brand + ' car is starting...');
    }
}

let car1 = new Car('Toyota', 'red');
car1.startEngine(); // Output: The Toyota car is starting...

L'héritage et le polymorphisme

L'héritage est un concept clé de la POO qui permet à une classe d'hériter des propriétés et des méthodes d'une autre classe. Cela permet de créer une hiérarchie de classes et de réutiliser du code.

Le polymorphisme est un autre concept de la POO qui permet à des objets de différentes classes d'être traités de manière interchangeable, en utilisant une interface commune.

Voici un exemple de code utilisant l'héritage et le polymorphisme en Javascript :

class Animal {
    constructor(name) {
        this.name = name;
    }

    makeSound() {
        console.log('The animal makes a sound.');
    }
}

class Dog extends Animal {
    makeSound() {
        console.log('The dog barks.');
    }
}

class Cat extends Animal {
    makeSound() {
        console.log('The cat meows.');
    }
}

let animal1 = new Animal('Animal');
let dog1 = new Dog('Dog');
let cat1 = new Cat('Cat');

animal1.makeSound(); // Output: The animal makes a sound.
dog1.makeSound(); // Output: The dog barks.
cat1.makeSound(); // Output: The cat meows.

Les modules et les namespaces

Les modules et les namespaces sont des concepts utilisés pour organiser et structurer le code en POO. Ils permettent de regrouper des fonctionnalités connexes et d'éviter les conflits de noms.

En Javascript, les modules peuvent être créés en utilisant les modules ES6 ou en utilisant des bibliothèques comme CommonJS ou AMD.

Voici un exemple de code utilisant les modules et les namespaces en Javascript :

// Module 1
let module1 = (function() {
    let privateVariable = 'Private variable';

    function privateFunction() {
        console.log('Private function');
    }

    return {
        publicVariable: 'Public variable',
        publicFunction: function() {
            console.log('Public function');
        }
    };
})();

// Module 2
let module2 = (function() {
    function privateFunction() {
        console.log('Private function');
    }

    return {
        publicVariable: 'Public variable',
        publicFunction: function() {
            console.log('Public function');
        }
    };
})();

module1.publicFunction(); // Output: Public function
module2.publicFunction(); // Output: Public function
