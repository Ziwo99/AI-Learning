Les modules en Javascript

Les modules en JavaScript sont des unités de code réutilisables qui permettent d'organiser et de structurer votre code de manière modulaire. Ils permettent d'encapsuler des variables et des fonctions dans un espace de noms distinct, évitant ainsi les conflits de noms et les collisions entre différentes parties de votre application.

Voici un exemple de module JavaScript :

const myModule = (function() {
    // Variables privées
    let privateVariable = 'Hello';
    
    // Fonctions privées
    function privateFunction() {
        console.log(privateVariable);
    }
    
    // Fonctions publiques
    return {
        publicFunction: function() {
            privateFunction();
        }
    };
})();

myModule.publicFunction(); // Affiche 'Hello' dans la console

Dans cet exemple, nous utilisons une fonction anonyme auto-exécutante pour créer un module. Les variables et les fonctions déclarées à l'intérieur de cette fonction sont privées et ne sont pas accessibles en dehors du module. Seule la fonction publique publicFunction est accessible depuis l'extérieur.

Les générateurs en Javascript

Les générateurs en JavaScript sont des fonctions spéciales qui permettent de contrôler l'exécution de manière itérative. Ils sont utilisés pour créer des itérations personnalisées, où vous pouvez mettre en pause et reprendre l'exécution à tout moment.

Voici un exemple de générateur JavaScript :

function* fibonacciGenerator() {
    let a = 0;
    let b = 1;
    
    while (true) {
        yield a;
        [a, b] = [b, a + b];
    }
}

const fibonacci = fibonacciGenerator();

console.log(fibonacci.next().value); // Affiche 0
console.log(fibonacci.next().value); // Affiche 1
console.log(fibonacci.next().value); // Affiche 1
console.log(fibonacci.next().value); // Affiche 2
console.log(fibonacci.next().value); // Affiche 3
// ...

Dans cet exemple, nous utilisons la syntaxe function* pour déclarer une fonction générateur. À l'intérieur de la fonction, nous utilisons le mot-clé yield pour mettre en pause l'exécution et renvoyer une valeur. Lorsque nous appelons la méthode next() sur l'objet générateur, l'exécution reprend à partir du point où elle s'était arrêtée.

Les Web Workers en Javascript

Les Web Workers en JavaScript permettent d'exécuter des scripts en arrière-plan, sans bloquer l'interface utilisateur principale. Ils sont utilisés pour effectuer des tâches intensives en calcul ou en traitement des données, tout en maintenant une expérience utilisateur fluide.

Voici un exemple d'utilisation des Web Workers en JavaScript :

// Fichier main.js
const worker = new Worker('worker.js');

worker.onmessage = function(event) {
    console.log('Résultat du Web Worker :', event.data);
};

worker.postMessage(42);

// Fichier worker.js
self.onmessage = function(event) {
    const result = event.data * 2;
    self.postMessage(result);
};

Dans cet exemple, nous créons un Web Worker en utilisant le constructeur Worker et lui attribuons un fichier JavaScript externe worker.js. Nous utilisons ensuite les méthodes onmessage et postMessage pour communiquer avec le Web Worker. Lorsque le Web Worker reçoit un message, il effectue un calcul (dans ce cas, il double la valeur reçue) et renvoie le résultat à l'aide de la méthode postMessage. Le résultat est ensuite affiché dans la console du navigateur.
