Section 5: Programmation asynchrone

Introduction à la programmation asynchrone

La programmation asynchrone est un concept clé en JavaScript qui permet d'exécuter des tâches de manière non bloquante. Contrairement à la programmation synchrone, où les tâches sont exécutées les unes après les autres, la programmation asynchrone permet d'exécuter plusieurs tâches en parallèle.

La programmation asynchrone est particulièrement utile lorsque nous devons effectuer des opérations qui prennent du temps, comme les appels réseau ou les accès à la base de données. Au lieu de bloquer l'exécution du programme jusqu'à ce que ces opérations soient terminées, nous pouvons les exécuter de manière asynchrone et continuer à exécuter d'autres tâches en attendant leur achèvement.

Les callbacks en Javascript

Les callbacks sont une technique couramment utilisée en JavaScript pour gérer la programmation asynchrone. Un callback est une fonction qui est passée en tant qu'argument à une autre fonction et qui sera appelée une fois que cette fonction aura terminé son exécution.

Voici un exemple d'utilisation d'un callback pour effectuer une requête AJAX :

function makeAjaxRequest(url, callback) {
    var xhr = new XMLHttpRequest();
    xhr.open('GET', url, true);
    xhr.onreadystatechange = function() {
        if (xhr.readyState === 4 && xhr.status === 200) {
            callback(xhr.responseText);
        }
    };
    xhr.send();
}

function handleResponse(response) {
    console.log('La réponse est : ' + response);
}

makeAjaxRequest('https://api.example.com/data', handleResponse);

Dans cet exemple, la fonction makeAjaxRequest effectue une requête AJAX asynchrone vers l'URL spécifiée et appelle le callback handleResponse une fois que la réponse est reçue. Cela permet d'exécuter d'autres tâches pendant que la requête est en cours d'exécution.

Les Promises en Javascript

Les Promises sont une autre façon de gérer la programmation asynchrone en JavaScript. Une Promise représente une valeur qui peut être disponible maintenant, dans le futur ou jamais. Elle permet de gérer les opérations asynchrones de manière plus lisible et plus expressive.

Voici un exemple d'utilisation d'une Promise pour effectuer une requête AJAX :

function makeAjaxRequest(url) {
    return new Promise(function(resolve, reject) {
        var xhr = new XMLHttpRequest();
        xhr.open('GET', url, true);
        xhr.onreadystatechange = function() {
            if (xhr.readyState === 4) {
                if (xhr.status === 200) {
                    resolve(xhr.responseText);
                } else {
                    reject(xhr.statusText);
                }
            }
        };
        xhr.send();
    });
}

makeAjaxRequest('https://api.example.com/data')
    .then(function(response) {
        console.log('La réponse est : ' + response);
    })
    .catch(function(error) {
        console.error('Une erreur s\'est produite : ' + error);
    });

Dans cet exemple, la fonction makeAjaxRequest retourne une Promise qui est résolue avec la réponse de la requête AJAX si celle-ci réussit, ou rejetée avec le statut de la requête si elle échoue. Les méthodes then et catch permettent de gérer respectivement la résolution et le rejet de la Promise.

Les async/await en Javascript

Les async/await sont une syntaxe introduite dans ECMAScript 2017 pour simplifier la gestion de la programmation asynchrone en JavaScript. Ils permettent d'écrire du code asynchrone de manière plus synchrone, en utilisant des mots-clés spéciaux.

Voici un exemple d'utilisation des async/await pour effectuer une requête AJAX :

async function makeAjaxRequest(url) {
    try {
        var response = await fetch(url);
        var data = await response.json();
        console.log('Les données sont : ' + JSON.stringify(data));
    } catch (error) {
        console.error('Une erreur s\'est produite : ' + error);
    }
}

makeAjaxRequest('https://api.example.com/data');

Dans cet exemple, la fonction makeAjaxRequest est déclarée avec le mot-clé async, ce qui permet d'utiliser le mot-clé await à l'intérieur de cette fonction. L'opérateur await permet d'attendre la résolution d'une Promise avant de continuer l'exécution du code. Cela permet d'écrire du code asynchrone de manière plus lisible et plus proche de la syntaxe synchrone.
