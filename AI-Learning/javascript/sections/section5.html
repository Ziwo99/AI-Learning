Section 5: JavaScript Objects

Object-oriented programming in JavaScript

Object-oriented programming (OOP) is a programming paradigm that allows developers to organize and structure their code by creating objects that encapsulate data and behavior. JavaScript supports object-oriented programming through its prototype-based inheritance model.

In JavaScript, objects are created using the Object constructor function or object literal notation. The properties and methods of an object can be accessed using dot notation or bracket notation. 

const person = {
  name: 'John',
  age: 30,
  greet: function() {
    console.log('Hello, my name is ' + this.name);
  }
};

console.log(person.name); // Output: John
console.log(person['age']); // Output: 30
person.greet(); // Output: Hello, my name is John

Creating and using objects

In JavaScript, objects can be created using the new keyword and a constructor function. Constructor functions are used to define the properties and methods of an object. The this keyword refers to the current instance of the object being created. 

function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function() {
  console.log('Hello, my name is ' + this.name);
};

const person = new Person('John', 30);
console.log(person.name); // Output: John
console.log(person.age); // Output: 30
person.greet(); // Output: Hello, my name is John

Prototypes and inheritance

In JavaScript, objects can inherit properties and methods from other objects through prototypes. Prototypes are a way to share common functionality between objects. When a property or method is accessed on an object, JavaScript will first look for it on the object itself, and if not found, it will look for it in the object's prototype chain. 

function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function() {
  console.log('Hello, my name is ' + this.name);
};

function Employee(name, age, job) {
  Person.call(this, name, age);
  this.job = job;
}

Employee.prototype = Object.create(Person.prototype);
Employee.prototype.constructor = Employee;

Employee.prototype.work = function() {
  console.log('Working as a ' + this.job);
};

const employee = new Employee('John', 30, 'developer');
console.log(employee.name); // Output: John
console.log(employee.age); // Output: 30
console.log(employee.job); // Output: developer
employee.greet(); // Output: Hello, my name is John
employee.work(); // Output: Working as a developer

Object methods and properties

JavaScript objects can have methods and properties. Methods are functions that are associated with an object and can be called on that object. Properties are values associated with an object. 

const person = {
  name: 'John',
  age: 30,
  greet: function() {
    console.log('Hello, my name is ' + this.name);
  }
};

console.log(person.hasOwnProperty('name')); // Output: true
console.log(person.hasOwnProperty('greet')); // Output: true
console.log(Object.keys(person)); // Output: ['name', 'age', 'greet']

In the example above, the hasOwnProperty() method is used to check if an object has a specific property. The Object.keys() method is used to retrieve an array of an object's own enumerable properties.
