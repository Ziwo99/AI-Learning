Les fonctions anonymes et les fonctions fléchées

Les fonctions anonymes, également appelées fonctions sans nom, sont des fonctions qui ne sont pas déclarées avec un nom spécifique. Elles sont souvent utilisées comme arguments de fonctions ou pour créer des fonctions auto-exécutables.

(function() {
    // Code à exécuter ici
})();

// Exemple d'utilisation d'une fonction anonyme comme argument de fonction
setTimeout(function() {
    console.log('Cette fonction sera exécutée après 2 secondes');
}, 2000);

Les fonctions fléchées, introduites dans ECMAScript 6, sont une syntaxe plus concise pour définir des fonctions. Elles sont souvent utilisées pour simplifier le code et éviter les problèmes de portée (scope) de this.

// Syntaxe d'une fonction fléchée
const addition = (a, b) => {
    return a + b;
};

// Exemple d'utilisation d'une fonction fléchée
const result = addition(2, 3);
console.log(result); // Affiche 5

Les fonctions de rappel (callbacks)

Les fonctions de rappel, également appelées callbacks, sont des fonctions passées en tant qu'arguments à d'autres fonctions. Elles sont utilisées pour exécuter du code de manière asynchrone ou pour définir des actions à effectuer après l'exécution d'une opération.

function fetchData(callback) {
    // Code pour récupérer des données
    const data = 'Données récupérées';

    // Appel de la fonction de rappel avec les données en paramètre
    callback(data);
}

// Utilisation d'une fonction de rappel
fetchData(function(data) {
    console.log('Données reçues : ' + data);
});

Les promesses et les fonctions asynchrones

Les promesses sont un moyen de gérer les opérations asynchrones en JavaScript. Elles permettent d'effectuer des actions en attente de résultats, puis d'exécuter du code en fonction du succès ou de l'échec de l'opération.

// Exemple d'utilisation de promesses
const fetchData = new Promise(function(resolve, reject) {
    // Code pour récupérer des données

    if (/* opération réussie */) {
        resolve('Données récupérées');
    } else {
        reject('Erreur lors de la récupération des données');
    }
});

// Utilisation des promesses
fetchData.then(function(data) {
    console.log('Données reçues : ' + data);
}).catch(function(error) {
    console.log('Erreur : ' + error);
});

Les fonctions asynchrones, introduites dans ECMAScript 8, sont une syntaxe plus simple pour gérer les opérations asynchrones en utilisant des promesses. Elles permettent d'écrire du code asynchrone de manière plus lisible et plus proche de la syntaxe synchrone.

// Syntaxe d'une fonction asynchrone
async function fetchData() {
    // Code pour récupérer des données

    if (/* opération réussie */) {
        return 'Données récupérées';
    } else {
        throw new Error('Erreur lors de la récupération des données');
    }
}

// Utilisation d'une fonction asynchrone
fetchData().then(function(data) {
    console.log('Données reçues : ' + data);
}).catch(function(error) {
    console.log('Erreur : ' + error);
});

Les générateurs et les itérables

Les générateurs sont des fonctions spéciales qui permettent de créer des itérables, c'est-à-dire des objets qui peuvent être parcourus à l'aide d'une boucle. Les générateurs utilisent le mot-clé yield pour retourner des valeurs de manière incrémentale.

// Exemple d'un générateur simple
function* countToFive() {
    yield 1;
    yield 2;
    yield 3;
    yield 4;
    yield 5;
}

// Utilisation d'un générateur
const iterator = countToFive();
console.log(iterator.next().value); // Affiche 1
console.log(iterator.next().value); // Affiche 2
console.log(iterator.next().value); // Affiche 3
// ...

// Exemple d'utilisation d'un générateur pour créer un itérable
const iterable = {
    [Symbol.iterator]: function*() {
        yield 'Première valeur';
        yield 'Deuxième valeur';
        yield 'Troisième valeur';
    }
};

// Parcours de l'itérable avec une boucle for...of
for (const value of iterable) {
    console.log(value);
}

Les itérables sont des objets qui implémentent la méthode [Symbol.iterator]() et qui peuvent être parcourus à l'aide d'une boucle for...of. Les tableaux, les chaînes de caractères et les objets Map et Set sont des exemples d'objets itérables.

// Exemple d'utilisation d'un itérable
const iterable = ['Première valeur', 'Deuxième valeur', 'Troisième valeur'];

// Parcours de l'itérable avec une boucle for...of
for (const value of iterable) {
    console.log(value);
}
