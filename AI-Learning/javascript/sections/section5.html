Section 5 : Les classes et l'héritage en Javascript

Introduction aux classes en Javascript

Les classes en Javascript sont un moyen de créer des objets réutilisables avec des propriétés et des méthodes communes. Elles permettent d'organiser et de structurer le code de manière plus claire et plus maintenable.

Pour déclarer une classe en Javascript, on utilise le mot-clé class suivi du nom de la classe. Par exemple :

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    sayHello() {
        console.log('Hello, my name is ' + this.name);
    }
}

Dans cet exemple, nous avons déclaré une classe Person avec un constructeur prenant deux paramètres : name et age. La méthode sayHello affiche un message de salutation avec le nom de la personne.

Déclaration et utilisation des classes en Javascript

Pour créer une instance d'une classe en Javascript, on utilise le mot-clé new suivi du nom de la classe et des arguments du constructeur. Par exemple :

let person1 = new Person('John', 25);
person1.sayHello(); // Affiche "Hello, my name is John"

Dans cet exemple, nous avons créé une instance de la classe Person appelée person1 avec les valeurs 'John' et 25 pour les paramètres name et age. Ensuite, nous avons appelé la méthode sayHello sur cette instance.

L'héritage en Javascript

L'héritage en Javascript permet à une classe d'hériter des propriétés et des méthodes d'une autre classe. Cela permet de créer une hiérarchie de classes et de réutiliser du code déjà existant.

Pour définir une classe qui hérite d'une autre classe, on utilise le mot-clé extends. Par exemple :

class Student extends Person {
    constructor(name, age, grade) {
        super(name, age);
        this.grade = grade;
    }

    study() {
        console.log(this.name + ' is studying.');
    }
}

Dans cet exemple, nous avons défini une classe Student qui hérite de la classe Person. Le constructeur de la classe Student appelle le constructeur de la classe Person à l'aide du mot-clé super et ajoute une propriété supplémentaire grade. La classe Student a également une méthode study qui affiche un message indiquant que l'étudiant étudie.

Les méthodes statiques en Javascript

Les méthodes statiques en Javascript sont des méthodes qui appartiennent à la classe elle-même plutôt qu'à ses instances. Elles sont utiles pour regrouper des fonctionnalités liées à la classe qui ne dépendent pas des instances spécifiques.

Pour déclarer une méthode statique en Javascript, on utilise le mot-clé static avant le nom de la méthode. Par exemple :

class MathUtils {
    static square(x) {
        return x * x;
    }
}

console.log(MathUtils.square(5)); // Affiche 25

Dans cet exemple, nous avons déclaré une classe MathUtils avec une méthode statique square qui calcule le carré d'un nombre. La méthode square peut être appelée directement sur la classe sans avoir besoin d'instancier un objet.

Les classes abstraites en Javascript

Les classes abstraites en Javascript sont des classes qui ne peuvent pas être instanciées directement, mais qui servent de modèles pour d'autres classes. Elles définissent des méthodes et des propriétés communes que les classes dérivées doivent implémenter.

Pour déclarer une classe abstraite en Javascript, on utilise le mot-clé abstract avant le nom de la classe. Par exemple :

abstract class Shape {
    constructor() {
        if (new.target === Shape) {
            throw new Error('Cannot instantiate abstract class Shape.');
        }
    }

    abstract calculateArea();
}

class Rectangle extends Shape {
    constructor(width, height) {
        super();
        this.width = width;
        this.height = height;
    }

    calculateArea() {
        return this.width * this.height;
    }
}

let rectangle = new Rectangle(5, 10);
console.log(rectangle.calculateArea()); // Affiche 50

Dans cet exemple, nous avons déclaré une classe abstraite Shape avec une méthode abstraite calculateArea. La classe Shape ne peut pas être instanciée directement, mais elle sert de modèle pour la classe dérivée Rectangle. La classe Rectangle implémente la méthode calculateArea en calculant l'aire du rectangle.

En conclusion, les classes et l'héritage en Javascript permettent de créer des structures de code plus organisées et réutilisables. Les classes permettent de regrouper des propriétés et des méthodes liées, tandis que l'héritage permet de créer une hiérarchie de classes et de réutiliser du code déjà existant. Les méthodes statiques sont utiles pour regrouper des fonctionnalités liées à la classe, tandis que les classes abstraites servent de modèles pour d'autres classes.
