Les fonctions anonymes et les fonctions fléchées

Les fonctions anonymes, également appelées fonctions sans nom, sont des fonctions qui ne sont pas déclarées avec un nom spécifique. Elles sont souvent utilisées comme arguments de fonctions ou pour créer des fonctions auto-exécutables.

(function() {
    // Code exécuté immédiatement
})();

// Fonction anonyme utilisée comme argument
setTimeout(function() {
  console.log('Hello, world!');
}, 1000);

// Fonction anonyme assignée à une variable
var greet = function() {
  console.log('Hello!');
};
greet();

Les fonctions fléchées sont une syntaxe plus concise pour définir des fonctions en JavaScript. Elles sont souvent utilisées pour simplifier le code et éviter les problèmes de portée (scope) de this.

// Syntaxe de la fonction fléchée
var add = (a, b) => {
  return a + b;
};

// Fonction fléchée avec un seul argument
var square = x => {
  return x * x;
};

// Fonction fléchée sans argument
var sayHello = () => {
  console.log('Hello!');
};

Les fonctions récursives

Une fonction récursive est une fonction qui s'appelle elle-même à l'intérieur de sa propre définition. Cela permet de résoudre des problèmes de manière itérative en les divisant en sous-problèmes plus petits.

function factorial(n) {
  if (n === 0) {
    return 1;
  } else {
    return n * factorial(n - 1);
  }
}

console.log(factorial(5)); // Résultat: 120

Les fonctions de rappel (callbacks)

Une fonction de rappel, ou callback en anglais, est une fonction qui est passée en argument à une autre fonction et qui est appelée ultérieurement. Les fonctions de rappel sont couramment utilisées pour gérer des opérations asynchrones ou des événements.

function fetchData(callback) {
  // Simulation d'une requête asynchrone
  setTimeout(function() {
    var data = 'Exemple de données';
    callback(data);
  }, 1000);
}

function processData(data) {
  console.log('Données traitées:', data);
}

fetchData(processData);

Les fonctions asynchrones (promises, async/await)

Les fonctions asynchrones sont utilisées pour gérer des opérations qui prennent du temps, comme les requêtes réseau ou les accès à la base de données. Elles permettent d'écrire du code de manière synchrone, ce qui le rend plus lisible et plus facile à comprendre.

// Utilisation de Promises
function fetchData() {
  return new Promise(function(resolve, reject) {
    // Simulation d'une requête asynchrone réussie
    setTimeout(function() {
      var data = 'Exemple de données';
      resolve(data);
    }, 1000);
  });
}

fetchData()
  .then(function(data) {
    console.log('Données récupérées:', data);
  })
  .catch(function(error) {
    console.error('Erreur:', error);
  });

// Utilisation de async/await
async function getData() {
  try {
    var data = await fetchData();
    console.log('Données récupérées:', data);
  } catch (error) {
    console.error('Erreur:', error);
  }
}

getData();
