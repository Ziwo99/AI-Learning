Les closures

Les closures sont un concept avancé en JavaScript qui permet de capturer l'état d'une fonction et de le conserver même après que la fonction ait terminé son exécution. Une closure est une fonction interne qui a accès aux variables de sa fonction parente, même après que la fonction parente ait été terminée.

Les closures sont souvent utilisées pour créer des fonctions avec des variables privées. Par exemple :

function createCounter() {
  let count = 0;

  return function() {
    count++;
    console.log(count);
  };
}

const counter = createCounter();
counter(); // Affiche 1
counter(); // Affiche 2
counter(); // Affiche 3

Dans cet exemple, la fonction createCounter crée une closure qui conserve la variable count. Chaque fois que la fonction interne est appelée, la variable count est incrémentée et affichée.

Les prototypes avancés

Les prototypes sont un mécanisme clé en JavaScript pour la création d'objets et l'héritage. Les prototypes avancés permettent de manipuler les prototypes d'objets de manière plus complexe.

Une des fonctionnalités avancées des prototypes est la possibilité de définir des méthodes sur le prototype d'un objet. Cela permet de partager des méthodes entre plusieurs instances d'objets sans les recréer à chaque fois. Par exemple :

function Person(name) {
  this.name = name;
}

Person.prototype.greet = function() {
  console.log('Hello, my name is ' + this.name);
};

const person1 = new Person('Alice');
const person2 = new Person('Bob');

person1.greet(); // Affiche "Hello, my name is Alice"
person2.greet(); // Affiche "Hello, my name is Bob"

Dans cet exemple, la méthode greet est définie sur le prototype de la fonction constructeur Person. Ainsi, toutes les instances de Person auront accès à cette méthode.

La programmation fonctionnelle en Javascript

La programmation fonctionnelle est un paradigme de programmation qui met l'accent sur l'utilisation de fonctions pures et l'évitement des effets de bord. En JavaScript, la programmation fonctionnelle est largement utilisée grâce à la présence de fonctions de première classe et de fonctions d'ordre supérieur.

La programmation fonctionnelle en JavaScript encourage l'utilisation de fonctions pures, qui ne modifient pas l'état global et ne dépendent que de leurs arguments. Les fonctions pures sont prévisibles et faciles à tester. Par exemple :

const numbers = [1, 2, 3, 4, 5];

const doubledNumbers = numbers.map(function(number) {
  return number * 2;
});

console.log(doubledNumbers); // Affiche [2, 4, 6, 8, 10]

Dans cet exemple, la méthode map est utilisée pour appliquer une fonction de transformation à chaque élément du tableau numbers. La fonction de transformation est une fonction pure qui double chaque nombre.

La programmation fonctionnelle en JavaScript utilise également des fonctions d'ordre supérieur, qui sont des fonctions qui prennent d'autres fonctions en tant qu'arguments ou renvoient des fonctions. Cela permet de créer des fonctions plus génériques et réutilisables. Par exemple :

function multiplyBy(factor) {
  return function(number) {
    return number * factor;
  };
}

const double = multiplyBy(2);
const triple = multiplyBy(3);

console.log(double(5)); // Affiche 10
console.log(triple(5)); // Affiche 15

Dans cet exemple, la fonction multiplyBy est une fonction d'ordre supérieur qui renvoie une fonction de multiplication par un facteur donné. Les fonctions double et triple sont créées en utilisant multiplyBy avec des facteurs différents.

La programmation fonctionnelle en JavaScript offre de nombreux avantages, tels que la réduction des effets de bord, la facilité de test et la réutilisabilité du code.
