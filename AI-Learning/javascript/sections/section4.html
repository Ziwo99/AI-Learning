Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet d'organiser et de structurer le code en utilisant des objets. En POO, un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets similaires.

La POO repose sur quatre principes fondamentaux : l'encapsulation, l'abstraction, l'héritage et le polymorphisme. Ces principes permettent de créer un code modulaire, réutilisable et facile à maintenir.

Classes et objets

En Javascript, les classes sont des modèles pour créer des objets. Une classe définit les propriétés et les méthodes communes à tous les objets qui en sont issus.

Voici un exemple de déclaration d'une classe en Javascript :

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  sayHello() {
    console.log("Hello, my name is " + this.name);
  }
}

// Création d'un objet de la classe Person
let person = new Person("John", 25);

// Appel de la méthode sayHello de l'objet person
person.sayHello(); // Output: Hello, my name is John

Dans cet exemple, la classe Person a deux propriétés (name et age) et une méthode (sayHello). L'objet person est créé à partir de la classe Person et peut accéder à ses propriétés et méthodes.

Héritage et polymorphisme

L'héritage est un concept clé de la POO qui permet à une classe d'hériter des propriétés et des méthodes d'une autre classe. La classe qui hérite est appelée la classe enfant, et la classe dont elle hérite est appelée la classe parent.

Voici un exemple d'héritage en Javascript :

class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(this.name + " makes a sound");
  }
}

class Dog extends Animal {
  speak() {
    console.log(this.name + " barks");
  }
}

let dog = new Dog("Max");
dog.speak(); // Output: Max barks

Dans cet exemple, la classe Dog hérite de la classe Animal. La classe Dog a sa propre implémentation de la méthode speak, qui remplace celle de la classe Animal. Ainsi, lorsque la méthode speak est appelée sur un objet de la classe Dog, la version de la classe Dog est exécutée.

Encapsulation et abstraction

L'encapsulation est un principe de la POO qui consiste à regrouper les données et les méthodes qui les manipulent au sein d'un même objet. Cela permet de cacher les détails d'implémentation et de protéger les données d'accès non autorisé.

L'abstraction est un autre principe de la POO qui consiste à représenter les objets du monde réel de manière simplifiée, en se concentrant sur les caractéristiques et les comportements essentiels. Cela permet de créer des modèles réutilisables et de faciliter la compréhension du code.

Voici un exemple d'encapsulation et d'abstraction en Javascript :

class BankAccount {
  constructor(accountNumber, balance) {
    this.accountNumber = accountNumber;
    this.balance = balance;
  }

  deposit(amount) {
    this.balance += amount;
  }

  withdraw(amount) {
    if (amount <= this.balance) {
      this.balance -= amount;
    } else {
      console.log("Insufficient funds");
    }
  }
}

let account = new BankAccount("123456789", 1000);
account.deposit(500);
account.withdraw(200);
console.log(account.balance); // Output: 1300

Dans cet exemple, la classe BankAccount encapsule les données relatives à un compte bancaire (accountNumber et balance) ainsi que les méthodes pour effectuer des dépôts et des retraits. L'utilisateur n'a pas besoin de connaître les détails d'implémentation de ces opérations, il peut simplement utiliser les méthodes fournies par la classe.
