Section 4: JavaScript Functions

Function declaration and expression

A function in JavaScript is a block of code that can be executed when called. There are two ways to define a function: function declaration and function expression.

Function declaration: A function declaration starts with the keyword "function" followed by the name of the function and a pair of parentheses. The code block of the function is enclosed in curly braces. 

function greet() {
    console.log("Hello, world!");
}

greet(); // Output: Hello, world!

Function expression: A function expression is similar to a function declaration, but the function is assigned to a variable. It can be anonymous or have a name. 

var greet = function() {
    console.log("Hello, world!");
};

greet(); // Output: Hello, world!

Function parameters and arguments

Functions can accept parameters, which are variables listed in the function's parentheses. These parameters act as placeholders for values that will be passed to the function when it is called. Arguments, on the other hand, are the actual values passed to the function. 

function greet(name) {
    console.log("Hello, " + name + "!");
}

greet("John"); // Output: Hello, John!

In the example above, the parameter "name" is defined in the function declaration. When the function is called with the argument "John", the value of "name" is set to "John" within the function's code block.

Return statements and function scope

The return statement is used to specify the value that a function should return. It is followed by an expression or a value that will be returned when the function is called. The return statement also ends the execution of the function. 

function add(a, b) {
    return a + b;
}

var result = add(3, 5);
console.log(result); // Output: 8

In the example above, the function "add" takes two parameters, "a" and "b", and returns their sum. The return statement is used to pass the result back to the caller.

Function scope refers to the visibility and accessibility of variables within a function. Variables declared inside a function are only accessible within that function, unless they are explicitly returned or assigned to a variable outside the function. 

function multiply(a, b) {
    var result = a * b;
    return result;
}

var product = multiply(2, 4);
console.log(result); // Output: ReferenceError: result is not defined

In the example above, the variable "result" is declared inside the "multiply" function and is not accessible outside of it. Trying to access it outside the function will result in a ReferenceError.

Higher-order functions and callbacks

In JavaScript, functions can be assigned to variables, passed as arguments to other functions, and returned from other functions. Functions that accept other functions as arguments or return functions are called higher-order functions. 

function doSomething(callback) {
    console.log("Doing something...");
    callback();
}

function sayHello() {
    console.log("Hello!");
}

doSomething(sayHello); // Output: Doing something... Hello!

In the example above, the function "doSomething" accepts a callback function as an argument and calls it within its code block. The function "sayHello" is passed as the callback, so it is executed when "doSomething" is called.

Callbacks are commonly used in asynchronous programming to handle the completion of an operation or to perform an action after a certain event occurs.
