Section 4: Les fonctions avancées en Javascript

Les fonctions anonymes et les fonctions fléchées

Les fonctions anonymes, également appelées fonctions sans nom, sont des fonctions qui ne sont pas déclarées avec un nom spécifique. Elles sont souvent utilisées comme arguments de fonctions ou pour créer des fonctions auto-exécutables. Voici un exemple de fonction anonyme :

(function() {
    console.log("Ceci est une fonction anonyme auto-exécutée.");
})();

Les fonctions fléchées sont une syntaxe plus concise pour définir des fonctions en Javascript. Elles sont souvent utilisées pour les fonctions de rappel (callbacks) ou pour simplifier la syntaxe des fonctions. Voici un exemple de fonction fléchée :

const addition = (a, b) => {
    return a + b;
};

console.log(addition(2, 3)); // Affiche 5

Les fonctions de rappel (callbacks)

Les fonctions de rappel sont des fonctions qui sont passées en tant qu'arguments à d'autres fonctions. Elles sont utilisées pour exécuter du code après l'exécution d'une autre fonction. Voici un exemple d'utilisation d'une fonction de rappel :

function calculer(a, b, operation) {
    return operation(a, b);
}

function addition(a, b) {
    return a + b;
}

console.log(calculer(2, 3, addition)); // Affiche 5

Les fonctions asynchrones et les promesses

Les fonctions asynchrones sont des fonctions qui permettent d'exécuter du code de manière asynchrone, c'est-à-dire sans bloquer l'exécution du reste du programme. Elles sont souvent utilisées pour effectuer des opérations longues ou des requêtes réseau. Les promesses sont un mécanisme utilisé pour gérer les résultats de fonctions asynchrones. Voici un exemple d'utilisation de fonctions asynchrones et de promesses :

function effectuerRequete(url) {
    return new Promise((resolve, reject) => {
        const xhr = new XMLHttpRequest();
        xhr.open('GET', url);
        xhr.onload = () => resolve(xhr.responseText);
        xhr.onerror = () => reject(xhr.statusText);
        xhr.send();
    });
}

effectuerRequete('https://api.example.com/data')
    .then((response) => {
        console.log(response);
    })
    .catch((error) => {
        console.error(error);
    });

Les générateurs et les itérables

Les générateurs sont des fonctions spéciales qui permettent de créer des itérables, c'est-à-dire des objets sur lesquels on peut itérer. Ils sont utilisés pour générer des séquences de valeurs de manière contrôlée. Voici un exemple d'utilisation d'un générateur :

function* genererSequence() {
    yield 1;
    yield 2;
    yield 3;
}

const sequence = genererSequence();

console.log(sequence.next().value); // Affiche 1
console.log(sequence.next().value); // Affiche 2
console.log(sequence.next().value); // Affiche 3

Les itérables sont des objets qui implémentent le protocole d'itération en fournissant une méthode [Symbol.iterator](). Voici un exemple d'utilisation d'un itérable :

const iterable = {
    [Symbol.iterator]() {
        let i = 0;
        return {
            next() {
                if (i < 3) {
                    return { value: i++, done: false };
                } else {
                    return { done: true };
                }
            }
        };
    }
};

for (const value of iterable) {
    console.log(value);
}
// Affiche 0, 1, 2
