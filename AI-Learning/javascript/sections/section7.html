Section 7 : Les promesses et les callbacks en Javascript

Introduction aux promesses en Javascript

Les promesses en Javascript sont un moyen de gérer les opérations asynchrones de manière plus claire et plus lisible. Elles permettent d'organiser le code de manière séquentielle, en évitant les callbacks en cascade.

Une promesse représente la réalisation future d'une opération asynchrone et peut être dans l'un des trois états suivants :

  • Pending : l'opération est en cours d'exécution
  • Fulfilled : l'opération s'est terminée avec succès
  • Rejected : l'opération a échoué

Pour créer une promesse, on utilise le constructeur Promise :

const myPromise = new Promise((resolve, reject) => {
    // Code de l'opération asynchrone
});

Utilisation des promesses en Javascript

Pour utiliser une promesse, on peut chaîner les méthodes then et catch. La méthode then est appelée lorsque la promesse est résolue avec succès, tandis que la méthode catch est appelée en cas d'échec.

Voici un exemple d'utilisation des promesses en Javascript :

myPromise
    .then((result) => {
        // Code à exécuter lorsque la promesse est résolue avec succès
    })
    .catch((error) => {
        // Code à exécuter en cas d'échec de la promesse
    });

Les callbacks en Javascript

Les callbacks sont une autre méthode couramment utilisée pour gérer les opérations asynchrones en Javascript. Un callback est une fonction qui est passée en argument à une autre fonction et qui sera appelée ultérieurement.

Voici un exemple d'utilisation des callbacks en Javascript :

function myFunction(callback) {
    // Code de l'opération asynchrone
    callback();
}

myFunction(() => {
    // Code à exécuter lorsque l'opération asynchrone est terminée
});

La gestion des erreurs avec les promesses et les callbacks

Les promesses et les callbacks permettent également de gérer les erreurs de manière efficace. En utilisant la méthode catch des promesses ou en passant une fonction de callback d'erreur, on peut capturer et traiter les erreurs survenues lors de l'exécution d'une opération asynchrone.

Voici un exemple de gestion des erreurs avec les promesses en Javascript :

myPromise
    .then((result) => {
        // Code à exécuter lorsque la promesse est résolue avec succès
    })
    .catch((error) => {
        // Code à exécuter en cas d'échec de la promesse
    });

Les promesses en chaîne et les promesses parallèles

Les promesses en Javascript peuvent être utilisées en chaîne ou en parallèle pour gérer des opérations asynchrones complexes.

Les promesses en chaîne permettent d'exécuter des opérations les unes après les autres, en utilisant la méthode then :

myPromise
    .then((result) => {
        // Code à exécuter lorsque la première promesse est résolue avec succès
        return anotherPromise;
    })
    .then((result) => {
        // Code à exécuter lorsque la deuxième promesse est résolue avec succès
    })
    .catch((error) => {
        // Code à exécuter en cas d'échec de l'une des promesses
    });

Les promesses parallèles permettent d'exécuter plusieurs opérations asynchrones en parallèle et d'attendre leur résolution avec la méthode Promise.all :

Promise.all([promise1, promise2, promise3])
    .then((results) => {
        // Code à exécuter lorsque toutes les promesses sont résolues avec succès
    })
    .catch((error) => {
        // Code à exécuter en cas d'échec de l'une des promesses
    });

Il est important de noter que les promesses en Javascript offrent une syntaxe plus claire et plus lisible pour gérer les opérations asynchrones, en évitant les callbacks en cascade et en permettant une meilleure gestion des erreurs.
