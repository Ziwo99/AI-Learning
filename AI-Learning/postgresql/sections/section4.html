Requêtes SELECT avec jointures

Les requêtes SELECT avec jointures permettent de combiner des données provenant de plusieurs tables en utilisant des critères de jointure. Cela permet d'obtenir des résultats plus complexes et plus précis.

Voici un exemple de requête SELECT avec une jointure :

SELECT orders.order_id, customers.customer_name
FROM orders
JOIN customers ON orders.customer_id = customers.customer_id;

Dans cet exemple, nous sélectionnons l'identifiant de commande (order_id) et le nom du client (customer_name) à partir des tables "orders" et "customers". La jointure est réalisée en utilisant la colonne "customer_id" qui est commune aux deux tables.

Il existe différents types de jointures, tels que INNER JOIN, LEFT JOIN, RIGHT JOIN et FULL JOIN, qui permettent de spécifier comment les enregistrements doivent être combinés. Chaque type de jointure a ses propres caractéristiques et est utilisé en fonction des besoins spécifiques.

Sous-requêtes et opérateurs logiques

Les sous-requêtes sont des requêtes SQL imbriquées à l'intérieur d'une autre requête. Elles permettent d'effectuer des opérations plus complexes en utilisant les résultats d'une requête comme critère de sélection dans une autre requête.

Voici un exemple de sous-requête :

SELECT customer_name
FROM customers
WHERE customer_id IN (SELECT customer_id FROM orders WHERE order_date = '2022-01-01');

Dans cet exemple, nous sélectionnons le nom des clients à partir de la table "customers" en utilisant une sous-requête. La sous-requête sélectionne les identifiants de client (customer_id) à partir de la table "orders" où la date de commande (order_date) est égale à '2022-01-01'.

Les opérateurs logiques, tels que AND, OR et NOT, peuvent également être utilisés avec les sous-requêtes pour créer des conditions plus complexes.

Agrégation de données avec GROUP BY

L'agrégation de données avec GROUP BY permet de regrouper les données en fonction de certaines colonnes et d'effectuer des opérations d'agrégation, telles que la somme, la moyenne, le maximum, le minimum, etc., sur les valeurs regroupées.

Voici un exemple d'utilisation de GROUP BY :

SELECT category_id, COUNT(*) as total_products
FROM products
GROUP BY category_id;

Dans cet exemple, nous sélectionnons l'identifiant de catégorie (category_id) et le nombre total de produits (total_products) à partir de la table "products". Les données sont regroupées par identifiant de catégorie en utilisant la clause GROUP BY.

Il est également possible d'utiliser des fonctions d'agrégation telles que SUM, AVG, MAX, MIN, etc., pour effectuer des calculs sur les valeurs regroupées.

Utilisation des fonctions d'agrégation

Les fonctions d'agrégation permettent d'effectuer des calculs sur un ensemble de valeurs. Elles sont souvent utilisées en conjonction avec la clause GROUP BY pour obtenir des résultats agrégés.

Voici un exemple d'utilisation de fonctions d'agrégation :

SELECT category_id, AVG(price) as average_price
FROM products
GROUP BY category_id;

Dans cet exemple, nous sélectionnons l'identifiant de catégorie (category_id) et le prix moyen (average_price) à partir de la table "products". Les données sont regroupées par identifiant de catégorie en utilisant la clause GROUP BY, puis la fonction AVG est utilisée pour calculer la moyenne des prix pour chaque catégorie.

Il existe de nombreuses autres fonctions d'agrégation telles que SUM, MAX, MIN, COUNT, etc., qui peuvent être utilisées en fonction des besoins spécifiques.

Manipulation de données avec INSERT, UPDATE et DELETE

Les instructions INSERT, UPDATE et DELETE permettent de manipuler les données dans une table.

Voici un exemple d'utilisation de l'instruction INSERT :

INSERT INTO customers (customer_name, email)
VALUES ('John Doe', 'john.doe@example.com');

Dans cet exemple, nous insérons un nouvel enregistrement dans la table "customers" en spécifiant les valeurs des colonnes "customer_name" et "email".

Voici un exemple d'utilisation de l'instruction UPDATE :

UPDATE customers
SET email = 'johndoe@example.com'
WHERE customer_id = 1;

Dans cet exemple, nous mettons à jour l'enregistrement dans la table "customers" en modifiant la valeur de la colonne "email" pour l'enregistrement dont l'identifiant de client (customer_id) est égal à 1.

Voici un exemple d'utilisation de l'instruction DELETE :

DELETE FROM customers
WHERE customer_id = 1;

Dans cet exemple, nous supprimons l'enregistrement de la table "customers" dont l'identifiant de client (customer_id) est égal à 1.

Il est important de noter que les instructions UPDATE et DELETE peuvent affecter plusieurs enregistrements en fonction des critères de sélection spécifiés.
