Les propriétés et les accesseurs

Les propriétés et les accesseurs sont des éléments clés de la programmation orientée objet en Objective-C. Ils permettent d'encapsuler les données et de contrôler l'accès à celles-ci.

Une propriété est une variable associée à une classe ou à un objet. Elle peut être définie avec différents attributs tels que "strong", "weak", "readonly", etc. Ces attributs définissent le comportement de la propriété.

Les accesseurs, également appelés "getters" et "setters", permettent d'accéder aux valeurs d'une propriété et de les modifier. En Objective-C, les accesseurs sont générés automatiquement par le compilateur lors de la déclaration d'une propriété.

Voici un exemple de déclaration d'une propriété avec ses accesseurs :

@interface MyClass : NSObject

@property (strong, nonatomic) NSString *name;

@end

@implementation MyClass

@end

Dans cet exemple, la classe MyClass possède une propriété "name" de type NSString. L'attribut "strong" indique que la propriété est une référence forte, ce qui signifie que l'objet auquel elle est associée ne sera pas libéré tant qu'il y aura une référence active.

Les accesseurs générés automatiquement permettent d'accéder à la valeur de la propriété et de la modifier :

MyClass *myObject = [[MyClass alloc] init];
myObject.name = @"John"; // Utilisation du setter
NSString *name = myObject.name; // Utilisation du getter

Il est également possible de personnaliser les accesseurs en les redéfinissant manuellement :

@interface MyClass : NSObject

@property (strong, nonatomic) NSString *name;

- (void)setName:(NSString *)name;
- (NSString *)name;

@end

@implementation MyClass

- (void)setName:(NSString *)name {
    _name = [name uppercaseString]; // Modification de la valeur avant l'assignation
}

- (NSString *)name {
    return [_name lowercaseString]; // Modification de la valeur avant la récupération
}

@end

Dans cet exemple, les accesseurs ont été redéfinis pour modifier la valeur de la propriété avant l'assignation et avant la récupération.

Les protocoles et les délégués

Les protocoles et les délégués sont des concepts importants en Objective-C pour la communication entre objets.

Un protocole est une liste de méthodes déclarées sans implémentation. Il définit un ensemble de comportements que les objets peuvent adopter. Un objet qui adopte un protocole doit implémenter toutes les méthodes déclarées dans ce protocole.

Voici un exemple de déclaration d'un protocole :

@protocol MyProtocol

- (void)doSomething;
- (void)doSomethingElse;

@end

Dans cet exemple, le protocole MyProtocol définit deux méthodes : "doSomething" et "doSomethingElse". Les objets qui adoptent ce protocole doivent implémenter ces deux méthodes.

Un délégué est un objet qui implémente les méthodes d'un protocole et qui est chargé de réaliser certaines tâches pour un autre objet. L'objet qui utilise le délégué peut appeler les méthodes du protocole pour communiquer avec le délégué.

Voici un exemple d'utilisation d'un délégué :

@interface MyObject : NSObject

@property (weak, nonatomic) id<MyProtocol> delegate;

- (void)performTask;

@end

@implementation MyObject

- (void)performTask {
    // Faire quelque chose
    [self.delegate doSomething];
    // Faire autre chose
    [self.delegate doSomethingElse];
}

@end

Dans cet exemple, la classe MyObject possède une propriété "delegate" de type id. L'objet qui utilise MyObject peut assigner un délégué à cette propriété et appeler les méthodes du protocole pour communiquer avec le délégué.

Les catégories et les extensions

Les catégories et les extensions permettent d'ajouter des méthodes à une classe existante sans avoir à la sous-classer.

Une catégorie est une déclaration supplémentaire d'une classe existante qui permet d'ajouter de nouvelles méthodes. Les méthodes ajoutées par une catégorie sont disponibles pour toutes les instances de la classe.

Voici un exemple de déclaration d'une catégorie :

@interface NSString (MyCategory)

- (NSString *)reversedString;

@end

@implementation NSString (MyCategory)

- (NSString *)reversedString {
    NSMutableString *reversedString = [NSMutableString string];
    NSInteger length = [self length];
    
    for (NSInteger i = length - 1; i >= 0; i--) {
        [reversedString appendString:[NSString stringWithFormat:@"%C", [self characterAtIndex:i]]];
    }
    
    return reversedString;
}

@end

Dans cet exemple, une catégorie MyCategory est ajoutée à la classe NSString. La catégorie ajoute une méthode "reversedString" qui permet de renvoyer une version inversée de la chaîne de caractères.

Une extension est similaire à une catégorie, mais elle est utilisée pour ajouter des méthodes privées à une classe existante. Les méthodes ajoutées par une extension ne sont pas visibles à l'extérieur de la classe.

Voici un exemple de déclaration d'une extension :

@interface MyClass ()

- (void)privateMethod;

@end

@implementation MyClass

- (void)privateMethod {
    // Faire quelque chose
}

@end

Dans cet exemple, une extension est ajoutée à la classe MyClass. L'extension ajoute une méthode "privateMethod" qui peut être utilisée uniquement à l'intérieur de la classe.

Les blocs et la programmation fonctionnelle

Les blocs sont des éléments de programmation fonctionnelle qui permettent de regrouper du code et de le traiter comme une valeur.

Un bloc est une structure de données qui contient un ensemble de code et qui peut être exécuté à un moment ultérieur. Les blocs peuvent être utilisés pour définir des callbacks, des itérations, des animations, etc.

Voici un exemple de déclaration et d'utilisation d'un bloc :

void (^myBlock)(void) = ^{
    NSLog(@"Hello, world!");
};

myBlock(); // Exécute le bloc

Dans cet exemple, un bloc est déclaré avec la syntaxe "void (^)(void)". Le bloc ne prend aucun argument et ne renvoie aucune valeur. Le code à exécuter est défini entre les accolades.

Les blocs peuvent également prendre des arguments et renvoyer des valeurs :

int (^add)(int, int) = ^(int a, int b) {
    return a + b;
};

int result = add(2, 3); // Appelle le bloc et récupère le résultat

Dans cet exemple, un bloc est déclaré avec la syntaxe "int (^)(int, int)". Le bloc prend deux arguments de type int et renvoie un résultat de type int.

Les blocs peuvent être utilisés dans de nombreux contextes, tels que les méthodes de tri, les animations, les opérations asynchrones, etc. Ils offrent une flexibilité et une expressivité accrues dans la programmation en Objective-C.
