Utilisation de l'état local

L'état local est utilisé pour stocker des données spécifiques à un composant React. Il est géré à l'intérieur du composant et peut être modifié à l'aide de la méthode setState(). Lorsque l'état local est mis à jour, le composant est automatiquement re-rendu pour refléter les changements.

Voici un exemple d'utilisation de l'état local dans un composant React :

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount(count + 1);
  };

  return (
    

      
Count: {count}

      
    

  );
}

export default Counter;

Dans cet exemple, nous utilisons le hook useState pour initialiser l'état local count à 0. La fonction setCount est utilisée pour mettre à jour la valeur de count lorsque le bouton est cliqué.

Utilisation de l'état global avec Redux

L'état global est utilisé pour stocker des données partagées entre plusieurs composants React. Redux est une bibliothèque populaire pour gérer l'état global dans une application React. Il utilise un concept de store centralisé pour stocker toutes les données de l'application.

Voici un exemple d'utilisation de Redux pour gérer l'état global :

// actions.js
export const increment = () => {
  return {
    type: 'INCREMENT'
  };
};

// reducers.js
const initialState = {
  count: 0
};

const counterReducer = (state = initialState, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return {
        ...state,
        count: state.count + 1
      };
    default:
      return state;
  }
};

export default counterReducer;

// App.js
import React from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { increment } from './actions';

function App() {
  const count = useSelector(state => state.count);
  const dispatch = useDispatch();

  const handleIncrement = () => {
    dispatch(increment());
  };

  return (
    

      
Count: {count}

      
    

  );
}

export default App;

Dans cet exemple, nous avons défini une action increment qui incrémente la valeur de count. Le reducer counterReducer gère les actions et met à jour l'état global en conséquence. Dans le composant App, nous utilisons les hooks useSelector et useDispatch pour accéder à l'état global et envoyer des actions.

Gestion des formulaires avec React

React facilite la gestion des formulaires en utilisant l'état local pour stocker les valeurs des champs de formulaire. Lorsque les valeurs des champs changent, l'état est mis à jour et le composant est re-rendu pour refléter les changements.

Voici un exemple de gestion d'un formulaire avec React :

import React, { useState } from 'react';

function Form() {
  const [name, setName] = useState('');
  const [email, setEmail] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    console.log('Name:', name);
    console.log('Email:', email);
  };

  return (
    

      
      
      
    

  );
}

export default Form;

Dans cet exemple, nous utilisons l'état local pour stocker les valeurs des champs de formulaire name et email. Lorsque les valeurs des champs changent, les fonctions setName et setEmail sont utilisées pour mettre à jour l'état. Lorsque le formulaire est soumis, la fonction handleSubmit est appelée pour traiter les données.

Utilisation des hooks pour gérer l'état

React fournit des hooks pour gérer l'état dans les composants fonctionnels. Les hooks les plus couramment utilisés sont useState et useEffect.

Voici un exemple d'utilisation des hooks pour gérer l'état :

import React, { useState, useEffect } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);

  const increment = () => {
    setCount(count + 1);
  };

  return (
    

      
Count: {count}

      
    

  );
}

export default Counter;

Dans cet exemple, nous utilisons le hook useState pour initialiser l'état local count à 0. Le hook useEffect est utilisé pour mettre à jour le titre de la page en fonction de la valeur de count. Lorsque count change, le code à l'intérieur de useEffect est exécuté.
