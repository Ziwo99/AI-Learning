Section 5: Gestion des formulaires avec Vue.js

Liaison de données avec les formulaires

La liaison de données est un concept clé dans Vue.js qui permet de lier les valeurs des éléments de formulaire à des propriétés dans l'instance Vue. Cela permet de synchroniser automatiquement les données du formulaire avec l'état de l'application.

Pour lier une valeur d'entrée de formulaire à une propriété dans l'instance Vue, nous utilisons la directive v-model. Par exemple, pour lier une entrée de texte à une propriété nom dans l'instance Vue, nous pouvons utiliser la syntaxe suivante : 

<input type="text" v-model="nom">

Dans cet exemple, chaque fois que l'utilisateur saisit du texte dans l'entrée, la propriété nom dans l'instance Vue sera automatiquement mise à jour avec la valeur saisie.

La liaison de données avec les formulaires fonctionne également avec d'autres types d'éléments de formulaire tels que les cases à cocher, les boutons radio et les menus déroulants. Il suffit d'utiliser la directive v-model avec l'élément approprié.

Validation des données

La validation des données est un aspect important de la gestion des formulaires. Vue.js offre plusieurs mécanismes pour valider les données saisies par l'utilisateur.

Une façon courante de valider les données est d'utiliser les attributs HTML5 tels que required, min, max, pattern, etc. Ces attributs peuvent être utilisés en conjonction avec la directive v-model pour effectuer une validation de base côté client.

Vue.js offre également une intégration transparente avec des bibliothèques de validation tierces telles que VeeValidate et Vuelidate. Ces bibliothèques fournissent des fonctionnalités avancées de validation des données telles que la validation conditionnelle, la validation asynchrone, les messages d'erreur personnalisés, etc.

Voici un exemple d'utilisation de la bibliothèque VeeValidate pour valider un champ de saisie de texte : 

<input type="text" v-model="email" v-validate="'required|email'">
<span v-show="errors.has('email')" class="error">{{ errors.first('email') }}</span>

Dans cet exemple, la directive v-validate est utilisée pour spécifier les règles de validation à appliquer au champ de saisie de texte. La méthode errors.has() est utilisée pour vérifier si des erreurs de validation existent, et la méthode errors.first() est utilisée pour afficher le premier message d'erreur.

Gestion des états de formulaire

La gestion des états de formulaire est un autre aspect important de la création de formulaires avec Vue.js. Vue.js fournit des fonctionnalités intégrées pour suivre l'état de validation, l'état de soumission et l'état de modification des formulaires.

Pour suivre l'état de validation d'un formulaire, nous pouvons utiliser la propriété $invalid de l'instance Vue. Cette propriété renvoie true si le formulaire contient des erreurs de validation et false sinon.

Pour suivre l'état de soumission d'un formulaire, nous pouvons utiliser la propriété $submitted de l'instance Vue. Cette propriété renvoie true si le formulaire a été soumis et false sinon.

Pour suivre l'état de modification d'un formulaire, nous pouvons utiliser la propriété $dirty de l'instance Vue. Cette propriété renvoie true si le formulaire a été modifié depuis sa dernière soumission et false sinon.

Voici un exemple d'utilisation de ces propriétés pour afficher un message d'état du formulaire : 

<p v-if="$invalid">Le formulaire contient des erreurs de validation.</p>
<p v-if="$submitted">Le formulaire a été soumis.</p>
<p v-if="$dirty">Le formulaire a été modifié.</p>

Utilisation de composants de formulaire

Vue.js offre la possibilité de créer des composants de formulaire réutilisables. Les composants de formulaire encapsulent la logique de gestion des formulaires et peuvent être utilisés dans plusieurs parties de l'application.

Pour créer un composant de formulaire, nous pouvons utiliser la directive v-model pour lier les valeurs des éléments de formulaire aux propriétés du composant. Nous pouvons également utiliser les directives v-bind et v-on pour transmettre des données et des événements entre le composant de formulaire et son parent.

Voici un exemple de création d'un composant de formulaire personnalisé pour la saisie d'un nom : 

<template>
  <div>
    <label for="nom">Nom:</label>
    <input type="text" v-model="nom">
  </div>
</template>

<script>
export default {
  data() {
    return {
      nom: ''
    };
  }
};
</script>

Dans cet exemple, le composant de formulaire contient un champ de saisie de texte lié à la propriété nom. Lorsque l'utilisateur saisit du texte dans le champ, la propriété nom sera automatiquement mise à jour.

Le composant de formulaire peut ensuite être utilisé dans d'autres parties de l'application en l'incluant avec la balise , où custom-form est le nom du composant.
