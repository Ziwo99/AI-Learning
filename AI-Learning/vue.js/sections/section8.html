Composants fonctionnels

Les composants fonctionnels sont une fonctionnalité de Vue.js qui permet d'améliorer les performances en réduisant la surcharge liée à la création et à la mise à jour des instances de composants. Les composants fonctionnels sont des composants sans état, ce qui signifie qu'ils n'ont pas de données réactives et ne gèrent pas d'état interne. Ils sont donc plus légers et plus rapides à rendre.

Voici un exemple de composant fonctionnel :

<template functional>
  <div>
    <p>Ceci est un composant fonctionnel.</p>
  </div>
</template>

Utilisation de la directive v-once

La directive v-once permet de rendre un élément ou un composant statique, c'est-à-dire qu'il ne sera pas mis à jour même si les données auxquelles il est lié changent. Cela peut améliorer les performances en évitant les rendus inutiles.

Voici un exemple d'utilisation de la directive v-once :

<template>
  <div v-once>
    <p>Ce paragraphe ne sera pas mis à jour même si les données changent.</p>
  </div>
</template>

Lazy loading des composants

Le lazy loading des composants est une technique qui consiste à charger les composants de manière asynchrone, uniquement lorsque cela est nécessaire. Cela permet de réduire le temps de chargement initial de l'application en ne chargeant que les composants requis pour la vue actuelle.

Voici un exemple d'utilisation du lazy loading des composants avec Vue Router :

const Foo = () => import('./Foo.vue');
const Bar = () => import('./Bar.vue');

const routes = [
  { path: '/foo', component: Foo },
  { path: '/bar', component: Bar }
];

Utilisation de la directive v-show vs v-if

La directive v-show et la directive v-if sont toutes deux utilisées pour conditionner l'affichage des éléments en fonction d'une expression. Cependant, elles fonctionnent de manière légèrement différente et ont un impact sur les performances.

La directive v-show rend l'élément en utilisant la propriété CSS "display" pour le cacher ou l'afficher. Cela signifie que l'élément est toujours présent dans le DOM, même s'il est caché.

La directive v-if, quant à elle, ajoute ou supprime l'élément du DOM en fonction de l'expression. Cela signifie que si l'expression est fausse, l'élément sera complètement supprimé du DOM.

En général, si vous avez des éléments qui sont souvent cachés et affichés, il est préférable d'utiliser la directive v-show pour éviter de manipuler le DOM à chaque fois. Si vous avez des éléments qui sont rarement affichés, il est préférable d'utiliser la directive v-if pour optimiser les performances en évitant d'avoir des éléments inutiles dans le DOM.

Voici un exemple d'utilisation de la directive v-show et de la directive v-if :

<template>
  <div>
    <div v-show="showElement">Cet élément est caché et affiché en utilisant v-show.</div>
    <div v-if="showElement">Cet élément est ajouté ou supprimé du DOM en utilisant v-if.</div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      showElement: true
    };
  }
};
</script>
