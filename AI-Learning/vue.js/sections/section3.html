Les propriétés réactives

Les propriétés réactives sont l'un des concepts clés de Vue.js. Elles permettent de lier les données à l'interface utilisateur de manière dynamique, de sorte que toute modification des données se reflète automatiquement dans l'interface utilisateur et vice versa.

Pour déclarer une propriété réactive dans Vue.js, vous pouvez utiliser l'objet data dans votre composant. Par exemple :

data() {
  return {
    message: 'Hello Vue.js!'
  }
}

Dans cet exemple, la propriété message est déclarée comme une propriété réactive. Vous pouvez ensuite l'utiliser dans votre template :

<template>
  <div>
    {{ message }}
  </div>
</template>

Chaque fois que la valeur de la propriété message change, l'interface utilisateur sera automatiquement mise à jour pour refléter cette modification.

Les méthodes et calculs

Les méthodes et calculs sont utilisés pour effectuer des opérations et des calculs sur les données dans Vue.js. Vous pouvez déclarer des méthodes dans votre composant en utilisant l'option methods. Par exemple :

methods: {
  sayHello() {
    console.log('Hello!');
  },
  calculateSum(a, b) {
    return a + b;
  }
}

Vous pouvez ensuite appeler ces méthodes dans votre template ou dans d'autres méthodes :

<template>
  <div>
    <button @click="sayHello">Say Hello</button>
    <p>{{ calculateSum(2, 3) }}</p>
  </div>
</template>

Les calculs sont similaires aux méthodes, mais ils sont utilisés pour effectuer des calculs basés sur les données réactives. Vous pouvez les déclarer en utilisant l'option computed. Par exemple :

computed: {
  fullName() {
    return this.firstName + ' ' + this.lastName;
  }
}

Dans cet exemple, la propriété calculée fullName est basée sur les propriétés réactives firstName et lastName. Vous pouvez ensuite l'utiliser dans votre template :

<template>
  <div>
    <p>Full Name: {{ fullName }}</p>
  </div>
</template>

Chaque fois que les valeurs des propriétés réactives firstName ou lastName changent, la propriété calculée fullName sera automatiquement mise à jour.

Les filtres

Les filtres sont utilisés pour formater et manipuler les données dans Vue.js. Ils peuvent être utilisés dans les templates pour modifier la valeur d'une expression avant de l'afficher. Par exemple :

<template>
  <div>
    <p>{{ message | capitalize }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      message: 'hello vue.js!'
    }
  },
  filters: {
    capitalize(value) {
      if (!value) return '';
      value = value.toString();
      return value.charAt(0).toUpperCase() + value.slice(1);
    }
  }
}
</script>

Dans cet exemple, le filtre capitalize est utilisé pour mettre en majuscule la première lettre de la valeur de la propriété message.

Les watchers

Les watchers sont utilisés pour effectuer des actions en réponse à des changements spécifiques dans les données réactives. Vous pouvez déclarer des watchers dans votre composant en utilisant l'option watch. Par exemple :

watch: {
  message(newValue, oldValue) {
    console.log('Message changed from ' + oldValue + ' to ' + newValue);
  }
}

Dans cet exemple, le watcher surveille les changements de la propriété réactive message. Chaque fois que la valeur de message change, la fonction de rappel sera exécutée.

Les computed properties

Les computed properties sont similaires aux propriétés calculées, mais elles sont utilisées pour effectuer des calculs plus complexes ou pour regrouper plusieurs propriétés réactives. Vous pouvez les déclarer en utilisant l'option computed. Par exemple :

computed: {
  fullName() {
    return this.firstName + ' ' + this.lastName;
  },
  fullNameLength() {
    return this.fullName.length;
  }
}

Dans cet exemple, la propriété calculée fullNameLength est basée sur la propriété calculée fullName. Vous pouvez ensuite l'utiliser dans votre template :

<template>
  <div>
    <p>Full Name: {{ fullName }}</p>
    <p>Full Name Length: {{ fullNameLength }}</p>
  </div>
</template>

Chaque fois que les valeurs des propriétés réactives firstName ou lastName changent, les propriétés calculées fullName et fullNameLength seront automatiquement mises à jour.
