Utilisation des rebases interactifs

Le rebase interactif est une fonctionnalité puissante de Git qui permet de réorganiser et de modifier l'historique des commits. Il est utile pour fusionner des branches, réorganiser l'ordre des commits, supprimer des commits indésirables, modifier des messages de commit, etc.

Voici comment utiliser le rebase interactif :

$ git rebase -i <commit>

Remplacez <commit> par le commit à partir duquel vous souhaitez commencer le rebase interactif. Cela ouvrira votre éditeur de texte par défaut avec une liste des commits à modifier.

Dans l'éditeur de texte, vous pouvez spécifier les actions à effectuer sur chaque commit :

  • pick : conserve le commit tel quel
  • reword : modifie le message de commit
  • edit : permet de modifier le contenu du commit
  • squash : fusionne le commit avec le commit précédent
  • fixup : fusionne le commit avec le commit précédent sans conserver le message de commit
  • drop : supprime le commit

Après avoir enregistré et fermé l'éditeur de texte, Git appliquera les modifications spécifiées lors du rebase interactif.

Utilisation des reflog

Le reflog (log des références) est un outil utile pour récupérer des commits perdus ou supprimés accidentellement. Il enregistre tous les mouvements de pointeurs de Git, tels que les commits, les branches, les tags, etc.

Voici comment utiliser le reflog :

$ git reflog

Cette commande affiche une liste chronologique de tous les mouvements de pointeurs enregistrés dans le reflog.

Pour restaurer un commit perdu ou supprimé, vous pouvez utiliser la commande git cherry-pick en spécifiant le hash du commit :

$ git cherry-pick <commit-hash>

Cela appliquera le commit spécifié à la branche actuelle.

Utilisation des commandes de recherche dans l'historique Git

Git propose plusieurs commandes pour rechercher des informations dans l'historique des commits :

  • git log : affiche l'historique des commits
  • git grep <pattern> : recherche un motif dans les fichiers de l'historique des commits
  • git bisect : permet de rechercher le commit introduisant un bogue en utilisant une recherche binaire

Par exemple, pour rechercher un motif spécifique dans les fichiers de l'historique des commits, vous pouvez utiliser la commande suivante :

$ git grep <pattern>

Cela affichera les fichiers contenant le motif spécifié, ainsi que les lignes correspondantes.

La commande git bisect est utile pour trouver le commit introduisant un bogue en effectuant une recherche binaire. Elle nécessite de spécifier un commit "bon" et un commit "mauvais" :

$ git bisect start
$ git bisect good <commit-good>
$ git bisect bad <commit-bad>

Git effectuera ensuite une recherche binaire en vous demandant de marquer chaque commit comme "bon" ou "mauvais" jusqu'à ce qu'il trouve le commit introduisant le bogue.

Ces commandes de recherche dans l'historique Git sont très utiles pour retrouver des informations spécifiques ou pour déboguer des problèmes.
