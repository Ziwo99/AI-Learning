Initialisation d'un dépôt Git

L'initialisation d'un dépôt Git est la première étape pour commencer à utiliser Git. Cela permet de créer un dépôt Git vide dans un répertoire existant.

Pour initialiser un dépôt Git, utilisez la commande suivante :

git init

Cette commande créera un nouveau répertoire .git dans le répertoire actuel, qui contiendra tous les fichiers et les informations nécessaires pour gérer les versions de votre projet.

Les commandes de base de Git (add, commit, push, pull)

Les commandes de base de Git sont essentielles pour gérer les versions de votre projet. Voici les principales commandes :

git add

La commande git add permet d'ajouter des fichiers à l'index de Git. Cela signifie que les modifications apportées à ces fichiers seront prises en compte lors du prochain commit.

Par exemple, pour ajouter tous les fichiers modifiés à l'index, utilisez la commande suivante :

git add .

git commit

La commande git commit permet de créer un nouveau commit, qui enregistre les modifications apportées aux fichiers de votre projet.

Pour créer un commit avec un message descriptif, utilisez la commande suivante :

git commit -m "Message descriptif du commit"

git push

La commande git push permet d'envoyer les commits locaux vers un dépôt distant, tel que GitHub ou GitLab.

Pour pousser vos commits vers un dépôt distant, utilisez la commande suivante :

git push origin nom_de_la_branche

git pull

La commande git pull permet de récupérer les derniers commits d'un dépôt distant et de les fusionner avec votre branche locale.

Pour récupérer les derniers commits d'une branche distante, utilisez la commande suivante :

git pull origin nom_de_la_branche

Gestion des branches

Les branches sont un élément clé de Git, car elles permettent de travailler sur des fonctionnalités ou des correctifs isolés, sans affecter la branche principale du projet.

Voici les principales commandes pour gérer les branches :

git branch

La commande git branch permet de lister toutes les branches de votre dépôt Git.

Pour afficher toutes les branches de votre dépôt, utilisez la commande suivante :

git branch

git checkout

La commande git checkout permet de basculer entre les branches de votre dépôt Git.

Pour basculer vers une autre branche, utilisez la commande suivante :

git checkout nom_de_la_branche

git merge

La commande git merge permet de fusionner une branche avec une autre branche.

Pour fusionner une branche avec la branche actuelle, utilisez la commande suivante :

git merge nom_de_la_branche

Fusionner des branches avec Git

La fusion de branches est une opération courante dans Git, qui permet de combiner les modifications apportées à une branche avec une autre branche.

Voici les étapes pour fusionner une branche avec Git :

  1. Assurez-vous d'être sur la branche dans laquelle vous souhaitez fusionner les modifications.
  2. Utilisez la commande git merge nom_de_la_branche pour fusionner la branche spécifiée avec la branche actuelle.
  3. Résolvez les éventuels conflits de fusion qui peuvent survenir.
  4. Validez la fusion en créant un nouveau commit.

Résolution des conflits

Les conflits surviennent lorsque Git ne peut pas fusionner automatiquement les modifications apportées à une branche avec une autre branche. Dans ce cas, il est nécessaire de résoudre manuellement les conflits.

Voici les étapes pour résoudre les conflits avec Git :

  1. Utilisez la commande git status pour afficher les fichiers en conflit.
  2. Ouvrez les fichiers en conflit et recherchez les sections marquées avec "<<<<<<<", "=======" et ">>>>>>>".
  3. Modifiez les fichiers en conflit pour résoudre les différences entre les versions.
  4. Utilisez la commande git add nom_du_fichier pour marquer les fichiers comme résolus.
  5. Utilisez la commande git commit pour créer un nouveau commit qui valide la résolution des conflits.
Ceci conclut la section 2 sur les bases de Git. Dans cette section, nous avons appris à initialiser un dépôt Git, à utiliser les commandes de base de Git, à gérer les branches, à fusionner des branches et à résoudre des conflits.
