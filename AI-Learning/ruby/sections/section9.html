Section 9 : Déploiement d'une application Ruby

Configuration d'un serveur d'hébergement

Lorsque vous êtes prêt à déployer votre application Ruby, la première étape consiste à configurer un serveur d'hébergement. Il existe de nombreuses options disponibles, mais nous allons nous concentrer sur l'utilisation d'un serveur Linux avec Apache et Passenger.

Pour commencer, vous devez vous assurer que votre serveur dispose de Ruby et de toutes les dépendances nécessaires installées. Vous pouvez utiliser des outils de gestion de versions comme RVM ou rbenv pour installer Ruby sur votre serveur.

Ensuite, vous devez installer Apache, qui est un serveur web populaire. Vous pouvez le faire en utilisant le gestionnaire de paquets de votre système d'exploitation. Par exemple, sur Ubuntu, vous pouvez exécuter la commande suivante :

sudo apt-get install apache2

Une fois Apache installé, vous devez installer Passenger, qui est un module pour Apache qui permet d'exécuter des applications Ruby. Vous pouvez l'installer en utilisant la gem Passenger :

gem install passenger

Après avoir installé Passenger, vous devez l'activer en exécutant la commande suivante :

sudo passenger-install-apache2-module

Cette commande vous guidera à travers le processus d'activation de Passenger pour Apache.

Déploiement d'une application Ruby

Une fois que votre serveur est configuré, vous pouvez procéder au déploiement de votre application Ruby. Voici les étapes à suivre :

  1. Assurez-vous que votre application est prête à être déployée. Cela signifie qu'elle doit être testée et fonctionnelle localement.
  2. Créez un répertoire sur votre serveur pour héberger votre application. Vous pouvez choisir n'importe quel emplacement, mais il est courant de le placer dans le répertoire /var/www.
  3. Transférez les fichiers de votre application vers le répertoire que vous avez créé sur le serveur. Vous pouvez utiliser des outils comme scp ou rsync pour cela.
  4. Configurez Apache pour qu'il serve votre application. Vous devez ajouter une configuration pour votre application dans le fichier de configuration d'Apache. Vous pouvez le trouver dans le répertoire /etc/apache2/sites-available. Voici un exemple de configuration :
<VirtualHost *:80>
  ServerName example.com
  DocumentRoot /var/www/your_app/public
  <Directory /var/www/your_app/public>
    AllowOverride all
    Options -MultiViews
    Require all granted
  </Directory>
</VirtualHost>

Assurez-vous de remplacer "example.com" par le nom de domaine de votre application et "/var/www/your_app" par le chemin vers le répertoire de votre application.

Ensuite, activez la configuration en exécutant la commande suivante :

sudo a2ensite your_app

Redémarrez Apache pour prendre en compte les modifications :

sudo service apache2 restart

Votre application Ruby est maintenant déployée et accessible via votre nom de domaine.

Gestion des mises à jour et de la maintenance

Une fois votre application déployée, vous devrez peut-être effectuer des mises à jour ou effectuer des tâches de maintenance régulières. Voici quelques bonnes pratiques à suivre :

  • Utilisez un système de gestion de versions comme Git pour suivre les modifications apportées à votre code source. Cela vous permettra de revenir à des versions antérieures en cas de problème.
  • Effectuez des sauvegardes régulières de votre base de données et de tout autre fichier important de votre application.
  • Surveillez les journaux d'erreurs de votre application pour détecter les problèmes potentiels.
  • Mettez à jour régulièrement les dépendances de votre application pour bénéficier des dernières fonctionnalités et correctifs de sécurité.
  • Planifiez des temps d'arrêt réguliers pour effectuer des tâches de maintenance, comme la mise à jour du système d'exploitation ou l'installation de correctifs de sécurité.

En suivant ces bonnes pratiques, vous pourrez maintenir votre application Ruby en bon état de fonctionnement et éviter les problèmes potentiels.
