Les tests unitaires avec Ruby

Les tests unitaires sont utilisés pour vérifier le bon fonctionnement des différentes unités de code, telles que les méthodes et les classes, de manière isolée. En Ruby, le framework de test le plus populaire est RSpec.

Voici un exemple de test unitaire en Ruby :

require 'rspec'

class Calculator
  def add(a, b)
    a + b
  end
end

RSpec.describe Calculator do
  describe '#add' do
    it 'returns the sum of two numbers' do
      calculator = Calculator.new
      result = calculator.add(2, 3)
      expect(result).to eq(5)
    end
  end
end

Dans cet exemple, nous avons une classe Calculator avec une méthode add qui additionne deux nombres. Le test vérifie si l'addition de 2 et 3 renvoie bien 5.

Les tests d'intégration avec Ruby

Les tests d'intégration permettent de vérifier le bon fonctionnement de l'ensemble du système en testant les interactions entre les différentes parties. En Ruby, les tests d'intégration peuvent être réalisés avec RSpec ou avec des frameworks spécifiques tels que Capybara pour les tests de navigation web.

Voici un exemple de test d'intégration en Ruby avec RSpec et Capybara :

require 'rspec'
require 'capybara/rspec'

Capybara.app = MyApp

RSpec.describe 'User management', type: :feature do
  it 'allows a user to sign up' do
    visit '/signup'
    fill_in 'Email', with: 'test@example.com'
    fill_in 'Password', with: 'password'
    click_button 'Sign up'
    expect(page).to have_content 'Welcome, test@example.com!'
  end
end

Dans cet exemple, nous avons un test d'intégration qui vérifie si un utilisateur peut s'inscrire sur une application web. Le test simule l'interaction de l'utilisateur avec le formulaire d'inscription en remplissant les champs email et mot de passe, puis en cliquant sur le bouton d'inscription. Ensuite, le test vérifie si la page affiche le message de bienvenue correspondant à l'adresse email de l'utilisateur.

Les outils de débogage en Ruby

Le débogage est une étape essentielle du processus de développement. En Ruby, il existe plusieurs outils de débogage disponibles, tels que Pry, Byebug et RubyMine.

Voici un exemple d'utilisation de l'outil de débogage Byebug :

require 'byebug'

def factorial(n)
  byebug
  if n == 0
    1
  else
    n * factorial(n-1)
  end
end

puts factorial(5)

Dans cet exemple, nous avons une fonction récursive factorial qui calcule la factorielle d'un nombre. L'instruction byebug permet de mettre en pause l'exécution du programme à cet endroit, ce qui permet d'inspecter les variables et d'exécuter des commandes pour comprendre le comportement du code.
