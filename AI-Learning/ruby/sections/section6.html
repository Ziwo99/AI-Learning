Connexion à une base de données

La connexion à une base de données est une étape essentielle dans le développement d'applications Ruby qui nécessitent une interaction avec une base de données. Ruby offre plusieurs options pour se connecter à différentes bases de données, telles que MySQL, PostgreSQL, SQLite, etc.

Pour se connecter à une base de données, nous devons d'abord installer le pilote approprié pour la base de données que nous utilisons. Par exemple, pour se connecter à une base de données MySQL, nous devons installer le pilote MySQL. Cela peut être fait en ajoutant la gem correspondante à notre fichier Gemfile et en exécutant la commande bundle install.

Une fois le pilote installé, nous pouvons établir une connexion à la base de données en utilisant les informations d'identification appropriées, telles que l'hôte, le nom d'utilisateur, le mot de passe, etc. Voici un exemple de code pour se connecter à une base de données MySQL :

require 'mysql2'

client = Mysql2::Client.new(
  host: "localhost",
  username: "root",
  password: "password",
  database: "mydatabase"
)

# Utilisation de la connexion à la base de données

Dans cet exemple, nous utilisons la gem mysql2 pour se connecter à une base de données MySQL. Nous créons une instance de la classe Mysql2::Client en passant les informations d'identification appropriées. Une fois la connexion établie, nous pouvons utiliser l'objet client pour exécuter des requêtes SQL et interagir avec la base de données.

Requêtes SQL avec Ruby

Une fois que nous sommes connectés à une base de données, nous pouvons exécuter des requêtes SQL pour récupérer, insérer, mettre à jour ou supprimer des données. Ruby offre plusieurs façons d'exécuter des requêtes SQL, notamment en utilisant des méthodes fournies par les pilotes de base de données ou en utilisant un ORM (Object-Relational Mapping).

Voici un exemple de code pour exécuter une requête SELECT pour récupérer des données à partir d'une table :

results = client.query("SELECT * FROM users")

results.each do |row|
  puts "ID: #{row['id']}, Name: #{row['name']}, Email: #{row['email']}"
end

Dans cet exemple, nous utilisons la méthode query de l'objet client pour exécuter une requête SELECT qui récupère toutes les lignes de la table "users". Les résultats de la requête sont renvoyés sous forme d'objet Mysql2::Result, que nous pouvons itérer pour accéder aux données de chaque ligne.

Utilisation d'un ORM (Object-Relational Mapping)

L'utilisation d'un ORM est une approche populaire pour interagir avec une base de données en Ruby. Un ORM permet de mapper les tables d'une base de données à des classes Ruby, ce qui facilite la manipulation des données en utilisant des objets plutôt que des requêtes SQL brutes.

Un exemple courant d'ORM en Ruby est ActiveRecord, qui est inclus dans le framework Ruby on Rails. Voici un exemple de code utilisant ActiveRecord pour récupérer toutes les lignes de la table "users" :

class User < ActiveRecord::Base
end

users = User.all

users.each do |user|
  puts "ID: #{user.id}, Name: #{user.name}, Email: #{user.email}"
end

Dans cet exemple, nous définissons une classe User qui hérite de la classe ActiveRecord::Base. ActiveRecord utilise des conventions de nommage pour mapper la classe User à la table "users" dans la base de données. Nous utilisons ensuite la méthode all pour récupérer toutes les lignes de la table "users" en tant qu'objets User. Nous pouvons ensuite itérer sur ces objets pour accéder aux données de chaque ligne.

L'utilisation d'un ORM offre de nombreux avantages, tels que la facilité d'utilisation, la gestion automatique des requêtes SQL et la possibilité de définir des relations entre les tables à l'aide de modèles d'objets.

Il est important de noter que chaque ORM peut avoir sa propre syntaxe et ses propres fonctionnalités spécifiques. Il est donc recommandé de consulter la documentation de l'ORM spécifique que vous utilisez pour obtenir des informations détaillées sur son utilisation.
