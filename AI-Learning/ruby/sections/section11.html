Section 11 : Avancé et sujets avancés en Ruby

Méta-programmation en Ruby

La méta-programmation en Ruby est une technique puissante qui permet de créer du code qui peut modifier ou générer du code lui-même. Cela permet aux développeurs de créer des bibliothèques et des frameworks flexibles et dynamiques.

Voici un exemple de méta-programmation en Ruby :

class MyClass
  define_method :my_dynamic_method do
    puts "This is a dynamic method created using meta-programming!"
  end
end

obj = MyClass.new
obj.my_dynamic_method

Dans cet exemple, nous utilisons la méthode define_method pour créer une méthode dynamique my_dynamic_method dans la classe MyClass. Cette méthode peut être appelée sur une instance de la classe et affiche un message à l'écran.

Utilisation de bibliothèques avancées

En Ruby, il existe de nombreuses bibliothèques avancées qui peuvent être utilisées pour étendre les fonctionnalités de base du langage. Ces bibliothèques offrent des fonctionnalités supplémentaires et facilitent le développement d'applications plus complexes.

Un exemple de bibliothèque avancée en Ruby est ActiveRecord, qui est utilisée dans le framework Ruby on Rails pour gérer les bases de données. Voici un exemple d'utilisation d'ActiveRecord :

require 'active_record'

class User < ActiveRecord::Base
  validates :name, presence: true
end

user = User.new
user.valid?  # Retourne false car le nom est manquant

Dans cet exemple, nous utilisons ActiveRecord pour définir une classe User qui représente une table dans une base de données. Nous utilisons également la validation presence pour nous assurer que le champ name n'est pas vide.

Optimisation des performances

L'optimisation des performances est un aspect important du développement en Ruby. Il existe plusieurs techniques et bonnes pratiques qui peuvent être utilisées pour améliorer les performances d'une application Ruby.

Un exemple de technique d'optimisation des performances en Ruby est l'utilisation de la méthode map au lieu de la méthode each pour itérer sur une collection :

numbers = [1, 2, 3, 4, 5]

# Utilisation de la méthode each
squared_numbers = []
numbers.each do |number|
  squared_numbers << number * number
end

# Utilisation de la méthode map
squared_numbers = numbers.map { |number| number * number }

Dans cet exemple, nous utilisons la méthode map pour créer un nouveau tableau squared_numbers qui contient les carrés des nombres du tableau numbers. Cette méthode est plus performante que l'utilisation de la méthode each avec un tableau vide et l'opérateur << pour ajouter les éléments au tableau.

Sécurité et bonnes pratiques de sécurité en Ruby

La sécurité est un aspect crucial du développement d'applications en Ruby. Il est important de suivre les bonnes pratiques de sécurité pour protéger les données sensibles et éviter les vulnérabilités.

Un exemple de bonne pratique de sécurité en Ruby est l'utilisation de la gem bcrypt pour le hachage sécurisé des mots de passe :

require 'bcrypt'

password = "my_password"
hashed_password = BCrypt::Password.create(password)

puts hashed_password

Dans cet exemple, nous utilisons la gem bcrypt pour créer un hachage sécurisé du mot de passe. Le mot de passe est stocké sous forme de hachage dans la variable hashed_password. Cela permet de stocker les mots de passe de manière sécurisée dans une base de données, sans les exposer en clair.
