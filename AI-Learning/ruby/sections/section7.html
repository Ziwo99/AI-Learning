Section 7 : Tests et débogage en Ruby

Introduction aux tests unitaires

Les tests unitaires sont une pratique essentielle dans le développement logiciel. Ils permettent de vérifier le bon fonctionnement des différentes parties d'un programme de manière isolée. En Ruby, il existe plusieurs frameworks de tests unitaires populaires, tels que RSpec et MiniTest.

Les tests unitaires consistent à écrire des cas de tests qui vérifient le comportement attendu d'une méthode ou d'une fonction. Ces cas de tests sont exécutés automatiquement et signalent toute déviation par rapport au comportement attendu.

Voici un exemple de test unitaire en utilisant le framework RSpec :

require 'rspec'

class Calculator
  def add(a, b)
    a + b
  end
end

RSpec.describe Calculator do
  describe '#add' do
    it 'returns the sum of two numbers' do
      calculator = Calculator.new
      result = calculator.add(2, 3)
      expect(result).to eq(5)
    end
  end
end

Utilisation de frameworks de tests (RSpec, MiniTest)

Les frameworks de tests tels que RSpec et MiniTest fournissent des outils et des structures pour écrire et exécuter des tests unitaires de manière organisée.

RSpec est un framework de tests très populaire en Ruby. Il utilise une syntaxe expressive et permet de décrire le comportement attendu d'une méthode ou d'une fonction de manière claire et lisible.

Voici un exemple d'utilisation de RSpec pour tester une classe Calculator :

require 'rspec'

class Calculator
  def add(a, b)
    a + b
  end
end

RSpec.describe Calculator do
  describe '#add' do
    it 'returns the sum of two numbers' do
      calculator = Calculator.new
      result = calculator.add(2, 3)
      expect(result).to eq(5)
    end
  end
end

MiniTest est un autre framework de tests intégré à Ruby. Il est plus léger que RSpec et utilise une syntaxe plus proche des tests unitaires traditionnels.

Voici un exemple d'utilisation de MiniTest pour tester une classe Calculator :

require 'minitest/autorun'

class Calculator
  def add(a, b)
    a + b
  end
end

class CalculatorTest < MiniTest::Test
  def test_add
    calculator = Calculator.new
    result = calculator.add(2, 3)
    assert_equal 5, result
  end
end

Débogage de code Ruby

Le débogage est une compétence essentielle pour identifier et résoudre les erreurs dans un programme. En Ruby, il existe plusieurs outils et techniques de débogage qui peuvent vous aider à comprendre le comportement d'un programme et à localiser les erreurs.

Un outil de débogage couramment utilisé en Ruby est le débogueur intégré (IRB). Il permet d'exécuter du code Ruby pas à pas et d'inspecter les valeurs des variables à chaque étape.

Voici un exemple d'utilisation du débogueur IRB :

def factorial(n)
  if n == 0
    1
  else
    n * factorial(n-1)
  end
end

result = factorial(5)
puts result

Pour déboguer ce code, vous pouvez insérer la ligne suivante avant l'appel à la méthode factorial :

require 'irb'; binding.irb

Cela ouvrira une console IRB à l'endroit où vous avez inséré la ligne. Vous pouvez alors exécuter du code Ruby et inspecter les valeurs des variables pour comprendre le comportement du programme.

En utilisant ces techniques de débogage, vous serez en mesure d'identifier et de résoudre les erreurs dans votre code Ruby de manière efficace.
