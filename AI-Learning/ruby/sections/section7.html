Introduction aux frameworks web Ruby (Ruby on Rails, Sinatra)

Les frameworks web Ruby sont des outils puissants qui facilitent le développement d'applications web en utilisant le langage de programmation Ruby. Deux des frameworks web les plus populaires en Ruby sont Ruby on Rails et Sinatra.

Ruby on Rails, également connu sous le nom de Rails, est un framework web complet qui suit le principe de convention plutôt que de configuration. Il fournit une structure solide pour le développement d'applications web et suit le modèle MVC (Modèle-Vue-Contrôleur) pour organiser le code.

Sinatra, en revanche, est un framework web minimaliste et léger. Il est souvent utilisé pour des projets plus simples ou des API RESTful. Sinatra est apprécié pour sa simplicité et sa flexibilité.

Création d'une application web simple

Pour créer une application web simple en Ruby, nous pouvons utiliser le framework Ruby on Rails. Voici les étapes pour créer une nouvelle application Rails :

$ rails new mon_app_web
$ cd mon_app_web

Ces commandes créent un nouveau projet Rails appelé "mon_app_web" et nous positionnent dans le répertoire du projet.

Ensuite, nous pouvons générer un contrôleur et une vue pour notre application :

$ rails generate controller Accueil index

Cette commande génère un contrôleur appelé "Accueil" avec une action "index" et crée également une vue correspondante.

Nous pouvons maintenant démarrer le serveur Rails pour voir notre application en action :

$ rails server

En accédant à l'URL "http://localhost:3000/accueil/index" dans notre navigateur, nous devrions voir le contenu de la vue "index" que nous avons générée.

Gestion des routes et des contrôleurs

Dans Ruby on Rails, les routes définissent les URL de notre application et les associent aux actions des contrôleurs correspondants. Les routes sont définies dans le fichier "config/routes.rb". Voici un exemple de définition de route :

get '/accueil', to: 'accueil#index'

Cette route indique que lorsque l'utilisateur accède à l'URL "/accueil", l'action "index" du contrôleur "Accueil" doit être exécutée.

Les contrôleurs dans Rails sont responsables de la logique métier de notre application web. Chaque action dans un contrôleur correspond à une URL spécifique et est responsable de la récupération des données nécessaires et de la préparation de la réponse à renvoyer au navigateur.

Voici un exemple de code pour une action dans un contrôleur Rails :

class AccueilController < ApplicationController
  def index
    @message = "Bienvenue sur notre site web !"
  end
end

Dans cet exemple, l'action "index" du contrôleur "Accueil" définit une instance de variable "@message" qui sera utilisée dans la vue correspondante pour afficher un message de bienvenue.

Utilisation de vues et de modèles

Les vues dans Ruby on Rails sont responsables de l'affichage des données aux utilisateurs. Elles utilisent des fichiers HTML avec des balises ERB (Embedded Ruby) pour intégrer du code Ruby dans les vues.

Voici un exemple de code pour une vue dans Rails :

<h1><%= @message %></h1>

Dans cet exemple, la vue affiche le contenu de la variable "@message" définie dans l'action du contrôleur.

Les modèles dans Rails sont responsables de l'accès aux données et de leur manipulation. Ils sont généralement associés à des tables de base de données et utilisent des requêtes SQL pour interagir avec la base de données.

Voici un exemple de code pour un modèle dans Rails :

class Utilisateur < ApplicationRecord
  validates :nom, presence: true
end

Dans cet exemple, le modèle "Utilisateur" utilise une validation pour s'assurer que le champ "nom" est présent avant de sauvegarder les données dans la base de données.

Ce sont là quelques-uns des concepts de base de la programmation web avec Ruby. En utilisant les frameworks web Ruby tels que Ruby on Rails et Sinatra, vous pouvez créer des applications web puissantes et élégantes.
