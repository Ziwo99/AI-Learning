Concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet d'organiser et de structurer le code en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles ou des plans pour créer des objets. La POO repose sur quatre concepts fondamentaux :

  • L'encapsulation : il s'agit de regrouper des données et des méthodes associées dans une même entité, appelée objet. L'encapsulation permet de cacher les détails internes de l'objet et de ne rendre accessibles que les fonctionnalités nécessaires à son utilisation.
  • L'abstraction : elle consiste à définir une interface claire et simplifiée pour interagir avec un objet, sans se soucier des détails de son implémentation interne. L'abstraction permet de se concentrer sur l'utilisation de l'objet plutôt que sur sa complexité interne.
  • L'héritage : il permet de créer de nouvelles classes en se basant sur des classes existantes. La classe nouvellement créée hérite des attributs et des méthodes de la classe parente, ce qui facilite la réutilisation du code et la création de hiérarchies de classes.
  • Le polymorphisme : il permet à des objets de différentes classes d'être traités de manière uniforme. Cela signifie qu'un objet peut être utilisé comme une instance de sa classe d'origine ou comme une instance d'une de ses classes dérivées.

Classes et objets

En Ruby, une classe est définie à l'aide du mot-clé class. Une classe est un modèle à partir duquel des objets peuvent être créés. Voici un exemple de définition de classe :

class Person
  attr_accessor :name, :age

  def initialize(name, age)
    @name = name
    @age = age
  end

  def say_hello
    puts "Hello, my name is #{@name} and I'm #{@age} years old."
  end
end

person = Person.new("John", 30)
person.say_hello

Dans cet exemple, nous avons défini une classe Person avec deux attributs name et age. La méthode initialize est un constructeur qui est appelé lors de la création d'un nouvel objet de la classe. La méthode say_hello affiche un message de salutation avec le nom et l'âge de la personne.

Pour créer un nouvel objet de la classe Person, nous utilisons la méthode new suivie des arguments requis par le constructeur. Ensuite, nous pouvons appeler les méthodes de l'objet créé.

Héritage et polymorphisme

En Ruby, l'héritage est réalisé en utilisant le symbole < pour indiquer que la classe enfant hérite de la classe parente. Voici un exemple :

class Animal
  def speak
    puts "I'm an animal."
  end
end

class Dog < Animal
  def speak
    puts "Woof!"
  end
end

class Cat < Animal
  def speak
    puts "Meow!"
  end
end

animal = Animal.new
dog = Dog.new
cat = Cat.new

animal.speak
dog.speak
cat.speak

Dans cet exemple, nous avons une classe parente Animal avec une méthode speak. Les classes enfants Dog et Cat héritent de la classe Animal et redéfinissent la méthode speak pour produire des sons spécifiques à chaque animal.

Le polymorphisme est illustré ici par le fait que nous pouvons appeler la méthode speak sur des objets de différentes classes, mais obtenir des résultats différents en fonction de la classe réelle de l'objet.

Encapsulation et abstraction

En Ruby, l'encapsulation est réalisée en utilisant des méthodes d'accès pour définir les attributs d'une classe. Les méthodes d'accès peuvent être définies à l'aide des mots-clés attr_reader, attr_writer ou attr_accessor. Voici un exemple :

class Car
  attr_accessor :brand, :model

  def initialize(brand, model)
    @brand = brand
    @model = model
  end
end

car = Car.new("Toyota", "Corolla")
puts car.brand
puts car.model

Dans cet exemple, nous avons une classe Car avec deux attributs brand et model. En utilisant attr_accessor, nous créons automatiquement des méthodes d'accès en lecture et en écriture pour ces attributs. Ainsi, nous pouvons lire et modifier les attributs de l'objet car en utilisant les méthodes générées.

L'abstraction est réalisée en définissant des méthodes publiques qui exposent uniquement les fonctionnalités nécessaires à l'utilisation de l'objet, tout en masquant les détails de son implémentation interne. Cela permet de simplifier l'utilisation de l'objet et de réduire la complexité du code client.

Modules et mixins

En Ruby, les modules sont des collections de méthodes et de constantes qui peuvent être incluses dans des classes pour leur ajouter des fonctionnalités supplémentaires. Voici un exemple :

module Greetings
  def say_hello
    puts "Hello!"
  end
end

class Person
  include Greetings
end

person = Person.new
person.say_hello

Dans cet exemple, nous avons défini un module Greetings avec une méthode say_hello. En incluant ce module dans la classe Person à l'aide du mot-clé include, nous ajoutons la méthode say_hello à la classe Person. Ainsi, les objets de la classe Person peuvent appeler la méthode say_hello.

Les mixins sont une forme de composition multiple en Ruby, ce qui signifie qu'une classe peut inclure plusieurs modules pour obtenir les fonctionnalités de chacun d'entre eux. Cela permet une grande flexibilité dans la conception et la réutilisation du code.
