Section 3 : Programmation orientée objet en Ruby

Concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles pour créer des objets avec des attributs et des méthodes.

En Ruby, tout est un objet, y compris les entiers, les chaînes de caractères et même les classes elles-mêmes. Pour créer une classe, on utilise le mot-clé class suivi du nom de la classe en commençant par une majuscule.

class Person
  # attributs et méthodes de la classe
end

Classes et objets

Une classe est un modèle pour créer des objets. Elle définit les attributs et les méthodes que les objets de cette classe auront.

Pour créer un nouvel objet à partir d'une classe, on utilise la méthode new de la classe :

person = Person.new

Les attributs d'un objet sont des variables qui lui sont propres. On peut les définir dans le constructeur de la classe :

class Person
  def initialize(name, age)
    @name = name
    @age = age
  end
end

person = Person.new("John", 25)

Héritage et polymorphisme

L'héritage permet de créer une nouvelle classe basée sur une classe existante. La nouvelle classe hérite des attributs et des méthodes de la classe parente.

En Ruby, on utilise le symbole < pour indiquer l'héritage :

class Student < Person
  # attributs et méthodes spécifiques aux étudiants
end

Le polymorphisme permet à des objets de différentes classes d'être traités de manière similaire. Par exemple, si une méthode attend un objet de la classe parente, on peut lui passer un objet de la classe enfant sans problème.

def print_name(person)
  puts person.name
end

person = Person.new("John", 25)
student = Student.new("Alice", 20)

print_name(person)  # affiche "John"
print_name(student) # affiche "Alice"

Modules et mixins

Les modules sont des collections de méthodes et de constantes qui peuvent être incluses dans une classe. Ils permettent de regrouper du code réutilisable.

En Ruby, on utilise le mot-clé module pour définir un module :

module MathUtils
  def self.square(x)
    x * x
  end
end

puts MathUtils.square(5) # affiche 25

Les mixins sont une façon d'inclure des modules dans une classe. Cela permet à la classe d'utiliser les méthodes du module comme si elles étaient ses propres méthodes.

class Calculator
  include MathUtils

  def square_root(x)
    Math.sqrt(x)
  end
end

calculator = Calculator.new
puts calculator.square(5)       # affiche 25
puts calculator.square_root(25) # affiche 5

Exceptions et gestion des erreurs

Les exceptions sont des événements qui se produisent lorsqu'une erreur se produit pendant l'exécution du programme. Elles permettent de gérer les erreurs de manière structurée.

En Ruby, on utilise le mot-clé rescue pour capturer une exception et le mot-clé raise pour lever une exception :

begin
  # code susceptible de lever une exception
rescue ExceptionType
  # code à exécuter en cas d'exception
end

def divide(x, y)
  raise "Division by zero" if y == 0
  x / y
rescue => e
  puts "An error occurred: #{e.message}"
end

puts divide(10, 2)  # affiche 5
puts divide(10, 0)  # affiche "An error occurred: Division by zero"
