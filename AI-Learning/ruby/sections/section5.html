Les exceptions prédéfinies en Ruby

En Ruby, les exceptions sont utilisées pour gérer les erreurs et les situations exceptionnelles qui peuvent survenir lors de l'exécution d'un programme. Ruby fournit plusieurs exceptions prédéfinies qui peuvent être utilisées pour capturer et gérer ces erreurs.

Voici quelques-unes des exceptions prédéfinies les plus couramment utilisées en Ruby :

  • StandardError : La classe de base pour toutes les exceptions couramment rencontrées.
  • ArgumentError : Levée lorsque les arguments d'une méthode sont incorrects.
  • TypeError : Levée lorsque le type d'un objet est incorrect.
  • ZeroDivisionError : Levée lorsque la division par zéro est effectuée.
  • FileNotFoundError : Levée lorsque le fichier spécifié n'est pas trouvé.

Chaque exception prédéfinie a un nom spécifique et peut être utilisée pour capturer des erreurs spécifiques dans votre code Ruby.

La gestion des exceptions avec les blocs begin-rescue-end

En Ruby, la gestion des exceptions est réalisée à l'aide des blocs begin, rescue et end. Le bloc begin est utilisé pour entourer le code qui pourrait potentiellement lever une exception. Le bloc rescue est utilisé pour capturer et gérer les exceptions levées dans le bloc begin. Le bloc end est utilisé pour marquer la fin du bloc begin.

Voici un exemple de gestion d'exception avec les blocs begin-rescue-end :

begin
    # Code susceptible de lever une exception
    result = 10 / 0
  rescue ZeroDivisionError
    # Gestion de l'exception ZeroDivisionError
    puts "Erreur : division par zéro"
  end

Dans cet exemple, le code à l'intérieur du bloc begin tente de diviser 10 par zéro, ce qui lève une exception ZeroDivisionError. Le bloc rescue capture cette exception et exécute le code à l'intérieur, qui affiche le message d'erreur "Erreur : division par zéro".

La création de ses propres exceptions

En plus des exceptions prédéfinies, Ruby permet également aux développeurs de créer leurs propres exceptions personnalisées. Cela peut être utile lorsque vous avez besoin de gérer des erreurs spécifiques à votre application.

Voici comment créer une exception personnalisée en Ruby :

class CustomException < StandardError
    # Ajoutez du code spécifique à votre exception personnalisée ici
  end

Dans cet exemple, nous avons créé une classe CustomException qui hérite de la classe de base StandardError. Vous pouvez ajouter du code spécifique à votre exception personnalisée à l'intérieur de cette classe.

Une fois que vous avez créé votre exception personnalisée, vous pouvez l'utiliser de la même manière que les exceptions prédéfinies pour capturer et gérer des erreurs spécifiques dans votre code.
