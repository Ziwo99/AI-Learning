Introduction aux exceptions

Les exceptions sont des événements qui se produisent lors de l'exécution d'un programme et qui interrompent le flux normal d'exécution. Elles sont utilisées pour gérer les erreurs et les situations exceptionnelles qui peuvent survenir pendant l'exécution d'un programme Ruby.

En Ruby, les exceptions sont des objets qui sont levés (ou "throw") lorsqu'une erreur se produit. Lorsqu'une exception est levée, le programme recherche un bloc de code appelé "rescue" qui peut gérer cette exception. Si aucun bloc rescue n'est trouvé, le programme s'arrête et affiche un message d'erreur.

Voici un exemple de levée d'exception :

begin
  # Code susceptible de lever une exception
  raise "Une erreur s'est produite"
rescue
  # Bloc de code pour gérer l'exception
  puts "Une exception a été levée"
end

Dans cet exemple, nous utilisons le mot-clé "raise" pour lever une exception avec le message "Une erreur s'est produite". Le bloc "rescue" est utilisé pour gérer cette exception et affiche le message "Une exception a été levée".

Utilisation des blocs begin-rescue-ensure

Les blocs "begin", "rescue" et "ensure" sont utilisés ensemble pour gérer les exceptions de manière plus précise.

Le bloc "begin" est utilisé pour entourer le code qui pourrait lever une exception. Le bloc "rescue" est utilisé pour gérer les exceptions levées dans le bloc "begin". Le bloc "ensure" est utilisé pour exécuter du code qui doit être exécuté, qu'une exception soit levée ou non.

Voici un exemple d'utilisation des blocs begin-rescue-ensure :

begin
  # Code susceptible de lever une exception
  raise "Une erreur s'est produite"
rescue
  # Bloc de code pour gérer l'exception
  puts "Une exception a été levée"
ensure
  # Bloc de code à exécuter quoi qu'il arrive
  puts "Fin de l'exécution"
end

Dans cet exemple, le bloc "begin" entoure le code qui peut lever une exception. Si une exception est levée, le bloc "rescue" est exécuté et affiche le message "Une exception a été levée". Ensuite, le bloc "ensure" est exécuté et affiche le message "Fin de l'exécution".

Création de ses propres exceptions

En plus des exceptions prédéfinies de Ruby, vous pouvez également créer vos propres exceptions personnalisées. Cela peut être utile lorsque vous souhaitez gérer des erreurs spécifiques à votre programme.

Pour créer une nouvelle exception, vous devez définir une classe qui hérite de la classe Exception. Voici un exemple :

class MonException < Exception
end

begin
  # Code susceptible de lever une exception
  raise MonException, "Une erreur s'est produite"
rescue MonException => e
  # Bloc de code pour gérer l'exception personnalisée
  puts "Une exception personnalisée a été levée : #{e.message}"
end

Dans cet exemple, nous définissons une nouvelle classe d'exception appelée "MonException" qui hérite de la classe Exception. Ensuite, nous utilisons le mot-clé "raise" pour lever une exception de cette classe avec le message "Une erreur s'est produite". Le bloc "rescue" est utilisé pour gérer cette exception personnalisée et affiche le message "Une exception personnalisée a été levée : Une erreur s'est produite".

Il est recommandé d'utiliser des exceptions personnalisées pour des erreurs spécifiques à votre programme, afin de faciliter la gestion des erreurs et d'améliorer la lisibilité de votre code.
