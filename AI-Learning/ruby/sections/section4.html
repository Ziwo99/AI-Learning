Section 4 : Manipulation de fichiers et dossiers

Lecture et écriture de fichiers

La lecture et l'écriture de fichiers sont des opérations courantes en programmation Ruby. Pour lire le contenu d'un fichier, vous pouvez utiliser la méthode File.open avec le mode de lecture approprié. Voici un exemple :

File.open('mon_fichier.txt', 'r') do |fichier|
  contenu = fichier.read
  puts contenu
end

Dans cet exemple, nous ouvrons le fichier mon_fichier.txt en mode lecture ('r') et nous utilisons la méthode read pour lire son contenu. Ensuite, nous affichons le contenu à l'aide de la méthode puts.

Pour écrire dans un fichier, vous pouvez utiliser la méthode File.open avec le mode d'écriture approprié. Voici un exemple :

File.open('mon_fichier.txt', 'w') do |fichier|
  fichier.write('Contenu à écrire dans le fichier')
end

Dans cet exemple, nous ouvrons le fichier mon_fichier.txt en mode écriture ('w') et nous utilisons la méthode write pour écrire le contenu spécifié dans le fichier.

Manipulation de dossiers

La manipulation de dossiers en Ruby peut être réalisée à l'aide de la classe Dir. Voici quelques opérations courantes :

Pour créer un nouveau dossier :

Dir.mkdir('nouveau_dossier')

Pour supprimer un dossier :

Dir.delete('dossier_a_supprimer')

Pour vérifier si un dossier existe :

if Dir.exist?('dossier')
  puts 'Le dossier existe'
else
  puts 'Le dossier n\'existe pas'
end

Gestion des erreurs liées aux fichiers et dossiers

Il est important de gérer les erreurs liées aux fichiers et dossiers lors de la manipulation. Voici un exemple de gestion d'erreur lors de la lecture d'un fichier :

begin
  File.open('fichier_inexistant.txt', 'r') do |fichier|
    contenu = fichier.read
    puts contenu
  end
rescue Errno::ENOENT
  puts 'Le fichier n\'existe pas'
end

Dans cet exemple, nous utilisons la structure begin/rescue pour capturer l'erreur Errno::ENOENT qui se produit lorsque le fichier spécifié n'existe pas. Dans le bloc rescue, nous affichons un message indiquant que le fichier n'existe pas.

Assurez-vous de toujours gérer les erreurs de manière appropriée pour éviter les problèmes lors de la manipulation de fichiers et de dossiers.
