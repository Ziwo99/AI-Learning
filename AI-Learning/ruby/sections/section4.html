Les concepts de base des classes et des objets

En Ruby, tout est un objet. Une classe est un modèle ou un plan pour créer des objets. Les objets sont des instances de classes. Les classes définissent les propriétés (attributs) et les comportements (méthodes) des objets.

La création d'une classe se fait en utilisant le mot-clé class, suivi du nom de la classe en commençant par une majuscule. Par exemple, pour créer une classe "Personne", on peut écrire :

class Personne
end

Pour créer un objet à partir d'une classe, on utilise la méthode new. Par exemple, pour créer un objet "personne1" à partir de la classe "Personne", on peut écrire :

personne1 = Personne.new

Les objets ont des attributs qui définissent leurs caractéristiques. On peut définir des attributs pour une classe en utilisant le mot-clé attr_accessor. Par exemple, pour définir les attributs "nom" et "age" pour la classe "Personne", on peut écrire :

class Personne
  attr_accessor :nom, :age
end

Les attributs peuvent être lus et modifiés en utilisant les méthodes générées automatiquement par attr_accessor. Par exemple, pour lire et modifier l'attribut "nom" de l'objet "personne1", on peut écrire :

personne1.nom = "Alice"
puts personne1.nom  # Affiche "Alice"

L'encapsulation et l'abstraction en Ruby

L'encapsulation est le principe de regrouper les données et les méthodes qui les manipulent dans une classe. Cela permet de cacher les détails d'implémentation et de protéger les données d'accès non autorisé.

En Ruby, l'encapsulation est réalisée en utilisant des méthodes d'accès (getters et setters) pour lire et modifier les attributs d'un objet. Par convention, les getters ont le même nom que l'attribut et les setters ont le même nom suivi d'un égal (=).

Par exemple, pour encapsuler l'attribut "nom" de la classe "Personne", on peut écrire :

class Personne
  def nom
    @nom
  end

  def nom=(nouveau_nom)
    @nom = nouveau_nom
  end
end

Ainsi, on peut lire et modifier l'attribut "nom" de la même manière qu'avec attr_accessor.

L'héritage et le polymorphisme

L'héritage est un mécanisme qui permet à une classe d'hériter des propriétés et des comportements d'une autre classe. La classe qui hérite est appelée la classe dérivée ou sous-classe, et la classe dont elle hérite est appelée la classe de base ou superclasse.

En Ruby, l'héritage est réalisé en utilisant le symbole <. Par exemple, pour définir une classe "Etudiant" qui hérite de la classe "Personne", on peut écrire :

class Etudiant < Personne
end

La classe "Etudiant" hérite maintenant des attributs et des méthodes de la classe "Personne". On peut également ajouter des attributs et des méthodes spécifiques à la classe "Etudiant".

Le polymorphisme est la capacité d'un objet à prendre différentes formes. En Ruby, le polymorphisme est réalisé en utilisant des méthodes avec le même nom mais des implémentations différentes dans les classes dérivées.

Par exemple, si la classe "Personne" a une méthode "presentation" qui affiche le nom de la personne, on peut redéfinir cette méthode dans la classe "Etudiant" pour afficher également le numéro d'étudiant :

class Personne
  attr_accessor :nom

  def presentation
    puts "Je m'appelle #{nom}."
  end
end

class Etudiant < Personne
  attr_accessor :numero_etudiant

  def presentation
    puts "Je m'appelle #{nom} et mon numéro d'étudiant est #{numero_etudiant}."
  end
end

Ainsi, on peut appeler la méthode "presentation" sur un objet de type "Personne" ou "Etudiant" et obtenir des résultats différents en fonction du type de l'objet.

Les modules et les mixins

Les modules sont des collections de méthodes et de constantes qui peuvent être incluses dans les classes. Ils permettent de regrouper des fonctionnalités communes et de les réutiliser dans plusieurs classes.

En Ruby, on peut définir un module en utilisant le mot-clé module. Par exemple, pour définir un module "Logger" qui ajoute des fonctionnalités de journalisation à une classe, on peut écrire :

module Logger
  def log(message)
    puts "[LOG] #{message}"
  end
end

Pour inclure un module dans une classe, on utilise le mot-clé include. Par exemple, pour inclure le module "Logger" dans la classe "Personne", on peut écrire :

class Personne
  include Logger

  attr_accessor :nom
end

Maintenant, la classe "Personne" a accès à la méthode "log" du module "Logger". On peut l'appeler sur un objet de type "Personne" :

personne1 = Personne.new
personne1.nom = "Alice"
personne1.log("Nouvelle personne créée")  # Affiche "[LOG] Nouvelle personne créée"

Les mixins sont une forme de composition multiple où une classe peut inclure plusieurs modules. Cela permet de combiner différentes fonctionnalités de manière flexible.

Par exemple, on peut définir un module "EmailSender" qui ajoute des fonctionnalités d'envoi d'e-mails à une classe :

module EmailSender
  def send_email(to, subject, body)
    # Code pour envoyer un e-mail
  end
end

Ensuite, on peut inclure à la fois le module "Logger" et le module "EmailSender" dans la classe "Personne" :

class Personne
  include Logger
  include EmailSender

  attr_accessor :nom
end

Maintenant, la classe "Personne" a accès à la fois à la méthode "log" du module "Logger" et à la méthode "send_email" du module "EmailSender". On peut les appeler sur un objet de type "Personne" :

personne1 = Personne.new
personne1.nom = "Alice"
personne1.log("Nouvelle personne créée")  # Affiche "[LOG] Nouvelle personne créée"
personne1.send_email("alice@example.com", "Nouveau message", "Bonjour Alice, vous avez un nouveau message.")  # Envoie un e-mail
