Section 5 : Gestion des données avec Ajax

Envoi et réception de données en format JSON

Lorsque vous travaillez avec Ajax, il est courant d'envoyer et de recevoir des données au format JSON (JavaScript Object Notation). JSON est un format de données léger et facile à lire et à écrire. Il est souvent utilisé pour échanger des données entre un serveur et une application web. Pour envoyer des données JSON avec Ajax, vous devez d'abord convertir vos données en une chaîne JSON à l'aide de la méthode JSON.stringify(). Ensuite, vous pouvez utiliser la méthode XMLHttpRequest pour envoyer cette chaîne JSON au serveur. Voici un exemple de code qui envoie des données JSON au serveur : 

var data = {
    name: "John Doe",
    age: 30,
    email: "johndoe@example.com"
};

var jsonData = JSON.stringify(data);

var xhr = new XMLHttpRequest();
xhr.open("POST", "https://example.com/api", true);
xhr.setRequestHeader("Content-Type", "application/json");
xhr.send(jsonData);

Pour recevoir des données JSON avec Ajax, vous devez spécifier le type de données attendu dans l'en-tête de la requête. Vous pouvez le faire en utilisant la méthode setRequestHeader() avec le paramètre "Accept" défini sur "application/json". Voici un exemple de code qui reçoit des données JSON du serveur : 

var xhr = new XMLHttpRequest();
xhr.open("GET", "https://example.com/api/data", true);
xhr.setRequestHeader("Accept", "application/json");

xhr.onreadystatechange = function() {
    if (xhr.readyState === 4 && xhr.status === 200) {
        var jsonData = JSON.parse(xhr.responseText);
        // Utilisez les données JSON ici
    }
};

xhr.send();

Utilisation de XML avec Ajax

En plus du format JSON, vous pouvez également utiliser XML (eXtensible Markup Language) pour échanger des données avec Ajax. XML est un langage de balisage qui permet de structurer les données de manière hiérarchique. Pour envoyer des données XML avec Ajax, vous devez créer une chaîne XML valide en utilisant les balises appropriées. Ensuite, vous pouvez utiliser la méthode XMLHttpRequest pour envoyer cette chaîne XML au serveur. Voici un exemple de code qui envoie des données XML au serveur : 

var xmlData = "<person><name>John Doe</name><age>30</age><email>johndoe@example.com</email></person>";

var xhr = new XMLHttpRequest();
xhr.open("POST", "https://example.com/api", true);
xhr.setRequestHeader("Content-Type", "application/xml");
xhr.send(xmlData);

Pour recevoir des données XML avec Ajax, vous pouvez utiliser la même méthode que pour les données JSON. Vous devez spécifier le type de données attendu dans l'en-tête de la requête en utilisant la méthode setRequestHeader() avec le paramètre "Accept" défini sur "application/xml". Voici un exemple de code qui reçoit des données XML du serveur : 

var xhr = new XMLHttpRequest();
xhr.open("GET", "https://example.com/api/data", true);
xhr.setRequestHeader("Accept", "application/xml");

xhr.onreadystatechange = function() {
    if (xhr.readyState === 4 && xhr.status === 200) {
        var xmlData = xhr.responseXML;
        // Utilisez les données XML ici
    }
};

xhr.send();

Utilisation de HTML avec Ajax

En plus des formats JSON et XML, vous pouvez également utiliser HTML pour échanger des données avec Ajax. Cela peut être utile lorsque vous souhaitez charger du contenu HTML dynamiquement dans une page web. Pour charger du contenu HTML avec Ajax, vous pouvez utiliser la méthode XMLHttpRequest pour envoyer une requête GET au serveur. Le serveur renverra alors le contenu HTML demandé, que vous pourrez insérer dans votre page web. Voici un exemple de code qui charge du contenu HTML avec Ajax : 

var xhr = new XMLHttpRequest();
xhr.open("GET", "https://example.com/content.html", true);

xhr.onreadystatechange = function() {
    if (xhr.readyState === 4 && xhr.status === 200) {
        var htmlData = xhr.responseText;
        // Insérez le contenu HTML dans votre page web ici
    }
};

xhr.send();

Notez que le contenu HTML renvoyé par le serveur peut contenir des balises <script> ou d'autres éléments interactifs. Assurez-vous de prendre les mesures de sécurité appropriées pour éviter les attaques XSS (Cross-Site Scripting).
