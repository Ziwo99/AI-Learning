Inventaire et gestion des hôtes

L'inventaire dans Ansible est un fichier qui répertorie tous les hôtes sur lesquels vous souhaitez exécuter des tâches. Il peut être au format INI ou YAML. L'inventaire peut être utilisé pour regrouper les hôtes en fonction de différents critères tels que les rôles, les environnements, etc.

Voici un exemple d'inventaire au format INI :

[servers]
  server1 ansible_host=192.168.0.1
  server2 ansible_host=192.168.0.2

  [database]
  db1 ansible_host=192.168.0.3
  db2 ansible_host=192.168.0.4

Une fois que vous avez défini votre inventaire, vous pouvez utiliser des modules Ansible pour gérer les hôtes. Par exemple, vous pouvez utiliser le module "ping" pour vérifier si les hôtes sont accessibles :

- name: Ping all hosts
  hosts: all
  tasks:
    - name: Ping host
      ping:

Playbooks et tâches

Les playbooks sont des fichiers YAML qui décrivent les tâches à exécuter sur les hôtes. Ils permettent de définir des scénarios d'automatisation en spécifiant les tâches à effectuer, les hôtes sur lesquels les exécuter et les variables à utiliser.

Voici un exemple de playbook qui installe Apache sur un groupe d'hôtes :

- name: Install Apache
  hosts: webservers
  tasks:
    - name: Install Apache
      apt:
        name: apache2
        state: present

Modules et plugins

Les modules sont des unités de travail autonomes dans Ansible. Ils sont utilisés pour effectuer des actions spécifiques sur les hôtes. Ansible fournit de nombreux modules prêts à l'emploi pour effectuer des tâches courantes telles que la gestion des packages, la configuration des services, etc.

Voici un exemple d'utilisation du module "copy" pour copier un fichier sur les hôtes :

- name: Copy file
  hosts: all
  tasks:
    - name: Copy file
      copy:
        src: /path/to/source/file
        dest: /path/to/destination/file

Variables et gestion des données

Les variables sont utilisées pour stocker des données dans Ansible. Elles peuvent être définies au niveau du playbook, de l'inventaire ou dans des fichiers séparés appelés "vars_files". Les variables peuvent être utilisées pour personnaliser les tâches en fonction des hôtes ou des groupes d'hôtes.

Voici un exemple d'utilisation de variables dans un playbook :

- name: Install package
  hosts: all
  vars:
    package_name: apache2
  tasks:
    - name: Install package
      apt:
        name: "{{ package_name }}"
        state: present

Gestion des erreurs et débogage

Ansible fournit des mécanismes pour gérer les erreurs et déboguer les playbooks. Vous pouvez utiliser des blocs "ignore_errors" pour ignorer les erreurs spécifiques et continuer l'exécution du playbook. Vous pouvez également utiliser le module "debug" pour afficher des informations de débogage pendant l'exécution du playbook.

Voici un exemple d'utilisation du bloc "ignore_errors" :

- name: Install package
  hosts: all
  tasks:
    - name: Install package
      apt:
        name: apache2
        state: present
      ignore_errors: yes
