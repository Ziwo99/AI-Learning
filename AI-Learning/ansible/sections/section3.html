Gestion des fichiers et répertoires

La gestion des fichiers et des répertoires est une tâche courante lors de l'automatisation des tâches avec Ansible. Ansible fournit plusieurs modules pour faciliter cette gestion.

Le module "file" permet de créer, supprimer ou modifier des fichiers. Par exemple, pour créer un fichier "example.txt" avec le contenu "Hello World", vous pouvez utiliser le code suivant :

- name: Créer un fichier
  hosts: localhost
  tasks:
    - name: Créer le fichier example.txt
      file:
        path: /path/to/example.txt
        state: touch
        mode: '0644'
        owner: user
        group: group
        content: "Hello World"

Le module "copy" permet de copier des fichiers d'une machine à une autre. Par exemple, pour copier le fichier "example.txt" du répertoire local vers le répertoire distant "/path/to/", vous pouvez utiliser le code suivant :

- name: Copier un fichier
  hosts: destination
  tasks:
    - name: Copier le fichier example.txt
      copy:
        src: /path/to/example.txt
        dest: /path/to/example.txt
        owner: user
        group: group
        mode: '0644'

Le module "template" permet de générer des fichiers à partir de modèles. Par exemple, pour générer un fichier de configuration à partir d'un modèle "example.conf.j2" avec des variables, vous pouvez utiliser le code suivant :

- name: Générer un fichier de configuration
  hosts: localhost
  tasks:
    - name: Générer le fichier example.conf
      template:
        src: /path/to/example.conf.j2
        dest: /path/to/example.conf
        owner: user
        group: group
        mode: '0644'

Gestion des services et processus

La gestion des services et des processus est une autre tâche courante lors de l'automatisation des tâches avec Ansible. Ansible fournit plusieurs modules pour faciliter cette gestion.

Le module "service" permet de démarrer, arrêter ou redémarrer des services. Par exemple, pour démarrer le service "apache2", vous pouvez utiliser le code suivant :

- name: Démarrer un service
  hosts: localhost
  tasks:
    - name: Démarrer le service apache2
      service:
        name: apache2
        state: started

Le module "systemd" permet de gérer les services systemd. Par exemple, pour activer le service "nginx" au démarrage du système, vous pouvez utiliser le code suivant :

- name: Activer un service systemd
  hosts: localhost
  tasks:
    - name: Activer le service nginx
      systemd:
        name: nginx
        enabled: yes

Le module "command" permet d'exécuter des commandes sur les hôtes distants. Par exemple, pour exécuter la commande "ls -l" sur l'hôte distant, vous pouvez utiliser le code suivant :

- name: Exécuter une commande
  hosts: localhost
  tasks:
    - name: Exécuter la commande ls -l
      command: ls -l

Configuration des réseaux

La configuration des réseaux est une autre tâche courante lors de l'automatisation des tâches avec Ansible. Ansible fournit plusieurs modules pour faciliter cette configuration.

Le module "lineinfile" permet de modifier des lignes dans un fichier. Par exemple, pour ajouter la ligne "nameserver 8.8.8.8" dans le fichier "/etc/resolv.conf", vous pouvez utiliser le code suivant :

- name: Modifier un fichier
  hosts: localhost
  tasks:
    - name: Ajouter la ligne nameserver 8.8.8.8
      lineinfile:
        path: /etc/resolv.conf
        line: "nameserver 8.8.8.8"

Le module "iptables" permet de configurer les règles de pare-feu. Par exemple, pour autoriser les connexions SSH entrantes, vous pouvez utiliser le code suivant :

- name: Configurer le pare-feu
  hosts: localhost
  tasks:
    - name: Autoriser les connexions SSH entrantes
      iptables:
        chain: INPUT
        protocol: tcp
        destination_port: 22
        jump: ACCEPT

Le module "hostname" permet de configurer le nom d'hôte d'un hôte distant. Par exemple, pour définir le nom d'hôte "webserver" sur l'hôte distant, vous pouvez utiliser le code suivant :

- name: Configurer le nom d'hôte
  hosts: localhost
  tasks:
    - name: Définir le nom d'hôte webserver
      hostname:
        name: webserver

Gestion des packages et logiciels

La gestion des packages et des logiciels est une autre tâche courante lors de l'automatisation des tâches avec Ansible. Ansible fournit plusieurs modules pour faciliter cette gestion.

Le module "apt" permet de gérer les packages sur les systèmes basés sur Debian. Par exemple, pour installer le package "nginx", vous pouvez utiliser le code suivant :

- name: Installer un package
  hosts: localhost
  tasks:
    - name: Installer le package nginx
      apt:
        name: nginx
        state: present

Le module "yum" permet de gérer les packages sur les systèmes basés sur Red Hat. Par exemple, pour installer le package "httpd", vous pouvez utiliser le code suivant :

- name: Installer un package
  hosts: localhost
  tasks:
    - name: Installer le package httpd
      yum:
        name: httpd
        state: present

Le module "pip" permet de gérer les packages Python. Par exemple, pour installer le package "requests", vous pouvez utiliser le code suivant :

- name: Installer un package Python
  hosts: localhost
  tasks:
    - name: Installer le package requests
      pip:
        name: requests
        state: present

Gestion des utilisateurs et des autorisations

La gestion des utilisateurs et des autorisations est une autre tâche courante lors de l'automatisation des tâches avec Ansible. Ansible fournit plusieurs modules pour faciliter cette gestion.

Le module "user" permet de créer, supprimer ou modifier des utilisateurs. Par exemple, pour créer un utilisateur "john" avec le répertoire personnel "/home/john", vous pouvez utiliser le code suivant :

- name: Créer un utilisateur
  hosts: localhost
  tasks:
    - name: Créer l'utilisateur john
      user:
        name: john
        state: present
        createhome: yes
        home: /home/john

Le module "group" permet de créer, supprimer ou modifier des groupes. Par exemple, pour créer un groupe "developers", vous pouvez utiliser le code suivant :

- name: Créer un groupe
  hosts: localhost
  tasks:
    - name: Créer le groupe developers
      group:
        name: developers
        state: present

Le module "file" permet de gérer les autorisations des fichiers et des répertoires. Par exemple, pour définir les autorisations "read" et "write" pour l'utilisateur "john" sur le fichier "/path/to/file.txt", vous pouvez utiliser le code suivant :

- name: Gérer les autorisations d'un fichier
  hosts: localhost
  tasks:
    - name: Définir les autorisations pour l'utilisateur john
      file:
        path: /path/to/file.txt
        owner: john
        group: john
        mode: '0644'
