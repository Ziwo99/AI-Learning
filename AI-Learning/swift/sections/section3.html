Introduction à la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet d'organiser et de structurer le code en utilisant des objets. Un objet est une instance d'une classe, qui est un modèle ou un plan pour créer des objets. La POO repose sur quatre principes fondamentaux : l'encapsulation, l'abstraction, l'héritage et le polymorphisme.

Classes et objets

En Swift, une classe est une structure qui définit les propriétés et les comportements d'un objet. Pour créer une classe, on utilise le mot-clé class. Un objet est une instance d'une classe, créée à partir du modèle défini par la classe. Pour créer un objet, on utilise le mot-clé let suivi du nom de la classe et de parenthèses vides.

class Person {
    var name: String
    var age: Int
    
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }
}

let person = Person(name: "John", age: 25)

Héritage et polymorphisme

L'héritage permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe qui hérite d'une autre classe est appelée une sous-classe ou une classe dérivée, tandis que la classe dont elle hérite est appelée une superclasse ou une classe de base. La sous-classe hérite des propriétés et des méthodes de sa superclasse, et peut également ajouter de nouvelles propriétés et méthodes spécifiques.

Le polymorphisme permet à un objet d'être traité comme s'il était de n'importe quel type de sa hiérarchie de classes. Cela permet d'écrire du code générique qui peut être utilisé avec différents types d'objets.

class Animal {
    func makeSound() {
        // Implementation générique du son d'un animal
    }
}

class Dog: Animal {
    override func makeSound() {
        print("Woof!")
    }
}

class Cat: Animal {
    override func makeSound() {
        print("Meow!")
    }
}

let animal: Animal = Dog()
animal.makeSound() // Affiche "Woof!"

let anotherAnimal: Animal = Cat()
anotherAnimal.makeSound() // Affiche "Meow!"

Encapsulation et abstraction

L'encapsulation est le principe qui consiste à regrouper les données et les méthodes qui les manipulent au sein d'une même entité, la classe. Cela permet de cacher les détails d'implémentation et de protéger les données sensibles en les rendant privées.

L'abstraction est le principe qui consiste à représenter les caractéristiques essentielles d'un objet sans se soucier des détails d'implémentation. Cela permet de simplifier la complexité du code en se concentrant sur les aspects les plus importants.

Les propriétés

Les propriétés sont des variables ou des constantes qui sont associées à une classe, et qui définissent les caractéristiques de l'objet. Elles peuvent être stockées (stockées en mémoire) ou calculées (calculées à partir d'autres propriétés ou méthodes).

class Rectangle {
    var width: Double
    var height: Double
    
    var area: Double {
        return width * height
    }
    
    init(width: Double, height: Double) {
        self.width = width
        self.height = height
    }
}

let rectangle = Rectangle(width: 10, height: 5)
print(rectangle.area) // Affiche "50.0"

Les méthodes

Les méthodes sont des fonctions qui sont associées à une classe, et qui définissent les actions que l'objet peut effectuer. Elles peuvent être des méthodes d'instance, qui agissent sur une instance spécifique de la classe, ou des méthodes de classe, qui agissent sur la classe elle-même.

class Counter {
    var count = 0
    
    func increment() {
        count += 1
    }
    
    static func reset() {
        count = 0
    }
}

let counter = Counter()
counter.increment()
print(counter.count) // Affiche "1"

Counter.reset()
print(Counter.count) // Affiche "0"

Les extensions

Les extensions permettent d'ajouter de nouvelles fonctionnalités à une classe existante, sans avoir à modifier son code source. Cela permet d'étendre les fonctionnalités d'une classe sans avoir à créer une sous-classe.

extension String {
    func reverse() -> String {
        return String(self.reversed())
    }
}

let message = "Hello, World!"
print(message.reverse()) // Affiche "!dlroW ,olleH"

Les protocoles

Les protocoles définissent un ensemble de méthodes et de propriétés que les types qui les adoptent doivent implémenter. Cela permet de définir des interfaces communes pour les types différents, et de les traiter de manière générique.

protocol Drawable {
    func draw()
}

class Circle: Drawable {
    func draw() {
        // Implémentation spécifique pour dessiner un cercle
    }
}

class Square: Drawable {
    func draw() {
        // Implémentation spécifique pour dessiner un carré
    }
}

func drawShapes(_ shapes: [Drawable]) {
    for shape in shapes {
        shape.draw()
    }
}

let circle = Circle()
let square = Square()

drawShapes([circle, square])
