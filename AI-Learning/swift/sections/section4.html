Gestion des erreurs avec les do-catch

La gestion des erreurs est une partie essentielle de la programmation en Swift. Elle permet de gérer les situations où une erreur peut se produire pendant l'exécution d'un programme. Swift utilise le mécanisme do-catch pour gérer les erreurs.

La structure do-catch permet d'exécuter un bloc de code et de capturer les erreurs qui peuvent se produire pendant son exécution. Voici un exemple de code utilisant la structure do-catch : 

do {
    // Code pouvant générer une erreur
} catch {
    // Gestion de l'erreur
}

Dans cet exemple, le code à l'intérieur du bloc do est exécuté. Si une erreur se produit, elle est capturée et le code à l'intérieur du bloc catch est exécuté pour gérer cette erreur.

Il est également possible de spécifier le type d'erreur que l'on souhaite capturer. Par exemple : 

do {
    // Code pouvant générer une erreur
} catch ErrorType1 {
    // Gestion de l'erreur de type ErrorType1
} catch ErrorType2 {
    // Gestion de l'erreur de type ErrorType2
}

Dans cet exemple, si une erreur de type ErrorType1 se produit, le premier bloc catch est exécuté. Si une erreur de type ErrorType2 se produit, le deuxième bloc catch est exécuté.

Utilisation des exceptions

En plus de la gestion des erreurs avec do-catch, Swift prend également en charge l'utilisation des exceptions pour gérer les erreurs. Les exceptions sont des objets qui représentent une erreur et qui peuvent être lancés (throw) et attrapés (catch).

Pour lancer une exception, on utilise le mot-clé throw suivi de l'objet exception. Voici un exemple de code utilisant les exceptions : 

func divide(_ a: Int, by b: Int) throws -> Int {
    if b == 0 {
        throw DivisionError.divideByZero
    }
    return a / b
}

Dans cet exemple, la fonction divide lance une exception de type DivisionError si le diviseur est égal à zéro.

Pour attraper une exception, on utilise la structure do-catch, comme dans l'exemple précédent. Voici un exemple de code attrapant une exception : 

do {
    let result = try divide(10, by: 0)
    print(result)
} catch DivisionError.divideByZero {
    print("Erreur : division par zéro")
}

Dans cet exemple, si une exception de type DivisionError est lancée, le bloc catch correspondant est exécuté.

Il est également possible de spécifier plusieurs blocs catch pour attraper différentes exceptions. Par exemple : 

do {
    let result = try divide(10, by: 0)
    print(result)
} catch DivisionError.divideByZero {
    print("Erreur : division par zéro")
} catch DivisionError.invalidOperation {
    print("Erreur : opération invalide")
}

Dans cet exemple, si une exception de type DivisionError.divideByZero est lancée, le premier bloc catch est exécuté. Si une exception de type DivisionError.invalidOperation est lancée, le deuxième bloc catch est exécuté.

Il est également possible de capturer toutes les exceptions non spécifiées en utilisant le bloc catch sans spécifier le type d'exception. Par exemple : 

do {
    let result = try divide(10, by: 0)
    print(result)
} catch {
    print("Erreur inattendue")
}

Dans cet exemple, si une exception non spécifiée est lancée, le bloc catch est exécuté.
