Les closures

Les closures sont des blocs de code auto-contenus qui peuvent être assignés à des variables ou passés en tant que paramètres à d'autres fonctions. Elles sont similaires aux lambdas ou aux fonctions anonymes dans d'autres langages de programmation.

Une closure peut capturer et stocker des références à des variables et à des constantes provenant du contexte dans lequel elle est définie. Elle peut ensuite être appelée ultérieurement dans un autre contexte.

Voici un exemple de closure en Swift :

let greet = {
    print("Hello, World!")
}

greet() // Affiche "Hello, World!"

Les opérations asynchrones

Les opérations asynchrones permettent d'exécuter des tâches en arrière-plan sans bloquer l'interface utilisateur. Cela est particulièrement utile lorsqu'il s'agit d'effectuer des opérations longues ou de télécharger des données depuis un serveur distant.

En Swift, les opérations asynchrones sont généralement gérées à l'aide de la classe DispatchQueue et de ses méthodes async et sync. Voici un exemple d'utilisation :

DispatchQueue.global().async {
    // Code exécuté de manière asynchrone
    print("Background task")
    
    DispatchQueue.main.async {
        // Code exécuté sur le thread principal
        print("UI update")
    }
}

Les promesses et les futures

Les promesses et les futures sont des concepts utilisés pour gérer les opérations asynchrones de manière plus fluide. Ils permettent de définir des actions à exécuter une fois qu'une opération asynchrone est terminée.

En Swift, il existe plusieurs bibliothèques tierces qui fournissent des fonctionnalités de promesses et de futures, telles que PromiseKit ou RxSwift. Voici un exemple d'utilisation avec PromiseKit :

func fetchData() -> Promise<Data> {
    return Promise { seal in
        URLSession.shared.dataTask(with: url) { data, _, error in
            if let error = error {
                seal.reject(error)
            } else if let data = data {
                seal.fulfill(data)
            }
        }.resume()
    }
}

fetchData().done { data in
    // Traitement des données
}.catch { error in
    // Gestion des erreurs
}

Ce code utilise PromiseKit pour effectuer une requête asynchrone à l'aide de URLSession et renvoie une promesse contenant les données téléchargées. Les blocs done et catch sont utilisés pour traiter les données ou gérer les erreurs une fois que la promesse est remplie ou rejetée.
