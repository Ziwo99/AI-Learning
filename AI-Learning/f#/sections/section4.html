Section 4: Programmation orientée objet en F#

Introduction à la programmation orientée objet en F#

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets qui interagissent les uns avec les autres. F# prend en charge la programmation orientée objet en fournissant des fonctionnalités telles que les classes, les objets, l'héritage et le polymorphisme.

Définition de classes et d'objets

En F#, une classe est une structure de données qui définit les propriétés et les comportements d'un objet. Pour définir une classe, utilisez le mot-clé type suivi du nom de la classe et de ses membres.

type Person =
    member val Name : string = ""
    member val Age : int = 0

    member this.SayHello() =
        printfn "Hello, my name is %s and I am %d years old." this.Name this.Age

Dans cet exemple, nous avons défini une classe Person avec deux propriétés, Name et Age, ainsi qu'une méthode SayHello qui affiche un message de salutation.

Pour créer un objet à partir d'une classe, utilisez le mot-clé new suivi du nom de la classe et des arguments nécessaires pour initialiser les propriétés.

let person = new Person(Name = "John", Age = 30)
person.SayHello()

Cela créera un nouvel objet de type Person avec les valeurs spécifiées pour les propriétés Name et Age, puis appellera la méthode SayHello de cet objet.

Héritage et polymorphisme

En F#, vous pouvez définir des classes dérivées à partir d'une classe de base existante en utilisant le mot-clé inherit. Cela permet à la classe dérivée d'hériter des propriétés et des comportements de la classe de base.

type Employee =
    inherit Person

    member val Salary : decimal = 0.0

    member this.GetSalary() =
        printfn "My salary is %f." this.Salary

Dans cet exemple, nous avons défini une classe Employee qui hérite de la classe Person. La classe Employee a une propriété supplémentaire Salary et une méthode GetSalary qui affiche le salaire de l'employé.

Vous pouvez également utiliser le polymorphisme en F# en définissant des méthodes virtuelles dans une classe de base et en les remplaçant dans les classes dérivées.

type Animal =
    abstract member MakeSound : unit -> unit

type Dog() =
    inherit Animal()

    override this.MakeSound() =
        printfn "Woof!"

type Cat() =
    inherit Animal()

    override this.MakeSound() =
        printfn "Meow!"

Dans cet exemple, nous avons défini une classe abstraite Animal avec une méthode virtuelle MakeSound. Les classes dérivées Dog et Cat remplacent cette méthode pour produire des sons spécifiques.

Interfaces et classes abstraites

En F#, vous pouvez définir des interfaces et des classes abstraites pour définir des contrats que les classes doivent respecter.

type IShape =
    abstract member GetArea : unit -> float

type Circle(radius : float) =
    interface IShape with
        member this.GetArea() =
            Math.PI * radius * radius

type Rectangle(width : float, height : float) =
    interface IShape with
        member this.GetArea() =
            width * height

Dans cet exemple, nous avons défini une interface IShape avec une méthode GetArea. Les classes Circle et Rectangle implémentent cette interface en fournissant une implémentation de la méthode GetArea.

Gestion des exceptions en F#

En F#, vous pouvez gérer les exceptions en utilisant les blocs try/with pour capturer et traiter les exceptions.

try
    // Code pouvant générer une exception
    let result = 10 / 0
with
    | :? System.DivideByZeroException as ex ->
        printfn "Une exception DivideByZeroException s'est produite : %s" ex.Message
    | ex ->
        printfn "Une exception s'est produite : %s" ex.Message

Dans cet exemple, nous essayons de diviser 10 par zéro, ce qui génère une exception DivideByZeroException. Nous capturons cette exception en utilisant le motif :? System.DivideByZeroException et affichons un message d'erreur approprié.

Il est également possible de lever des exceptions en utilisant le mot-clé raise.

let divide x y =
    if y = 0 then
        raise (System.DivideByZeroException())
    else
        x / y

Dans cet exemple, nous définissons une fonction divide qui lève une exception DivideByZeroException si le diviseur est zéro.
