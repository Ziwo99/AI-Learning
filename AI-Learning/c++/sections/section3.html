Les classes et les objets

Les classes et les objets sont les éléments fondamentaux de la programmation orientée objet en C++. Une classe est une structure qui définit les caractéristiques et les comportements d'un objet. Un objet est une instance d'une classe, créée à partir de la définition de la classe.

Une classe est définie à l'aide du mot-clé class, suivi du nom de la classe et d'un bloc de code entre accolades. Les membres d'une classe, tels que les variables et les fonctions, sont déclarés à l'intérieur de ce bloc de code.

class Person {
    // Variables membres
    string name;
    int age;
    
    // Fonctions membres
    void setName(string n) {
        name = n;
    }
    
    void setAge(int a) {
        age = a;
    }
    
    void displayInfo() {
        cout << "Name: " << name << endl;
        cout << "Age: " << age << endl;
    }
};

Pour créer un objet à partir d'une classe, on utilise le nom de la classe suivi d'un nom d'objet et de parenthèses. On peut ensuite accéder aux membres de l'objet à l'aide de l'opérateur point (.).

Person person1;
person1.setName("John");
person1.setAge(25);
person1.displayInfo();

L'encapsulation et l'abstraction

L'encapsulation est un concept clé de la programmation orientée objet qui consiste à regrouper les données et les fonctions qui les manipulent dans une classe. L'encapsulation permet de cacher les détails internes de la classe et de fournir une interface publique pour interagir avec les objets.

L'abstraction est un autre concept important qui consiste à représenter les objets du monde réel sous forme de classes et d'ignorer les détails non pertinents. L'abstraction permet de simplifier la complexité en se concentrant sur les aspects essentiels.

En C++, l'encapsulation est réalisée en déclarant les membres d'une classe comme privés (private) par défaut. Les membres privés ne peuvent être accédés que par les fonctions membres de la classe. Les fonctions membres qui fournissent un accès aux membres privés sont généralement déclarées comme publiques (public).

class BankAccount {
private:
    string accountNumber;
    double balance;
    
public:
    void deposit(double amount) {
        balance += amount;
    }
    
    void withdraw(double amount) {
        if (amount <= balance) {
            balance -= amount;
        } else {
            cout << "Insufficient balance." << endl;
        }
    }
    
    double getBalance() {
        return balance;
    }
};

En utilisant cette classe, on peut créer des objets de compte bancaire et effectuer des opérations telles que le dépôt et le retrait d'argent.

BankAccount account1;
account1.deposit(1000);
account1.withdraw(500);
cout << "Balance: " << account1.getBalance() << endl;

L'héritage et le polymorphisme

L'héritage est un mécanisme qui permet à une classe d'hériter des caractéristiques et des comportements d'une autre classe. La classe qui hérite est appelée classe dérivée, et la classe dont elle hérite est appelée classe de base.

Le polymorphisme est un concept qui permet à un objet d'être traité comme s'il était de différents types. Cela permet de créer des hiérarchies de classes et d'utiliser des objets dérivés de manière générique.

En C++, l'héritage est réalisé en utilisant le mot-clé class suivi du nom de la classe dérivée, suivi du mot-clé : public et du nom de la classe de base.

class Shape {
public:
    virtual double getArea() = 0;
};

class Rectangle : public Shape {
private:
    double width;
    double height;
    
public:
    Rectangle(double w, double h) : width(w), height(h) {}
    
    double getArea() {
        return width * height;
    }
};

class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(double r) : radius(r) {}
    
    double getArea() {
        return 3.14 * radius * radius;
    }
};

En utilisant ces classes, on peut créer des objets de différentes formes et calculer leurs aires.

Shape* shape1 = new Rectangle(5, 3);
Shape* shape2 = new Circle(2.5);

cout << "Area of shape1: " << shape1->getArea() << endl;
cout << "Area of shape2: " << shape2->getArea() << endl;

delete shape1;
delete shape2;

Les templates et la programmation générique

Les templates sont un mécanisme puissant en C++ qui permet de créer des fonctions et des classes génériques, capables de travailler avec différents types de données. La programmation générique permet de réutiliser le code et d'augmenter la flexibilité.

En C++, les templates sont définis à l'aide du mot-clé template, suivi d'une liste de paramètres de type entre chevrons (<>). Ces paramètres de type peuvent être utilisés dans la définition de la fonction ou de la classe.

template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

int maxInt = maximum(10, 5);
double maxDouble = maximum(3.14, 2.71);

En utilisant cette fonction générique, on peut trouver le maximum de deux entiers ou de deux nombres à virgule flottante.

La gestion de la mémoire et les pointeurs intelligents

La gestion de la mémoire est un aspect important de la programmation en C++, car il est de la responsabilité du programmeur d'allouer et de libérer la mémoire utilisée par les objets.

Les pointeurs intelligents sont des classes spéciales en C++ qui aident à gérer automatiquement la mémoire. Ils s'occupent de l'allocation et de la libération de la mémoire, ce qui évite les fuites de mémoire et les erreurs de désallocation.

En C++, les pointeurs intelligents les plus couramment utilisés sont unique_ptr, shared_ptr et weak_ptr.

unique_ptr<int> ptr1(new int(5));
shared_ptr<int> ptr2 = make_shared<int>(10);
weak_ptr<int> ptr3 = ptr2;

cout << *ptr1 << endl;
cout << *ptr2 << endl;
cout << *ptr3.lock() << endl;

En utilisant ces pointeurs intelligents, on peut allouer et libérer automatiquement la mémoire sans se soucier des détails de gestion de la mémoire.
