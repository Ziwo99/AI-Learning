Les classes et les objets

Les classes et les objets sont des concepts fondamentaux en programmation orientée objet (POO) en C++. Une classe est un modèle ou un plan pour créer des objets, tandis qu'un objet est une instance spécifique d'une classe.

Une classe est définie en utilisant le mot-clé class, suivi du nom de la classe et d'un bloc de code entre accolades. Les membres d'une classe, tels que les attributs et les méthodes, sont définis à l'intérieur de ce bloc de code.

class Person {
    // Attributs
    string name;
    int age;
    
    // Méthodes
    void setName(string n) {
        name = n;
    }
    
    void setAge(int a) {
        age = a;
    }
    
    void printInfo() {
        cout << "Name: " << name << endl;
        cout << "Age: " << age << endl;
    }
};

Pour créer un objet à partir d'une classe, on utilise le nom de la classe suivi d'un nom d'objet et de parenthèses. Les parenthèses peuvent être utilisées pour appeler un constructeur, qui est une méthode spéciale utilisée pour initialiser les attributs de l'objet.

Person person1; // Crée un objet de la classe Person
person1.setName("John");
person1.setAge(25);
person1.printInfo(); // Affiche "Name: John" et "Age: 25"

L'encapsulation et l'abstraction

L'encapsulation est un concept qui consiste à regrouper des données et des méthodes connexes dans une classe, en limitant l'accès direct aux données depuis l'extérieur de la classe. Cela permet de protéger les données et de garantir leur intégrité.

L'abstraction est un concept qui consiste à cacher les détails d'implémentation d'une classe et à fournir une interface simplifiée pour interagir avec celle-ci. Cela permet aux utilisateurs de la classe de se concentrer sur ce qu'elle fait plutôt que sur comment elle le fait.

En C++, l'encapsulation est réalisée en déclarant les attributs d'une classe comme privés (private) et en fournissant des méthodes publiques (public) pour accéder et modifier ces attributs.

class BankAccount {
private:
    string accountNumber;
    double balance;
    
public:
    void deposit(double amount) {
        balance += amount;
    }
    
    void withdraw(double amount) {
        if (amount <= balance) {
            balance -= amount;
        } else {
            cout << "Insufficient funds." << endl;
        }
    }
    
    double getBalance() {
        return balance;
    }
};

Les utilisateurs de la classe BankAccount peuvent uniquement interagir avec elle en utilisant les méthodes publiques, telles que deposit, withdraw et getBalance. Les détails de l'implémentation, tels que les attributs accountNumber et balance, sont cachés.

L'héritage et le polymorphisme

L'héritage est un concept qui permet de créer de nouvelles classes (appelées classes dérivées) à partir d'une classe existante (appelée classe de base). Les classes dérivées héritent des attributs et des méthodes de la classe de base, ce qui permet de réutiliser du code et de créer une hiérarchie de classes.

Le polymorphisme est un concept qui permet à un objet d'une classe dérivée d'être traité comme un objet de la classe de base. Cela permet d'écrire du code générique qui peut être utilisé avec des objets de différentes classes dérivées, ce qui facilite l'extension et la maintenance du code.

En C++, l'héritage est réalisé en utilisant le signe de deux points (::) pour spécifier la classe de base lors de la définition d'une classe dérivée.

class Shape {
protected:
    double width;
    double height;
    
public:
    void setWidth(double w) {
        width = w;
    }
    
    void setHeight(double h) {
        height = h;
    }
    
    virtual double getArea() {
        return 0.0;
    }
};

class Rectangle : public Shape {
public:
    double getArea() override {
        return width * height;
    }
};

class Triangle : public Shape {
public:
    double getArea() override {
        return 0.5 * width * height;
    }
};

La classe Shape est la classe de base, tandis que les classes Rectangle et Triangle sont des classes dérivées. Les classes dérivées héritent des attributs et des méthodes de la classe de base, et peuvent également redéfinir (override) les méthodes virtuelles pour fournir une implémentation spécifique.

Le polymorphisme peut être utilisé pour traiter des objets de différentes classes dérivées comme des objets de la classe de base. Par exemple :

Shape* shape1 = new Rectangle();
Shape* shape2 = new Triangle();

shape1->setWidth(5.0);
shape1->setHeight(3.0);
cout << "Area of shape1: " << shape1->getArea() << endl;

shape2->setWidth(4.0);
shape2->setHeight(6.0);
cout << "Area of shape2: " << shape2->getArea() << endl;

Cela permet d'écrire du code générique qui peut être utilisé avec des objets de différentes classes dérivées, sans avoir à connaître le type spécifique de l'objet.

Les templates et la programmation générique

Les templates sont un mécanisme en C++ qui permet de créer des fonctions et des classes génériques, c'est-à-dire des fonctions et des classes qui peuvent être utilisées avec différents types de données. Cela permet de réutiliser du code et d'écrire des algorithmes génériques.

En utilisant des templates, on peut définir des fonctions et des classes qui prennent des paramètres de type générique, représentés par des paramètres de template. Les paramètres de template sont déclarés entre chevrons (<>) et peuvent être utilisés à l'intérieur de la fonction ou de la classe pour définir des types de données.

template <typename T>
T maximum(T a, T b) {
    return (a > b) ? a : b;
}

int maxInt = maximum(5, 10);
double maxDouble = maximum(3.14, 2.71);
string maxString = maximum("hello", "world");

Dans cet exemple, la fonction maximum est définie comme un template avec un paramètre de type générique T. La fonction peut être appelée avec différents types de données, tels que des entiers, des nombres à virgule flottante ou des chaînes de caractères.

Les exceptions et la gestion des erreurs

Les exceptions sont un mécanisme en C++ qui permet de gérer les erreurs et les situations exceptionnelles de manière structurée. Une exception est une valeur ou un objet qui est lancé (thrown) lorsqu'une erreur se produit, et qui peut être attrapé (caught) et géré par un bloc de code spécifique.

En utilisant des exceptions, on peut séparer la logique normale du programme de la gestion des erreurs, ce qui rend le code plus lisible et plus maintenable. Les exceptions peuvent être lancées à l'aide du mot-clé throw et attrapées à l'aide du mot-clé try suivi d'un bloc de code catch.

try {
    int result = divide(10, 0);
    cout << "Result: " << result << endl;
} catch (const char* error) {
    cout << "Error: " << error << endl;
}

int divide(int a, int b) {
    if (b == 0) {
        throw "Division by zero.";
    }
    return a / b;
}

Dans cet exemple, la fonction divide lance une exception avec le message "Division by zero" si le diviseur est égal à zéro. L'exception est attrapée par le bloc de code catch, qui affiche le message d'erreur correspondant.

Il est également possible de définir ses propres types d'exceptions en créant des classes dérivées de la classe de base std::exception.
