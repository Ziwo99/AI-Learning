Les concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de modéliser des concepts du monde réel en utilisant des objets. En C++, la POO repose sur trois principaux concepts : l'encapsulation, l'héritage et le polymorphisme.

L'encapsulation consiste à regrouper des données et des fonctions associées dans une même entité appelée classe. Une classe est un modèle à partir duquel on peut créer des objets. Les données sont représentées par des variables membres et les fonctions par des méthodes.

L'héritage permet de créer de nouvelles classes à partir de classes existantes. Une classe dérivée hérite des caractéristiques (variables membres et méthodes) de la classe de base. Cela permet de réutiliser du code et de créer des hiérarchies de classes.

Le polymorphisme permet à un objet d'être traité comme un objet d'un type différent. Cela permet de manipuler des objets de différentes classes de manière uniforme. Le polymorphisme peut être statique (résolu à la compilation) ou dynamique (résolu à l'exécution).

Les classes et les objets

En C++, une classe est définie à l'aide du mot-clé class. Elle peut contenir des variables membres (attributs) et des méthodes (fonctions membres). Un objet est une instance d'une classe, créée à l'aide du mot-clé new.

Voici un exemple de définition d'une classe Personne avec des variables membres nom et age :

class Personne {
    public:
        string nom;
        int age;
};

Pour créer un objet de la classe Personne :

Personne personne1;

Les variables membres d'un objet peuvent être accédées à l'aide de l'opérateur .. Par exemple, pour accéder à la variable nom de l'objet personne1 :

personne1.nom = "Jean";
cout << personne1.nom << endl;

L'héritage et le polymorphisme

En C++, l'héritage est réalisé à l'aide du symbole :. Une classe dérivée hérite des membres publics et protégés de la classe de base. Voici un exemple de classe dérivée Etudiant qui hérite de la classe de base Personne :

class Etudiant : public Personne {
    public:
        string matricule;
};

L'objet de la classe dérivée Etudiant possède à la fois les variables membres de la classe de base Personne et de la classe dérivée Etudiant. Par exemple :

Etudiant etudiant1;
etudiant1.nom = "Alice";
etudiant1.age = 20;
etudiant1.matricule = "123456";

Le polymorphisme en C++ peut être réalisé à l'aide de fonctions virtuelles et de classes abstraites. Une fonction virtuelle est une fonction déclarée dans une classe de base et redéfinie dans une classe dérivée. Une classe abstraite est une classe qui contient au moins une fonction virtuelle pure, déclarée à l'aide du mot-clé virtual et suivi de = 0.

Voici un exemple de classe abstraite Forme avec une fonction virtuelle pure aire() :

class Forme {
    public:
        virtual double aire() const = 0;
};

Une classe dérivée de la classe abstraite Forme doit redéfinir la fonction virtuelle pure aire(). Par exemple :

class Rectangle : public Forme {
    private:
        double largeur;
        double hauteur;
    
    public:
        double aire() const {
            return largeur * hauteur;
        }
};

Le polymorphisme permet d'utiliser un pointeur ou une référence de la classe de base pour manipuler des objets de différentes classes dérivées. Par exemple :

Forme* forme1 = new Rectangle();
double aire = forme1->aire();

Ici, le pointeur forme1 est de type Forme*, mais il pointe vers un objet de type Rectangle. L'appel à la fonction aire() se fait de manière polymorphique, c'est-à-dire que la version de la fonction dans la classe dérivée Rectangle est appelée.
