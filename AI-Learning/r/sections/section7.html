Shiny : création d'applications interactives

Shiny est un package R qui permet de créer des applications web interactives directement à partir de R. Avec Shiny, vous pouvez facilement transformer votre code R en une application web interactive sans avoir à apprendre des langages de programmation web tels que HTML, CSS ou JavaScript.

Pour commencer à utiliser Shiny, vous devez d'abord installer le package en utilisant la fonction install.packages("shiny"). Une fois le package installé, vous pouvez le charger dans votre environnement R en utilisant la fonction library(shiny).

Une application Shiny est composée de deux parties principales : l'interface utilisateur (UI) et le serveur. L'UI définit la mise en page et les éléments interactifs de l'application, tandis que le serveur contient le code R qui contrôle le comportement de l'application.

Voici un exemple simple d'une application Shiny qui affiche un histogramme interactif : 

library(shiny)

# Définition de l'interface utilisateur (UI)
ui <- fluidPage(
  titlePanel("Mon application Shiny"),
  sidebarLayout(
    sidebarPanel(
      sliderInput("bins",
                  "Nombre de bins:",
                  min = 1,
                  max = 50,
                  value = 30)
    ),
    mainPanel(
      plotOutput("histogram")
    )
  )
)

# Définition du serveur
server <- function(input, output) {
  output$histogram <- renderPlot({
    hist(rnorm(1000, mean = 0, sd = 1), 
         breaks = input$bins, 
         col = 'steelblue', 
         border = 'white')
  })
}

# Lancement de l'application
shinyApp(ui = ui, server = server)

Dans cet exemple, l'interface utilisateur (UI) est définie à l'aide de la fonction fluidPage(), qui crée une mise en page réactive qui s'adapte automatiquement à la taille de l'écran. Nous avons ajouté un titre à l'application avec la fonction titlePanel() et un panneau latéral avec un curseur pour ajuster le nombre de bins dans l'histogramme avec la fonction sliderInput(). Le panneau principal affiche l'histogramme avec la fonction plotOutput().

Dans le serveur, nous avons utilisé la fonction renderPlot() pour générer l'histogramme en utilisant la fonction hist() de base de R. Nous avons également utilisé la variable input$bins pour récupérer la valeur du curseur dans le panneau latéral.

Pour lancer l'application Shiny, vous pouvez utiliser la fonction shinyApp() en spécifiant l'interface utilisateur (UI) et le serveur. L'application sera alors exécutée dans votre navigateur web par défaut.

R Markdown : création de rapports dynamiques

R Markdown est une extension du langage Markdown qui permet d'intégrer du code R et de générer des rapports dynamiques à partir de ce code. Avec R Markdown, vous pouvez facilement combiner du texte, des graphiques et du code R dans un seul document, ce qui en fait un outil puissant pour la création de rapports reproductibles et personnalisables.

Pour commencer à utiliser R Markdown, vous devez d'abord installer le package en utilisant la fonction install.packages("rmarkdown"). Une fois le package installé, vous pouvez le charger dans votre environnement R en utilisant la fonction library(rmarkdown).

Un document R Markdown est composé de plusieurs sections, y compris une section d'en-tête YAML, une section de texte et des sections de code R. Vous pouvez utiliser des balises Markdown pour formater le texte et insérer des images, des liens, des tableaux, etc.

Voici un exemple simple d'un document R Markdown qui génère un rapport avec un graphique : 

---
title: "Mon rapport R Markdown"
output: html_document
---

# Introduction

Ce rapport présente une analyse exploratoire des données.

## Chargement des données

```{r}
data <- read.csv("data.csv")
```

## Visualisation des données

```{r}
plot(data$variable)
```

## Conclusion

Ce rapport montre l'évolution de la variable au fil du temps.

Dans cet exemple, la section d'en-tête YAML spécifie le titre du rapport et le format de sortie (dans ce cas, un document HTML). La section de texte utilise des balises Markdown pour créer des titres et des sous-titres, ainsi que pour insérer du texte descriptif.

Les sections de code R sont délimitées par des balises ```{r} et ```. Le code R à l'intérieur de ces balises sera exécuté et les résultats seront affichés dans le rapport final. Dans cet exemple, nous avons chargé les données à partir d'un fichier CSV avec la fonction read.csv() et nous avons tracé un graphique avec la fonction plot().

Pour générer le rapport R Markdown, vous pouvez utiliser la fonction rmarkdown::render() en spécifiant le nom du fichier R Markdown en entrée et le format de sortie souhaité. Le rapport sera alors généré dans le répertoire de travail actuel.

Package development : création de packages R

La création de packages R est un moyen efficace d'organiser et de distribuer du code R réutilisable. Un package R est une collection de fonctions, de données et de documentation qui peut être facilement installée et chargée dans un environnement R.

Pour créer un package R, vous devez d'abord installer le package devtools en utilisant la fonction install.packages("devtools"). Une fois le package installé, vous pouvez le charger dans votre environnement R en utilisant la fonction library(devtools).

La création d'un package R implique plusieurs étapes, notamment la création de la structure du package, la définition des fonctions et des données, la documentation des fonctions et la génération d'un fichier de description du package.

Voici un exemple simple de création d'un package R avec une fonction qui calcule la moyenne d'un vecteur : 

# Création de la structure du package
create_package("monpackage")

# Définition de la fonction
monpackage::mean_vector <- function(x) {
  mean(x)
}

# Documentation de la fonction
?monpackage::mean_vector

# Génération du fichier de description du package
devtools::document("monpackage")

Dans cet exemple, nous avons utilisé la fonction create_package() du package devtools pour créer la structure de base du package. Cela crée un répertoire avec les fichiers nécessaires pour le package, y compris un fichier de description du package.

Ensuite, nous avons défini une fonction mean_vector() qui calcule la moyenne d'un vecteur en utilisant la fonction mean() de base de R. Nous avons également documenté la fonction en utilisant la syntaxe ?monpackage::mean_vector.

Enfin, nous avons utilisé la fonction devtools::document() pour générer le fichier de description du package à partir des fichiers de documentation et des fonctions définies.

Une fois que vous avez créé votre package R, vous pouvez l'installer en utilisant la fonction devtools::install(). Une fois installé, vous pouvez le charger dans votre environnement R en utilisant la fonction library(monpackage).
