Création et utilisation de composants

Les composants sont les éléments de base d'une application Angular. Ils permettent de regrouper du code, des styles et des templates pour créer des fonctionnalités réutilisables.

Pour créer un nouveau composant, vous pouvez utiliser la commande suivante :

ng generate component nom-du-composant

Cela générera automatiquement les fichiers nécessaires pour votre composant, y compris le fichier TypeScript, le fichier HTML et le fichier CSS.

Une fois que votre composant est créé, vous pouvez l'utiliser dans d'autres parties de votre application en l'important et en l'ajoutant à la liste des déclarations dans le module correspondant.

Par exemple, si vous avez un composant nommé "MonComposant", vous pouvez l'importer dans un autre fichier TypeScript de la manière suivante :

import { MonComposant } from './mon-composant.component';

Ensuite, vous pouvez l'ajouter à la liste des déclarations dans le module :

@NgModule({
  declarations: [
    MonComposant
  ],
  ...
})
export class MonModule { }

Communication entre composants

La communication entre composants est essentielle dans une application Angular. Il existe plusieurs façons de réaliser cette communication, notamment en utilisant les propriétés d'entrée et de sortie, ainsi que les services.

Les propriétés d'entrée permettent de passer des données d'un composant parent à un composant enfant. Pour définir une propriété d'entrée dans un composant enfant, vous pouvez utiliser le décorateur @Input :

@Input() maPropriete: string;

Dans le composant parent, vous pouvez ensuite lier cette propriété à une valeur :

<app-mon-composant [maPropriete]="valeur"></app-mon-composant>

Les propriétés de sortie permettent à un composant enfant d'émettre des événements vers son composant parent. Pour définir une propriété de sortie dans un composant enfant, vous pouvez utiliser le décorateur @Output :

@Output() monEvenement: EventEmitter<string> = new EventEmitter<string>();

Dans le composant enfant, vous pouvez ensuite émettre un événement :

this.monEvenement.emit('valeur');

Dans le composant parent, vous pouvez écouter cet événement :

<app-mon-composant (monEvenement)="maFonction($event)"></app-mon-composant>

Utilisation de templates et directives structurelles

Les templates et les directives structurelles permettent de manipuler la structure du DOM en fonction des données et de la logique de votre application.

Un template est une représentation HTML d'un composant. Il peut contenir des balises HTML, des directives et des expressions Angular.

Les directives structurelles permettent de conditionner l'affichage de certaines parties du template en fonction de conditions spécifiques. Par exemple, la directive *ngIf permet d'afficher ou de masquer un élément en fonction d'une expression booléenne :

<div *ngIf="condition">Contenu conditionnel</div>

La directive *ngFor permet de répéter un élément pour chaque élément d'un tableau :

<ul>
  <li *ngFor="let item of items">{{ item }}</li>
</ul>

Il existe de nombreuses autres directives structurelles, telles que *ngSwitch, *ngFor et *ngClass, qui permettent de manipuler la structure du DOM de manière dynamique.

Vous avez appris à créer et à utiliser des composants, à communiquer entre eux et à utiliser des templates et des directives structurelles pour manipuler la structure du DOM.
