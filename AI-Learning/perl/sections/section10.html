Gestion des erreurs avec die et warn

En Perl, la gestion des erreurs peut être réalisée à l'aide des fonctions die et warn. La fonction die est utilisée pour générer une erreur fatale qui arrête immédiatement l'exécution du programme, tandis que la fonction warn génère un avertissement qui ne stoppe pas l'exécution du programme.

Voici un exemple d'utilisation de la fonction die :

my $num = 10;

if ($num > 5) {
    die "Le nombre ne peut pas être supérieur à 5.";
}

print "Ce message ne sera pas affiché car le programme s'est arrêté.";

Dans cet exemple, si la condition $num > 5 est vraie, le programme s'arrête immédiatement et affiche le message d'erreur spécifié dans la fonction die.

Voici un exemple d'utilisation de la fonction warn :

my $num = 10;

if ($num > 5) {
    warn "Le nombre est supérieur à 5.";
}

print "Ce message sera affiché malgré l'avertissement.";

Dans cet exemple, si la condition $num > 5 est vraie, un avertissement est généré mais le programme continue son exécution normalement.

Utilisation des blocs eval

En Perl, les blocs eval sont utilisés pour capturer et gérer les exceptions. Un bloc eval permet d'exécuter du code Perl tout en capturant les erreurs qui pourraient survenir.

Voici un exemple d'utilisation d'un bloc eval :

eval {
    # Code susceptible de générer une exception
    my $result = 10 / 0;
};

if ($@) {
    # Gestion de l'exception
    print "Une erreur s'est produite : $@";
}

Dans cet exemple, le code à l'intérieur du bloc eval tente de diviser le nombre 10 par zéro, ce qui génère une exception. L'exception est capturée par la variable spéciale $@ et peut être utilisée pour afficher un message d'erreur personnalisé.

Création de classes d'exceptions personnalisées

En Perl, il est possible de créer des classes d'exceptions personnalisées en utilisant le mécanisme d'héritage. Cela permet de définir des exceptions spécifiques à notre application et de les gérer de manière appropriée.

Voici un exemple de création d'une classe d'exception personnalisée :

package MonException;

use strict;
use warnings;

use base 'Error::Simple';

sub new {
    my ($class, $message) = @_;
    my $self = $class->SUPER::new($message);
    return $self;
}

1;

Dans cet exemple, nous créons une classe MonException qui hérite de la classe Error::Simple. La méthode new est redéfinie pour permettre la création d'une instance de notre classe d'exception personnalisée.

Voici comment utiliser notre classe d'exception personnalisée :

use MonException;

eval {
    # Code susceptible de générer une exception
    if (...) {
        die MonException->new("Une erreur s'est produite.");
    }
};

if ($@) {
    # Gestion de l'exception
    if (ref $@ eq 'MonException') {
        print "Une exception de type MonException a été capturée.";
    } else {
        print "Une autre exception a été capturée.";
    }
}

Dans cet exemple, nous utilisons la fonction die pour générer une instance de notre classe d'exception personnalisée MonException. L'exception est ensuite capturée et nous pouvons vérifier son type à l'aide de la fonction ref.
