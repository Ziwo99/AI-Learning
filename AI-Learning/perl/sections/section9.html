Concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Un objet est une instance d'une classe, qui est une structure de données contenant des attributs (variables) et des méthodes (fonctions). La POO offre de nombreux avantages, tels que la modularité, la réutilisabilité du code et la facilité de maintenance.

En Perl, la POO est mise en œuvre en utilisant des packages. Un package est une collection de variables et de sous-routines (fonctions) qui peuvent être utilisées pour créer des objets. Pour créer une classe en Perl, il suffit de déclarer un package et de définir les attributs et les méthodes de la classe.

package Person;

# Définition de la classe Person
sub new {
    my $class = shift;
    my $self = {
        name => shift,
        age => shift,
    };
    bless $self, $class;
    return $self;
}

sub getName {
    my ($self) = @_;
    return $self->{name};
}

sub getAge {
    my ($self) = @_;
    return $self->{age};
}

1; # Fin du package

Dans cet exemple, nous avons créé une classe "Person" avec deux attributs "name" et "age" ainsi que deux méthodes "getName" et "getAge". La méthode "new" est utilisée pour créer une nouvelle instance de la classe et initialiser les attributs.

Définition de classes et d'objets

Pour créer un objet à partir d'une classe en Perl, il suffit d'appeler la méthode "new" de la classe et de passer les paramètres nécessaires. Par exemple :

my $person = Person->new("John Doe", 30);

Dans cet exemple, nous avons créé un nouvel objet "person" de la classe "Person" avec le nom "John Doe" et l'âge 30.

Pour accéder aux attributs et aux méthodes d'un objet, nous utilisons l'opérateur "->". Par exemple :

my $name = $person->getName();
my $age = $person->getAge();

Dans cet exemple, nous utilisons les méthodes "getName" et "getAge" de l'objet "person" pour obtenir respectivement le nom et l'âge.

Héritage et polymorphisme en Perl

En Perl, l'héritage permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe enfant hérite des attributs et des méthodes de sa classe parent. Cela permet de réutiliser le code et de créer une hiérarchie de classes.

Pour définir une classe enfant en Perl, il suffit de déclarer un package pour la classe enfant et d'utiliser la fonction "use base" pour spécifier la classe parent. Par exemple :

package Employee;
use base 'Person';

# Définition de la classe Employee
sub new {
    my $class = shift;
    my $self = $class->SUPER::new(@_);
    $self->{salary} = shift;
    return $self;
}

sub getSalary {
    my ($self) = @_;
    return $self->{salary};
}

1; # Fin du package

Dans cet exemple, nous avons créé une classe "Employee" qui hérite de la classe "Person". La classe "Employee" a un attribut supplémentaire "salary" et une méthode supplémentaire "getSalary". La méthode "new" est redéfinie pour ajouter l'attribut "salary" à l'objet créé par la classe parent.

Le polymorphisme en Perl permet à une classe enfant de redéfinir les méthodes de sa classe parent. Cela permet d'utiliser une méthode spécifique à la classe enfant lorsqu'elle est appelée sur un objet de cette classe. Par exemple :

my $employee = Employee->new("Jane Smith", 35, 50000);
my $name = $employee->getName(); # Appelle la méthode getName de la classe Person
my $salary = $employee->getSalary(); # Appelle la méthode getSalary de la classe Employee

Dans cet exemple, nous utilisons les méthodes "getName" et "getSalary" de l'objet "employee". La méthode "getName" est héritée de la classe "Person" et la méthode "getSalary" est définie dans la classe "Employee".
