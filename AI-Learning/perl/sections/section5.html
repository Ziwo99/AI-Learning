Définition et appel de fonctions

En Perl, une fonction est un bloc de code réutilisable qui effectue une tâche spécifique. Elle peut prendre des paramètres en entrée et renvoyer une valeur en sortie. Voici comment définir et appeler une fonction en Perl :

sub maFonction {
    # Code de la fonction
}

# Appel de la fonction
maFonction();

Dans cet exemple, nous avons défini une fonction nommée "maFonction" en utilisant le mot-clé "sub". Le code de la fonction est placé entre des accolades {}. Pour appeler la fonction, nous utilisons simplement son nom suivi de parenthèses ().

Paramètres de fonctions

Les fonctions en Perl peuvent prendre des paramètres en entrée. Ces paramètres sont des valeurs que vous pouvez transmettre à la fonction pour qu'elle les utilise dans son code. Voici comment définir une fonction avec des paramètres :

sub maFonction {
    my ($parametre1, $parametre2) = @_;
    # Code de la fonction utilisant les paramètres
}

# Appel de la fonction avec des valeurs pour les paramètres
maFonction($valeur1, $valeur2);

Dans cet exemple, nous avons défini une fonction nommée "maFonction" avec deux paramètres, "$parametre1" et "$parametre2". Lors de l'appel de la fonction, nous passons les valeurs des paramètres entre parenthèses. À l'intérieur de la fonction, nous utilisons la variable spéciale "@_" pour accéder aux valeurs des paramètres.

Variables locales et globales

En Perl, vous pouvez déclarer des variables locales à l'intérieur d'une fonction en utilisant le mot-clé "my". Ces variables ne sont accessibles qu'à l'intérieur de la fonction. Voici un exemple :

sub maFonction {
    my $variableLocale = "valeur";
    # Code de la fonction utilisant la variable locale
}

# Appel de la fonction
maFonction();

# Tentative d'accéder à la variable locale en dehors de la fonction (erreur)
print $variableLocale;

Dans cet exemple, nous avons déclaré une variable locale "$variableLocale" à l'intérieur de la fonction "maFonction". Cette variable n'est pas accessible en dehors de la fonction, donc la tentative d'imprimer sa valeur en dehors de la fonction générera une erreur.

En revanche, les variables déclarées en dehors de toutes les fonctions sont des variables globales et peuvent être utilisées à la fois à l'intérieur et à l'extérieur des fonctions :

$variableGlobale = "valeur";

sub maFonction {
    # Code de la fonction utilisant la variable globale
}

# Appel de la fonction
maFonction();

# Accès à la variable globale en dehors de la fonction
print $variableGlobale;

Fonctions récursives

Une fonction récursive est une fonction qui s'appelle elle-même à l'intérieur de son propre code. Cela peut être utile pour résoudre des problèmes qui peuvent être décomposés en sous-problèmes plus petits. Voici un exemple de fonction récursive en Perl :

sub maFonctionRecursive {
    my ($n) = @_;
    
    if ($n == 0) {
        return 1;
    } else {
        return $n * maFonctionRecursive($n - 1);
    }
}

# Appel de la fonction récursive
my $resultat = maFonctionRecursive(5);
print $resultat;

Dans cet exemple, nous avons défini une fonction récursive "maFonctionRecursive" qui calcule la factorielle d'un nombre. La fonction s'appelle elle-même avec un argument réduit à chaque itération jusqu'à ce que la condition d'arrêt soit atteinte. Le résultat final est renvoyé à chaque appel de la fonction.

Il est important de noter que les fonctions récursives peuvent entraîner une consommation excessive de mémoire si elles sont mal utilisées. Il est donc essentiel de s'assurer que la condition d'arrêt est atteinte à un moment donné.
