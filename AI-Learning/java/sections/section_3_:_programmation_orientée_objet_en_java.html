Section 3 : Programmation orientée objet en Java

Concepts fondamentaux de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles pour créer des objets. Les concepts fondamentaux de la POO comprennent :

  • L'encapsulation : il s'agit de regrouper les données et les méthodes associées dans une classe.
  • L'abstraction : il s'agit de simplifier la complexité en utilisant des classes et des objets pour représenter des entités du monde réel.
  • L'héritage : il s'agit de créer de nouvelles classes en utilisant les caractéristiques d'une classe existante.
  • Le polymorphisme : il s'agit de la capacité d'un objet à prendre différentes formes en fonction du contexte.

Classes et objets

Une classe est un modèle pour créer des objets. Elle définit les attributs (variables) et les méthodes (fonctions) qui seront utilisés par les objets. Un objet est une instance d'une classe. Voici un exemple de définition de classe en Java :

public class Person {
    private String name;
    private int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void sayHello() {
        System.out.println("Hello, my name is " + name + " and I am " + age + " years old.");
    }
}

Dans cet exemple, la classe "Person" a deux attributs (name et age) et une méthode (sayHello) qui affiche un message de salutation. Pour créer un objet de cette classe, nous utilisons le mot-clé "new" :

Person person = new Person("John", 25);
person.sayHello();

Cela affichera "Hello, my name is John and I am 25 years old."

Encapsulation et modificateurs d'accès

L'encapsulation est un concept clé de la POO qui permet de regrouper les données et les méthodes associées dans une classe. Les modificateurs d'accès permettent de contrôler l'accès aux membres d'une classe. Les modificateurs d'accès en Java sont :

  • public : accessible depuis n'importe quelle classe.
  • private : accessible uniquement depuis la classe elle-même.
  • protected : accessible depuis la classe elle-même, les classes dérivées et les classes du même package.
  • default : accessible uniquement depuis les classes du même package.

Voici un exemple :

public class BankAccount {
    private double balance;
    
    public BankAccount(double initialBalance) {
        balance = initialBalance;
    }
    
    public void deposit(double amount) {
        balance += amount;
    }
    
    public void withdraw(double amount) {
        balance -= amount;
    }
    
    public double getBalance() {
        return balance;
    }
}

Dans cet exemple, la variable "balance" est déclarée comme privée, ce qui signifie qu'elle ne peut être accédée directement depuis une autre classe. Pour y accéder, nous utilisons les méthodes "deposit", "withdraw" et "getBalance".

Héritage et polymorphisme

L'héritage est un mécanisme qui permet de créer de nouvelles classes en utilisant les caractéristiques d'une classe existante. La classe existante est appelée classe parent ou superclasse, et la nouvelle classe est appelée classe enfant ou sous-classe. Le polymorphisme est la capacité d'un objet à prendre différentes formes en fonction du contexte.

Voici un exemple :

public class Animal {
    public void makeSound() {
        System.out.println("The animal makes a sound");
    }
}

public class Dog extends Animal {
    public void makeSound() {
        System.out.println("The dog barks");
    }
}

public class Cat extends Animal {
    public void makeSound() {
        System.out.println("The cat meows");
    }
}

Dans cet exemple, la classe "Animal" est la classe parent, et les classes "Dog" et "Cat" sont des classes enfants. Chaque classe enfant redéfinit la méthode "makeSound" pour produire un son spécifique. Ainsi, lorsque nous appelons la méthode "makeSound" sur un objet de type "Animal", le son approprié sera produit en fonction du type réel de l'objet.

Interfaces et classes abstraites

Les interfaces et les classes abstraites sont des mécanismes de la POO qui permettent de définir des contrats ou des modèles pour les classes qui les implémentent ou les étendent. Une interface est une collection de méthodes abstraites (sans implémentation), tandis qu'une classe abstraite est une classe qui peut contenir des méthodes abstraites et des méthodes avec une implémentation.

Voici un exemple d'interface :

public interface Shape {
    double getArea();
    double getPerimeter();
}

Dans cet exemple, l'interface "Shape" définit deux méthodes abstraites : "getArea" et "getPerimeter". Toute classe qui implémente cette interface doit fournir une implémentation pour ces méthodes.

Voici un exemple de classe abstraite :

public abstract class Animal {
    public abstract void makeSound();
    
    public void sleep() {
        System.out.println("The animal is sleeping");
    }
}

Dans cet exemple, la classe abstraite "Animal" contient une méthode abstraite "makeSound" qui doit être redéfinie par les classes qui étendent cette classe. Elle contient également une méthode "sleep" avec une implémentation par défaut.

Gestion de la mémoire et du garbage collector

En Java, la gestion de la mémoire est automatisée grâce au garbage collector. Le garbage collector est responsable de la libération de la mémoire occupée par les objets qui ne sont plus utilisés. Il fonctionne en recherchant les objets qui ne sont plus référencés par d'autres objets et en les marquant comme éligibles à la suppression.

Voici un exemple :

public class Example {
    public static void main(String[] args) {
        Person person1 = new Person("John", 25);
        Person person2 = new Person("Jane", 30);
        
        person1 = null;
        
        // À ce stade, l'objet "John" n'est plus référencé par aucune variable
        
        // Le garbage collector peut maintenant libérer la mémoire occupée par l'objet "John"
    }
}

Dans cet exemple, l'objet "person1" est défini comme null, ce qui signifie qu'il ne référence plus l'objet "John". Par conséquent, l'objet "John" devient éligible à la suppression par le garbage collector.
