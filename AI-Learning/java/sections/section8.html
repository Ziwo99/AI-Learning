Les threads en Java

Les threads en Java permettent d'exécuter plusieurs tâches simultanément, améliorant ainsi les performances et la réactivité des applications. Un thread est une unité d'exécution indépendante qui peut être exécutée en parallèle avec d'autres threads.

En Java, la création et la gestion des threads peuvent être réalisées en utilisant la classe Thread ou l'interface Runnable. Voici un exemple de création d'un thread en utilisant la classe Thread :

public class MyThread extends Thread {
    public void run() {
        // Code exécuté par le thread
    }
}

// Création et démarrage du thread
MyThread thread = new MyThread();
thread.start();

Et voici un exemple de création d'un thread en utilisant l'interface Runnable :

public class MyRunnable implements Runnable {
    public void run() {
        // Code exécuté par le thread
    }
}

// Création et démarrage du thread
Thread thread = new Thread(new MyRunnable());
thread.start();

Il est également possible de contrôler l'exécution des threads en utilisant des méthodes telles que sleep(), join() et interrupt(). La méthode sleep() permet de mettre en pause l'exécution d'un thread pendant une durée spécifiée. La méthode join() permet d'attendre la fin de l'exécution d'un thread avant de continuer l'exécution du thread actuel. La méthode interrupt() permet d'interrompre l'exécution d'un thread en lui envoyant un signal d'interruption.

Les annotations en Java

Les annotations en Java sont des métadonnées qui peuvent être ajoutées au code source pour fournir des informations supplémentaires sur les classes, les méthodes, les variables, etc. Elles peuvent être utilisées par le compilateur, les outils de développement et d'autres frameworks pour effectuer des actions spécifiques.

En Java, les annotations sont définies à l'aide de l'annotation @interface. Voici un exemple de définition d'une annotation personnalisée :

import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.METHOD)
public @interface MyAnnotation {
    String value();
}

// Utilisation de l'annotation
@MyAnnotation("Exemple")
public void myMethod() {
    // Code de la méthode
}

Dans cet exemple, l'annotation @MyAnnotation est définie avec une seule valeur de type String. L'annotation peut ensuite être utilisée sur des méthodes en spécifiant une valeur pour cette annotation.

Les annotations prédéfinies en Java incluent @Override, @Deprecated et @SuppressWarnings, entre autres. L'annotation @Override est utilisée pour indiquer qu'une méthode est censée remplacer une méthode héritée de sa classe parente. L'annotation @Deprecated est utilisée pour indiquer qu'un élément (méthode, classe, etc.) est déprécié et ne doit plus être utilisé. L'annotation @SuppressWarnings est utilisée pour supprimer les avertissements du compilateur pour un élément spécifique.

La réflexion en Java

La réflexion en Java permet d'inspecter et de manipuler les classes, les méthodes, les champs, etc. à l'exécution. Elle permet d'accéder aux informations sur les classes et les objets, d'appeler des méthodes dynamiquement et de manipuler les champs privés.

En Java, la réflexion est réalisée à l'aide de la classe Class et de ses méthodes. Voici un exemple d'utilisation de la réflexion pour obtenir des informations sur une classe :

import java.lang.reflect.*;

public class MyClass {
    public static void main(String[] args) {
        Class myClass = MyClass.class;
        
        // Obtention du nom de la classe
        String className = myClass.getName();
        System.out.println("Nom de la classe : " + className);
        
        // Obtention des méthodes de la classe
        Method[] methods = myClass.getMethods();
        System.out.println("Méthodes de la classe :");
        for (Method method : methods) {
            System.out.println(method.getName());
        }
        
        // Obtention des champs de la classe
        Field[] fields = myClass.getFields();
        System.out.println("Champs de la classe :");
        for (Field field : fields) {
            System.out.println(field.getName());
        }
    }
}

Dans cet exemple, la classe MyClass utilise la réflexion pour obtenir son nom, ses méthodes et ses champs. Les méthodes getClass(), getMethods() et getFields() sont utilisées pour obtenir les informations souhaitées.

Les expressions lambda en Java

Les expressions lambda en Java permettent de simplifier l'écriture de certaines interfaces fonctionnelles en utilisant une syntaxe concise. Elles sont principalement utilisées avec les interfaces fonctionnelles, qui sont des interfaces ne contenant qu'une seule méthode abstraite.

En Java, les expressions lambda sont définies en utilisant la syntaxe (paramètres) -> expression. Voici un exemple d'utilisation d'une expression lambda avec l'interface fonctionnelle Runnable :

Runnable runnable = () -> {
    // Code exécuté par le Runnable
};

// Utilisation du Runnable
Thread thread = new Thread(runnable);
thread.start();

Dans cet exemple, une expression lambda est utilisée pour définir le comportement du Runnable. L'interface fonctionnelle Runnable ne contient qu'une seule méthode abstraite, run(), qui est implémentée par l'expression lambda.

Les expressions lambda peuvent également être utilisées avec d'autres interfaces fonctionnelles telles que Comparator, Predicate, Consumer, etc. Elles permettent de simplifier l'écriture du code en évitant la création de classes anonymes.
