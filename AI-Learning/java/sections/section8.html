Les threads et la concurrence en Java

En Java, un thread est une unité d'exécution indépendante qui peut être exécutée en parallèle avec d'autres threads. La concurrence est la capacité d'exécuter plusieurs threads simultanément, ce qui permet d'améliorer les performances et la réactivité d'une application.

Pour créer un thread en Java, vous pouvez soit étendre la classe Thread, soit implémenter l'interface Runnable. Voici un exemple de création d'un thread en étendant la classe Thread :

public class MyThread extends Thread {
    public void run() {
        // Code à exécuter dans le thread
    }
}

Une fois que vous avez créé un thread, vous pouvez le démarrer en appelant la méthode start(). Cela permettra à Java de planifier l'exécution du thread et d'appeler la méthode run() que vous avez définie :

MyThread myThread = new MyThread();
myThread.start();

La synchronisation en Java

La synchronisation est un mécanisme qui permet de contrôler l'accès concurrent à des ressources partagées. En Java, vous pouvez utiliser le mot-clé synchronized pour synchroniser des blocs de code ou des méthodes. Cela garantit qu'un seul thread peut accéder à la ressource partagée à la fois, évitant ainsi les problèmes de concurrence.

Voici un exemple de synchronisation d'une méthode en Java :

public synchronized void myMethod() {
    // Code synchronisé
}

Vous pouvez également synchroniser des blocs de code en utilisant un objet de verrouillage :

public void myMethod() {
    synchronized (lock) {
        // Code synchronisé
    }
}

Les threads et les ressources partagées en Java

Lorsque plusieurs threads accèdent à des ressources partagées, il est important de les synchroniser correctement pour éviter les problèmes de concurrence. En Java, vous pouvez utiliser des mécanismes de synchronisation tels que les verrous, les sémaphores ou les moniteurs pour garantir un accès sûr aux ressources partagées.

Voici un exemple de synchronisation d'une ressource partagée en utilisant un verrou :

public class SharedResource {
    private Object lock = new Object();
    private int count = 0;

    public void increment() {
        synchronized (lock) {
            count++;
        }
    }

    public int getCount() {
        synchronized (lock) {
            return count;
        }
    }
}

Les threads et les interruptions en Java

En Java, vous pouvez interrompre l'exécution d'un thread en appelant la méthode interrupt(). Cela envoie un signal au thread pour lui demander de s'arrêter. Le thread peut choisir de répondre à cette interruption en arrêtant son exécution ou en continuant à s'exécuter.

Voici un exemple d'utilisation de l'interruption d'un thread en Java :

public class MyThread extends Thread {
    public void run() {
        while (!Thread.currentThread().isInterrupted()) {
            // Code à exécuter dans le thread
        }
    }
}

MyThread myThread = new MyThread();
myThread.start();

// Interruption du thread
myThread.interrupt();

Il est important de noter que l'interruption d'un thread ne garantit pas son arrêt immédiat. Le thread doit vérifier régulièrement s'il a été interrompu et décider de s'arrêter en conséquence.
