Les concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de modéliser le monde réel en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles ou des plans pour créer des objets. La POO repose sur quatre concepts fondamentaux :

  1. L'encapsulation : Il s'agit de la capacité de regrouper des données et des méthodes dans une seule entité appelée classe. L'encapsulation permet de cacher les détails internes de la classe et de fournir une interface publique pour interagir avec l'objet.
  2. L'héritage : Il s'agit d'un mécanisme qui permet à une classe d'hériter des propriétés et des comportements d'une autre classe appelée classe parent. L'héritage favorise la réutilisation du code et permet de créer des hiérarchies de classes.
  3. Le polymorphisme : Il s'agit de la capacité d'un objet à prendre différentes formes ou à se comporter de différentes manières en fonction du contexte. Le polymorphisme permet de traiter des objets de différentes classes de manière uniforme.
  4. L'abstraction : Il s'agit de la capacité de représenter des concepts complexes en utilisant des entités simplifiées. L'abstraction permet de se concentrer sur les aspects essentiels d'un objet et d'ignorer les détails non pertinents.

Les classes et les objets en Java

En Java, une classe est un modèle ou un plan pour créer des objets. Une classe définit les propriétés (variables) et les comportements (méthodes) d'un objet. Voici un exemple de définition de classe en Java :

public class Person {
    // Propriétés
    private String name;
    private int age;
    
    // Constructeur
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // Méthodes
    public void sayHello() {
        System.out.println("Hello, my name is " + name + " and I am " + age + " years old.");
    }
}

Une fois que nous avons défini une classe, nous pouvons créer des objets à partir de cette classe en utilisant le mot-clé "new". Voici un exemple de création d'objet en Java :

Person person1 = new Person("John", 25);
Person person2 = new Person("Jane", 30);

Chaque objet créé à partir d'une classe possède ses propres valeurs pour les propriétés et peut appeler les méthodes définies dans la classe.

L'encapsulation en Java

L'encapsulation en Java permet de regrouper des données et des méthodes dans une seule entité appelée classe. Les données sont généralement déclarées comme des variables d'instance privées, ce qui signifie qu'elles ne peuvent être accédées directement depuis l'extérieur de la classe. Pour accéder ou modifier les données encapsulées, nous utilisons des méthodes publiques appelées accesseurs (getters) et mutateurs (setters). Voici un exemple :

public class Person {
    private String name;
    private int age;
    
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
}

Ainsi, pour accéder ou modifier les propriétés encapsulées d'un objet Person, nous utilisons les méthodes getName(), setName(), getAge() et setAge().

L'héritage en Java

L'héritage en Java permet à une classe d'hériter des propriétés et des comportements d'une autre classe appelée classe parent. La classe qui hérite est appelée classe enfant ou sous-classe, et la classe dont elle hérite est appelée classe parent ou superclasse. Pour hériter d'une classe, nous utilisons le mot-clé "extends". Voici un exemple :

public class Student extends Person {
    private String major;
    
    public Student(String name, int age, String major) {
        super(name, age);
        this.major = major;
    }
    
    public String getMajor() {
        return major;
    }
    
    public void setMajor(String major) {
        this.major = major;
    }
}

Dans cet exemple, la classe Student hérite de la classe Person. Elle ajoute une propriété supplémentaire appelée "major" et des méthodes pour accéder et modifier cette propriété.

Le polymorphisme en Java

Le polymorphisme en Java permet à un objet d'être traité comme un objet d'un type différent dans une hiérarchie de classes. Cela signifie qu'un objet peut prendre différentes formes ou se comporter de différentes manières en fonction du contexte. Le polymorphisme est souvent utilisé en conjonction avec l'héritage et les classes abstraites ou les interfaces. Voici un exemple :

public interface Shape {
    double getArea();
}

public class Circle implements Shape {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

public class Rectangle implements Shape {
    private double width;
    private double height;
    
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
    
    public double getArea() {
        return width * height;
    }
}

Dans cet exemple, nous avons une interface Shape qui définit une méthode getArea(). Les classes Circle et Rectangle implémentent cette interface et fournissent une implémentation spécifique de la méthode getArea(). Ainsi, nous pouvons traiter un objet Circle ou Rectangle comme un objet Shape et appeler la méthode getArea() de manière uniforme.
