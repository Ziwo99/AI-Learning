Les classes et les objets en Java

En Java, une classe est un modèle ou un plan pour créer des objets. Une classe définit les propriétés et les comportements communs à un groupe d'objets. Les objets sont des instances d'une classe spécifique.

Pour déclarer une classe en Java, utilisez le mot-clé class suivi du nom de la classe. Par exemple :

public class Person {
    // Propriétés
    String name;
    int age;
    
    // Méthodes
    public void sayHello() {
        System.out.println("Hello, my name is " + name);
    }
}

Dans cet exemple, nous avons déclaré une classe Person avec deux propriétés : name de type String et age de type int. Nous avons également défini une méthode sayHello() qui affiche un message de salutation.

Les attributs et les méthodes en Java

Les attributs, également appelés variables d'instance, sont des variables déclarées à l'intérieur d'une classe mais à l'extérieur de toute méthode. Ils représentent les caractéristiques ou les données d'un objet.

Les méthodes, également appelées fonctions, sont des blocs de code qui définissent le comportement d'un objet. Elles peuvent être utilisées pour effectuer des opérations sur les attributs de l'objet.

Voici un exemple de classe avec des attributs et des méthodes :

public class Circle {
    // Attributs
    double radius;
    
    // Méthodes
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    public double calculatePerimeter() {
        return 2 * Math.PI * radius;
    }
}

Dans cet exemple, nous avons une classe Circle avec un attribut radius de type double. Nous avons également deux méthodes : calculateArea() qui calcule l'aire du cercle et calculatePerimeter() qui calcule le périmètre du cercle.

L'encapsulation en Java

L'encapsulation est un concept clé en programmation orientée objet qui consiste à regrouper les attributs et les méthodes d'une classe en une seule entité. Cela permet de contrôler l'accès aux données et de protéger les attributs d'une classe contre les modifications non autorisées.

En Java, l'encapsulation est réalisée en définissant les attributs d'une classe comme privés (private) et en fournissant des méthodes publiques (public) pour accéder et modifier ces attributs. Ces méthodes sont appelées des accesseurs (getters) et des mutateurs (setters).

Voici un exemple de classe avec des attributs privés et des méthodes publiques :

public class BankAccount {
    // Attributs privés
    private double balance;
    
    // Méthodes publiques
    public double getBalance() {
        return balance;
    }
    
    public void deposit(double amount) {
        balance += amount;
    }
    
    public void withdraw(double amount) {
        if (amount <= balance) {
            balance -= amount;
        } else {
            System.out.println("Insufficient funds");
        }
    }
}

Dans cet exemple, nous avons une classe BankAccount avec un attribut privé balance. Nous avons également des méthodes publiques getBalance() pour obtenir le solde du compte, deposit() pour effectuer un dépôt et withdraw() pour effectuer un retrait.

L'héritage en Java

L'héritage est un mécanisme en programmation orientée objet qui permet à une classe d'hériter des propriétés et des comportements d'une autre classe. La classe qui hérite est appelée la classe dérivée ou sous-classe, et la classe dont elle hérite est appelée la classe de base ou superclasse.

En Java, l'héritage est réalisé en utilisant le mot-clé extends. Voici un exemple :

public class Animal {
    // Méthode
    public void eat() {
        System.out.println("The animal is eating");
    }
}

public class Dog extends Animal {
    // Méthode
    public void bark() {
        System.out.println("The dog is barking");
    }
}

Dans cet exemple, nous avons une classe Animal avec une méthode eat(). Nous avons également une classe Dog qui hérite de la classe Animal et ajoute une méthode bark().

Grâce à l'héritage, un objet de la classe Dog peut utiliser à la fois les méthodes de la classe Dog et de la classe Animal.

Le polymorphisme en Java

Le polymorphisme est un concept en programmation orientée objet qui permet à un objet d'être traité comme s'il était de plusieurs types différents. Cela permet de créer du code plus générique et réutilisable.

En Java, le polymorphisme est réalisé en utilisant des classes et des méthodes abstraites, ainsi que des interfaces. Voici un exemple :

public abstract class Shape {
    // Méthode abstraite
    public abstract double calculateArea();
}

public class Circle extends Shape {
    // Attribut
    double radius;
    
    // Méthode
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
}

public class Rectangle extends Shape {
    // Attributs
    double width;
    double height;
    
    // Méthode
    public double calculateArea() {
        return width * height;
    }
}

Dans cet exemple, nous avons une classe abstraite Shape avec une méthode abstraite calculateArea(). Nous avons également deux classes concrètes, Circle et Rectangle, qui héritent de la classe Shape et implémentent la méthode calculateArea().

Grâce au polymorphisme, nous pouvons traiter un objet de type Circle ou Rectangle comme un objet de type Shape, ce qui nous permet d'utiliser la méthode calculateArea() de manière générique.
