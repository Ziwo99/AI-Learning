Les classes et les objets en Java

En Java, la programmation orientée objet repose sur le concept de classes et d'objets. Une classe est un modèle ou un plan qui définit les caractéristiques et les comportements d'un objet. Un objet est une instance d'une classe, créée à partir de ce modèle.

Pour déclarer une classe en Java, utilisez le mot-clé class suivi du nom de la classe. Par exemple :

public class Person {
    // attributs
    String name;
    int age;
    
    // méthodes
    public void sayHello() {
        System.out.println("Hello, my name is " + name);
    }
}

Dans cet exemple, nous avons déclaré une classe Person avec deux attributs (name et age) et une méthode (sayHello). Les attributs représentent les données de l'objet, tandis que les méthodes définissent les actions qu'un objet peut effectuer.

Pour créer un objet à partir d'une classe, utilisez le mot-clé new suivi du nom de la classe et des parenthèses. Par exemple :

Person person = new Person();
person.name = "John";
person.age = 25;

Dans cet exemple, nous avons créé un nouvel objet person de type Person et nous avons attribué des valeurs à ses attributs.

Une fois que vous avez créé un objet, vous pouvez appeler ses méthodes en utilisant la notation pointée. Par exemple :

person.sayHello();

Cette ligne de code appelle la méthode sayHello de l'objet person, qui affiche le message "Hello, my name is John".

L'encapsulation et l'abstraction en Java

L'encapsulation et l'abstraction sont des concepts clés de la programmation orientée objet en Java.

L'encapsulation consiste à regrouper les données et les méthodes qui agissent sur ces données dans une classe. Les données sont généralement déclarées comme privées (private) pour les protéger de l'accès direct et les méthodes sont déclarées comme publiques (public) pour permettre l'accès contrôlé aux données.

Voici un exemple :

public class BankAccount {
    private double balance;
    
    public void deposit(double amount) {
        balance += amount;
    }
    
    public void withdraw(double amount) {
        if (amount <= balance) {
            balance -= amount;
        } else {
            System.out.println("Insufficient funds");
        }
    }
    
    public double getBalance() {
        return balance;
    }
}

Dans cet exemple, la classe BankAccount encapsule la donnée balance et les méthodes deposit, withdraw et getBalance permettent de manipuler cette donnée de manière contrôlée.

L'abstraction consiste à cacher les détails d'implémentation d'une classe et à fournir une interface simplifiée pour interagir avec l'objet. Cela permet de réduire la complexité et de faciliter la maintenance du code.

Dans notre exemple, l'utilisateur de la classe BankAccount n'a pas besoin de connaître les détails de l'implémentation de la méthode deposit ou withdraw. Il lui suffit d'appeler ces méthodes pour effectuer des opérations sur le compte bancaire.

L'héritage et le polymorphisme en Java

L'héritage est un mécanisme qui permet à une classe d'hériter des caractéristiques (attributs et méthodes) d'une autre classe. La classe qui hérite est appelée classe dérivée ou sous-classe, et la classe dont elle hérite est appelée classe de base ou superclasse.

En Java, l'héritage est réalisé à l'aide du mot-clé extends. Par exemple :

public class Animal {
    public void eat() {
        System.out.println("Animal is eating");
    }
}

public class Dog extends Animal {
    public void bark() {
        System.out.println("Dog is barking");
    }
}

Dans cet exemple, la classe Dog hérite de la classe Animal. La classe Dog hérite donc de la méthode eat de la classe Animal et ajoute sa propre méthode bark.

Le polymorphisme est la capacité d'un objet à prendre différentes formes. En Java, cela signifie qu'un objet d'une classe dérivée peut être traité comme un objet de sa classe de base.

Par exemple :

Animal animal = new Dog();
animal.eat(); // Appelle la méthode eat() de la classe Dog
((Dog) animal).bark(); // Appelle la méthode bark() de la classe Dog

Dans cet exemple, nous avons créé un objet animal de type Dog et nous l'avons traité comme un objet de type Animal. Cela nous permet d'appeler la méthode eat de la classe Dog. En utilisant une conversion de type ((Dog) animal), nous pouvons également appeler la méthode bark spécifique à la classe Dog.

Les interfaces en Java

Une interface en Java est une collection de méthodes abstraites (méthodes sans implémentation) qui définissent un contrat que les classes qui implémentent cette interface doivent respecter.

Pour déclarer une interface en Java, utilisez le mot-clé interface. Par exemple :

public interface Shape {
    double getArea();
    double getPerimeter();
}

Dans cet exemple, nous avons déclaré une interface Shape avec deux méthodes abstraites : getArea et getPerimeter. Toute classe qui implémente cette interface doit fournir une implémentation pour ces deux méthodes.

Pour implémenter une interface en Java, utilisez le mot-clé implements. Par exemple :

public class Circle implements Shape {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return Math.PI * radius * radius;
    }
    
    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}

Dans cet exemple, la classe Circle implémente l'interface Shape et fournit une implémentation pour les méthodes getArea et getPerimeter.

Les interfaces permettent de définir des contrats communs pour des classes qui peuvent avoir des implémentations différentes. Elles favorisent également la modularité et la réutilisabilité du code.

Les packages en Java

Un package en Java est un mécanisme qui permet d'organiser les classes en groupes logiques. Il aide à éviter les conflits de noms et facilite la gestion des classes.

Pour déclarer un package en Java, utilisez le mot-clé package suivi du nom du package. Par exemple :

package com.example.myapp;

Dans cet exemple, nous avons déclaré un package com.example.myapp. Les classes qui sont déclarées dans ce fichier Java appartiennent à ce package.

Pour utiliser une classe d'un autre package, vous devez importer ce package en utilisant le mot-clé import. Par exemple :

import com.example.myapp.MyClass;

Dans cet exemple, nous importons la classe MyClass du package com.example.myapp. Cela nous permet d'utiliser cette classe dans notre code.

Les packages aident à organiser le code, à le rendre plus lisible et à éviter les conflits de noms. Ils sont largement utilisés dans les projets Java pour structurer le code en modules logiques.
