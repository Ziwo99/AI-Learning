Section 3 : Programmation orientée objet en Java

Les concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer le code en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles pour créer des objets avec des attributs et des comportements spécifiques.

Les concepts de base de la POO en Java sont :

  • Encapsulation : il s'agit de regrouper les données et les méthodes qui les manipulent dans une classe.
  • Héritage : il permet de créer de nouvelles classes en se basant sur des classes existantes, en héritant de leurs attributs et méthodes.
  • Polymorphisme : il permet à un objet d'être utilisé comme s'il était d'un type différent.
  • Abstraction : il permet de définir des classes abstraites qui servent de modèles pour d'autres classes.

Les classes et les objets avancés

Les classes et les objets sont les éléments fondamentaux de la POO en Java. Une classe est un modèle pour créer des objets, qui sont des instances de cette classe. Les objets ont des attributs, qui sont des variables qui définissent leur état, et des méthodes, qui définissent leur comportement.

Voici un exemple de classe en Java :

public class Person {
    private String name;
    private int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void sayHello() {
        System.out.println("Hello, my name is " + name + " and I am " + age + " years old.");
    }
}

Et voici comment créer un objet à partir de cette classe :

Person person = new Person("John", 25);
person.sayHello();

L'héritage et le polymorphisme en Java

L'héritage permet de créer de nouvelles classes en se basant sur des classes existantes. Une classe qui hérite d'une autre classe est appelée une classe dérivée ou une sous-classe, tandis que la classe dont elle hérite est appelée une classe de base ou une superclasse.

Le polymorphisme permet à un objet d'être utilisé comme s'il était d'un type différent. Cela permet de traiter des objets de différentes classes de manière uniforme, en utilisant des méthodes communes définies dans une classe de base.

Voici un exemple d'héritage et de polymorphisme en Java :

public class Animal {
    public void makeSound() {
        System.out.println("The animal makes a sound.");
    }
}

public class Dog extends Animal {
    public void makeSound() {
        System.out.println("The dog barks.");
    }
}

public class Cat extends Animal {
    public void makeSound() {
        System.out.println("The cat meows.");
    }
}

Animal animal1 = new Animal();
Animal animal2 = new Dog();
Animal animal3 = new Cat();

animal1.makeSound();
animal2.makeSound();
animal3.makeSound();

Les interfaces et les classes abstraites en Java

Les interfaces et les classes abstraites sont des mécanismes utilisés en Java pour définir des contrats et des modèles pour d'autres classes.

Une interface est une collection de méthodes abstraites, qui sont des méthodes sans implémentation. Une classe peut implémenter une ou plusieurs interfaces, en fournissant une implémentation pour toutes les méthodes définies dans ces interfaces.

Une classe abstraite est une classe qui ne peut pas être instanciée, mais qui peut être utilisée comme modèle pour d'autres classes. Elle peut contenir des méthodes abstraites, qui doivent être implémentées par les classes dérivées.

Voici un exemple d'interface et de classe abstraite en Java :

public interface Shape {
    double getArea();
    double getPerimeter();
}

public abstract class AbstractShape implements Shape {
    protected String color;
    
    public AbstractShape(String color) {
        this.color = color;
    }
    
    public abstract double getArea();
    
    public abstract double getPerimeter();
}

public class Circle extends AbstractShape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    public double getArea() {
        return Math.PI * radius * radius;
    }
    
    public double getPerimeter() {
        return 2 * Math.PI * radius;
    }
}

Les packages et la modularité en Java

Les packages sont utilisés en Java pour organiser les classes en modules logiques. Ils permettent de regrouper les classes qui ont une fonctionnalité ou une relation commune.

La modularité en Java consiste à découper un programme en modules indépendants, qui peuvent être développés, testés et maintenus séparément. Cela facilite la réutilisation du code et la gestion des dépendances entre les modules.

Voici un exemple d'utilisation des packages et de la modularité en Java :

package com.example.myapp;

import com.example.mylibrary.MyClass;

public class MyApp {
    public static void main(String[] args) {
        MyClass myObject = new MyClass();
        myObject.doSomething();
    }
}

package com.example.mylibrary;

public class MyClass {
    public void doSomething() {
        System.out.println("Doing something...");
    }
}
