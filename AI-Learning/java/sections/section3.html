Section 3 : Programmation orientée objet en Java

Principes de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer et d'organiser le code en utilisant des objets. Les objets sont des instances de classes, qui sont des modèles ou des plans pour créer des objets. La POO repose sur quatre principes fondamentaux :

  • Encapsulation : Il s'agit de regrouper les données et les méthodes qui les manipulent au sein d'une même entité appelée classe. L'encapsulation permet de cacher les détails d'implémentation et de protéger les données sensibles.
  • Héritage : Il permet de créer de nouvelles classes en se basant sur des classes existantes. La classe héritante (ou sous-classe) hérite des attributs et des méthodes de la classe parente (ou superclasse) et peut les modifier ou en ajouter de nouveaux.
  • Polymorphisme : Il permet à un objet d'être utilisé comme s'il était d'un type différent. Cela permet de traiter des objets de différentes classes de manière uniforme, en utilisant des méthodes communes.
  • Abstraction : Il s'agit de simplifier la complexité en se concentrant sur les aspects essentiels et en ignorant les détails non pertinents. L'abstraction permet de créer des classes abstraites et des interfaces, qui définissent des comportements communs pour les classes qui les implémentent.

Classes et objets

Une classe est un modèle ou un plan pour créer des objets. Elle définit les attributs (variables) et les méthodes (fonctions) qui caractérisent les objets de cette classe. Pour créer un objet à partir d'une classe, on utilise le mot-clé new. Par exemple :

public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void sayHello() {
        System.out.println("Hello, my name is " + name + " and I am " + age + " years old.");
    }
}

// Création d'un objet Person
Person person = new Person("John", 25);
person.sayHello();

Encapsulation, héritage et polymorphisme

L'encapsulation permet de regrouper les données et les méthodes qui les manipulent au sein d'une même classe. Les données peuvent être déclarées comme privées pour les protéger de l'accès direct et les méthodes peuvent être utilisées pour les manipuler de manière contrôlée.

L'héritage permet de créer de nouvelles classes en se basant sur des classes existantes. La classe héritante hérite des attributs et des méthodes de la classe parente et peut les modifier ou en ajouter de nouveaux. Cela permet de réutiliser du code et de créer une hiérarchie de classes.

Le polymorphisme permet à un objet d'être utilisé comme s'il était d'un type différent. Cela permet de traiter des objets de différentes classes de manière uniforme, en utilisant des méthodes communes. Par exemple :

public class Animal {
    public void makeSound() {
        System.out.println("The animal makes a sound.");
    }
}

public class Dog extends Animal {
    public void makeSound() {
        System.out.println("The dog barks.");
    }
}

public class Cat extends Animal {
    public void makeSound() {
        System.out.println("The cat meows.");
    }
}

Animal animal1 = new Animal();
Animal animal2 = new Dog();
Animal animal3 = new Cat();

animal1.makeSound(); // Output: The animal makes a sound.
animal2.makeSound(); // Output: The dog barks.
animal3.makeSound(); // Output: The cat meows.

Interfaces et classes abstraites

Les interfaces et les classes abstraites sont des mécanismes de l'orienté objet qui permettent de définir des comportements communs pour les classes qui les implémentent ou les étendent.

Une interface est une collection de méthodes abstraites (sans implémentation) et de constantes. Une classe peut implémenter plusieurs interfaces. Par exemple :

public interface Drawable {
    void draw();
}

public class Circle implements Drawable {
    public void draw() {
        System.out.println("Drawing a circle.");
    }
}

public class Rectangle implements Drawable {
    public void draw() {
        System.out.println("Drawing a rectangle.");
    }
}

Drawable circle = new Circle();
Drawable rectangle = new Rectangle();

circle.draw(); // Output: Drawing a circle.
rectangle.draw(); // Output: Drawing a rectangle.

Une classe abstraite est une classe qui ne peut pas être instanciée et qui peut contenir des méthodes abstraites (sans implémentation) et des méthodes concrètes (avec implémentation). Une classe peut étendre une seule classe abstraite. Par exemple :

public abstract class Shape {
    public abstract double getArea();

    public void printArea() {
        System.out.println("The area is " + getArea());
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    public double getArea() {
        return Math.PI * radius * radius;
    }
}

Shape circle = new Circle(5);
circle.printArea(); // Output: The area is 78.53981633974483.
