Section 3: Programmation orientée objet en Java

Les concepts de base de la programmation orientée objet

La programmation orientée objet (POO) est un paradigme de programmation qui permet de structurer un programme en utilisant des objets qui interagissent entre eux. Les principaux concepts de la POO sont :

  • L'encapsulation : il s'agit de regrouper des données et des méthodes dans une même entité appelée objet. L'encapsulation permet de cacher les détails internes de l'objet et de fournir une interface pour interagir avec lui.
  • L'héritage : il permet de créer de nouvelles classes en se basant sur des classes existantes. La classe héritante (ou sous-classe) hérite des attributs et des méthodes de la classe parente (ou superclasse) et peut les modifier ou en ajouter de nouveaux.
  • Le polymorphisme : il permet à un objet d'être utilisé de différentes manières, en fonction du contexte. Cela permet d'écrire du code générique et réutilisable.
  • L'abstraction : il s'agit de simplifier un concept complexe en le représentant par un modèle plus simple. En POO, cela se traduit par la création de classes abstraites et d'interfaces.

Les classes et les objets en Java

En Java, une classe est un modèle ou un plan pour créer des objets. Une classe définit les attributs (variables) et les méthodes (fonctions) qui caractérisent un objet. Voici un exemple de déclaration de classe en Java :

public class Person {
    // Attributs
    private String name;
    private int age;
    
    // Méthodes
    public void setName(String name) {
        this.name = name;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
    
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
}

Une fois qu'une classe est définie, on peut créer des objets à partir de cette classe. Voici comment créer un objet de la classe Person :

Person person = new Person();
person.setName("John");
person.setAge(25);
System.out.println(person.getName()); // Output: John
System.out.println(person.getAge()); // Output: 25

L'encapsulation en Java

L'encapsulation en Java permet de regrouper des données et des méthodes dans une même entité appelée classe. Les attributs d'une classe peuvent être déclarés avec différents niveaux d'accès (public, private, protected) pour contrôler leur visibilité depuis d'autres classes. Voici un exemple :

public class Person {
    private String name;
    private int age;
    
    public void setName(String name) {
        this.name = name;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
    
    public String getName() {
        return name;
    }
    
    public int getAge() {
        return age;
    }
}

Dans cet exemple, les attributs name et age sont déclarés comme privés (private), ce qui signifie qu'ils ne peuvent être accédés directement depuis d'autres classes. Pour y accéder, on utilise les méthodes publiques setName, setAge, getName et getAge.

L'héritage en Java

L'héritage en Java permet de créer de nouvelles classes en se basant sur des classes existantes. La classe héritante (ou sous-classe) hérite des attributs et des méthodes de la classe parente (ou superclasse) et peut les modifier ou en ajouter de nouveaux. Voici un exemple :

public class Student extends Person {
    private String studentId;
    
    public void setStudentId(String studentId) {
        this.studentId = studentId;
    }
    
    public String getStudentId() {
        return studentId;
    }
}

Dans cet exemple, la classe Student hérite de la classe Person. Elle ajoute un nouvel attribut studentId et les méthodes correspondantes. Les instances de la classe Student auront donc accès aux attributs et aux méthodes de la classe Person, en plus de ceux spécifiques à la classe Student.

Le polymorphisme en Java

Le polymorphisme en Java permet à un objet d'être utilisé de différentes manières, en fonction du contexte. Cela permet d'écrire du code générique et réutilisable. Voici un exemple :

public interface Shape {
    double getArea();
}

public class Circle implements Shape {
    private double radius;
    
    public Circle(double radius) {
        this.radius = radius;
    }
    
    public double getArea() {
        return Math.PI * radius * radius;
    }
}

public class Rectangle implements Shape {
    private double width;
    private double height;
    
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
    
    public double getArea() {
        return width * height;
    }
}

Dans cet exemple, l'interface Shape définit une méthode getArea. Les classes Circle et Rectangle implémentent cette interface et fournissent une implémentation spécifique de la méthode getArea. On peut alors utiliser des objets de type Shape pour calculer l'aire de différentes formes, sans se soucier de leur type concret :

Shape circle = new Circle(5);
Shape rectangle = new Rectangle(3, 4);

System.out.println(circle.getArea()); // Output: 78.53981633974483
System.out.println(rectangle.getArea()); // Output: 12.0

Ce contenu détaillé sur la programmation orientée objet en Java vous permettra de comprendre les concepts fondamentaux de la POO et de les appliquer dans vos programmes Java. Vous serez en mesure de créer des classes, d'utiliser l'héritage et le polymorphisme, et de tirer parti de l'encapsulation pour créer des programmes modulaires et réutilisables.
