Polymorphisme et liaison dynamique

Le polymorphisme est un concept clé de la programmation orientée objet (POO) qui permet à un objet d'être traité comme une instance de sa classe de base ou de l'une de ses classes dérivées. Cela signifie qu'un objet peut prendre différentes formes en fonction du contexte dans lequel il est utilisé.

La liaison dynamique est le mécanisme par lequel le compilateur détermine quelle méthode appeler en fonction du type réel de l'objet au moment de l'exécution. Cela permet d'appeler la méthode appropriée même si l'objet est référencé par un type de base.

Voici un exemple pour illustrer le polymorphisme et la liaison dynamique :

public class Animal {
    public void makeSound() {
        System.out.println("L'animal fait un son.");
    }
}

public class Dog extends Animal {
    public void makeSound() {
        System.out.println("Le chien aboie.");
    }
}

public class Cat extends Animal {
    public void makeSound() {
        System.out.println("Le chat miaule.");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal animal1 = new Animal();
        Animal animal2 = new Dog();
        Animal animal3 = new Cat();

        animal1.makeSound();
        animal2.makeSound();
        animal3.makeSound();
    }
}

Résultat :

L'animal fait un son.
Le chien aboie.
Le chat miaule.

Dans cet exemple, nous avons une classe de base "Animal" et deux classes dérivées "Dog" et "Cat". Chacune de ces classes a une méthode "makeSound()" qui est redéfinie.

Dans la méthode "main()", nous créons trois objets : un de type "Animal", un de type "Dog" et un de type "Cat". Lorsque nous appelons la méthode "makeSound()" sur chaque objet, la méthode appropriée est appelée en fonction du type réel de l'objet.

Classes internes et anonymes

En Java, il est possible de définir des classes à l'intérieur d'autres classes. Ces classes sont appelées classes internes. Les classes internes peuvent être utilisées pour encapsuler la logique liée à une classe spécifique et pour accéder aux membres privés de cette classe.

Voici un exemple de classe interne :

public class OuterClass {
    private int x = 10;

    public class InnerClass {
        public void printX() {
            System.out.println(x);
        }
    }
}

public class Main {
    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        OuterClass.InnerClass inner = outer.new InnerClass();

        inner.printX();
    }
}

Résultat :

10

Dans cet exemple, nous avons une classe externe "OuterClass" qui contient une classe interne "InnerClass". La classe interne a accès aux membres privés de la classe externe, comme la variable "x".

Pour créer une instance de la classe interne, nous devons d'abord créer une instance de la classe externe, puis utiliser cette instance pour créer une instance de la classe interne.

En plus des classes internes, Java prend également en charge les classes anonymes. Une classe anonyme est une classe sans nom qui est déclarée et instanciée en même temps. Les classes anonymes sont souvent utilisées pour implémenter des interfaces ou des classes abstraites de manière concise.

Voici un exemple de classe anonyme :

public interface Greeting {
    void sayHello();
}

public class Main {
    public static void main(String[] args) {
        Greeting greeting = new Greeting() {
            public void sayHello() {
                System.out.println("Bonjour !");
            }
        };

        greeting.sayHello();
    }
}

Résultat :

Bonjour !

Dans cet exemple, nous avons une interface "Greeting" avec une méthode "sayHello()". Nous créons une instance de cette interface en utilisant une classe anonyme qui implémente la méthode "sayHello()".

Généricité

La généricité est un mécanisme qui permet de créer des classes et des méthodes qui peuvent être utilisées avec différents types de données. Cela permet de réutiliser le code et d'améliorer la sécurité et la lisibilité du code.

Voici un exemple de classe générique :

public class Box<T> {
    private T content;

    public void setContent(T content) {
        this.content = content;
    }

    public T getContent() {
        return content;
    }
}

public class Main {
    public static void main(String[] args) {
        Box<String> box1 = new Box<>();
        box1.setContent("Contenu de la boîte 1");
        String content1 = box1.getContent();

        Box<Integer> box2 = new Box<>();
        box2.setContent(42);
        int content2 = box2.getContent();

        System.out.println(content1);
        System.out.println(content2);
    }
}

Résultat :

Contenu de la boîte 1
42

Dans cet exemple, nous avons une classe générique "Box" qui peut contenir n'importe quel type d'objet. La classe utilise le paramètre de type "T" pour définir le type du contenu.

Dans la méthode "main()", nous créons deux instances de la classe "Box" : une avec le type "String" et une avec le type "Integer". Nous pouvons ensuite définir et obtenir le contenu de chaque boîte en utilisant les méthodes appropriées.

Réflexion et annotations

La réflexion est un mécanisme qui permet à un programme d'examiner et de modifier sa propre structure à l'exécution. Cela permet d'accéder aux informations sur les classes, les méthodes et les champs, et d'appeler des méthodes ou d'accéder à des champs dynamiquement.

Les annotations sont des métadonnées qui peuvent être ajoutées au code source pour fournir des informations supplémentaires sur les classes, les méthodes, les champs, etc. Les annotations peuvent être utilisées par le compilateur, l'IDE ou d'autres outils pour générer du code, effectuer des vérifications statiques ou fournir des fonctionnalités supplémentaires.

Voici un exemple d'utilisation de la réflexion et des annotations :

import java.lang.reflect.Method;

public class MyClass {
    @MyAnnotation
    public void myMethod() {
        System.out.println("Méthode appelée.");
    }
}

public @interface MyAnnotation {
    String value() default "";
}

public class Main {
    public static void main(String[] args) throws Exception {
        MyClass myObject = new MyClass();
        Method method = myObject.getClass().getMethod("myMethod");

        if (method.isAnnotationPresent(MyAnnotation.class)) {
            MyAnnotation annotation = method.getAnnotation(MyAnnotation.class);
            System.out.println(annotation.value());
        }

        method.invoke(myObject);
    }
}

Résultat :


Méthode appelée.

Dans cet exemple, nous avons une classe "MyClass" avec une méthode "myMethod()" annotée avec "@MyAnnotation". La classe "MyAnnotation" est une annotation personnalisée avec une valeur par défaut.

Dans la méthode "main()", nous créons une instance de "MyClass" et utilisons la réflexion pour obtenir la méthode "myMethod()". Nous vérifions si la méthode est annotée avec "@MyAnnotation" et, si c'est le cas, nous obtenons la valeur de l'annotation et l'affichons.

Enfin, nous appelons la méthode "myMethod()" en utilisant la réflexion.

Ce sont là quelques-uns des concepts avancés de la programmation orientée objet en Java. Ils vous permettront de développer des applications plus flexibles, réutilisables et évolutives.
