Section 4 : Gestion des exceptions

Introduction aux exceptions

En programmation Java, une exception est un événement anormal qui se produit pendant l'exécution d'un programme et qui interrompt le flux normal d'exécution. Les exceptions sont utilisées pour gérer les erreurs et les situations exceptionnelles qui peuvent survenir lors de l'exécution d'un programme.

Java fournit une hiérarchie de classes d'exceptions prédéfinies qui peuvent être utilisées pour gérer différents types d'exceptions. Les exceptions peuvent être levées (ou "jetées") à l'aide du mot-clé throw et capturées à l'aide du bloc try-catch.

Gestion des exceptions avec try-catch

Le bloc try-catch est utilisé pour capturer et gérer les exceptions levées dans un bloc de code. Le bloc try contient le code qui peut générer une exception, tandis que le bloc catch contient le code qui est exécuté lorsque l'exception est levée.

Voici un exemple de structure de base d'un bloc try-catch :

try {
    // Code susceptible de générer une exception
} catch (TypeException e) {
    // Code exécuté lorsque l'exception de type TypeException est levée
}

Dans cet exemple, si une exception de type TypeException est levée dans le bloc try, le code dans le bloc catch sera exécuté pour gérer cette exception.

Création de ses propres exceptions

En plus des exceptions prédéfinies, il est également possible de créer ses propres exceptions personnalisées en définissant des classes d'exceptions. Pour créer une exception personnalisée, il suffit de créer une classe qui étend la classe Exception ou l'une de ses sous-classes.

Voici un exemple de création d'une exception personnalisée :

public class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }
}

Dans cet exemple, nous avons créé une classe CustomException qui étend la classe Exception. La classe CustomException a un constructeur qui prend un message en paramètre et appelle le constructeur de la classe Exception avec ce message.

Une fois que vous avez créé votre exception personnalisée, vous pouvez la lever (ou "jeter") à l'aide du mot-clé throw :

throw new CustomException("Une erreur personnalisée s'est produite.");

Dans cet exemple, nous avons levé une exception de type CustomException avec un message d'erreur personnalisé.

Il est également possible de capturer et de gérer une exception personnalisée à l'aide d'un bloc try-catch :

try {
    // Code susceptible de générer une exception personnalisée
} catch (CustomException e) {
    // Code exécuté lorsque l'exception de type CustomException est levée
}

Dans cet exemple, si une exception de type CustomException est levée dans le bloc try, le code dans le bloc catch sera exécuté pour gérer cette exception.
