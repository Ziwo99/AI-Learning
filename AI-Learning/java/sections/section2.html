Les structures de contrôle (boucles, conditions)

Les structures de contrôle permettent de contrôler le flux d'exécution d'un programme en Java. Elles comprennent les boucles et les conditions.

Les boucles permettent de répéter un bloc de code plusieurs fois. Il existe trois types de boucles en Java :

while (condition) {
    // code à exécuter tant que la condition est vraie
}

do {
    // code à exécuter au moins une fois, puis tant que la condition est vraie
} while (condition);

for (initialisation; condition; mise à jour) {
    // code à exécuter tant que la condition est vraie
}

Les conditions permettent d'exécuter un bloc de code en fonction d'une condition donnée. Les opérateurs de comparaison et les opérateurs logiques sont utilisés pour évaluer les conditions. Voici quelques exemples :

if (condition) {
    // code à exécuter si la condition est vraie
} else if (autreCondition) {
    // code à exécuter si la première condition est fausse et l'autre condition est vraie
} else {
    // code à exécuter si toutes les conditions précédentes sont fausses
}

switch (variable) {
    case valeur1:
        // code à exécuter si la variable a la valeur 1
        break;
    case valeur2:
        // code à exécuter si la variable a la valeur 2
        break;
    default:
        // code à exécuter si la variable n'a aucune des valeurs précédentes
        break;
}

Les tableaux en Java

Un tableau est une structure de données qui permet de stocker plusieurs valeurs de même type dans une seule variable. En Java, les tableaux sont indexés à partir de zéro.

// Déclaration et initialisation d'un tableau de nombres
int[] tableau = {1, 2, 3, 4, 5};

// Accès aux éléments du tableau
int premierElement = tableau[0];
int dernierElement = tableau[tableau.length - 1];

// Parcours d'un tableau avec une boucle for
for (int i = 0; i < tableau.length; i++) {
    // code à exécuter pour chaque élément du tableau
}

Les fonctions et les méthodes en Java

Les fonctions et les méthodes permettent de regrouper un ensemble d'instructions et de les réutiliser à plusieurs endroits dans un programme. En Java, les fonctions sont définies en dehors des classes, tandis que les méthodes sont définies à l'intérieur des classes.

// Définition d'une fonction
int addition(int a, int b) {
    return a + b;
}

// Appel d'une fonction
int resultat = addition(3, 4);

// Définition d'une méthode
public class MaClasse {
    int multiplication(int a, int b) {
        return a * b;
    }
}

// Appel d'une méthode
MaClasse maInstance = new MaClasse();
int resultat = maInstance.multiplication(3, 4);

La gestion des exceptions en Java

Les exceptions permettent de gérer les erreurs et les situations exceptionnelles qui peuvent se produire lors de l'exécution d'un programme en Java. Les exceptions peuvent être levées (throw) et capturées (catch).

try {
    // code susceptible de générer une exception
} catch (TypeException e) {
    // code à exécuter si une exception de type TypeException est levée
} catch (AutreTypeException e) {
    // code à exécuter si une exception de type AutreTypeException est levée
} finally {
    // code à exécuter quoi qu'il arrive, même s'il y a une exception
}

Les entrées/sorties en Java

Les entrées/sorties (E/S) permettent de lire des données depuis un périphérique d'entrée (comme le clavier) et d'écrire des données vers un périphérique de sortie (comme l'écran ou un fichier). En Java, les E/S sont gérées à l'aide de classes et de méthodes spécifiques.

// Lecture d'une ligne de texte depuis le clavier
Scanner scanner = new Scanner(System.in);
String ligne = scanner.nextLine();

// Écriture d'une ligne de texte vers la console
System.out.println("Hello, World!");

// Lecture et écriture de fichiers
try {
    BufferedReader lecteur = new BufferedReader(new FileReader("fichier.txt"));
    String ligne;
    while ((ligne = lecteur.readLine()) != null) {
        System.out.println(ligne);
    }
    lecteur.close();

    BufferedWriter ecrivain = new BufferedWriter(new FileWriter("fichier.txt"));
    ecrivain.write("Hello, World!");
    ecrivain.close();
} catch (IOException e) {
    e.printStackTrace();
}
