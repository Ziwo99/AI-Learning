Introduction aux exceptions

En programmation, une exception est une condition anormale ou une erreur qui se produit pendant l'exécution d'un programme. Les exceptions sont utilisées pour gérer les erreurs et les situations exceptionnelles de manière structurée.

En Java, les exceptions sont représentées par des objets qui héritent de la classe Throwable. Il existe deux types d'exceptions : les exceptions vérifiées et les exceptions non vérifiées.

Les exceptions vérifiées sont celles qui doivent être gérées explicitement par le programmeur en utilisant des blocs try-catch ou en les déclarant dans la signature de la méthode. Les exceptions non vérifiées sont celles qui n'ont pas besoin d'être gérées explicitement.

Les exceptions peuvent être levées (ou "throw") à l'aide du mot-clé throw et capturées (ou "catch") à l'aide du mot-clé catch.

Gestion des exceptions avec try-catch

La gestion des exceptions en Java se fait généralement à l'aide des blocs try-catch. Un bloc try est utilisé pour entourer le code qui peut potentiellement générer une exception, tandis que le bloc catch est utilisé pour capturer et gérer l'exception.

Voici la syntaxe générale d'un bloc try-catch :

try {
    // Code pouvant générer une exception
} catch (TypeException1 exception1) {
    // Gestion de l'exception1
} catch (TypeException2 exception2) {
    // Gestion de l'exception2
} finally {
    // Code exécuté toujours, qu'il y ait eu une exception ou non
}

Le bloc finally est optionnel et est utilisé pour spécifier un code qui doit être exécuté toujours, qu'il y ait eu une exception ou non.

Création de ses propres exceptions

En plus des exceptions prédéfinies dans Java, il est possible de créer ses propres exceptions personnalisées en définissant une classe qui hérite de la classe Exception ou d'une de ses sous-classes.

Voici un exemple de création d'une exception personnalisée :

public class CustomException extends Exception {
    public CustomException(String message) {
        super(message);
    }
}

Dans cet exemple, la classe CustomException hérite de la classe Exception et définit un constructeur prenant un message en paramètre. Le mot-clé super est utilisé pour appeler le constructeur de la classe parente.

Pour lever une exception personnalisée, on peut utiliser le mot-clé throw :

throw new CustomException("Une erreur personnalisée s'est produite.");

Il est également possible de capturer et de gérer une exception personnalisée de la même manière que les exceptions prédéfinies.
