Définition d'interfaces

Une interface en Java est une collection de méthodes abstraites (méthodes sans implémentation) et de constantes. Elle définit un contrat que les classes qui l'implémentent doivent respecter. Une interface est définie à l'aide du mot-clé interface.

Voici un exemple de définition d'une interface Calculable :

public interface Calculable {
    double calculer(double a, double b);
    void afficherResultat(double resultat);
}

Dans cet exemple, l'interface Calculable définit deux méthodes abstraites : calculer et afficherResultat. Toute classe qui implémente cette interface doit fournir une implémentation pour ces méthodes.

Implémentation d'interfaces

Pour implémenter une interface en Java, une classe utilise le mot-clé implements suivi du nom de l'interface. La classe doit alors fournir une implémentation pour toutes les méthodes définies dans l'interface.

Voici un exemple d'implémentation de l'interface Calculable :

public class Calculatrice implements Calculable {
    public double calculer(double a, double b) {
        return a + b;
    }
    
    public void afficherResultat(double resultat) {
        System.out.println("Le résultat est : " + resultat);
    }
}

Dans cet exemple, la classe Calculatrice implémente l'interface Calculable et fournit une implémentation pour les méthodes calculer et afficherResultat.

Classes abstraites et héritage multiple

Une classe abstraite en Java est une classe qui ne peut pas être instanciée directement, mais qui peut être utilisée comme classe de base pour d'autres classes. Elle peut contenir des méthodes abstraites et des méthodes avec une implémentation.

Voici un exemple de définition d'une classe abstraite Forme :

public abstract class Forme {
    protected String couleur;
    
    public Forme(String couleur) {
        this.couleur = couleur;
    }
    
    public abstract double calculerSurface();
    
    public void afficherCouleur() {
        System.out.println("La couleur de la forme est : " + couleur);
    }
}

Dans cet exemple, la classe abstraite Forme définit un attribut couleur et deux méthodes : calculerSurface (méthode abstraite) et afficherCouleur (méthode avec implémentation).

Une classe peut hériter d'une seule classe abstraite, mais peut implémenter plusieurs interfaces. Cela permet d'obtenir une forme d'héritage multiple en Java.

Voici un exemple de classe Rectangle qui hérite de la classe abstraite Forme et implémente l'interface Calculable :

public class Rectangle extends Forme implements Calculable {
    private double longueur;
    private double largeur;
    
    public Rectangle(String couleur, double longueur, double largeur) {
        super(couleur);
        this.longueur = longueur;
        this.largeur = largeur;
    }
    
    public double calculerSurface() {
        return longueur * largeur;
    }
    
    public void afficherResultat(double resultat) {
        System.out.println("La surface du rectangle est : " + resultat);
    }
}

Dans cet exemple, la classe Rectangle hérite de la classe abstraite Forme et implémente l'interface Calculable. Elle fournit une implémentation pour la méthode abstraite calculerSurface et pour la méthode de l'interface afficherResultat.
